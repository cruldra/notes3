软件测试

软件测试是为了发现和预防软件缺陷，确保软件质量的过程。它通过系统化的测试用例设计和执行，验证软件功能是否符合需求，并评估软件的性能、可靠性、安全性等非功能特性。

📄️ Mock测试的意义与实践

Mock（模拟）是软件测试中的一种技术，它创建一个虚假但可控的替代品来代替真实的依赖项。简单来说，就是用假的东西替换真的东西，让测试更加可控和可靠。

📄️ Testing Library

Testing Library 是一个简单且完整的测试工具集合，它鼓励良好的测试实践。

📄️ Vitest

Vitest 是一个现代化的 JavaScript/TypeScript 测试框架，由 Vite 团队开发。它专为 Vite 生态系统设计，但也可以在非 Vite 项目中使用。Vitest 继承了 Vite 的快速构建和热重载特性，为前端开发者提供了极速的测试体验。

📄️ jsdom详解

jsdom 是一个用纯 JavaScript 实现的 DOM 和 HTML 标准库，它在 Node.js 环境中模拟浏览器的 DOM API。简单来说，jsdom 让你可以在没有浏览器的服务器环境中操作 DOM 元素。

📄️ 基本概念

软件测试是一个系统性的过程，用于评估和验证软件应用程序或系统是否满足指定的需求，并确保其按预期工作。测试的主要目标是发现缺陷、错误或不符合预期的行为。
