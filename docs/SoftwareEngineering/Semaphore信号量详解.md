# Semaphore（信号量）详解

> 本文使用费曼学习法的思想，用最直白的语言把 Semaphore（信号量）讲清楚。

---

## 一、什么是 Semaphore？

**Semaphore（信号量）**是一个**计数器**，用来控制同时访问某个特定资源的线程数量。

它的核心思想很简单：手里有几张"通行证"，就允许几个人进去；证发完了，后来的人必须排队，等里面有人出来还回一张证，才能进入。

---

## 二、生活中的类比：自习室模型

想象一个**高级自习室**，里面只有 **3 个座位**。

### 初始状态

门口挂着一个牌子，上面写着数字 **3**（这就是信号量的初始值，表示有 3 张通行证）。

### 进入（Wait / P 操作）

- 第一个学生来了，看到牌子写着 **3**（还有名额），拿走一张，牌子变成 **2**，进去学习。
- 第二个学生同理，牌子变成 **1**。
- 第三个学生进入，牌子变成 **0**。

### 阻塞（Blocking）

- 第四个学生来了，看到牌子是 **0**，说明没位子了。他不能进去，**只能在门口原地坐下等待**。

### 离开（Signal / V 操作）

- 自习室里有人学累了，推门出来，把名额还回去，牌子从 **0** 变回 **1**。
- **关键动作：** 牌子变回 1 的瞬间，门口等待的学生被唤醒，拿走这个名额进去，牌子再次变回 **0**。

---

## 三、核心机制：两个动作

在代码里，信号量只需要记住两个操作：

### Wait / P 操作（获取通行证）

```
检查计数器：
  ├─ 如果 > 0 → 计数器减 1，继续执行（拿到通行证）
  └─ 如果 = 0 → 线程进入休眠，等待被唤醒
```

### Signal / V 操作（归还通行证）

```
计数器加 1：
  ├─ 如果有线程在排队 → 唤醒其中一个线程
  └─ 如果没有线程排队 → 只是计数器加 1
```

---

## 四、两种常见的类型

### 二值信号量（Binary Semaphore）

- 计数器只有 **0 和 1**
- 就像厕所只有一个隔间：要么有人（0），要么没人（1）
- 行为和**互斥锁（Mutex）**非常相似，但信号量可以跨线程使用

### 计数信号量（Counting Semaphore）

- 计数器可以是任何非负整数（如 3、10、100）
- 用于管理有**多个副本**的资源
- 典型场景：**数据库连接池**（只允许同时有 10 个连接）、**线程池**（限制并发工作线程数）

---

## 五、为什么不直接用锁？

假设有 100 个线程同时访问只有 3 个名额的资源：

| 方案 | 结果 |
|------|------|
| 不用任何控制 | 100 个线程抢 3 个名额，数据混乱，系统过载 |
| 用互斥锁 | 所有线程排队等一把锁，串行执行，效率低 |
| **用信号量** | 3 张通行证，公平分配，排队有序，空出来就放行一个 |

**信号量的意义在于"优雅的限流"：**
- 确保系统不会因为过载而崩溃
- 保证资源的公平分配（先来的先等，空出来就进）
- 允许多个线程同时持有"通行证"并发执行

## 六、代码示例

### Go 语言

```go
func main() {
    sem := semaphore.NewWeighted(3) // 最多 3 个并发

    for i := 0; i < 10; i++ {
        go func(id int) {
            sem.Acquire(context.Background(), 1) // P 操作：获取通行证
            defer sem.Release(1)                  // V 操作：归还通行证

            fmt.Printf("线程 %d 正在执行\n", id)
            time.Sleep(time.Second)
        }(i)
    }

    time.Sleep(15 * time.Second)
}
```

### Python 语言

```python
import threading

sem = threading.Semaphore(3)  # 最多 3 个并发

def worker(id):
    with sem:  # 自动 Wait / Signal
        print(f"线程 {id} 正在执行")
        time.sleep(1)

threads = [threading.Thread(target=worker, args=(i,)) for i in range(10)]
for t in threads:
    t.start()
```

## 七、一句话总结

> **Semaphore = 有限资源的"看门人"。核心逻辑是：有票进场，没票排队，还票唤醒。**

## 八、拓展阅读

- **互斥锁（Mutex）**：二值信号量的近亲，适合单资源独占场景
- **条件变量（Condition Variable）**：常与互斥锁配合使用，适用于"等某个条件成立"的场景
- **Go Weighted semaphore**：Go 标准库提供的带权信号量，可精细控制并发数量
