Pinia

Pinia是Vue的官方状态管理库,作为Vuex的继任者.

安装​

npm install pinia

配置​

// main.ts
import { createPinia } from 'pinia'
const pinia = createPinia()
app.use(pinia)

创建状态仓库​

  • 原生
  • Setup
// store/counter.ts
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', {
  // 状态
  state: () => ({
    count: 0,
    name: 'Eduardo'
  }),

  // 计算属性
  getters: {
    doubleCount: (state) => state.count * 2
  },

  // 方法
  actions: {
    increment() {
      this.count++
    },
    async fetchData() {
      const data = await api.get('...')
      this.someData = data
    }
  }
})
// store/counter.ts
import { defineStore } from 'pinia'

export const useCounterStore = defineStore('counter', () => {
    const count = ref(0)
    const doubleCount = computed(() => count.value * 2)

    function increment() {
        count.value++
    }

    return {count, doubleCount, increment}
})

在组件中使用​

<script setup>
import { useCounterStore } from '@/stores/counter'

const store = useCounterStore()

// 访问 state
console.log(store.count)

// 调用 action
store.increment()

// 使用 getter
console.log(store.doubleCount)
</script>

解构导致丢失响应性的问题​

在使用React的useState时,经常会使用解构的方式来获取状态,例如:

const [count, setCount] = useState(0)

如果在使用Pinia时也使用解构的方式,会导致状态失去响应性,例如:

<script setup lang="ts">
import { useAuthStore } from "@/stores/auth.ts"

// ❌ 错误方式：失去响应性
const { loginType } = useAuthStore()
console.log(loginType) // 得到当前值，但不会随 store 更新

// ✅ 正确方式1：直接使用 store
const store = useAuthStore()
console.log(store.loginType) // 会随 store 更新

// ✅ 正确方式2：使用 storeToRefs
import { storeToRefs } from 'pinia'
const store = useAuthStore()
const { loginType } = storeToRefs(store)
console.log(loginType.value) // 会随 store 更新
</script>

<template>
  <!-- ❌ 错误方式 -->
  <div>{{ loginType }}</div>

  <!-- ✅ 正确方式1 -->
  <div>{{ store.loginType }}</div>

  <!-- ✅ 正确方式2 -->
  <div>{{ loginType }}</div>
</template>
提示

解构赋值会丢失响应性是因为其本质上是创建了一个新的变量,并将值复制给这个变量,从而断开了与原始响应式对象的连接.

持久化​

持久化是指将状态存储在本地,以便在页面刷新或关闭后仍然保留状态.需要用到Pinia Persistedstate插件

  1. 安装
pnpm i pinia-plugin-persistedstate
  1. 配置存储介质
const store = createPinia();

//uniapp
store.use(createPersistedState({
    storage: {
        getItem: uni.getStorageSync,
        setItem: uni.setStorageSync
    }
}))
  1. 然后在创建store时,添加persist: true选项
export const useUserStore = defineStore('user',() => {
    const user = ref(null)
    return {
        user,
    }
}, {
    persist: true
})
