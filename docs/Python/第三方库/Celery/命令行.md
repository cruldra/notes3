> 详细解析 `celery -A smart_sales.celery_app beat --loglevel=info` 的每个部分

---

## 命令拆解

```bash
celery -A smart_sales.celery_app beat --loglevel=info
```

### 1. `celery` —— 主命令

这是 Celery 框架的 CLI 入口。安装 `celery` 包后，系统会注册这个可执行命令。

---

### 2. `-A smart_sales.celery_app` —— 指定应用实例

**`-A`** 是 `--app` 的简写，用于告诉 Celery **你的应用在哪里**。

**语法解析**：
```
-A 模块路径.变量名
   │           │
   │           └── celery_app 实例名
   └── smart_sales.celery_app 模块路径
```

**对应代码结构**：
```python
# src/smart_sales/celery_app.py
from celery import Celery

celery_app = Celery("smart_sales", ...)  # ← 这就是要找的实例
```

**为什么需要 `-A`？**

Celery 是个框架，它需要知道：
1. 你的 Broker 配置是什么（Redis 地址）
2. 你的任务定义在哪里
3. 你的调度表是什么

`-A` 就是指向那个**已经配置好的 Celery 应用实例**。

---

### 3. `beat` —— 子命令（启动模式）

Celery 有多种运行模式，通过子命令指定：

| 子命令 | 作用 | 类比 |
|--------|------|------|
| **`worker`** | 启动任务消费者 | 工厂工人开始干活 |
| **`beat`** | 启动定时调度器 | 监工开始打节拍 |
| `flower` | 启动监控界面 | 工厂监控大屏 |
| `call` | 手动调用任务 | 临时派活 |
| `inspect` | 查看 Worker 状态 | 查岗 |

**为什么叫 `beat` 而不是 `scheduler`？**

遵循 Unix 哲学：命令要短。而且 `beat` 更形象——就像音乐节拍器一样，"滴答滴答"地触发任务。

---

### 4. `--loglevel=info` —— 日志级别

控制日志输出详细程度：

| 级别 | 输出内容 | 适用场景 |
|------|---------|----------|
| `debug` | 最详细，包括内部状态 | 开发调试 |
| **`info`** | 关键事件（任务触发、执行）| **生产推荐** |
| `warning` | 警告和错误 | 安静模式 |
| `error` | 仅错误 | 极简模式 |
| `critical` | 仅致命错误 | 紧急模式 |

## Worker vs Beat 命令对比

同样是 Celery，Worker 启动命令略有不同：

```bash
# Beat 调度器
celery -A smart_sales.celery_app beat --loglevel=info

# Worker 执行器  
celery -A smart_sales.celery_app worker --loglevel=info --concurrency=4
```

**区别**：

| 维度 | Beat | Worker |
|------|------|--------|
| **命令** | `beat` | `worker` |
| **作用** | 只发送任务 | 只执行任务 |
| **读取** | 调度表 | 任务队列 |
| **写入** | 任务队列 | 执行结果 |
| **特有参数** | 无 | `--concurrency=4` |
| **可扩展** | 单实例 | 多实例 |

**`--concurrency=4`**：Worker 特有参数，表示同时处理 4 个任务（并发数）。

## 常用命令速查

```bash
# ═══════════════════════════════════════════════════════════════
# 基础启动
# ═══════════════════════════════════════════════════════════════

# 启动 Worker（4并发，生产推荐）
celery -A smart_sales.celery_app worker -l info -c 4

# 启动 Beat（定时调度）
cery -A smart_sales.celery_app beat -l info

# ═══════════════════════════════════════════════════════════════
# 开发调试用
# ═══════════════════════════════════════════════════════════════

# 同时启动 Worker + Beat（仅开发用，不推荐生产）
celery -A smart_sales.celery_app worker -B -l info

# 调试模式（最详细日志）
celery -A smart_sales.celery_app worker -l debug

# ═══════════════════════════════════════════════════════════════
# 监控管理
# ═══════════════════════════════════════════════════════════════

# 查看 Worker 状态
celery -A smart_sales.celery_app inspect active

# 查看已注册的任务列表
celery -A smart_sales.celery_app inspect registered

# Ping Worker 检查存活
celery -A smart_sales.celery_app inspect ping

# ═══════════════════════════════════════════════════════════════
# 任务操作
# ═══════════════════════════════════════════════════════════════

# 手动触发任务
celery -A smart_sales.celery_app call smart_sales.tasks.sync_chat_records

# 查看任务结果（需要配置 result backend）
celery -A smart_sales.celery_app result <task_id>

# ═══════════════════════════════════════════════════════════════
# 监控界面
# ═══════════════════════════════════════════════════════════════

# 启动 Flower 监控（需先安装 flower）
celery -A smart_sales.celery_app flower --port=5555

# 访问 http://localhost:5555 查看监控面板
```

## 简写形式

Celery 支持多种简写，提高打字效率：

| 完整形式 | 简写 | 示例 |
|---------|------|------|
| `--app` | `-A` | `-A smart_sales.celery_app` |
| `--loglevel` | `-l` | `-l info` |
| `--concurrency` | `-c` | `-c 4` |
| `--broker` | `-b` | `-b redis://localhost:6379/0` |

**简写版命令示例**：
```bash
celery -A smart_sales.celery_app worker -l info -c 4
celery -A smart_sales.celery_app beat -l info
```

## 配置文件方式（推荐生产使用）

除了在命令行指定参数，还可以在代码中配置：

```python
# celery_app.py
celery_app.conf.update(
    task_serializer="json",
    accept_content=["json"],
    result_serializer="json",
    timezone="Asia/Shanghai",
    enable_utc=True,
    worker_prefetch_multiplier=1,  # 相当于 --concurrency 的配套设置
    result_expires=3600,
)
```

这样命令行可以保持简洁：
```bash
celery -A smart_sales.celery_app worker
celery -A smart_sales.celery_app beat
```
