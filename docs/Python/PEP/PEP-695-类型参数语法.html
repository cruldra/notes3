PEP-695-类型参数语法

PEP 695 在 Python 3.12 中引入了一套全新的类型参数（type parameter）声明语法，用方括号原生表达泛型函数、泛型类和泛型类型别名，并新增了 type 软关键字用于声明带参数的类型别名。这一改动让 Python 的泛型写法终于在视觉上接近 TypeScript / Kotlin / Rust，不再需要先在外层导入并手工创建一个 TypeVar。

设计目标：让"声明类型变量"和"使用类型变量"在同一处出现，作用域显式而非全局。

三种新语法​

# 1. 泛型函数
def first[T](items: list[T]) -> T:
    return items[0]

# 2. 泛型类
class Stack[T]:
    def __init__(self) -> None:
        self._items: list[T] = []
    def push(self, item: T) -> None:
        self._items.append(item)
    def pop(self) -> T:
        return self._items.pop()

# 3. 泛型类型别名（配合新 type 软关键字）
type Vector[T] = list[tuple[T, T]]

与旧写法的对比​

旧语法（PEP 484 / PEP 612 / PEP 646）需要先在模块顶层声明类型变量，再在使用处引用：

# 旧写法（仍然可用，但不推荐用于新代码）
from typing import TypeVar, Generic, TypeAlias

T = TypeVar("T")

def first(items: list[T]) -> T:
    return items[0]

class Stack(Generic[T]):
    ...

Vector: TypeAlias = list[tuple[T, T]]

新写法的优势：

维度旧写法新写法
声明位置模块全局紧贴使用处
作用域模糊：靠类型检查器推断明确：仅在该函数/类/别名内部可见
是否需要 Generic[T] 基类需要不需要
是否需要 TypeAlias 注解需要直接使用 type 关键字
类型变量复用不同函数共用同一个全局 T每个签名拥有独立的 T，互不干扰

type 软关键字​

type 是 Python 3.12 引入的"软关键字"——在普通表达式中它仍然是内置函数 type()，只有在语句开头紧跟一个标识符时才被识别为别名声明：

type UserId = int
type Pair[T] = tuple[T, T]
type ListOrSet[T] = list[T] | set[T]

它和 X = int 这种简单赋值的本质差异：

  • 类型别名是惰性求值的，右侧表达式直到被访问 __value__ 时才计算，天然支持前向引用： 
    type Tree[T] = T | list[Tree[T]]   # 自引用合法
  • 别名对象的运行时类型是 typing.TypeAliasType，可以通过 Pair.__type_params__ 拿到声明的类型参数。

三种类型参数​

方括号里支持三种类型参数，对应 TypeVar / TypeVarTuple / ParamSpec：

def f[T](x: T) -> T: ...                            # 普通类型变量
def g[*Ts](*args: *Ts) -> tuple[*Ts]: ...           # 可变参数类型（PEP 646）
def h[**P](fn: Callable[P, int], *a: P.args, **kw: P.kwargs): ...  # 参数规约（PEP 612）

约束与上界​

新语法保留了对类型变量的约束（constraints）与上界（bound）能力：

# bound：T 必须是 Number 的子类
def total[T: Number](xs: list[T]) -> T:
    return sum(xs)

# constraints：T 只能是 str 或 bytes
def join[T: (str, bytes)](sep: T, parts: list[T]) -> T:
    return sep.join(parts)

注意区分语法：T: Number 是单一上界，T: (str, bytes) 是约束集合（必须是元组）。

默认值（PEP 696，3.13 起）​

Python 3.13 通过 PEP 696 为类型参数补齐了默认值能力，配合 PEP 695 的语法非常自然：

class Box[T = int]:
    def __init__(self, value: T = 0) -> None:
        self.value = value

type Pair[T = str] = tuple[T, T]

reveal_type(Box())          # Box[int]
reveal_type(Pair)           # tuple[str, str]

运行时反射​

新语法引入了一致的反射入口 __type_params__：

def first[T](xs: list[T]) -> T: ...
class Stack[T]: ...
type Vector[T] = list[T]

print(first.__type_params__)   # (T,)
print(Stack.__type_params__)   # (T,)
print(Vector.__type_params__)  # (T,)

兼容性​

  • 新语法仅在 Python 3.12+ 可用。如果库需要兼容 3.11 及以下，仍需使用旧的 TypeVar / Generic 写法。
  • 新旧两套写法可以混用，但同一个泛型签名内不要混。
  • 主流类型检查器（mypy 1.8+、Pyright、Pylance）均已完整支持。

何时用 / 何时不用​

  • 推荐使用：新写的 3.12+ 项目、内部代码库、教程示例
  • 暂时回避：需要兼容 3.11 及以下的库代码、需要将类型变量在多个签名间共享的场景（这种"共享"的诉求往往是设计味道）
