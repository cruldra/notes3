开源选股平台调研报告

调研目的: 为客户"价值罗盘"项目寻找可直接部署的开源股票分析平台，作为二次开发基座 调研日期: 2026年3月3日 商业策略: 部署开源平台 → 客户试用 → 按功能模块收费定制

一、执行摘要​

核心结论​

经过对 GitHub、Gitee 等平台的全面调研，共发现 12个 与客户需求相关的开源项目。根据功能覆盖度、部署难度、技术栈匹配度和社区活跃度，推荐以下方案：

推荐等级项目核心优势客户需求覆盖率
🥇 首选StockAnal_SysFlask+AKShare，功能最匹配，有Web UI~80%
🥈 备选Aquantstock-selection-systemStreamlit多因子选股，一键Docker部署~70%
🥉 备选Bstock-monitorAI价值投资分析，巴菲特/格雷厄姆视角~60%

推荐策略​

方案一（快速交付，2-4周）: 以 StockAnal_Sys 为基座，补充 DCF 估值和组合管理模块 方案二（混合架构，4-6周）: StockAnal_Sys 核心 + qstock 选股算法 + Backtrader 回测引擎 方案三（专业路线，2-3月）: QUANTAXIS/vnpy 后端 + 自建 React 前端

二、客户需求回顾​

根据《深度调研报告》和《技术可行性评估报告》，客户核心需求如下：

功能模块具体需求优先级
基础选股低PE/PB、高ROIC、高股息率筛选P0
财报可视化利润表/资产负债表/现金流图表化展示P0
AI护城河评估大模型分析企业竞争优势P1
DCF估值自由现金流折现模型计算内在价值P1
策略回测历史数据验证选股策略收益P1
组合监控持仓管理、盈亏追踪、再平衡提醒P2
财务风险预警异常财务指标自动报警P2

技术栈偏好: Python + FastAPI 后端, React 前端, AKShare 数据源 目标市场: 沪深A股（后期扩展港股） 商业模式: 免费基础版 + 付费专业版（¥499/月）

cp config.template.py config.py

pip install -r requirements.txt
python run.py
# 访问 http://localhost:8000

二次开发评估:

  • FastAPI 后端是最大亮点，与客户需求技术栈完全匹配
  • AI分析功能可直接复用（客户要求的"AI护城河评估"）
  • 需要添加选股筛选、回测、DCF估值等模块

五、部署策略详细方案​

方案一：快速交付路线（推荐）​

基座: StockAnal_Sys 工期: 2-4 周 预算: ¥15,000-25,000

阶段1（第1周）: 部署 + 界面优化
├── 部署 StockAnal_Sys 到客户服务器
├── 调整 UI 风格（品牌色、Logo、布局）
└── 确保基础选股功能正常运行

阶段2（第2-3周）: 核心功能补充
├── 集成 AI 分析（参考 stock-monitor 的豆包集成）
├── 添加 DCF 估值计算器（纯Python实现）
└── 优化财报可视化图表

阶段3（第3-4周）: 测试 + 交付
├── 集成 Backtrader 回测模块（参考 stock-backtrader-web-app）
├── 用户测试 + Bug修复
└── 部署上线

收费模型:

模块初始部署（含在基础费用中）定制开发（按模块收费）
基础选股筛选✅ 已有-

基座: StockAnal_Sys + qstock算法 + Backtrader回测 工期: 4-6 周 预算: ¥30,000-40,000

基座: QUANTAXIS 后端 + 自建 React 前端 工期: 2-3 个月 预算: ¥80,000-120,000（超出客户预算）

八、风险评估​

风险项等级说明应对措施
开源项目停更🟡 中Stars 较少的项目可能不再维护选择活跃项目，提前Fork
数据源API变更🟡 中AKShare/Tushare API可能变更抽象数据层，支持多数据源切换
客户需求膨胀🔴 高客户可能不断追加需求严格按模块收费，签订明确合同
法律合规🟡 中金融类产品需注意监管明确标注"仅供学习研究，不构成投资建议"
性能瓶颈🟢 低单用户/少用户场景初期无需考虑，后期可引入Redis缓存
