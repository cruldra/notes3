# 开源选股平台调研报告

> **调研目的**: 为客户"价值罗盘"项目寻找可直接部署的开源股票分析平台，作为二次开发基座
> **调研日期**: 2026年3月3日
> **商业策略**: 部署开源平台 → 客户试用 → 按功能模块收费定制

---

## 一、执行摘要

### 核心结论

经过对 GitHub、Gitee 等平台的全面调研，共发现 **12个** 与客户需求相关的开源项目。根据功能覆盖度、部署难度、技术栈匹配度和社区活跃度，推荐以下方案：

| 推荐等级 | 项目 | 核心优势 | 客户需求覆盖率 |
|---------|------|---------|-------------|
| 🥇 首选 | StockAnal_Sys | Flask+AKShare，功能最匹配，有Web UI | ~80% |
| 🥈 备选A | quantstock-selection-system | Streamlit多因子选股，一键Docker部署 | ~70% |
| 🥉 备选B | stock-monitor | AI价值投资分析，巴菲特/格雷厄姆视角 | ~60% |

### 推荐策略

**方案一（快速交付，2-4周）**: 以 StockAnal_Sys 为基座，补充 DCF 估值和组合管理模块
**方案二（混合架构，4-6周）**: StockAnal_Sys 核心 + qstock 选股算法 + Backtrader 回测引擎
**方案三（专业路线，2-3月）**: QUANTAXIS/vnpy 后端 + 自建 React 前端

## 二、客户需求回顾

根据《深度调研报告》和《技术可行性评估报告》，客户核心需求如下：

| 功能模块 | 具体需求 | 优先级 |
|---------|---------|-------|
| 基础选股 | 低PE/PB、高ROIC、高股息率筛选 | P0 |
| 财报可视化 | 利润表/资产负债表/现金流图表化展示 | P0 |
| AI护城河评估 | 大模型分析企业竞争优势 | P1 |
| DCF估值 | 自由现金流折现模型计算内在价值 | P1 |
| 策略回测 | 历史数据验证选股策略收益 | P1 |
| 组合监控 | 持仓管理、盈亏追踪、再平衡提醒 | P2 |
| 财务风险预警 | 异常财务指标自动报警 | P2 |

**技术栈偏好**: Python + FastAPI 后端, React 前端, AKShare 数据源
**目标市场**: 沪深A股（后期扩展港股）
**商业模式**: 免费基础版 + 付费专业版（¥499/月）
