项目规划

高级搜索​

目的​

高级搜索是指从多个维度对器物进行搜索,包括器物的名称(包括俗称)、类型、年代、朝代、材料、制作工艺、颜色、形状、拍卖公司、拍卖时间等.

实现思路​

建模

对器物进行现代化建模，用于存储各个维度的信息以便后续检索

建模是最底层但也最重要的部分,会先对现有的器物数据库结构进行分类调整,参考故宫博物院、雅昌艺术品拍卖网等平台上有关器物数据模型的部分

建模还涉及到数据库软件系统选型,会组合2到3种数据库系统,具体选型需要进一步了解实际需求后再决定.

数字化

对现有的器物数据进行数字化,将其录入到数据库中.

这部分是工作量最大的部分,需要对器物数据进行对接、分类、迁移或翻译等操作,并将其数字化到数据库中.

这里会大量利用到现成或单独定制的ai工具以提高数字化效率.

关于目前app中的一部分器物数据,如果其底层结构支持高级现代化搜索,则直接使用,否则仍需要对其按上面的步骤进行数字化.

用户界面

数据准备就绪后还需要一个用户界面来展示和检索器物数据,通过微信小程序的形式提供.

图像检索​

目的​

图像检索通常是指用户通过拍照等方式上传器物图片,系统从图中提取特征并进行匹配,以找到与查询图片相似的器物图片.

实现思路​

这部分涉及到计算机视觉和机器学习等技术,以下是大致思路:

  • 选择预训练模型(如ResNet、VGG等)
  • 准备GPU计算资源
  • 建立特征向量数据库
  • 高清拍摄
  • 多角度采集
  • 光照标准化
  • 图像预处理(统一尺寸、去噪等)
  • 提取特征向量
  • 存入向量数据库

模型训练和微调​

自己训练器物大模型具备以下优势:

  1. 做成聊天大模型可以面向所有专业或非专业用户,体验更好
  2. 数据以非结构化形式提供,难以被竞争对手复制

但训练大模型需要大量数据和计算资源,成本较高.

成本预估​

整个系统开发中涉及到的付费大概有:

  • 应用、数据库等服务器
  • 向量数据库需要商业版权费用
  • 小程序域名、备案
  • GPU计算资源
  • 软件开发、数据处理涉及到的人力成本
  • 如果涉及大模型训练及微调,还需要额外的计算资源和人力成本

人员配置​

核心开发团队：

  • 后端工程师 x 2
  • 前端工程师 x 1
  • 算法工程师 x 1
  • 运维工程师 x 1

如需训练模型：

  • 数据标注人员 x 3
  • 模型优化工程师 x 1

开发周期​

  • 仅高级搜索图片检索: 预计4个月左右
  • 如果训练模型: 预计6个月左右
