智能销售系统架构视图

智能销售系统架构视图

基于 “三层两翼” 设计理念 • 事件驱动与状态机决策模型

Human-in-the-loop (交互)

🦅 左翼：销售实操工作台

企微侧边栏 (H5)
实时漏斗仪表盘
人工干预/改派
待办任务中心

1. 感知层 (Perception)

事件驱动信号总线 (Event Bus)

系统的“耳目”。负责实时监听全渠道的异构信号，清洗并标准化为内部事件。

Webhook 接收器
直播埋点清洗
事件去重/防抖
Kafka 消息队列

2. 决策层 (Decision)

FSM 状态机 & 策略路由 (The Brain)

系统的“大脑”。维护客户状态(FSM)，基于规则引擎和原子事实做出判断，驱动 AI 生成策略。

动态客户状态机 (FSM)
分层策略路由引擎
冲突消解中心
AI Agent & RAG

3. 执行层 (Execution)

通道抽象网关 (Channel Gateway)

系统的“手脚”。屏蔽底层 API 差异，负责安全、合规地将指令触达给客户。

统一发送指令接口
高频防护 & 熔断
企微/短信/外呼适配器
通道自动切换
Configuration (规则)

🦅 右翼：运营配置中心

策略画布配置
原子事实管理 (价格/政策)
人群包与LTV配置
审计与风控日志
