Leader 键​

OpenCode 大多数快捷键使用 leader 键。这样可以避免与您的终端冲突。

默认情况下，ctrl+x 是 leader 键，大多数操作需要您先按 leader 键，然后再按快捷键。例如，要开始新会话，您需要先按 ctrl+x，然后按 n。

您不一定要为快捷键使用 leader 键，但我们建议这样做。

禁用快捷键​

您可以通过在配置中添加键并将其值设置为 "none" 来禁用快捷键。

{
  "$schema": "https://opencode.ai/config.json",
  "keybinds": {
    "session_compact": "none"
  }
}

桌面应用提示符快捷键​

OpenCode 桌面应用的提示输入支持常见的 Readline/Emacs 风格的文本编辑快捷键。这些是内置的，目前无法通过 opencode.json 配置。

快捷键操作
ctrl+a移动到当前行的开头
ctrl+e移动到当前行的末尾
ctrl+b光标向后移动一个字符
ctrl+f光标向前移动一个字符
alt+b光标向后移动一个单词
alt+f光标向前移动一个单词
ctrl+d删除光标下的字符
ctrl+k删除到行尾
ctrl+u删除到行首
ctrl+w删除前一个单词
alt+d删除下一个单词
ctrl+t转置字符
ctrl+g取消弹出窗口 / 中止正在运行的响应

Shift+Enter​

某些终端默认不会发送带有 Enter 键的修饰键。您可能需要配置终端以将 Shift+Enter 作为转义序列发送。

Windows Terminal​

打开您的 settings.json，位于：

%LOCALAPPDATA%\Packages\Microsoft.WindowsTerminal_8wekyb3d8bbwe\LocalState\settings.json

在根级别的 actions 数组中添加以下内容：

"actions": [
  {
    "command": {
      "action": "sendInput",
      "input": "\\u001b[13;2u"
    },
    "id": "User.sendInput.ShiftEnterCustom"
  }
]

在根级别的 keybindings 数组中添加以下内容：

"keybindings": [
  {
    "keys": "shift+enter",
    "id": "User.sendInput.ShiftEnterCustom"
  }
]

保存文件并重新启动 Windows Terminal 或打开新标签页。
