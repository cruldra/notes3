## 场景一：接手新项目

### 背景
你刚加入团队，需要快速理解一个微服务架构的代码库。

### 步骤

#### 1. 索引项目

```bash
cd microservices-project
gitnexus analyze
```

#### 2. 让 AI 生成架构概览

**提问**：
```
请帮我理解这个项目的整体架构，包括：
1. 有哪些主要服务/模块？
2. 它们之间是如何交互的？
3. 数据流是怎样的？
```

**AI 使用工具链**：
1. `list_repos` - 发现可用仓库
2. `query({query: "microservice architecture service modules"})` - 查找服务模块
3. `query({query: "API endpoints controllers routes"})` - 查找 API 端点
4. `query({query: "database models entities"})` - 查找数据模型
5. `context({name: "各主要服务类"})` - 深入了解关键服务

**预期输出**：
```markdown
## 项目架构概览

### 主要服务
1. **User Service** (src/services/user/)
   - 处理用户注册、登录、资料管理
   - 依赖：Database, Redis, Email Service
   - API：RESTful endpoints at /api/users

2. **Order Service** (src/services/order/)
   - 处理订单创建、支付、状态跟踪
   - 依赖：Database, Payment Gateway, Notification Service
   - API：RESTful endpoints at /api/orders

3. **Inventory Service** (src/services/inventory/)
   - 管理库存、商品信息
   - 依赖：Database, Cache
   - API：RESTful endpoints at /api/inventory

### 服务间通信
- **同步**：REST API 调用（Order → User, Order → Inventory）
- **异步**：消息队列（所有服务 → Notification Service）

### 数据流
1. 用户下单 → Order Service
2. Order Service 验证用户 → User Service
3. Order Service 检查库存 → Inventory Service
4. 扣减库存 → Inventory Service
5. 处理支付 → Payment Gateway
6. 发送通知 → Notification Service (异步)
```

#### 3. 深入理解特定流程

**提问**：
```
帮我详细分析订单创建的完整流程，从 API 入口到数据库操作
```

**AI 使用工具链**：
1. `query({query: "order creation flow create order"})` - 查找相关代码
2. `context({name: "createOrder"})` - 查看核心函数上下文
3. `context({name: "OrderController"})` - 查看控制器

**预期输出**：
```markdown

**Step 1: API 入口**
- File: src/services/order/controllers/orderController.ts:45
- Function: `createOrder(req, res)`
- 验证请求参数，提取用户 ID、商品 ID、数量

- 调用: `userService.validateUser(userId)`
```

```markdown
### 流程分析

**正常流程**：

- File: src/controllers/paymentCallbackController.ts:34
```

```typescript
// src/controllers/paymentCallbackController.ts

async handleCallback(payload: PaymentCallbackPayload): Promise<void> {
  // 1. 验证回调签名
  const isValid = await this.paymentService.verifyCallbackSignature(payload);
  if (!isValid) {
    throw new Error('Invalid callback signature');
  }
  
  // 2. 幂等性检查 - 新增
  const processed = await this.idempotencyService.isProcessed(payload.callbackId);
  if (processed) {
    logger.info(`Callback ${payload.callbackId} already processed`);
    return;
  }
  
  // 3. 获取订单并验证状态 - 新增
  const order = await this.orderService.getOrder(payload.orderId);
  if (!order) {
    throw new Error(`Order ${payload.orderId} not found`);
  }
  
  const allowedTransitions = ['pending', 'payment_pending'];
  if (!allowedTransitions.includes(order.status)) {
    logger.warn(`Cannot update order ${order.id} with status ${order.status}`);
    return;
  }
  
  // 4. 使用事务处理 - 修改
  await this.database.transaction(async (trx) => {
    // 记录支付
    await this.paymentService.recordPayment(payload, trx);
    
    // 更新订单状态
    await this.orderService.updateOrderStatus(
      payload.orderId, 
      'paid', 
      { transaction: trx }
    );
    
    // 标记回调已处理
    await this.idempotencyService.markProcessed(payload.callbackId, trx);
  });
  
  // 5. 发送通知
  await this.notificationService.sendPaymentSuccess(order);
}
```
