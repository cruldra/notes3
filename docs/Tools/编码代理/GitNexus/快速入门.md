# GitNexus 快速入门指南

> 5 分钟上手 GitNexus，让你的 AI 编码代理拥有代码库的深度理解能力

---

## 第一步：安装

```bash
# 全局安装（推荐）
npm install -g gitnexus

# 或使用 npx（无需安装）
npx gitnexus
```

---

## 第二步：索引你的代码库

```bash
# 进入你的项目目录
cd your-project

# 一键索引
gitnexus analyze
```

完成！✅

这会：
- 解析整个代码库结构
- 构建知识图谱
- 安装 AI 代理技能
- 创建 `AGENTS.md` / `CLAUDE.md`

---

## 第三步：配置 MCP（一次即可）

### 自动配置（推荐）

```bash
gitnexus setup
```

自动检测并配置你的编辑器。

### 手动配置

**Claude Code**：
```bash
claude mcp add gitnexus -- npx -y gitnexus@latest mcp
```

**Cursor**（`~/.cursor/mcp.json`）：
```json
{
  "mcpServers": {
    "gitnexus": {
      "command": "npx",
      "args": ["-y", "gitnexus@latest", "mcp"]
    }
  }
}
```

**OpenCode**（`~/.config/opencode/config.json`）：
```json
{
  "mcp": {
    "gitnexus": {
      "command": "npx",
      "args": ["-y", "gitnexus@latest", "mcp"]
    }
  }
}
```

---

## 第四步：开始使用

### 常用命令

```bash
# 检查索引状态
gitnexus status

# 列出所有索引的仓库
gitnexus list

# 更新索引
gitnexus analyze

# 强制重新索引
gitnexus analyze --force

# 生成项目 Wiki
gitnexus wiki

# 清理索引
gitnexus clean
```

## 第五步：让 AI 代理帮你分析代码

### 询问影响范围

```
"如果修改 UserService.validate()，会影响哪些代码？"
```

AI 会使用 `impact` 工具分析所有依赖关系。

### 理解代码结构

```
"帮我理解这个项目的架构"
```

AI 会使用知识图谱生成架构文档。

### 追踪 Bug

```
"这个错误的调用链是怎样的？"
```

AI 会追踪从入口点到问题代码的完整路径。

### 安全重构

```
"将 validateUser 重命名为 verifyUser"
```

AI 会协调多文件修改。

### 提交前检查

```
"检查我的变更会影响哪些部分"
```

AI 会分析 Git diff 的影响范围。

## 常见问题

### Q: 索引需要多长时间？

取决于代码库大小。小型项目几秒，大型项目几分钟。

### Q: 支持哪些语言？

TypeScript, JavaScript, Python, Java, Kotlin, C, C++, C#, Go, Rust, PHP, Swift。

### Q: 数据安全吗？

完全本地处理，无代码上传到任何服务器。

### Q: 索引文件会提交到 Git 吗？

不会，`.gitnexus/` 目录已自动添加到 `.gitignore`。

### Q: 可以免费使用吗？

是的，GitNexus 是开源的，使用 PolyForm Noncommercial 许可证。
