Serena 提供了完整 agent 的功能，其中一个实用的方面就是 Serena 的记忆系统。
尽管设计简洁，但我们收到了许多用户的积极反馈，他们倾向于将其与 agent 内部的记忆管理机制（如 `AGENTS.md` 文件）结合使用。

## 记忆

记忆是简单、人类可读的 Markdown 文件，你和你的 agent 都可以创建、读取和编辑。

Serena 区分以下两种记忆：

- **项目专属记忆**：存储在项目文件夹中的 `.serena/memories/` 目录下
- **全局记忆**：跨所有项目共享，默认存储在 `~/.serena/memories/global/` 目录下

LLM 会被告知记忆的存在，并被指示在适当的时候读取它们——适当性由文件名推断。
当 agent 开始在某个项目上工作时，它会收到可用记忆的列表。
用户应在适当的时候指示 agent 更新记忆。

### 组织记忆

记忆可以通过在名称中使用 `/` 来按**主题**组织（例如 `modules/frontend`）。
这种结构映射到文件系统，主题对应子目录。
`list_memories` 工具可以按主题过滤，使 agent 能够以结构化的方式浏览大量记忆。

### 全局记忆

全局记忆使用顶级主题 `global`，即每当记忆名称以 `global/` 开头时，它就会被存储在全局记忆目录中，并在所有项目之间共享。

默认情况下，允许删除和编辑全局记忆。
如果你想保护它们免受 agent 意外修改，可以在全局或项目级[配置](050_configuration)中添加正则表达式到 `read_only_memory_patterns`。例如，设置 `"global/.*"` 将把所有全局记忆标记为只读。agent 会被告知哪些记忆是只读的。

由于全局记忆不会随项目文件一起版本化，将全局记忆用 git 跟踪（即将 `~/.serena/memories/` 设为 git 仓库）会很有帮助，这样可以保留变更历史，并在需要时回退。

### 忽略记忆

对于积累了大量归档记忆文件的项目，可以使用 `ignored_memory_patterns` 将它们从 `list_memories` 和 `activate_project` 的输出中排除。在全局或项目级[配置](050_configuration)中添加正则表达式：

```yaml
ignored_memory_patterns: ["_archive/.*", "_episodes/.*"]
```

被忽略的记忆将被完全排除——无法通过 `read_memory`、`write_memory` 或任何其他记忆工具访问它们。要读取被忽略的记忆文件，请使用 `read_file` 工具直接访问原始文件路径（例如 `.serena/memories/_archive/2026-03/some-topic.md`）。

与 `read_only_memory_patterns` 一样，全局和项目级配置中的模式会合并叠加。

### 手动编辑记忆

你可以直接在文件系统中使用你喜欢的文本编辑器或 IDE 编辑记忆。
也可以通过 [Serena 仪表盘](060_dashboard) 访问它们，该仪表盘提供了图形化界面，让你在 Serena 运行时查看、创建、编辑和删除记忆。

## 入门引导

默认情况下，Serena 在首次遇到某个项目时（即尚不存在项目记忆时）会执行**入门引导流程**。
入门引导的目标是让 Serena 熟悉项目——包括项目结构、构建系统、测试设置以及其他关键方面——并将这些知识存储为记忆，以供后续交互使用。

在后续的项目激活中，Serena 会通过检查是否存在项目记忆来判断是否已完成入门引导，如果找到记忆则跳过入门引导流程。

### 入门引导的工作方式

1. 当项目被激活时，Serena 会检查是否已完成入门引导（通过检查是否存在任何记忆）。
2. 如果没有找到记忆，Serena 会触发入门引导流程，读取关键文件和目录以了解项目。
3. 收集到的信息会被写入项目专属的记忆文件中（见上文）。

### 入门引导提示

- **上下文用量**：入门引导流程会读取项目中的大量内容，填满上下文窗口。因此建议在入门引导完成后**切换到新的对话**。
- **LLM 失败**：如果 LLM 未能完成入门引导且没有实际将相应的记忆写入磁盘，你可能需要明确要求它这样做。
- **检查结果**：入门引导完成后，建议快速浏览生成的记忆，并根据需要编辑或添加新的记忆。

## 禁用记忆与入门引导

如果你不需要本节描述的功能，可以选择性地禁用它。

- 要禁用所有记忆相关工具（包括入门引导），在 Serena 的[全局配置](050_configuration)中的 `base_modes` 里添加 `no-memories`。
- 类似地，要仅禁用入门引导，在 `base_modes` 中添加 `no-onboarding`。
