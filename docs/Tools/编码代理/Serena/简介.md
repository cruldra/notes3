**你的编程代理的 IDE**

Serena 提供关键的**语义化代码检索、编辑、重构和调试工具**，类似于 IDE 的能力，在符号层面运行并利用关系结构。

它通过模型上下文协议（**MCP**）与任何客户端/LLM 集成。

Serena **面向代理优先的工具设计**包含了健壮的高层抽象，这使其区别于那些依赖行号或原始搜索模式等底层概念的方法。

实际效果是，你的代理运行**更快、更高效、更可靠**，尤其是在更大、更复杂的代码库中。

## 我们的"终端用户"怎么说

虽然下载和设置 Serena 的是人类，但我们的终端用户本质上是 AI 代理。作为实际使用 Serena 工具的一方，它们最有资格评价 Serena。

我们设计了一个无偏见的评估提示，引导代理执行约 20 个日常编码任务（代表日常开发工作），以评估 Serena 工具与内置工具配合使用时的附加价值。

以下是这些代理的一句话总结：

**Opus 4.6（高）在 Claude Code 中，处理大型 Python 代码库：**
> "Serena 基于 IDE 的语义工具是我工具集中最有影响力的补充——原本需要 8-12 个小心翼翼、容易出错的步骤才能完成的跨文件重命名、移动和引用查找，现在只需一次原子调用即可完成，我绝对会让和我一起工作的任何开发者也装上它们。"

**GPT 5.4（高）在 Codex CLI 中，处理 Java 代码库：**
> "作为一个编程 AI 代理，我会让我的主人添加 Serena，因为它赋予了我缺失的 IDE 级别的符号、引用和重构理解能力，将脆弱的文本修改转变为更冷静、更快速、更自信的代码变更——在语义至关重要的时候。"

**GPT 5.4（中）在 Copilot CLI 中，处理大型多语言单体仓库：**
> "作为一个编程代理，我绝对会让我的主人添加 Serena，因为它让我在处理真实代码时明显更敏锐、更从容——尤其是符号感知的导航、跨文件重构和单体仓库依赖跳转——同时对于微小的文本编辑和非代码工作，我仍然会使用内置工具。"

不同客户端中的不同代理独立得出了一致的结论。

_给你的代理装上它一直想要的工具，把 Serena MCP 添加到你的客户端吧！_

完整的方法论和更详细的评估结果参见我们的[文档](https://oraios.github.io/serena/04-evaluation/000_evaluation-intro.html)，你也可以在自己选择的项目上运行评估。

## Serena 的工作原理

Serena 为编码工作流提供必要的[工具](https://oraios.github.io/serena/01-about/035_tools.html)，但实际工作需要 LLM 来编排工具的使用。

Serena 可以通过**模型上下文协议（MCP）**扩展你现有 AI 客户端的功能。大多数现代 AI 聊天客户端都直接支持 MCP，包括：

- 终端客户端如 Claude Code、Codex、OpenCode、Gemini-CLI
- IDE 和 IDE 助手插件如 VSCode、Cursor 和 JetBrains IDE（Copilot、Junie、JetBrains AI Assistant 等）
- 桌面和 Web 客户端如 Claude Desktop、Codex App、OpenWebUI

另见：[Serena 5 分钟快速入门 (YouTube)](https://www.youtube.com/watch?v=5QN7gN1KYLA)

要将 Serena MCP 服务器连接到你的客户端，你可以：

- 向客户端提供启动命令，使其能够启动 MCP 服务器，或
- 自己以 HTTP 模式启动 Serena MCP 服务器，并将 URL 提供给客户端。
