Serena 基于对代码的符号化理解，为多种编程语言提供了一套通用的代码查询和编辑功能。凭借这些能力，Serena 能像一位熟练使用 IDE 的资深开发者一样发现和编辑代码。即使在非常庞大和复杂的项目中，Serena 也能高效地找到正确的上下文并做出正确的操作。

有两种替代技术为这些能力提供动力：

- **语言服务器**——实现语言服务器协议（LSP），是免费/开源的方案。
- **Serena JetBrains 插件**——利用 JetBrains IDE 强大的代码分析和编辑能力。

详见[功能特性](025_features)部分，了解 JetBrains 插件与语言服务器的详细能力对比。

## 语言服务器

Serena 集成了一个强大的抽象层，用于整合实现了语言服务器协议（LSP）的语言服务器。它甚至支持并行使用多个语言服务器，以应对多语言项目。

语言服务器本身通常是开源项目（和 Serena 一样），或至少可以免费使用。

我们目前为以下编程语言提供了开箱即用的直接支持。部分语言需要额外安装或设置步骤，已注明。

- **Ada / SPARK**（使用 AdaCore 的 [Ada Language Server (ALS)](https://github.com/AdaCore/ada_language_server)，自动下载；支持 `.ads`、`.adb` 和 `.ada` 文件；最好在仓库根目录放置 `.gpr` GNAT 项目文件；SPARK 由同一服务器透明处理——两者均设置语言为 `ada`。如需使用预装的 ALS（如来自 Alire、GNAT Studio 或 VS Code Ada 扩展），请设置 `ls_specific_settings.ada.ls_path`。）
- **AL**
- **Angular**（实验性；需要 Node.js + npm，且项目根目录已执行 `npm install`，使 `@angular/core` 可解析——否则模板感知功能会静默返回空结果；对 `.ts`/`.html` 文件涵盖了 `typescript` 和 `html`，因此无需重复列出）
- **Ansible**（实验性；需要 Node.js 和 npm；自动安装 `@ansible/ansible-language-server`；必须在 `project.yml` 的 `languages` 条目中显式指定；需要 `ansible` 在 PATH 中以获得完整功能。上游 `@ansible/ansible-language-server@1.2.3` 支持悬停、补全、跳转定义、语义标记和验证；文档符号、工作区符号、引用和重命名在此版本中不受支持。）
- **Bash**
- **BSL**（1C:Enterprise / OneScript）（需要 Java 21+ 在 PATH 中；使用 1c-syntax 的 [bsl-language-server](https://github.com/1c-syntax/bsl-language-server)；JAR 包自动下载并通过 SHA-256 校验；支持 `.bsl` 和 `.os` 文件；可在 `ls_specific_settings.bsl` 下配置可选的 `ls_path` 或 `bsl_ls_version`）
- **C#**（默认使用 Roslyn 语言服务器（语言 `csharp`），需要 [.NET v10+](https://dotnet.microsoft.com/en-us/download/dotnet)，Windows 上还需要 `pwsh`（[PowerShell 7+](https://learn.microsoft.com/en-us/powershell/scripting/install/install-powershell-on-windows?view=powershell-7.5)）；设置语言为 `csharp_omnisharp` 可改用 OmniSharp）
- **C/C++**（默认使用 clangd 语言服务器（语言 `cpp`），同时也支持 ccls（语言 `cpp_ccls`）；为获得最佳效果，请在仓库根目录提供 `compile_commands.json`；详见 [C/C++ 设置指南](../03-special-guides/cpp_setup)。）
- **Clojure**
- **Crystal**（需要安装 [Crystalline](https://github.com/elbywan/crystalline) 语言服务器并在 PATH 中可用；注意：Crystalline 的跳转定义支持有限，且不支持查找引用）
- **Dart**
- **Elixir**（需要安装 Elixir；Expert 语言服务器自动下载）
- **Elm**（需要 Elm 编译器）
- **Erlang**（需要安装 beam 和 [erlang_ls](https://github.com/erlang-ls/erlang_ls)；实验性，可能较慢或卡死）
- **F#**（需要 [.NET v8.0+](https://dotnet.microsoft.com/en-us/download/dotnet)；使用 FsAutoComplete/Ionide，自动安装；macOS 上通过 Homebrew 安装 .NET 时，请在环境中设置 DOTNET_ROOT）
- **Fortran**（需要安装 fortls：`pip install fortls`）
- **GDScript**（Godot 引擎）（需要 Godot 编辑器运行并启用内置 LSP——默认端口 6008；Serena 通过 TCP 连接，不会自行启动 Godot；详见 [GDScript 设置指南](../03-special-guides/godot_gdscript_setup_guide_for_serena)）
- **Go**（需要安装 `gopls`）
- **Groovy**（需要通过 `GROOVY_LS_JAR_PATH` 或配置进行本地 groovy-language-server.jar 设置）
- **Haskell**（自动通过 ghcup、stack 或系统 PATH 定位 HLS；支持 Stack 和 Cabal 项目）
- **Haxe**（需要 Haxe 编译器 3.4.0+ 和 Node.js；使用 [vshaxe language server](https://github.com/vshaxe/haxe-language-server)；从 Open VSX 自动下载，或从 vshaxe VSCode 扩展中发现）
- **HLSL / GLSL / WGSL**（使用 [shader-language-server](https://github.com/antaalt/shader-sense)（语言 `hlsl`）；自动下载；macOS 上需要 Rust 工具链以从源码构建；注意：此语言服务器不支持引用搜索）
- **HTML**（实验性；需要 Node.js + npm）
- **Java**
- **JavaScript**（通过 TypeScript 语言服务器支持，即 JavaScript 和 TypeScript 均使用语言 `typescript`）
- **Julia**
- **Kotlin**（使用预 alpha 阶段的[官方 kotlin LS](https://github.com/Kotlin/kotlin-lsp)，可能出现一些问题）
- **Lean 4**（需要通过 [elan](https://github.com/leanprover/elan) 安装 `lean` 和 `lake`；使用内置的 Lean 4 LSP；项目必须是 Lake 项目，使用前需运行 `lake build`）
- **Lua**
- **Luau**
- **Markdown**（必须显式启用语言 `markdown`，主要适用于文档密集型项目）
- **mSL**（mIRC 脚本语言）（自动安装；无需外部依赖——使用 Serena 自带的基于 pygls 的自定义 LSP 服务器；支持 `.mrc` 文件中的别名、事件、菜单、对话框和 CTCP 处理器的文档符号、工作区符号、引用和跳转定义）
- **Nix**（需要安装 nixd）
- **OCaml**（需要手动安装 opam 和 ocaml-lsp-server；详见 OCaml 设置指南）
- **Pascal**（使用 Pascal/Lazarus，自动下载；设置 `PP` 和 `FPCDIR` 环境变量以进行源码导航）
- **Perl**（需要安装 Perl::LanguageServer）
- **PHP**（默认使用 Intelephense 语言服务器（语言 `php`），设置 `INTELEPHENSE_LICENSE_KEY` 环境变量以使用高级功能；同时支持 [Phpactor](https://github.com/phpactor/phpactor)（语言 `php_phpactor`），需要 PHP 8.1+）
- **Python**
- **R**（需要安装 `languageserver` R 包）
- **Ruby**（默认使用 [ruby-lsp](https://github.com/Shopify/ruby-lsp)（语言 `ruby`）；使用语言 `ruby_solargraph` 可改用 Solargraph。）
- **Rust**（需要 [rustup](https://rustup.rs/)——使用工具链中的 rust-analyzer）
- **Scala**（需要一些手动设置；使用 Metals LSP）
- **SCSS / Sass / CSS**（实验性；需要 Node.js + npm；使用 [some-sass-language-server](https://github.com/wkillerud/some-sass) 处理 `.scss`、`.sass` 和 `.css`）
- **Solidity**（实验性；需要 Node.js 和 npm；自动安装 `@nomicfoundation/solidity-language-server`；最好在项目根目录放置 `foundry.toml` 或 `hardhat.config.js`）
- **Svelte**（需要 Node.js v18+ 和 npm；通过 `svelte-language-server` 支持 `.svelte` 单文件组件以及 TypeScript/JavaScript 文件；自动生成配套的 `typescript-language-server` + `typescript-svelte-plugin`，用于跨 `.ts`/`.js` 和 `.svelte` 文件的跨文件重命名、跳转定义和引用；Svelte 项目使用语言 `svelte`，无需再启用 `typescript`）
- **Swift**
- **TypeScript**
- **Vue**（3.x 配合 TypeScript；需要 Node.js v18+ 和 npm；支持 .vue 单文件组件，具备单体仓库检测能力）
- **YAML**
- **JSON**（实验性；必须显式添加到语言列表中；需要 Node.js 和 npm）
- **Zig**（需要安装 ZLS——Zig 语言服务器）

更多语言的支持可以通过为新语言服务器实现提供浅层适配器来轻松添加，参见 Serena 的[相关记忆](https://github.com/oraios/serena/blob/main/.serena/memories/adding_new_language_support_guide.md)。

## Serena JetBrains 插件

[Serena JetBrains 插件](https://plugins.jetbrains.com/plugin/28946-serena/)利用了 JetBrains IDE 强大的代码分析能力。该插件天然支持 JetBrains IDE 所支持的所有编程语言和框架。

使用插件时，Serena 通过插件连接到你正在运行的 JetBrains IDE 实例。对于已经在使用 JetBrains IDE 的用户来说，这意味着 Serena 无缝集成到你通常已经打开的 IDE 实例中，除插件本身外无需额外的设置或配置。

- 参见 [JetBrains 插件文档](../02-usage/025_jetbrains_plugin)了解其优势和用法细节的高层概述。
- 参见[功能特性](025_features)部分，了解 JetBrains 插件与语言服务器的详细能力对比。
