Serena 提供了广泛的工具，用于高效的代码检索、编辑和重构，以及一个面向长期代理工作流的记忆系统。

鉴于其广泛的功能范围，Serena 通过多层配置系统来适应你的需求。

## 检索

Serena 的检索工具允许代理在符号层面探索代码库，理解结构和关系，而无需阅读整个文件。

| 能力 | 语言服务器 | JetBrains 插件 |
|------|-----------|----------------|
| 查找符号 | 支持 | 支持 |
| 符号概览（文件大纲） | 支持 | 支持 |
| 查找引用符号 | 支持 | 支持 |
| 搜索项目依赖 | -- | 支持 |
| 类型层次结构 | -- | 支持 |
| 查找声明 | 支持* | 支持 |
| 查找实现 | 支持** | 支持 |
| 查询外部项目 | 支持 | 支持 |
| 诊断/检查 | 支持 | 支持 |

\*：通常无法在外部依赖中查找声明。

\**：仅部分语言可用，受语言服务器功能限制。

## 重构

没有精确的重构工具，代理只能诉诸不可靠且代价高昂的搜索替换操作。

| 能力 | 语言服务器 | JetBrains 插件 |
|------|-----------|----------------|
| 重命名 | 支持（仅符号） | 支持（符号、文件、目录） |
| 移动（符号、文件、目录） | -- | 支持 |
| 内联 | -- | 支持 |
| 传播删除（移除未使用的代码） | -- | 支持 |

## 符号化编辑

Serena 的符号化编辑工具比典型的替代方案更不容易出错，且 token 效率更高。

| 能力 | 语言服务器 | JetBrains 插件 |
|------|-----------|----------------|
| 替换符号体 | 支持 | 支持 |
| 在符号之后插入 | 支持 | 支持 |
| 在符号之前插入 | 支持 | 支持 |
| 安全删除 | 支持 | 支持 |

## 交互式调试

Serena 支持一个高度通用的调试工具（JetBrains 插件专属），允许代理设置断点、检查变量、求值表达式以及通过持久化的 REPL 风格界面控制执行流程。

## 基础功能

除了语义化能力之外，Serena 还包含一组基础工具以保持完整性。当 Serena 在诸如 Claude Code 或 Codex 等代理框架中使用时，这些工具通常默认禁用，因为外围框架已经提供了重叠的文件、搜索和 Shell 能力。

- **`search_for_pattern`**——跨代码库的灵活正则搜索
- **`replace_content`**——面向代理优化的基于正则和文本字面量的替换
- **`list_dir` / `find_file`**——目录列出和文件搜索
- **`read_file`**——读取文件或文件片段
- **`execute_shell_command`**——运行 Shell 命令（如构建、测试、代码检查）

## 记忆管理

记忆系统是长期代理工作流的基础要素，尤其是当知识需要跨会话、跨用户和跨项目共享时。尽管设计简洁，但我们收到了许多用户的积极反馈，他们倾向于将 Serena 的记忆管理系统与代理的内部系统（如 `AGENTS.md` 文件）结合使用。如果你更倾向于使用其他方案，也可以轻松禁用它。

## 可配置性

Serena 的活动工具、工具描述、提示词、语言后端细节以及许多其他方面，都可以通过简单调整几行 YAML 来根据具体情况灵活配置。为此，Serena 提供了多个层级的（可组合的）配置：

- 全局配置
- MCP 启动命令（CLI）配置
- 逐项目配置（带本地覆盖）
- 特定执行上下文的配置（如针对特定客户端）
- 动态可组合的配置片段（模式）
