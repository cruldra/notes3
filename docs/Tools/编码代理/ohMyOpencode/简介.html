简介

1. 项目定位：打破单一模型锁定的“多代理开发团队”​

Oh My OpenCode 不是单纯的“强化版 Claude Code”，而是一个多模型代理编排框架（Multi-model agent orchestration harness）。它的核心理念是不绑定任何单一的 AI 供应商（如 Anthropic 或 OpenAI），而是将单个 AI 代理转化为一支各司其职的“虚拟开发团队”。它会让 Claude 负责编排、GPT 负责深度推理、Gemini 负责前端视觉、Haiku 负责简单快速的任务，实现自动化的完美协作。

2. 核心工作模式​

系统为开发者提供了两种截然不同的交互方式：

  • Ultrawork 模式（“懒人”全自动模式）： 只需要输入 ultrawork（或 ulw），AI 就会全权接管。它会自动探索代码库、研究模式、实现功能并运行诊断验证，直到任务完全结束。
  • Prometheus 模式（精确控制与规划模式）： 按下 Tab 键进入。此时 AI 会像一位资深工程师一样对你进行“访谈”，提出澄清问题，明确需求边界和边缘情况，制定出详细的开发计划。随后通过 /start-work 让执行代理（Atlas）接手，适合跨日期的复杂项目、重大重构或需要留下决策记录的场景。

3. “开发团队”的代理角色设定​

系统不是让一个模型包揽一切，而是设计了不同的专门角色：

  • Sisyphus（西西弗斯 - 主编排器）：团队的主管。它永不停歇地进行计划、委派任务给专家代理，并积极推动并行执行。最适合搭配 Claude 系列（如 Opus/Sonnet 4.6）或 Kimi K2.5 等擅长指令遵循的模型。
  • Hephaestus（赫菲斯托斯 - 纯粹工匠）：一个基于 GPT-5.3 Codex 的原生代理。当遇到需要深度架构推理、跨多文件复杂调试时，他能发挥极大优势。
  • Prometheus（普罗米修斯）：战略规划师，负责挖掘用户意图并制定严谨计划。
  • Atlas（阿特拉斯）：指挥官，负责执行 Prometheus 制定好的计划，分配任务并独立验证结果。
  • Oracle（神谕）：高智商的只读架构顾问，专门应对陌生的设计模式、安全顾虑或系统级权衡。
  • 辅助阵容：如 Metis（查漏补缺）、Momus（严厉的审查员）、Explore（快速代码检索）、Librarian（文档与开源库研究）等。

4. 智能的模型动态路由 (Agent Model Matching)​

Sisyphus 在分配任务时，并不是硬编码某个模型，而是按任务类别委派：

  • 视觉工程 (visual-engineering)：自动路由给 Gemini 3 Pro。
  • 超级大脑 (ultrabrain)：路由给 GPT-5.3 Codex 进行深度推演。
  • 敏捷任务 (quick)：分配给 Claude Haiku 等廉价极速模型。 系统不仅默认配置了最佳匹配，还支持回退机制（如果首选 API 挂了，自动寻找备用），并允许用户在 JSON 中完全自定义模型和分类组合。

5. 相比于纯 Claude Code 的绝对优势​

文档在最后指出了 Oh My OpenCode 架构的几个碾压级优势：

  • 并行执行 (Parallel execution)：单一模型只能单线思考，而 Oh My OpenCode 可以同时唤醒 5 个后台代理进行代码编写、模式研究和文档查阅，效率成倍提升。
  • 哈希锚定编辑 (Hash-anchored edits)：通过特定的行号与内容哈希校验代码修改，极大避免了 AI 修改代码时“找错行”或“覆盖错误”的问题。
  • 意图网关 (Intent Gate)：在行动前先分析用户的真实意图（是想做研究、写代码、排查还是修复？），而不是拿到提示词就盲目开始。
  • LSP + AST 工具链：具备真正的 IDE 级精度，支持工作区重命名、跳转定义、查找引用以及 AST 感知的代码重写。
  • 纪律强制器 (Discipline enforcement)：内置防偷懒机制，清理 AI 产生的冗余注释（AI slop），并强制代理一直工作直到达到 100% 的完成度。

总结来说，这篇文档展示了 Oh My OpenCode 如何通过“多模型协同编排”和“工程化的纪律约束”，真正落实其宣言中“消除人类干预、交付生产级代码”的终极目标。
