OpenCLI 子命令速览

OpenCLI 把任意网站、桌面应用或第三方 CLI 包装成统一的 opencli <site> <command> 入口，方便人和 Agent 调用。本文按用途分组列出所有顶层子命令的核心用法。

命令分类​

分组子命令解决什么问题
发现与诊断list、validate、verify、convention-audit、doctor看有什么、是否能用
浏览器驱动browser <子命令>没现成适配器时手动操作页面
适配器管理adapter status / eject / reset修改或还原内置站点适配器
配置与运行时profile、daemon、completionChrome 多账号、守护进程、补全脚本
第三方扩展plugin <子命令>从 Git 仓库安装别人写的扩展
外部 CLI 透传external install / register / list把 gh、docker 这类工具纳入统一入口
站点适配器调用opencli <site> <cmd>直接执行一个具体任务

发现与诊断​

opencli list​

列出所有可用命令，按站点分组。

opencli list                 # 表格输出，人看
opencli list -f json         # 结构化，给 Agent
opencli list | grep twitter  # 找某个站点

-f 支持 table、json、yaml、md、csv。Agent 应当总是用 -f json，里面带 site / name / strategy / browser / args / columns 等字段，比读文档可靠。

opencli validate [target]​

只读检查适配器定义是否合法：缺少描述、字段名重复、func / pipeline 三选一是否满足，等等。不联网、不开浏览器。

opencli validate                 # 全量
opencli validate bilibili        # 单站点
opencli validate bilibili/feed   # 单命令

opencli verify [target] [--smoke]​

validate 的超集：先做语义检查，再用合成参数跑一次 func。--smoke 启用真实 smoke 测试。

opencli verify hackernews/top --smoke

opencli convention-audit [target]​

检查适配器是否符合 agent-native 输出规范（columns 与返回对象 key 一一对齐、列顺序、错误返回方式等）。

opencli convention-audit --site bilibili --strict

--strict 让它在发现问题时返回非零退出码，可放进 CI。

opencli doctor​

诊断浏览器桥：daemon 是否在跑、Chrome 扩展是否连上、版本是否匹配。

opencli doctor              # 完整检查
opencli doctor --no-live    # 跳过 live 测试
opencli doctor --sessions   # 列出活跃自动化会话

只有 cookie / header / intercept / ui 策略的命令需要它绿灯。public 和 local 不依赖。

浏览器驱动 opencli browser​

没有现成适配器、需要边走边看时用。所有子命令都打到一个 Chrome 自动化窗口。

导航与观察​

opencli browser open https://example.com   # 打开 URL
opencli browser back                       # 后退
opencli browser scroll down                # 滚动
opencli browser state                      # 当前 URL/标题/可交互元素 [N]
opencli browser get url                    # 单独取页面属性
opencli browser screenshot /tmp/x.png      # 截图
opencli browser console                    # 最近控制台消息

state 给出的 [N] 索引是后续 click N / type N 的参照系。

交互​

opencli browser click 3                            # 点 [3]
opencli browser type 'input[name=q]' "搜索词"       # 选择器 + 文本
opencli browser select '#country' "China"          # 下拉选项
opencli browser keys Enter                         # 按键
opencli browser dialog accept                      # 处理 alert/confirm

抓数据​

opencli browser find 'a.title'             # CSS 选择器，返回 {matches_n, entries[]}
opencli browser extract                    # 一把抓页面常见结构
opencli browser eval "document.title"      # 任意 JS
opencli browser network                    # 列出捕获到的 JSON API 请求
opencli browser network --detail 0         # 看第 0 条的完整响应体

写新适配器前，靠 network 和 eval 摸清接口，再落到 TS 文件里。

多标签 / 工作区​

opencli browser tab list / create / close
opencli browser bind --workspace bound:work     # 绑定到当前真实标签
opencli browser unbind --workspace bound:work   # 解绑（不关页面）

--workspace 让多个流程并行而不互相踩；bound:* 则是把自动化挂到用户已经登录好的标签上。

适配器开发辅助​

opencli browser init bilibili/myfeed     # 生成适配器骨架
opencli browser verify bilibili/myfeed   # 端到端 smoke
opencli browser analyze https://x.com    # 反爬识别 + 推荐策略
opencli browser wait 'css' '.loaded'     # 等元素出现
opencli browser frames                   # 列出 iframe target

适配器管理 opencli adapter​

opencli adapter status            # 哪些站点用了本地覆盖
opencli adapter eject bilibili    # 把官方适配器拷到 ~/.opencli/clis/ 改
opencli adapter reset bilibili    # 还原成官方版本

适合"我想微调官方某个命令但不污染主仓"的场景。

配置与运行时​

opencli profile — Chrome 多账号​

opencli profile list                       # 已连扩展的 Chrome profiles
opencli profile rename ctx-7f3a personal   # 起别名
opencli profile use personal               # 设默认

随后 opencli --profile personal <site> <cmd> 即指定走哪个账号。

opencli daemon — 守护进程​

opencli daemon status / stop / restart

正常不用动；浏览器桥卡住时 restart 一下。

opencli completion <shell>​

输出 bash / zsh / fish 的补全脚本到 stdout。按各自 shell 的方式 source 即可。

第三方扩展 opencli plugin​

从 Git 仓库安装别人写的扩展，独立于内置适配器。

opencli plugin install github:user/repo
opencli plugin list -f json
opencli plugin update [name] / --all
opencli plugin uninstall <name>
opencli plugin create my-plugin   # 生成脚手架

外部 CLI 透传 opencli external​

把已存在的命令行工具（gh、docker、自家脚本）登记到 OpenCLI 入口，stdio 直通、退出码原样传递。

opencli external install gh         # 内置清单里查表，自动 brew/apt/npm 装

opencli external register my-tool \
    --binary my-tool \
    --install "npm i -g my-tool" \
    --desc "我的内部 CLI"

opencli external list               # 看注册了哪些
opencli gh pr list --limit 5        # 透传调用

内置清单：src/external-clis.yaml；用户清单：~/.opencli/external-clis.yaml。

透传 ≠ 适配器：输出原样返回，不会变成结构化 JSON。需要结构化就写一个 local 策略的适配器去包装。

站点适配器调用 opencli <site> <cmd>​

最终用户的主要路径。每个命令都支持两个通用 flag：

flag作用
-f, --formattable(TTY 默认) / yaml(非 TTY 默认) / json / plain / md / csv
-v, --verbose调试日志，失败时打印堆栈
opencli <site> --help              # 看一个站点的命令清单
opencli <site> <cmd> --help        # 看命令的位置参数和专属 flag
opencli hackernews top --limit 10 -f json
opencli twitter user-tweets jack --tab tweets

每条适配器命令带一个 strategy 标签（PUBLIC / COOKIE / HEADER / INTERCEPT / UI / LOCAL）。PUBLIC 走纯 HTTP 不开浏览器，其他几类需要 Chrome + Browser Bridge 扩展登录目标站。

失败时的自修复​

适配器命令报错时，CLI 会提示 # AutoFix: re-run with OPENCLI_DIAGNOSTIC=1 ...。照做后会得到一份结构化 RepairContext（含 adapter.sourcePath），按里头的提示改适配器、重试。最多 3 轮。

常用环境变量​

变量默认用途
OPENCLI_DAEMON_PORT19825daemon ↔ 扩展端口
OPENCLI_BROWSER_COMMAND_TIMEOUT60单条命令超时（秒）
OPENCLI_CDP_ENDPOINT—接外部 Chrome / Electron 时手动指定 CDP
OPENCLI_CACHE_DIR~/.opencli/cache网络抓包和页面快照缓存
OPENCLI_VERBOSEfalse与 -v 等价
OPENCLI_DIAGNOSTICfalse失败时输出 RepairContext JSON

速查口诀​

  • 不知道有什么 → opencli list -f json
  • 写新命令之前 → opencli browser open + opencli browser network
  • 调通了之后 → opencli verify --smoke
  • 浏览器命令报错 → opencli doctor
  • 多账号 → opencli profile use <name>
  • 想把现成 CLI 也一起管理 → opencli external register
