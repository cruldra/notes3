# 配置详解

OMP 提供丰富的配置选项，让你根据项目需求和个人偏好定制体验。

## 配置文件位置

### 用户级配置

```
~/.omp/
├── settings.json          # 主配置文件
├── config.json            # 别名
├── lsp.json              # LSP 配置
├── mcp.json              # MCP 服务器配置
├── ssh.json              # SSH 主机配置
├── agent/
│   ├── commands/         # 自定义命令
│   ├── agents/           # 自定义代理
│   ├── modules/          # Python 模块
│   └── skills/           # 自定义技能
└── logs/                 # 日志文件
```

### 项目级配置

```
project/
├── .omp/
│   ├── config.json       # 项目配置
│   ├── lsp.json          # 项目 LSP 配置
│   ├── mcp.json          # 项目 MCP 配置
│   ├── ssh.json          # 项目 SSH 配置
│   ├── rules/            # TTSR 规则
│   ├── commands/         # 项目命令
│   └── agents/           # 项目代理
└── src/
```

## 项目上下文文件

### AGENTS.md

项目级 AI 指导文件：

```markdown
# 项目规范

## 技术栈
- React 18 + TypeScript
- Tailwind CSS
- Vite

## 代码规范
1. 使用函数组件 + Hooks
2. Props 类型必须显式定义
3. 组件文件使用 PascalCase
4. 工具函数使用 camelCase

## 文件组织
```
src/
├── components/     # UI 组件
├── hooks/         # 自定义 hooks
├── utils/         # 工具函数
├── types/         # 类型定义
└── services/      # API 服务
```

## 注意事项
- 不要直接修改 node_modules
- 所有 API 调用通过 services/
- 错误处理使用统一的 ErrorBoundary
```

## 配置命令

### 查看配置

```bash
# 列出所有配置
omp config list

# 获取特定配置
omp config get models.roles.default.model
# 输出: claude-3-5-sonnet-20241022

# 查看配置路径
omp config path
# 输出: ~/.omp/settings.json
```

### 修改配置

```bash
# 设置配置值
omp config set display.theme "dracula"

# 重置为默认值
omp config reset display.theme

# 重置所有配置
omp config reset --all
```

### 配置验证

```bash
omp config validate

检查配置文件...
✓ JSON 格式正确
✓ 所有必需字段存在
⚠️ 警告: models.smol 未配置
✓ 没有错误
```

## 配置优先级

配置按以下顺序加载，后加载的覆盖前者：

```
1. 内置默认配置
2. ~/.omp/settings.json (用户级)
3. .omp/config.json (项目级)
4. 环境变量
5. 命令行参数
```

示例：

```bash
# 配置文件中设置
display.theme = "catppuccin"

# 环境变量覆盖
export OMP_THEME="dracula"

# 命令行参数最终覆盖
omp --theme "tokyo-night"
```

## 故障排除配置

### 问题：配置不生效

**检查：**

```bash
# 确认配置文件路径
omp config path

# 验证 JSON 格式
omp config validate

# 查看实际生效的配置
omp config dump
```

### 问题：环境变量不生效

**检查：**

```bash
# 确认变量已设置
echo $OPENAI_API_KEY

# 检查拼写
# 正确: OPENAI_API_KEY
# 错误: OPENAI_API-KEY, openai_api_key

# 重新加载 shell
source ~/.bashrc
```

### 问题：项目级配置被忽略

**检查：**

```bash
# 确认文件存在
ls -la .omp/config.json

# 检查权限
chmod 644 .omp/config.json

# 验证 JSON 格式
jsonlint .omp/config.json
```
