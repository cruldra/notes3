# 安装指南

## 系统要求

### 必需条件
- **Node.js**: 18.0.0 或更高版本
- **Bun**: 1.0.0 或更高版本（推荐运行时）
- **Git**: 用于版本控制集成

### 可选依赖
- **Python**: 3.8+（如需使用 Python 工具）
- **Rust**: 如需从源码构建

### 支持的平台
| 平台 | 架构 | 状态 |
|------|------|------|
| Linux | x64, arm64 | ✅ 完全支持 |
| macOS | x64, arm64 | ✅ 完全支持 |
| Windows | x64 | ✅ 完全支持 |

## 快速安装

### 使用 npm 安装

```bash
npm install -g @oh-my-pi/pi-coding-agent
```

### 使用 Bun 安装（推荐）

```bash
bun install -g @oh-my-pi/pi-coding-agent
```

### 使用 npx 运行（无需安装）

```bash
npx @oh-my-pi/pi-coding-agent
```

## 从源码构建

### 1. 克隆仓库

```bash
git clone https://github.com/can1357/oh-my-pi.git
cd oh-my-pi
```

### 2. 安装依赖

```bash
# 安装 Bun（如果尚未安装）
curl -fsSL https://bun.sh/install | bash

# 安装 Node.js 依赖
bun install
```

### 3. 构建 Rust 原生模块

```bash
# 需要 Rust 工具链
rustup target add wasm32-unknown-unknown

# 构建
bun run build:rust
```

### 4. 编译项目

```bash
bun run build
```

### 5. 链接到全局

```bash
bun link
```

## 安装后配置

### 1. 运行诊断检查

```bash
omp doctor
```

此命令会检查：
- Node.js/Bun 版本
- 原生模块加载
- API 密钥配置
- 可选依赖状态

### 2. 安装可选组件

#### Python 支持

```bash
omp setup python
```

这将安装：
- IPython 内核
- Jupyter 支持
- 常用的数据科学库

#### 浏览器自动化

```bash
omp setup browser
```

这将安装 Puppeteer 和必要的 Chromium 依赖。

### 3. 配置 API 密钥

#### 方式一：交互式配置

```bash
omp config
```

使用交互式向导配置：
- 默认 AI 提供商
- API 密钥
- 模型选择

#### 方式二：环境变量

```bash
# OpenAI
export OPENAI_API_KEY="sk-..."

# Anthropic
export ANTHROPIC_API_KEY="sk-ant-..."

# Google (Gemini)
export GOOGLE_API_KEY="..."

# OpenRouter
export OPENROUTER_API_KEY="..."
```

#### 方式三：配置文件

编辑 `~/.omp/settings.json`：

```json
{
  "providers": {
    "openai": {
      "apiKey": "sk-...",
      "model": "gpt-4o"
    },
    "anthropic": {
      "apiKey": "sk-ant-...",
      "model": "claude-3-5-sonnet-20241022"
    }
  }
}
```

## 故障排除

### 安装失败

#### 问题：原生模块编译失败

**解决方案：**

```bash
# 确保 Rust 已安装
rustc --version

# 如果未安装
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 重新构建
bun run build:rust
```

#### 问题：权限错误

**解决方案：**

```bash
# 使用 sudo（Linux/macOS）
sudo npm install -g @oh-my-pi/pi-coding-agent

# 或者更改 npm 前缀
npm config set prefix '~/.local'
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
```

#### 问题：Bun 找不到

**解决方案：**

```bash
# 确保 Bun 在 PATH 中
export PATH="$HOME/.bun/bin:$PATH"

# 添加到 shell 配置文件
echo 'export PATH="$HOME/.bun/bin:$PATH"' >> ~/.bashrc
```

### 运行时问题

#### 问题：API 密钥未找到

**检查步骤：**

1. 确认环境变量已设置
2. 检查配置文件路径：`~/.omp/settings.json`
3. 验证密钥格式（不要有多余空格）

#### 问题：LSP 无法启动

**解决方案：**

```bash
# 检查语言服务器是否安装
which typescript-language-server

# 手动安装（以 TypeScript 为例）
npm install -g typescript-language-server

# 在 OMP 中重新加载
/reload
```
