# 快速开始

本指南将帮助你在 15 分钟内掌握 Oh My Pi 的基本使用方法。

## 启动 OMP

在你的项目目录中打开终端，运行：

```bash
omp
```

你将看到 OMP 的欢迎界面，包括：
- Logo 和版本信息
- 快捷键提示
- 最近的会话列表

## 第一次对话

### 基础提问

```
> 你好，请帮我解释这个项目的结构
```

OMP 会自动读取当前目录的文件结构并提供分析。

### 使用 @ 引用文件

```
> @package.json 这个项目的依赖有哪些？
> @src/main.ts 解释这段代码的功能
```

在提示中使用 `@` 符号可以快速引用文件内容。

## 核心功能体验

### 1. 代码编辑

让 OMP 帮你修改代码：

```
> 在 src/utils.ts 中添加一个计算斐波那契数列的函数
```

OMP 会：
1. 读取目标文件
2. 理解现有代码结构
3. 生成符合规范的代码
4. 应用更改

### 2. 使用工具

#### Bash 工具

```
> 运行测试
```

OMP 会自动执行 `npm test` 或 `pytest` 等命令。

#### 文件操作

```
> 查找所有包含 "TODO" 的文件
```

OMP 会使用 grep 工具搜索项目。

#### 代码搜索

```
> 搜索项目中所有使用了 useState 的地方
```

### 3. 斜杠命令

#### 常用命令

```
/commit    # AI 生成提交信息并提交
/agents    # 管理子代理
/models    # 切换模型
/config    # 查看/修改配置
/help      # 显示帮助
```

#### 使用示例

```
> /commit
```

OMP 会分析你的变更，生成符合 Conventional Commits 规范的提交信息。

## 实战示例

### 示例 1：修复 Bug

假设你有一个 JavaScript 错误：

```
> 我遇到了这个错误：TypeError: Cannot read property 'name' of undefined
   at src/components/UserCard.tsx:15
```

OMP 会：
1. 读取错误堆栈指定的文件
2. 分析问题原因
3. 提供修复方案
4. 应用修复（如果你同意）

### 示例 2：重构代码

```
> 将 src/old-code.js 重构成 TypeScript，使用现代 ES6+ 语法
```

OMP 会：
1. 分析现有代码
2. 创建新的 .ts 文件
3. 添加类型注解
4. 更新导入/导出语法
5. 确保功能不变

### 示例 3：添加测试

```
> 为 src/calculator.js 编写单元测试
```

OMP 会：
1. 识别测试框架（Jest、Mocha、Vitest 等）
2. 分析代码的边界情况
3. 生成全面的测试用例
4. 创建测试文件

## 效率技巧

### 键盘快捷键

| 快捷键 | 功能 |
|--------|------|
| `Tab` | 自动补全 |
| `Ctrl+C` | 取消当前操作 |
| `Ctrl+D` | 退出 OMP |
| `Ctrl+R` | 搜索历史记录 |
| `Ctrl+T` | 切换 Todo 面板 |
| `↑/↓` | 浏览历史 |
| `?` | 显示快捷键帮助 |

### 提示词技巧

#### 明确具体

✅ **好**："将 src/auth.js 中的 login 函数重命名为 authenticateUser"
❌ **差**："改一下登录相关的代码"

#### 提供上下文

```
> @src/database.ts @src/models/user.ts 
  为 User 模型添加 email 验证方法
```

#### 分步骤请求

对于复杂任务，分解为多个步骤：

```
> 第一步：读取 src/app.js 了解当前路由结构
> 第二步：在 routes 目录下创建新的 API 路由文件
> 第三步：在 app.js 中注册新路由
```

### 使用子代理

对于大型任务，使用子代理并行处理：

```
> 使用子代理并行分析：
  1. 检查 src/components 目录的组件结构
  2. 分析 src/utils 中的工具函数
  3. 查看测试覆盖率
```

## 会话管理

### 保存和恢复会话

```
> /save my-feature-work
```

稍后恢复：

```bash
omp --resume my-feature-work
```

### 分支会话

```
> /branch experiment-approach-b
```

创建一个独立的会话分支，尝试不同的实现方案。

## 最佳实践

### 1. 从小任务开始

初次使用 OMP 时，先尝试简单的代码修改和查询，逐步熟悉其能力边界。

### 2. 审查所有更改

尽管 OMP 很智能，但你应该：
- 查看 diff 后再确认应用
- 运行测试验证功能
- 理解修改的逻辑

### 3. 使用版本控制

始终在 Git 仓库中使用 OMP：

```bash
git add .
git commit -m "Before OMP changes"
omp
```

这样可以在需要时回滚。

### 4. 善用配置

创建项目级配置文件 `.omp/config.json`：

```json
{
  "rules": [
    "使用 TypeScript 严格模式",
    "遵循项目现有的代码风格",
    "所有新函数必须包含 JSDoc 注释"
  ]
}
```
