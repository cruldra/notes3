模型角色系统

OMP 的模型角色系统允许为不同任务配置专门的 AI 模型，实现成本与质量的灵活平衡。

核心概念​

什么是模型角色？​

不同任务需要不同的模型能力：

快速查询 → 小型模型（快、便宜）
代码生成 → 中型模型（平衡）
架构设计 → 大型模型（强、贵）

角色系统让你自动路由到最适合的模型。

内置角色​

角色用途默认模型类型
default一般任务平衡型
smol快速查询轻量型
slow高质量输出强大型
plan任务规划推理型
commit提交信息生成轻量型

配置角色​

方式一：环境变量​

# 设置各角色使用的模型
export PI_DEFAULT_MODEL=anthropic:claude-3-5-sonnet-20241022
export PI_SMOL_MODEL=openai:gpt-4o-mini
export PI_SLOW_MODEL=anthropic:claude-3-opus-20240229
export PI_PLAN_MODEL=google:gemini-1.5-pro
export PI_COMMIT_MODEL=openai:gpt-4o-mini

方式二：配置文件​

// ~/.omp/settings.json
{
  "models": {
    "roles": {
      "default": {
        "provider": "anthropic",
        "model": "claude-3-5-sonnet-20241022",
        "temperature": 0.7
      },
      "smol": {
        "provider": "openai",
        "model": "gpt-4o-mini",
        "temperature": 0.5
      },
      "slow": {
        "provider": "anthropic",
        "model": "claude-3-opus-20240229",
        "temperature": 0.3
      },
      "plan": {
        "provider": "google",
        "model": "gemini-1.5-pro",
        "temperature": 0.2
      },
      "commit": {
        "provider": "openai",
        "model": "gpt-4o-mini"
      }
    }
  }
}

使用角色​

命令行参数​

# 使用 smol 模式（快速/便宜）
omp --smol

# 使用 slow 模式（高质量）
omp --slow

# 使用 plan 模式（规划任务）
omp --plan

会话中切换​

> /models use smol
切换到 smol 角色 (gpt-4o-mini)

> /models use default
切换回 default 角色 (claude-3-5-sonnet)

子代理中指定​

> 创建子代理：
  类型: explore
  模型: pi/smol  # 使用 smol 角色
  任务: 快速扫描项目结构

> 创建子代理：
  类型: reviewer
  模型: pi/slow  # 使用 slow 角色
  任务: 深度代码审查

自动角色选择​

智能路由​

OMP 根据任务类型自动选择角色：

用户: "你好"
→ 路由到 default

用户: "快速总结一下这个文件"
→ 路由到 smol

用户: "设计一个微服务架构"
→ 路由到 slow

用户: "/commit"
→ 路由到 commit

用户: "/plan 实现用户系统"
→ 路由到 plan

自定义路由规则​

// ~/.omp/settings.json
{
  "models": {
    "routing": {
      "rules": [
        {
          "pattern": "^/(commit|fix|typo)",
          "role": "smol"
        },
        {
          "pattern": "(架构|设计|重构)",
          "role": "slow"
        },
        {
          "pattern": "^/plan",
          "role": "plan"
        }
      ]
    }
  }
}

支持的提供商​

Anthropic​

{
  "provider": "anthropic",
  "model": "claude-3-5-sonnet-20241022",
  "apiKey": "${ANTHROPIC_API_KEY}"
}

可用模型：

  • claude-3-opus-20240229 - 最强能力
  • claude-3-5-sonnet-20241022 - 平衡选择
  • claude-3-5-haiku-20241022 - 快速经济

OpenAI​

{
  "provider": "openai",
  "model": "gpt-4o",
  "apiKey": "${OPENAI_API_KEY}"
}

可用模型：

  • gpt-4o - 旗舰模型
  • gpt-4o-mini - 经济高效
  • o1-preview - 推理专用

Google​

{
  "provider": "google",
  "model": "gemini-1.5-pro",
  "apiKey": "${GOOGLE_API_KEY}"
}

可用模型：

  • gemini-1.5-pro - 长上下文
  • gemini-1.5-flash - 快速响应

OpenRouter​

{
  "provider": "openrouter",
  "model": "anthropic/claude-3.5-sonnet",
  "apiKey": "${OPENROUTER_API_KEY}"
}

访问多个提供商的模型。

Cursor​

{
  "provider": "cursor",
  "auth": "oauth"  // 浏览器 OAuth 登录
}

使用 Cursor Pro 订阅。

采样参数​

        "temperature": 0.7,

扩展推理 (Extended Thinking)​

Anthropic 模型的深度推理模式：

{
  "models": {
    "roles": {
      "slow": {
        "provider": "anthropic",
        "model": "claude-3-5-sonnet-20241022",
        "extendedThinking": {
          "enabled": true,
          "budgetTokens": 16000
        }
      }
    }
  }
}

多凭证支持​

{
  "models": {
    "credentials": [
      {
        "provider": "openai",
        "apiKey": "sk-...-key-1"
      },

故障排除​

问题：模型无响应​

检查：

> /models test default

测试 default 角色...
❌ 失败: API key 无效

检查环境变量：
  ANTHROPIC_API_KEY: 未设置

问题：角色切换无效​

检查配置优先级：

配置加载顺序（后加载覆盖前者）：
1. 默认配置
2. ~/.omp/settings.json
3. .omp/config.json (项目级)
4. 环境变量
5. 命令行参数

问题：成本过高​

优化建议：

1. 更多使用 smol 角色
2. 启用缓存
3. 设置用量警报
4. 定期审查 stats

💡 提示：合理使用角色可以在保持质量的同时显著降低成本！
