浏览器自动化工具

OMP 集成了 Puppeteer 浏览器自动化，配备 14 种反检测脚本，支持复杂的网页交互和数据提取。

核心特性​

14 种 Stealth 插件​

插件功能
chrome-runtime修复 Chrome Runtime 指纹
iframe-content-window隐藏 iframe 自动化标记
media-codec模拟真实编解码器支持
navigator-languages设置真实语言列表
navigator-permissions模拟权限 API
navigator-plugins添加真实插件列表
navigator-vendor设置正确的 vendor
navigator-webdriver移除 webdriver 标记
sourceurl隐藏源代码 URL
user-agent-override自定义 User Agent
webgl修改 WebGL 指纹
window-outerdimensions设置窗口尺寸
chrome-csi模拟 Chrome CSI API
chrome-load-times模拟加载时间 API

User Agent 模拟​

自动移除 HeadlessChrome 标识：

原始: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 
      (KHTML, like Gecko) HeadlessChrome/120.0.0.0 Safari/537.36

模拟: Mozilla/5.0 (X11; Linux x86_64) AppleWebKit/537.36 
      (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36

示例 2：数据抓取​

> 打开 https://example.com/products
> 向下滚动到页面底部
> 提取所有产品价格：

执行 JS:
const prices = document.querySelectorAll('.price');
return Array.from(prices).map(p => ({
  product: p.closest('.product').querySelector('h2').textContent,
  price: p.textContent
}));

结果:
[
  {"product": "产品A", "price": "¥199"},
  {"product": "产品B", "price": "¥299"},
  {"product": "产品C", "price": "¥399"}
]

模式切换​

无头模式 / 可见模式​

> /browser headless false

切换到可见模式，浏览器窗口将显示。

> /browser headless true

切换回无头模式。

配置默认模式​

// ~/.omp/settings.json
{
  "browser": {
    "headless": true,
    "viewport": {
      "width": 1920,
      "height": 1080
    },
    "userAgent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64)...",
    "stealth": true
  }
}

等待策略​

显式等待​

> 等待 3 秒
> 等待元素 "#loading" 消失
> 等待元素 ".result" 出现

网络等待​

> 等待网络空闲
> 等待导航完成

故障排除​

问题：被检测为机器人​

加强 stealth：

{
  "browser": {
    "stealth": true,
    "extraPlugins": ["user-agent-override", "webgl"],
    "viewport": {
      "width": 1920,
      "height": 1080
    }
  }
}

添加延迟：

> 随机延迟 1-3 秒
> 模拟人类鼠标移动

问题：元素找不到​

使用无障碍树：

> 获取无障碍快照
> 使用显示的 ID 交互

等待元素：

> 等待 ".dynamic-content" 出现
> 超时 10 秒

问题：页面加载超时​

{
  "browser": {
    "navigationTimeout": 60000,
    "waitUntil": "networkidle2"
  }
}

安全注意事项​

⚠️ 重要提醒：

  1. 不要在公共网站滥用：遵守网站的 robots.txt 和服务条款
  2. 保护敏感信息：不要记录密码或 API 密钥
  3. 遵守法律：仅用于合法的数据抓取和测试目的
  4. 注意频率：避免对同一网站频繁请求
