LSP 集成（语言服务器协议）

OMP 提供完整的 IDE 级代码智能，支持 40+ 种编程语言，通过 Language Server Protocol (LSP) 实现。

核心功能​

11 种 LSP 操作​

操作描述使用场景
diagnostics诊断检查代码错误、警告
definition跳转到定义查看函数/变量定义
type_definition类型定义查看类型别名定义
implementation实现跳转查看接口实现
references查找引用查找所有使用位置
hover悬停信息查看文档和类型
symbols文档符号文件大纲、导航
rename重命名安全重命名符号
code_actions代码操作快速修复、重构
status服务器状态检查 LSP 健康
reload重新加载重启语言服务器

自动格式化​

保存文件时自动格式化：

> 格式化 src/utils.ts

OMP 使用：
- TypeScript → prettier / tsfmt
- Rust → rustfmt
- Go → gofmt
- Python → black / autopep8

实时诊断​

编辑后立即显示错误：

// 输入时
function greet(name: string) {
  return "Hello " + name;
}

greet(123);  // ❌ Error: Argument of type 'number' is not assignable to 'string'

配置语言服务器​

在 ~/.omp/lsp.json 中配置：

{
  "servers": {
    "typescript": {
      "command": ["typescript-language-server", "--stdio"],
      "filetypes": ["typescript", "javascript", "typescriptreact", "javascriptreact"],
      "rootPatterns": ["package.json", "tsconfig.json"],
      "settings": {
        "typescript": {
          "format": {
            "enable": true
          },
          "suggest": {
            "completeFunctionCalls": true
          }
        }
      }
    },
    "python": {
      "command": ["pylsp"],
      "filetypes": ["python"],
      "rootPatterns": ["setup.py", "pyproject.toml", ".git"],
      "settings": {
        "pylsp": {
          "plugins": {
            "pycodestyle": {
              "maxLineLength": 100
            }
          }
        }
      }
    }
  }
}

符号消歧​

当一行有多个相同符号时，使用 occurrence 参数：

// 第 10 行
const result = process(result);  // 两个 "result"
> 重命名第 10 行的第一个 result

OMP 使用 occurrence: 0 定位第一个 result

在 OMP 中使用​

直接使用 LSP 工具​

> 查找 getUser 函数的定义

# OMP 使用 LSP:definition
结果: src/services/user.ts:45:10

代码编辑时自动触发​

> 添加错误处理到 fetchData 函数

OMP 执行：
1. LSP:symbols - 获取文件大纲
2. 找到 fetchData 函数位置
3. 分析函数签名 (LSP:hover)
4. 检查返回类型 (LSP:type_definition)
5. 应用修改
6. LSP:diagnostics - 验证修改

重构工作流​

> 将 UserService 类重命名为 UserManager

OMP 执行：
1. LSP:references - 查找所有引用 (12 处)
2. 逐个确认
3. LSP:rename - 安全重命名
4. LSP:diagnostics - 确保无错误

故障排除​

问题：LSP 未启动​

诊断：

> /lsp status

TypeScript: ❌ Not running
  Reason: typescript-language-server not found
  Install: npm install -g typescript-language-server

解决：

npm install -g typescript-language-server
# 在 OMP 中
/reload

问题：诊断不显示​

检查：

// ~/.omp/settings.json
{
  "lsp": {
    "diagnosticsOnEdit": true,
    "diagnosticsOnSave": true
  }
}

问题：格式化失败​

排查：

  1. 检查语言服务器支持：
> /lsp status typescript

Capabilities: [..., "documentFormattingProvider", ...]
  1. 手动指定格式化器：
{
  "formatters": {
    "typescript": ["prettier", "--stdin-filepath", "{file}"]
  }
}

性能优化​

延迟初始化​

大型项目延迟启动 LSP：

{
  "lsp": {
    "lazyLoad": true,
    "startupTimeout": 30000
  }
}

诊断节流​

控制诊断频率：

{
  "lsp": {
    "diagnosticsDebounce": 500,  // ms
    "maxConcurrentChecks": 5
  }
}

最佳实践​

  1. 项目级配置：在 .omp/lsp.json 中配置，确保团队一致
  2. 按需启用：大型项目只启用必要的语言服务器
  3. 定期更新：保持语言服务器最新以获得最佳支持
  4. 结合工具：LSP + grep + AST 工具 = 强大的代码分析能力

💡 提示：使用 /lsp 命令查看所有 LSP 相关操作！
