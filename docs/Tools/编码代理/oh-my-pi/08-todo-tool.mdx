# 任务追踪工具 (Todo Tool)

OMP 提供结构化的任务管理系统，支持分阶段任务列表、实时进度跟踪和自动提醒。

## 基本用法

### 创建任务列表

```
> 创建任务列表来实现用户认证功能：

OMP 生成：
┌─ Todo List ──────────────────────────────────┐
│ 📋 实现用户认证功能                           │
│                                              │
│ 🔄 Phase 1: 数据库设计                        │
│    ☐ 创建 users 表                           │
│    ☐ 添加索引和约束                          │
│    ☐ 创建 migration                          │
│                                              │
│ ⏳ Phase 2: API 开发                          │
│    ☐ 实现注册接口                            │
│    ☐ 实现登录接口                            │
│    ☐ 实现 JWT 生成                           │
│                                              │
│ ⏳ Phase 3: 前端集成                          │
│    ☐ 创建登录表单                            │
│    ☐ 添加表单验证                            │
│    ☐ 处理认证状态                            │
│                                              │
│ ⏳ Phase 4: 测试                              │
│    ☐ 单元测试                                │
│    ☐ 集成测试                                │
│    ☐ E2E 测试                                │
└──────────────────────────────────────────────┘
```

### 更新任务状态

```
> 标记 "创建 users 表" 为进行中

OMP 更新：
🔄 Phase 1: 数据库设计
   🟡 创建 users 表  ← 进行中
   ☐ 添加索引和约束
   ☐ 创建 migration
```

### 完成任务

```
> 完成当前任务

OMP：
✅ 标记 "创建 users 表" 为完成
🔄 自动将 "添加索引和约束" 设为进行中
```

## 五种操作

### 1. replace（替换）

完全替换任务列表：

```
> 替换任务列表：
  Phase 1:
    - 任务 A
    - 任务 B
  Phase 2:
    - 任务 C
```

### 2. add_phase（添加阶段）

添加新阶段：

```
> 添加阶段 "Phase 5: 部署"

添加：
⏳ Phase 5: 部署
   ☐ 配置生产环境
   ☐ 设置 CI/CD
```

### 3. add_task（添加任务）

在现有阶段添加任务：

```
> 在 Phase 2 添加任务 "实现密码重置"

更新：
🔄 Phase 2: API 开发
   ✅ 实现注册接口
   ✅ 实现登录接口
   🟡 实现 JWT 生成  ← 进行中
   ☐ 实现密码重置   ← 新增
```

### 4. update（更新状态）

更新任务状态：

```
状态选项：
- pending（待办）
- in_progress（进行中）
- completed（已完成）
- abandoned（已放弃）

> 将 "任务 A" 标记为 completed
> 放弃 "任务 B"
```

### 5. remove_task（删除任务）

删除任务：

```
> 删除 "Phase 1" 中的 "任务 B"
```

## 四种任务状态

| 状态 | 图标 | 含义 |
|------|------|------|
| `pending` | ☐ | 待办，尚未开始 |
| `in_progress` | 🟡 | 正在进行中 |
| `completed` | ✅ | 已完成 |
| `abandoned` | ❌ | 已放弃/取消 |

## 自动规范化

### 唯一进行中任务

系统确保**任何时候只有一个任务处于 `in_progress` 状态**：

```
当前状态：
   🟡 任务 A（进行中）
   ☐ 任务 B

> 将任务 B 设为进行中

OMP 自动：
✅ 将任务 A 设为完成（或 pending）
🟡 将任务 B 设为进行中
```

### 阶段自动推进

当一个阶段的所有任务完成，自动激活下一阶段：

```
Phase 1: 数据库设计
   ✅ 所有任务完成

自动激活：
🔄 Phase 2: API 开发
   🟡 实现注册接口（自动设为进行中）
```

## 完成提醒

### 未完成任务警告

当尝试结束会话时有未完成的任务：

```
⚠️  你有 3 个未完成的任务：

  ☐ 实现密码重置
  ☐ 单元测试
  ☐ 集成测试

选项：
  [c] 继续工作
  [a] 放弃所有并退出
  [s] 保存状态并退出
  [v] 查看任务列表
```

### 配置提醒

```json
// ~/.omp/settings.json
{
  "todo": {
    "reminders": true,        // 启用提醒
    "reminderThreshold": 3,   // 超过3个未完成任务时提醒
    "autoSave": true          // 自动保存任务状态
  }
}
```

## 与代码集成

### 从代码生成任务

```
> 扫描代码中的 TODO 注释并生成任务列表

发现：
- src/auth.ts:15  // TODO: 添加 rate limiting
- src/api.ts:42   // TODO: 处理超时错误
- src/db.ts:88    // TODO: 添加索引

已生成对应的任务列表。
```

### 任务关联文件

```
> 创建任务 "重构 utils.ts"，关联文件 src/utils.ts

OMP 记录关联，在任务面板上显示：
🟡 重构 utils.ts  [src/utils.ts]
```

## 导入导出

### 导出任务

```
> 导出任务列表到 project-todos.json

格式：
{
  "project": "用户认证",
  "phases": [
    {
      "name": "Phase 1",
      "tasks": [
        {"name": "创建 users 表", "status": "completed"}
      ]
    }
  ]
}
```

### 导入任务

```
> 从 jira-export.csv 导入任务

OMP 解析并创建对应的任务列表。
```

## 最佳实践

### 1. 任务粒度

✅ **好**：15-30 分钟能完成的任务
❌ **差**：过大（"完成整个功能"）或过小（"输入一个字符"）

### 2. 命名清晰

✅ **好**："实现 JWT 生成函数"
❌ **差**："做 JWT"

### 3. 定期更新

保持任务状态实时更新，反映实际进度。

### 4. 阶段划分

按逻辑分组，每个阶段有明确的目标。

### 5. 优先级管理

将高优先级任务放在前面阶段。

## 故障排除

### 问题：任务状态不同步

**解决：**

```
> /todo refresh

OMP 重新加载并同步任务状态。
```

### 问题：面板不显示

**检查：**

```json
{
  "display": {
    "showTodoPanel": true,
    "todoPanelPosition": "top"  // 或 "bottom"
  }
}
```

### 问题：丢失任务历史

**恢复：**

```
> /todo history

显示最近的任务列表修改历史。
```
