SSH 远程执行工具

OMP 的 SSH 工具支持持久连接、自动 OS/Shell 检测和 SSHFS 挂载，让你在远程服务器上像本地一样工作。

核心特性​

  • 持久连接：跨命令重用 SSH 连接，减少延迟
  • 自动检测：自动识别远程 OS 和 Shell 类型
  • SSHFS 挂载：可选的远程目录本地挂载
  • 兼容模式：完整的 Windows 主机支持

配置 SSH 主机​

方式一：配置文件​

在项目根目录创建 ssh.json 或 .ssh.json：

{
  "hosts": {
    "production": {
      "hostname": "prod.example.com",
      "user": "deploy",
      "port": 22,
      "identityFile": "~/.ssh/prod_key",
      "forwardAgent": true
    },
    "staging": {
      "hostname": "staging.internal",
      "user": "ubuntu",
      "password": "${STAGING_PASSWORD}",
      "mount": {
        "remotePath": "/var/www/app",
        "localPath": "./remote-staging"
      }
    },
    "bastion": {
      "hostname": "jump-server.company.com",
      "user": "admin",
      "proxyJump": "bastion-gateway"
    }
  }
}

方式二：交互式配置​

> /ssh add

用户名: deploy

  [2] 密码
  [3] SSH 代理

列出主机​

> /ssh list

连接/断开​

> /ssh connect production
✓ 已连接到 production

> /ssh disconnect production
✓ 已断开连接

删除主机​

> /ssh remove staging
⚠️  确定删除主机 "staging"？[y/N]
y
✓ 已删除

SSHFS 挂载​

自动挂载​

配置中启用 mount：

{
  "hosts": {
    "dev": {
      "hostname": "dev-server",
      "user": "developer",
      "mount": {
        "remotePath": "/home/developer/project",
        "localPath": "./mnt/dev",
        "options": ["follow_symlinks", "allow_other"]
      }
    }
  }
}

连接时自动挂载：

> /ssh connect dev

✓ 已连接到 dev-server
✓ 已挂载 /home/developer/project 到 ./mnt/dev

现在可以本地编辑：
vim ./mnt/dev/src/app.js

手动挂载​

> /ssh mount production /var/log ./logs

✓ 已挂载 production:/var/log 到 ./logs

卸载​

> /ssh unmount ./logs
✓ 已卸载

高级功能​

代理跳转 (ProxyJump)​

通过跳板机连接内网服务器：

{
  "hosts": {
    "gateway": {
      "hostname": "jump.company.com",
      "user": "admin"
    },
    "internal-db": {
      "hostname": "10.0.1.50",
      "user": "dbadmin",
      "proxyJump": "gateway"
    }
  }
}

使用：

> 在 internal-db 上运行 psql -c "SELECT version();"

OMP:
通过 gateway 连接到 internal-db...
✓ 连接成功

执行命令...

环境变量​

在配置中引用环境变量：

{
  "hosts": {
    "secure": {
      "hostname": "secure-server.example.com",
      "user": "admin",
      "password": "${SECURE_SERVER_PASSWORD}",
      "identityFile": "${SSH_KEY_PATH}"
    }
  }
}

连接选项​

{
  "hosts": {
    "special": {
      "hostname": "special.example.com",
      "user": "user",
      "options": {
        "ServerAliveInterval": 60,
        "ServerAliveCountMax": 3,
        "Compression": true,
        "CompressionLevel": 6
      }
    }
  }
}

使用场景​

场景 1：日志分析​

> 在 production 上：
  1. grep "ERROR" /var/log/app/app.log
  2. 显示最后 100 行 /var/log/nginx/error.log
  3. 统计 500 错误的数量

OMP 批量执行并汇总结果。

场景 2：部署验证​

> 在 production 上：
  - 检查服务状态: systemctl status myapp
  - 验证版本: /opt/app/bin/version
  - 健康检查: curl localhost:8080/health

场景 3：远程开发​

> 连接 dev 服务器并挂载项目目录
> 在本地编辑代码（通过 SSHFS）
> 在 dev 上运行测试
> 在 dev 上查看日志

故障排除​

问题：连接超时​

检查：

# 测试连接
ssh -v user@hostname

# 检查防火墙
nc -zv hostname 22

增加超时：

{
  "hosts": {
    "slow": {
      "hostname": "slow-server.com",
      "connectionTimeout": 30000
    }
  }
}

问题：密钥认证失败​

检查权限：

chmod 600 ~/.ssh/id_rsa
chmod 700 ~/.ssh

使用 SSH 代理：

{
  "hosts": {
    "secure": {
      "hostname": "secure.com",
      "user": "admin",
      "forwardAgent": true
    }
  }
}

问题：SSHFS 挂载失败​

检查：

# macOS
brew install macfuse sshfs

# Linux
sudo apt install sshfs

# 检查挂载点
mkdir -p ./mnt/remote

安全配置​

最佳实践​

  1. 使用密钥而非密码

    {
      "identityFile": "~/.ssh/id_ed25519"
    }

  2. 启用主机密钥验证

    {
      "strictHostKeyChecking": true
    }

  3. 限制配置文件权限

    chmod 600 ssh.json

  4. 使用环境变量存储敏感信息

    {
      "password": "${SSH_PASSWORD}"
    }

💡 提示：SSH 工具与子代理结合，可以同时在多台服务器上并行执行任务！
