用代理视图管理多个代理

从单一界面调度并管理多个 Claude Code 会话。代理视图（agent view）展示每个会话正在做什么，以及哪些会话需要你的介入。

代理视图通过 claude agents 打开，是所有后台会话的统一界面：哪些在运行、哪些需要你输入、哪些已完成。你可以调度新会话、一目了然地观察它们的状态，而不必逐条翻阅对话记录，仅在某个会话需要你时介入。即使没有终端附着，会话也会在后台持续运行。

当你有多个 Claude 可以并行处理的独立任务时，比如修复一个 bug、审查一个 pull request 或分析一份日志，就使用代理视图。当你想协同解决某个问题时，附着到该会话并像往常一样交互式使用 Claude Code。

代理视图中的会话各自独立运行，仅向你汇报。要与子代理（subagent）、代理团队（agent team）和工作树（worktree）做对比，参见并行运行代理。

注意：代理视图是研究预览功能，需要 Claude Code v2.1.139 或更高版本。使用 claude --version 检查你的版本。界面和键盘快捷键可能会随功能演进而变化，管理员可以通过 disableAgentView 受管设置为整个组织禁用代理视图。

本页涵盖：

  • 快速开始
  • 用代理视图监控会话，包括状态图标、查看和回复、附着、整理列表以及键盘快捷键
  • 调度新代理：从代理视图中、从会话内部、或从 shell
  • 从 shell 管理会话
  • 由 supervisor 进程承载的后台会话承载方式

快速开始​

本指引带你打开代理视图、调度一个会话、从查看面板回复，以及附着以查看完整对话。

步骤 1：打开代理视图​

在 shell 中运行：

claude agents

代理视图打开，底部是一个输入框，会话启动后表格会逐步填满。任何时候按 Esc 退出。你的会话会继续运行。

步骤 2：调度一个会话​

在输入框中输入 prompt 然后按 Enter。一个新的会话启动，并以一行的形式出现，显示它是在工作中、等待你输入还是已完成。重复操作可以并行运行多个会话。每个会话独立消耗你的订阅配额，因此在大量调度前请参见限制。

步骤 3：查看与回复​

用方向键选中一行后按 Space，可以看到该会话正在做什么或需要你提供什么。输入回复后按 Enter 即可发送，无需离开代理视图。

步骤 4：附着与脱离​

在某行上按 Enter 或 → 可以附着以查看完整对话。该会话会像你直接运行 claude 一样接管终端。在空 prompt 上按 ← 即可脱离并返回表格。

要把已有的交互式会话纳入代理视图，可以在会话中运行 /bg，或在空 prompt 上按 ← 一步将会话送入后台并打开代理视图。会话会在后台继续运行并以一行的形式出现。要从 shell 直接启动一个新的后台会话，运行 claude --bg "<prompt>"。

你可以将 claude agents 作为主要入口替代 claude：从代理视图调度每个任务，需要查看完整对话时附着，按 ← 返回表格。

用代理视图监控会话​

运行 claude agents 打开代理视图。它会接管整个终端，按状态分组列出每个会话，被置顶的会话和需要你介入的会话排在最上方。每行显示会话名称、当前活动以及上次状态变更的时间。

列表覆盖位于你 config 目录下的所有后台会话，无论它们在哪个项目或工作树中工作——所以一个在某仓库启动的会话和另一个在不同工作树启动的会话会一起出现。在其他终端打开的交互式会话不会出现，除非你将其送入后台；会话内运行的子代理也不会作为单独的行列出。

Pinned
  ✽ clawd walk cycle          Write assets/sprites/clawd-walk.png           3m

Ready for review
  ∙ jump physics              github.com/anthropics/example/pull/2048       2h

Needs input
  ✻ power-up design           needs input: double jump or wall climb?       1m

  ✽ collision detection       Edit src/physics/CollisionSystem.ts           2m

  ✻ title screen              result: menu, options, and credits done       9m
  ∙ sound effects             result: 14 SFX exported to assets/audio       4h
  … 6 more

每行的图标承载两个信号。指示符告诉你会话的状态，图标的形状告诉你底层进程是否仍在运行。状态包括：

指示符状态含义
动画工作中Claude 正在主动运行工具或生成响应
黄色需要输入Claude 在等你输入，通常是权限决定或回答
暗淡空闲会话在等待输入但没有被某个具体问题阻塞
绿色已完成任务成功完成
红色失败任务以错误结束
灰色已停止会话被 Ctrl+X 或 claude stop 停止

图标的形状告诉你底层进程是否仍在运行。✻，或者 Claude 工作时动画的 ✽，意味着会话仍存活，你可以立即回复。∙ 表示进程已退出，但你仍然可以查看、回复或附着：Claude 会从中断处恢复会话。✢ 表示一个 /loop 会话正在迭代之间睡眠，行内会显示运行次数和距离下次迭代的倒计时。

后台会话不需要任何打开的终端就能继续工作。一个独立的 supervisor 进程会运行它们，因此你可以关闭代理视图、关闭 shell 或开启新的交互式会话，已调度的工作仍会继续推进。

会话会持久化到磁盘上：关闭终端或自动更新都不会丢失它们，重新打开 claude agents 会显示全部。如果机器进入睡眠或关机，正在运行的会话会停止；用 claude respawn --all 重新启动它们。

每行的一行摘要由你配置的 Haiku 级别模型生成，以便该行能告诉你会话正在做什么、需要什么或产生了什么，无需打开对话记录。当会话正在工作时，摘要至多每 15 秒刷新一次，并在每个回合结束时刷新一次。每次刷新都是一个通过常规 provider 发出的 Haiku 级别短请求，按照与会话相同的数据使用条款计费和处理。

当某个会话开启了 pull request，行内会显示 PR 链接和它的 CI 检查状态。对大多数任务而言，这一行就是你拿到结果的地方：在 PR 检查通过后审查并合并它。

查看和回复​

在选中行上按 Space 打开查看面板。它会显示该会话需要你输入什么、最近的输出，以及它开启的任何 pull request。多数情况下这就够了，你完全无需打开完整对话记录。

在查看面板中输入回复，按 Enter 即可发送给该会话。当会话在询问一个多选题时，查看面板会显示选项，你可以按数字键选择。对于其他被阻塞的会话，按 Tab 让输入框填入一个建议的回复，可以编辑后再发送。回复以 ! 开头则会作为 Bash 命令发送。

用 ↑ 和 ↓ 在不关闭面板的情况下查看相邻会话，或按 → 附着。

附着到会话​

在选中行上按 Enter 或 → 来附着，或按 Alt+1 到 Alt+9 直接附着到当前分组中的第 N 个会话。代理视图会被替换为完整的交互式会话，与你在该目录运行 claude 完全一致。附着时，Claude 会发布一段简短的回顾，告诉你你不在时发生了什么。

附着期间，会话与任何其他 Claude Code 会话表现一致：每个命令、键盘快捷键和功能都可用。

在空 prompt 上按 ← 即可脱离并返回代理视图。如果某个对话框获得了焦点而没有响应 ←，按 Ctrl+Z 立即脱离。

脱离绝不会停止后台会话：←、Ctrl+C、Ctrl+D、Ctrl+Z 和 /exit 都让会话保持运行。要从会话内部结束它，运行 /stop。

在你调度或将会话送入后台之后，在空 prompt 上按 ← 不仅适用于你从代理视图附着的会话，对任何 Claude Code 会话都生效。它会把当前会话送入后台并打开代理视图，并预选中该会话，这样你就可以在不离开终端的情况下切换会话。你可以在 /config 中关闭这个快捷键。

整理列表​

代理视图按状态分组排列会话，需要输入的会话排在工作中或已完成的会话之上。按 Ctrl+S 切换为按目录分组。你的选择会在多次运行间保留。在分组内，用 Ctrl+T 将一个会话置顶，用 Shift+↑ 和 Shift+↓ 重新排序，或在分组标题上按 Enter 折叠它。要移除一个会话，按 Ctrl+X 停止它，并在两秒内再按一次 Ctrl+X 删除它。在分组标题上按 Ctrl+X 会在确认后删除该分组中的每个会话。

较旧的已完成会话会折叠成一行 "… N more"，让列表保持简短。失败的会话和带有未关闭 pull request 的会话总是保持可见。

过滤列表​

在调度输入框中输入文本即可过滤而非调度：

过滤器显示
a:<name>运行指定代理的会话
s:<state>处于指定状态的会话，例如 s:blocked 表示需要你的会话
#<number> 或 PR URL正在处理该 pull request 的会话

键盘快捷键​

在代理视图中按 ? 查看每个快捷键。最常用的有：

快捷键动作
↑ / ↓在行间移动
Enter附着到选中的会话；若输入框中有文本则进行调度
Space为选中的会话打开或关闭查看面板
Shift+Enter调度并立即附着
→附着到选中的会话
Alt+1..Alt+9附着到当前分组中的第 N 个会话
Tab浏览所有子代理，或应用高亮的建议
Ctrl+S在按状态和按目录分组之间切换
Ctrl+T置顶或取消置顶选中的会话
Ctrl+R重命名选中的会话
Ctrl+G在你的 $EDITOR 中打开调度 prompt
Ctrl+X停止会话；两秒内再按一次将其删除
Shift+↑ / Shift+↓重新排序选中的会话

?显示所有快捷键

调度新代理​

你可以从代理视图调度新的后台会话，把现有的交互式会话送入后台，或直接从 shell 启动一个。

从代理视图​

在代理视图底部输入框中输入 prompt 然后按 Enter 启动一个新的后台会话。会话会自动从 prompt 命名。之后可以用 Ctrl+R 重命名。把图片粘贴到 prompt 中，可以为任务附带截图或图表。

为 prompt 添加前缀或在其中提到某些部分，可以控制会话的启动方式：

输入效果
<agent-name> <prompt>如果第一个词与某个自定义子代理名称匹配，该子代理会作为会话的主代理运行，并使用其 frontmatter 中的配置
@<agent-name>在 prompt 任意位置提到某个自定义子代理，使其作为主代理运行
@<repo>提到你打开代理视图所在目录下的某个仓库，让会话在那里运行
/<skill>建议作为 prompt 调度的技能
#<number> 或 PR URL如果已有会话在处理该 PR，则选中它而非调度新会话
Shift+Enter调度并立即附着到新会话

输入 / 来调度一个技能。把重复任务打包为技能，能让你从代理视图多次启动同一工作流而无需重新输入 prompt。在空输入框中按 Tab 浏览每个可调度的子代理，或在显示建议时按 Tab 应用高亮的建议。

当同一个 @name 同时匹配某个子代理和同级仓库时，子代理优先。无 @ 的首词形式同样适用于任何子代理名称——如果 prompt 的第一个词匹配你某个子代理名称，就会调度那个子代理。当你想要明确指定时，使用 @ 形式。

调度到特定目录​

新会话在你打开代理视图的目录中运行。要指定不同的目录：

  • 在该目录中打开 claude agents。
  • 在包含多个仓库的父目录中打开 claude agents，并在 prompt 中用 @<repo> 提到其中一个，让会话在那里运行。
  • 从 shell 中 cd 进入该目录后运行 claude --bg "<prompt>"。

当代理视图按目录分组时，高亮行的目录会成为调度目标，因此你可以滚动到某个分组并直接调度，无需重新输入路径。

从会话内部​

运行 /background 或它的别名 /bg 可以让当前对话脱离并继续运行。可以传递一个 prompt，例如 /bg run the test suite and fix any failures，在脱离前发送多一条指令。

从 shell​

传递 --bg 来启动一个直接进入后台的会话：

claude --bg "investigate the flaky SettingsChangeDetector test"

要让特定子代理作为会话的主代理运行，将 --bg 与 --agent 组合使用：

claude --agent code-reviewer --bg "address review comments on PR 1234"

送入后台后，Claude 会打印会话的短 ID 以及管理它的命令：

backgrounded · 7c5dcf5d
  claude agents             list sessions
  claude attach 7c5dcf5d    open in this terminal
  claude logs 7c5dcf5d      show recent output
  claude stop 7c5dcf5d      stop this session

文件编辑如何隔离​

每个后台会话——无论是从代理视图、/bg 还是 claude --bg 启动——都从你的工作目录中开始运行，但被禁止向该目录写入文件。当会话需要编辑文件时，Claude 会自动将其移入 .claude/worktrees/ 下的隔离 git 工作树，这样并行会话可以读取同一份检出，但各自写入自己的工作树。当会话已经在某个工作树中、当工作目录不是 git 仓库时，或对工作目录之外的写入，该限制不适用。

工作树会在你删除会话时被移除，所以删除前请先合并或推送你想保留的更改。要查找某个会话的工作树路径，对其进行查看（peek）或附着并检查它的工作目录。

要让某个子代理无论如何启动都始终在自己的工作树中运行，在它的 frontmatter 中设置 isolation: worktree。

权限模式与设置​

被调度的会话会从其运行的目录读取设置和权限模式，与你在该目录启动 claude 时一样。从代理视图输入框调度时不会传入权限模式，所以会话会使用该目录设置中的 defaultMode 或被调度子代理 frontmatter 中的 permissionMode。

要从 shell 设置模式，在 claude --bg 时传递 --permission-mode。以这种方式使用 bypassPermissions 或 auto 会被拒绝，除非你曾经以该模式交互式地运行过 claude 一次接受了它，因为这些模式会让你没有观察的会话在没有审批的情况下执行操作。

从 shell 管理会话​

每个后台会话都有一个短 ID，你可以从 shell 中使用。这些命令对脚本编写或不想打开代理视图时很有用。

命令用途
claude agents打开代理视图
claude attach <id>在当前终端附着到一个会话
claude logs <id>打印该会话的近期输出
claude stop <id>停止一个会话。也接受 claude kill
claude respawn <id>在保留对话记录的情况下重启一个已停止的会话
claude respawn --all重启每个已停止的会话
claude rm <id>从列表中移除一个会话

后台会话的承载方式​

后台会话由一个用户级别的 supervisor 进程承载，它独立于你的终端和代理视图。当你第一次将会话送入后台或打开代理视图时，它会自动启动，你无需直接管理。supervisor 及其会话使用与你的交互式会话相同的凭据进行认证，并且除模型 API 之外不会建立任何额外的网络连接。

每个后台会话都是它自己的 Claude Code 进程，挂在 supervisor 之下而非你的终端之下。一个仍在工作、等待你输入或有终端附着的会话，其进程会保持运行。一旦会话完成且未附着大约一小时，supervisor 会停止其进程以释放资源。对话记录和状态保留在磁盘上，下次你附着、查看或回复时，supervisor 会从中断处启动一个新进程。当每个会话都已完成且没有终端连接时，supervisor 自身会退出，等到下次你将会话送入后台或打开代理视图时再启动。

supervisor 监视磁盘上已安装的 Claude Code 二进制文件，并在常规自动更新器替换它后重启进入新版本。这是本地文件监视，而非网络检查。后台会话是独立进程，所以它们会跨重启继续运行，新的 supervisor 会重新连接到它们。

会话状态存储在你的 Claude Code config 目录下。如果你设置了 CLAUDE_CONFIG_DIR，supervisor 会使用该目录而非 ~/.claude，并作为独立实例运行，拥有自己的会话集合。

路径内容
~/.claude/daemon.logsupervisor 日志
~/.claude/daemon/roster.json正在运行的后台会话列表，用于重启后重新连接
~/.claude/jobs/<id>/state.json代理视图中显示的每会话状态

要完全关闭后台代理和代理视图，将 disableAgentView 设置设为 true 或设置 CLAUDE_CODE_DISABLE_AGENT_VIEW 环境变量。管理员可以通过受管设置强制执行此规则。

故障排查​

代理视图打开后没有会话​

代理视图在你调度第一个会话之前是空的。在底部输入框输入 prompt 然后按 Enter。

机器从睡眠唤醒后会话显示为已停止​

后台会话无法在睡眠或关机后存活。对任何已停止的会话进行附着、查看或回复，它就会从中断处重启。要一次重启所有会话，运行 claude respawn --all。

附着后会话响应缓慢​

会话完成并未附着大约一小时后，supervisor 会停止其进程以释放资源。附着会从中断处启动一个新进程，需要片刻时间。仍在工作或等待你输入的会话不会被这样停止。

.claude/worktrees/ 在变满​

工作树在你删除创建它的会话时被移除。如果某个会话结束时未清理，在项目目录中用 git worktree list 列出残留条目，并用 git worktree remove <path> 各自移除。参见清理工作树。

限制​

代理视图是研究预览功能。需要注意当前的限制：

  • 适用速率限制：后台会话与交互式会话一样消耗你的订阅用量，因此并行运行 10 个代理消耗配额的速度大致是运行 1 个的 10 倍。
  • 会话是本地的：后台会话在你的机器上运行，并在机器睡眠或关机时停止。
  • 工作树会随会话一起被删除：在删除一个在其自己工作树中编辑过文件的会话之前，先合并或推送更改。

下一步​

了解了代理视图后，探索这些相关功能：

  • 并行运行代理：将代理视图与子代理、代理团队和工作树作对比
  • 子代理：定义可复用的代理配置，包含自定义 prompt、工具和隔离
  • 代理团队：协调多个相互发送消息的会话
  • Web 上的 Claude Code：在受管的云环境中而非本地运行会话
