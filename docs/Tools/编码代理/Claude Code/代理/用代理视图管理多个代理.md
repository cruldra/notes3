> 从一个屏幕调度并管理多个 Claude Code 会话。代理视图会显示每个会话正在做什么，以及哪些会话需要你的输入。

通过 `claude agents` 打开的代理视图，是所有后台会话的一个统一屏幕：哪些正在运行、哪些需要你的输入、哪些已经完成。你可以调度新会话，一眼查看它们的状态，而不用在对话记录里来回滚动，只在某个会话需要你时介入。每个后台会话都是一个完整的 Claude Code 对话，即使没有终端附着也会持续运行，所以你可以打开它、回复它，然后随时离开。

当你有几个独立任务可以交给 Claude 处理，而且不需要一直盯着每一步时，就使用代理视图。把一个 bug 修复、一个 pull request 审查和一个 flaky 测试调查调度成三行，继续在另一个窗口工作，等某一行显示需要你或已经有结果时再回来查看。

当你想更直接地处理某个代理会话时，可以附着到该行，进入完整对话。

要比较代理视图与子代理、代理团队和工作树，请参阅[并行运行代理](/en/agents)。

> **注意**
>
> 代理视图处于研究预览阶段，需要 Claude Code v2.1.139 或更高版本。使用 `claude --version` 检查版本。随着功能演进，界面和键盘快捷键可能会变化。

* [快速开始](#快速开始)：给 Claude 一个后台任务、检查进度，并在需要时介入

* [调度新代理](#调度新代理)：从代理视图、会话内部或 shell 调度
* [从 shell 管理会话](#从-shell-管理会话)
* [后台会话如何承载](#后台会话如何承载)：由 supervisor 进程承载

## 快速开始

这个演练覆盖代理视图的核心流程：调度一个任务，在 Claude 工作时观察它的行状态更新，通过查看面板检查并回复它，以及附着进入完整对话。你调度的会话在关闭代理视图后仍会继续运行，所以你可以离开后再回来。

### 步骤 1：打开代理视图

在 shell 中运行：

```bash
claude agents
```

代理视图会打开，底部有一个输入框；会话启动后，表格会逐步填充。任何时候按 `Esc` 都可以回到 shell。你离开时，会话会继续运行；下次打开代理视图时，它们会重新出现。

### 步骤 2：调度一个会话

输入描述任务的 prompt，然后按 `Enter`。一个新的后台会话会开始处理该任务，并显示为一行，告诉你它是在工作、等待你输入，还是已经完成。新会话会使用代理视图页眉显示的模型，以及你在该目录运行 `claude` 时会得到的同一[权限模式](#权限模式与设置)。

你在这里输入的每个 prompt 都会启动自己的新会话。输入另一个 prompt 并按 `Enter`，会并行启动第二个会话，而不是把它作为追问发送给第一个会话。你可以用这种方式并行运行多个会话。

每个会话会独立消耗你的订阅配额，所以在一次调度很多会话之前，请先查看[限制](#限制)。

### 步骤 3：查看并回复

用方向键选择一行，然后按 `Space` 打开查看面板。它显示该会话最近的输出，或它正在等待的问题，而不是完整对话记录。输入回复并按 `Enter`，即可在不离开代理视图的情况下发送。

### 步骤 4：附着和脱离

在某行上按 `Enter` 或 `→`，可以在你想查看完整对话时附着进去。该会话会接管终端，就像你运行了 `claude` 一样。在空 prompt 上按 `←`，即可脱离并返回表格。

### 步骤 5：带入现有会话

要把已经打开的会话移入代理视图，在该会话中运行 `/bg`，或者在空 prompt 上按 `←`，一步将它转入后台并打开代理视图。该会话会继续运行，并与其他调度的会话一起显示为一行。

你可以把 `claude agents` 作为主要入口，而不是 `claude`：从代理视图调度每个任务，需要完整对话时附着，按 `←` 返回表格。

## 用代理视图监控会话

运行 `claude agents` 打开代理视图。它会接管整个终端，按状态列出每个会话，置顶会话和需要你输入的会话排在最前面。每行显示会话名称、当前活动，以及距离上次变化过了多久。

列表会显示你启动过的所有后台会话，跨所有项目显示。在一个仓库中工作的会话，和在另一个工作树中工作的会话，都会出现在这里，不管你从哪个目录打开代理视图。你在其他终端里打开的交互式会话不会出现，除非你将它们[转入后台](#从会话内部)。某个会话生成的[子代理](/en/sub-agents)和[队友](/en/agent-teams)不会作为单独的行列出。

```text
Pinned
  ✽ clawd walk cycle          Write assets/sprites/clawd-walk.png           3m

Ready for review
  ∙ jump physics              github.com/example/game/pull/2048          ●  2h

Needs input
  ✻ power-up design           needs input: double jump or wall climb?       1m

Working
  ✽ collision detection       Edit src/physics/CollisionSystem.ts           2m
  ✢ playtest level 3          run 12 · all checkpoints cleared           in 4m

Completed
  ✻ title screen              result: menu, options, and credits done       9m
  ∙ sound effects             result: 14 SFX exported to assets/audio       4h
  … 6 more
```

### 读取会话状态

每行开头都有一个图标，它的颜色和动画显示会话状态：

| 状态 | 图标显示为 | 含义 |
| :--- | :--- | :--- |
| Working | 动画 | Claude 正在主动运行工具或生成响应 |
| Needs input | 黄色 | Claude 正在等待你的特定问题回答或权限决定 |
| Idle | 暗淡 | 会话没有待办事项，已准备好接收你的下一个 prompt |
| Completed | 绿色 | 任务已成功完成 |
| Failed | 红色 | 任务以错误结束 |
| Stopped | 灰色 | 会话已被 `Ctrl+X` 或 `claude stop` 停止 |

另外，图标形状显示底层进程是否仍在运行：

| 形状 | 含义 |
| :--- | :--- |
| `✻` 或动画 `✽` | 会话进程仍然存活，会立即回复 |
| `∙` | 进程已经退出。你仍可查看、回复或附着，Claude 会从离开的位置重启 |
| `✢` | 一个 [`/loop`](/en/scheduled-tasks) 会话正在两次迭代之间睡眠。该行会显示运行次数和倒计时 |

后台会话不需要任何终端保持打开就能继续工作。一个独立的 [supervisor 进程](#supervisor-进程)会运行它们，所以你可以关闭代理视图、关闭 shell，或启动一个新的交互式会话；已调度的工作仍会继续。

会话状态会通过自动更新和 supervisor 重启持久化到磁盘。如果你的机器睡眠或关机，正在运行的会话会停止；使用 `claude respawn --all` 重启它们。

### 行摘要

每行中的单行摘要由 [Haiku 级别模型](/en/model-config)生成，因此无需打开对话记录，该行就能告诉你会话正在做什么、需要什么，或产出了什么。当会话正在工作时，摘要最多每 15 秒刷新一次，并会在每个回合结束时再刷新一次。

每次刷新都是一次通过你常规 provider 发起的短 Haiku 级别请求，计费和处理遵循与会话本身相同的[数据使用条款](/en/data-usage)。

### Pull request 状态

当某个会话打开 pull request 时，行的右侧会出现一个状态点；在支持超链接的终端中，它会链接到该 pull request。当会话打开了多个 pull request 时，状态点前会显示数量，颜色反映最需要关注的那一个。

| 点颜色 | Pull request 状态 |
| :--- | :--- |
| 黄色 | 正在等待检查或审查，或检查失败 |
| 绿色 | 检查已通过，且没有阻塞性审查 |
| 紫色 | 已合并 |
| 灰色 | 草稿或已关闭 |

对多数任务来说，这一行就是你拿到结果的地方：当状态点变绿时，审查并合并 pull request。

### 查看和回复

在选中的行上按 `Space` 打开查看面板。它会显示该会话需要你做什么、最近输出，以及它打开的任何 pull request。多数情况下这就足够了，你不需要打开完整对话记录。

在查看面板中输入回复并按 `Enter`，即可发送给该会话。当会话提出多选问题时，查看面板会显示选项，你可以按数字键选择。对于其他被阻塞的会话，按 `Tab` 可以把建议回复填入输入框，你可以编辑后发送。回复前加 `!`，会把它作为 Bash 命令发送。

使用 `↑` 和 `↓` 可以在不关闭面板的情况下查看相邻会话；按 `→` 可以附着。

### 附着到会话

在选中行上按 `Enter` 或 `→` 附着。代理视图会被完整交互式会话替换，就像你在该目录运行了 `claude` 一样。附着时，Claude 会发一段简短回顾，说明你离开时发生了什么。

附着期间，会话的行为与任何其他 Claude Code 会话相同：每个[命令](/en/commands)、键盘快捷键和功能都可用。

在空 prompt 上按 `←` 可以脱离并返回代理视图。如果某个对话框获得焦点且没有响应 `←`，按 `Ctrl+Z` 立即脱离。

脱离永远不会停止后台会话：`←`、`Ctrl+C`、`Ctrl+D`、`Ctrl+Z` 和 `/exit` 都会让它继续运行。要从会话内部结束它，运行 `/stop`。

在你调度或后台化一个会话后，从任何 Claude Code 会话的空 prompt 上按 `←` 都有效，不只限于从代理视图附着的会话。它会把当前会话转入后台，并打开代理视图且预选中该会话，让你无需离开终端即可切换会话。你可以在 `/config` 中关闭这个快捷键。

### 整理列表

代理视图会对会话分组，让需要输入的会话排在顶部，`Ready for review` 和 `Needs input` 位于 `Working` 和 `Completed` 之上。这些分组名称并不与上面的[状态](#读取会话状态)一一对应：当会话有打开的 pull request 时，它会移动到 `Ready for review`；`Completed` 会收集已完成、失败和已停止的会话。按 `Ctrl+S` 可以改为按目录分组。你的选择会跨运行持久保存。

在分组内：

* 按 `Ctrl+T` 将会话置顶
* 按 `Shift+↑` 或 `Shift+↓` 重新排序会话
* 按 `Ctrl+R` 重命名会话
* 在分组标题上按 `Enter` 折叠分组

要从列表中移除会话，按 `Ctrl+X` 停止它，并在两秒内再按一次 `Ctrl+X` 删除它。在分组标题上按 `Ctrl+X`，会在确认后删除该分组中的每个会话。

删除会将该会话从代理视图中移除，并清理它的[工作树](#文件编辑如何隔离)，包括其中任何未提交的更改。因此，删除前请先推送或提交你想保留的工作。对话记录会留在磁盘上，并可通过 `claude --resume` 继续访问。

较早的已完成会话会折叠进 `… N more` 行，以保持列表简短。失败的会话和带有打开 pull request 的会话总会保持可见。

### 过滤会话

在调度输入框中输入内容会进行过滤，而不是调度：

| 过滤器 | 显示 |
| :--- | :--- |
| `a:<name>` | 运行指定代理的会话 |
| `s:<state>` | 处于指定状态的会话，例如 `s:working`。也接受 `s:blocked`，表示所有正在等待你的会话 |
| `#<number>` 或 PR URL | 正在处理该 pull request 的会话 |

### 键盘快捷键

在代理视图中按 `?`，可以查看当前上下文中的所有快捷键。下表汇总了它们。

| 快捷键 | 操作 |
| :--- | :--- |
| `↑` / `↓` | 在行之间移动 |
| `Enter` | 附着到选中的会话；如果输入框中有文本，则调度 |
| `Space` | 为选中的会话打开或关闭查看面板 |
| `Shift+Enter` | 调度并立即附着 |
| `→` | 附着到选中的会话 |
| `Alt+1`..`Alt+9` | 附着到当前分组中的第 1 到第 9 个会话 |
| `Tab` | 空输入框中浏览所有子代理；否则应用高亮建议 |
| `Ctrl+S` | 在按状态分组和按目录分组之间切换 |
| `Ctrl+T` | 置顶或取消置顶选中的会话 |
| `Ctrl+R` | 重命名选中的会话 |
| `Ctrl+G` | 在你的 `$EDITOR` 中打开调度 prompt |
| `Ctrl+X` | 停止会话；两秒内再按一次删除它 |
| `Shift+↑` / `Shift+↓` | 重新排序选中的会话 |
| `Esc` | 关闭查看面板、清空输入框或退出 |
| `Ctrl+C` | 清空输入框；按两次退出 |
| `?` | 显示所有快捷键 |

## 调度新代理

你可以从代理视图调度新的后台会话，将现有交互式会话送到后台，或直接从 shell 启动一个。

### 从代理视图

在代理视图底部输入框中输入 prompt，然后按 `Enter` 启动一个新的后台会话。会话会根据 prompt 自动命名；之后可以用 `Ctrl+R` 重命名。

把图片粘贴到 prompt 中，可以随任务附带截图或图表。

给 prompt 添加前缀，或在其中提到某些内容，可以控制会话如何启动：

| 输入 | 效果 |
| :--- | :--- |
| `<agent-name> <prompt>` | 如果第一个词匹配某个自定义[子代理](/en/sub-agents)名称，该子代理会作为会话的主代理运行，并使用其 frontmatter 中的配置 |
| `@<agent-name>` | 在 prompt 任意位置提到自定义子代理，让它作为主代理运行 |
| `@<repo>` | 提到你打开代理视图所在目录下的某个仓库，让会话在该仓库运行 |
| `/<skill>` | 建议可作为 prompt 调度的[技能](/en/skills) |
| `#<number>` 或 pull request URL | 如果已有会话在处理该 PR，则选中它，而不是调度新会话 |
| `Shift+Enter` | 调度并立即附着到新会话 |

将重复任务打包为[技能](/en/skills)，可以让你从代理视图反复启动同一工作流，而不用重新输入 prompt。

当同一个 `@name` 同时匹配某个子代理和同级仓库时，子代理优先。裸首词匹配同样适用，所以如果某个 prompt 恰好以你的某个子代理名称开头，就会调度该子代理，而不是把这个词当作普通文本。想明确指定时使用 `@` 形式；或者让 prompt 以另一个词开头，避免匹配。

#### 调度到特定目录

新会话会在你打开代理视图的目录中运行。要指定另一个目录：

* 在该目录中打开 `claude agents`。
* 在包含多个仓库的父目录中打开 `claude agents`，并在 prompt 中提到 `@<repo>`，让会话在那里运行。
* 从 shell 中 `cd` 到该目录，然后运行 `claude --bg "<prompt>"`。

当代理视图按目录分组时，高亮行的目录会成为调度目标，所以你可以滚动到某个分组并直接向其中调度，而不用重新输入路径。

### 从会话内部

运行 `/background` 或其别名 `/bg`，将当前对话移入后台会话。可以传入一个 prompt，例如 `/bg run the test suite and fix any failures`，在后台化前再给一条指令。

从交互式会话后台化时，会启动一个新进程，从已保存的对话继续。因此，正在运行的子代理和后台命令不会转移过去。当有这些任务正在运行时，Claude 会在后台化前请求你确认。进入后台后，该会话可以启动新的子代理和后台命令，并且这些任务会在之后的脱离和重新附着之间继续运行。

### 从 shell

传入 `--bg` 可以启动一个直接进入后台的会话：

```bash
claude --bg "investigate the flaky SettingsChangeDetector test"
```

要让特定子代理作为会话主代理运行，把 `--bg` 与 `--agent` 组合使用：

```bash
claude --agent code-reviewer --bg "address review comments on PR 1234"
```

后台化后，Claude 会打印会话短 ID 和管理命令：

```text
backgrounded · 7c5dcf5d
  claude agents             list sessions
  claude attach 7c5dcf5d    open in this terminal
  claude logs 7c5dcf5d      show recent output
  claude stop 7c5dcf5d      stop this session
```

### 文件编辑如何隔离

每个后台会话，无论是从代理视图、`/bg` 还是 `claude --bg` 启动，都会从你的工作目录开始。编辑文件前，Claude 会将该会话移入 `.claude/worktrees/` 下的隔离 [git 工作树](/en/worktrees)，这样并行会话可以读取同一个检出，但各自写入自己的工作树。如果会话已经位于 `.claude/worktrees/` 下、工作目录不是 git 仓库，或写入发生在工作目录之外，Claude 会跳过这一步。

在 git 仓库之外，会话会直接写入工作目录，并且彼此不隔离，所以不要调度会编辑同一文件的并行会话。

工作树会在你删除会话时被移除，所以删除前请先合并或推送你想保留的更改。要找到某个会话的工作树路径，查看该会话或附着进去并检查它的工作目录。

要让某个子代理无论如何启动都始终在自己的工作树中运行，在它的 frontmatter 中设置 [`isolation: worktree`](/en/sub-agents#supported-frontmatter-fields)。

### 设置模型

代理视图页眉中显示的模型名称是调度默认值。你从输入框启动的新会话会使用这个模型，它与任意会话中 [`/model`](/en/model-config) 控制的设置相同。

每个后台会话可以运行在不同模型上。要为某个会话覆盖模型：

* 从 shell 使用 `claude --bg` 时传入 `--model`。
* 附着到正在运行的会话，在其中运行 `/model`。如果会话被 respawn，该更改会保留。
* 调度一个在 frontmatter 中设置了 `model` 字段的[子代理](/en/sub-agents)。

### 权限模式与设置

被调度的会话会从它运行的目录读取[设置](/en/settings)和[权限模式](/en/permissions)，就像你从该目录启动 `claude` 一样。从代理视图输入框调度不会传入权限模式，所以会话会使用该目录设置中的 `defaultMode`，或被调度[子代理 frontmatter](/en/sub-agents#supported-frontmatter-fields) 中的 `permissionMode`。

要从 shell 设置模式，在 `claude --bg` 中传入 `--permission-mode`。以这种方式使用 `bypassPermissions` 或 `auto` 会被拒绝，直到你已经通过交互式运行 `claude` 接受过该模式，因为这些模式允许一个你没有盯着的会话无需批准就采取行动。

## 从 shell 管理会话

每个后台会话都有一个短 ID，你可以从 shell 使用。使用 `claude --bg` 启动会话时会打印这个 ID；每个会话的 ID 也是 `~/.claude/jobs/` 下的目录名。当你要写脚本，或不想打开代理视图时，这些命令很有用。

| 命令 | 用途 |
| :--- | :--- |
| `claude agents` | 打开代理视图 |
| `claude attach <id>` | 在此终端附着到一个会话 |
| `claude logs <id>` | 打印该会话最近输出 |
| `claude stop <id>` | 停止会话。也接受 `claude kill` |
| `claude respawn <id>` | 在保留对话的情况下重启已停止会话 |
| `claude respawn --all` | 重启每个已停止会话 |
| `claude rm <id>` | 从列表中移除会话。如果没有未提交更改，也会清理它的工作树 |

## 后台会话如何承载

代理视图列出的每个会话都被视为后台会话，不管你当前是否附着在其中。相比之下，直接运行 `claude` 启动的会话会绑定到该终端，并在终端关闭时结束，除非你[将它送入后台](#从会话内部)。

### Supervisor 进程

后台会话由一个每用户的 supervisor 进程承载，它独立于你的终端和代理视图。第一次后台化会话或打开代理视图时，supervisor 会自动启动，你不需要直接管理它。

Supervisor 及其会话会使用与你的交互式会话相同的凭据认证，除了模型 API 之外不会建立其他网络连接。

每个后台会话都是自己的 Claude Code 进程，由 supervisor 管理，而不是绑定到你的终端。正在主动工作、等待你输入或有终端附着的会话会保持进程运行。

一旦某个会话完成并且未附着约一小时，supervisor 会停止其进程以释放资源。对话记录和状态仍保留在磁盘上；下次你附着、查看或回复时，supervisor 会从停止的位置启动一个新进程。当所有会话都已完成且没有终端连接时，supervisor 本身会退出，并在下次你需要它时再次启动。

Supervisor 会监视磁盘上已安装的 Claude Code 二进制文件，并在常规[自动更新器](/en/setup#auto-updates)替换它后重启到新版本。这是本地文件监视，不是网络检查。后台会话是脱离终端的进程，所以它们会在 supervisor 重启期间继续运行，新 supervisor 会重新连接到它们。

### 状态存储位置

会话状态存储在你的 Claude Code 配置目录下。如果设置了 [`CLAUDE_CONFIG_DIR`](/en/env-vars)，supervisor 会使用该目录，而不是 `~/.claude`，并作为一个独立实例运行，拥有自己的会话。

| 路径 | 内容 |
| :--- | :--- |
| `~/.claude/daemon.log` | Supervisor 日志 |
| `~/.claude/daemon/roster.json` | 正在运行的后台会话列表，用于重启后重新连接 |
| `~/.claude/jobs/<id>/state.json` | 代理视图中显示的每会话状态 |

### 关闭代理视图

要完全关闭后台代理和代理视图，将 [setting](/en/settings) 中的 `disableAgentView` 设为 `true`，或设置 `CLAUDE_CODE_DISABLE_AGENT_VIEW` 环境变量。管理员可以通过[受管设置](/en/permissions#managed-settings)强制执行。

## 故障排查

### `claude agents` 列出子代理，而不是打开代理视图

如果 `claude agents` 打印一个数量，随后列出你配置的子代理，然后退出，说明代理视图在你的环境中不可用。早期版本并非在所有环境中都会打开代理视图，包括通过 Bedrock、Vertex AI 或 Foundry 连接时。运行 `claude update` 安装最新版本。

如果更新后代理视图仍未打开，检查它是否已被某个设置或环境变量[关闭](#关闭代理视图)。

### 代理视图打开后没有会话

在你调度第一个会话之前，代理视图是空的。在底部输入框输入 prompt 并按 `Enter`。

### 因后台任务正在运行而无法打开 agents

如果按 `←` 后台化当前会话时显示 `Cannot open agents — N background task(s) running`，说明该会话有正在进行的工作，例如子代理、workflow 或后台 shell 命令；该快捷键不会静默放弃它。运行 `/tasks` 查看正在运行的内容，然后运行 `/bg` 确认放弃它们。参阅[从会话内部](#从会话内部)，了解后台化时哪些内容会转移、哪些不会。

### Prompt 因太短被拒绝

调度输入框期望的是任务描述，不是寒暄式开场。少于四个字符的 prompt 会被拒绝，并显示 `Too short` 提示，避免误按键启动会话。请描述你希望会话做什么，例如 `investigate the flaky checkout test`。

### 机器唤醒后会话显示失败

后台会话无法跨睡眠或关机存活，所以机器唤醒后，正在运行的会话会显示为失败。附着、查看或回复其中任何一个，会话就会从离开的位置重启。要一次重启全部会话，运行 `claude respawn --all`。

### 附着后会话响应缓慢

会话完成并且未附着约一小时后，supervisor 会停止其进程以释放资源。附着时会从离开的位置启动新进程，需要一点时间。正在工作或等待你的会话不会被这样停止。

### `.claude/worktrees/` 占用越来越多

工作树会在你删除创建它的会话时被移除。如果某个会话结束后没有清理，在项目目录中用 `git worktree list` 列出遗留条目，并分别用 `git worktree remove <path>` 移除。参阅[清理工作树](/en/worktrees#clean-up-worktrees)。

## 限制

代理视图处于研究预览阶段，有以下限制：

* **速率限制适用**：后台会话与交互式会话一样消耗你的订阅用量，因此并行运行十个代理，消耗配额的速度大约是运行一个代理的十倍。
* **会话是本地的**：后台会话在你的机器上运行，如果机器睡眠或关机，它们会停止。
* **工作树会随会话删除**：删除在自己工作树中编辑过文件的会话前，请先合并或推送更改。
