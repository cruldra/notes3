用频道向运行中的会话推送事件

使用频道从 MCP 服务器向你的 Claude Code 会话推送消息、警报和 webhook。转发 CI 结果、聊天消息和监控事件，这样 Claude 可以在你离开时做出响应。

注意：频道处于研究预览阶段，需要 Claude Code v2.1.80 或更高版本。它们需要 claude.ai 登录。不支持 Console 和 API 密钥认证。Team 和 Enterprise 组织必须显式启用它们。

频道是一个 MCP 服务器，将事件推送到你正在运行的 Claude Code 会话中，这样 Claude 可以对你不在终端时发生的事情做出响应。频道可以是双向的：Claude 读取事件并通过同一频道回复，就像聊天桥接。事件仅在会话打开时到达，因此对于始终在线的设置，你在后台进程或持久终端中运行 Claude。

与生成全新云会话或等待轮询的集成不同，事件到达你已经打开的会话中：见频道对比。

你将频道作为插件安装并用你自己的凭据配置它。Telegram、Discord 和 iMessage 包含在研究预览中。

当 Claude 通过频道回复时，你会在终端中看到入站消息但看不到回复文本。终端显示工具调用和确认（如"sent"），实际回复出现在另一个平台上。

本页涵盖：

  • 支持的频道：Telegram、Discord 和 iMessage 设置
  • 安装和运行频道：使用 fakechat（本地主机演示）
  • 谁可以推送消息：发送者白名单和配对方式
  • 为你的组织启用频道：Team 和 Enterprise
  • 频道对比：与 web 会话、Slack、MCP 和远程控制的比较

要构建你自己的频道，见频道参考。

支持的频道​

每个支持的频道都是一个需要 Bun 的插件。在连接真实平台之前体验插件流程，尝试 fakechat 快速入门。

Telegram

查看完整的 Telegram 插件源码。

步骤 1：创建 Telegram 机器人

在 Telegram 中打开 BotFather 并发送 /newbot。给它一个显示名称和以 bot 结尾的唯一用户名。复制 BotFather 返回的 token。

步骤 2：安装插件

在 Claude Code 中运行：

/plugin install telegram@claude-plugins-official

如果 Claude Code 报告在任何市场中找不到插件，你的市场要么缺失要么过时。运行 /plugin marketplace update claude-plugins-official 刷新它，或者如果你之前没有添加过，运行 /plugin marketplace add anthropics/claude-plugins-official。然后重试安装。

安装后，运行 /reload-plugins 激活插件的配置命令。

步骤 3：配置你的 token

使用 BotFather 的 token 运行配置命令：

/telegram:configure <token>

这将其保存到 ~/.claude/channels/telegram/.env。你也可以在启动 Claude Code 之前在 shell 环境中设置 TELEGRAM_BOT_TOKEN。

步骤 4：重启并启用频道

退出 Claude Code 并用频道标志重启。这会启动 Telegram 插件，开始从你的机器人轮询消息：

claude --channels plugin:telegram@claude-plugins-official

步骤 5：配对你的账户

打开 Telegram 并向你的机器人发送任何消息。机器人回复一个配对码。

如果你的机器人没有响应，确保 Claude Code 在上一步中带 --channels 运行。机器人仅在频道激活时才能回复。

回到 Claude Code，运行：

/telegram:access pair <code>

然后锁定访问，使只有你的账户可以发送消息：

/telegram:access policy allowlist
Discord

查看完整的 Discord 插件源码。

步骤 1：创建 Discord 机器人

前往 Discord 开发者门户，点击 New Application 并命名。在 Bot 部分，创建用户名，然后点击 Reset Token 并复制 token。

步骤 2：启用消息内容意图

在你的机器人设置中，滚动到 Privileged Gateway Intents 并启用 Message Content Intent。

步骤 3：邀请机器人到你的服务器

前往 OAuth2 > URL Generator。选择 bot 作用域并启用这些权限：

  • View Channels
  • Send Messages
  • Send Messages in Threads
  • Read Message History
  • Attach Files
  • Add Reactions

打开生成的 URL 将机器人添加到你的服务器。

步骤 4：安装插件

在 Claude Code 中运行：

/plugin install discord@claude-plugins-official

如果 Claude Code 报告在任何市场中找不到插件，你的市场要么缺失要么过时。运行 /plugin marketplace update claude-plugins-official 刷新它，或者如果你之前没有添加过，运行 /plugin marketplace add anthropics/claude-plugins-official。然后重试安装。

安装后，运行 /reload-plugins 激活插件的配置命令。

步骤 5：配置你的 token

使用你复制的机器人 token 运行配置命令：

/discord:configure <token>

这将其保存到 ~/.claude/channels/discord/.env。你也可以在启动 Claude Code 之前在 shell 环境中设置 DISCORD_BOT_TOKEN。

步骤 6：重启并启用频道

退出 Claude Code 并用频道标志重启。这会连接 Discord 插件，使你的机器人可以接收和响应消息：

claude --channels plugin:discord@claude-plugins-official

步骤 7：配对你的账户

在 Discord 上私信你的机器人。机器人回复一个配对码。

如果你的机器人没有响应，确保 Claude Code 在上一步中带 --channels 运行。机器人仅在频道激活时才能回复。

回到 Claude Code，运行：

/discord:access pair <code>

然后锁定访问，使只有你的账户可以发送消息：

/discord:access policy allowlist
iMessage

查看完整的 iMessage 插件源码。

iMessage 频道直接读取你的 Messages 数据库并通过 AppleScript 发送回复。它需要 macOS，不需要机器人 token 或外部服务。

步骤 1：授予完全磁盘访问权限

位于 ~/Library/Messages/chat.db 的 Messages 数据库受 macOS 保护。服务器首次读取它时，macOS 会提示访问：点击 Allow。提示命名启动 Bun 的应用，如 Terminal、iTerm 或你的 IDE。

如果提示没有出现或你点击了 Don't Allow，在系统设置 > 隐私与安全性 > 完全磁盘访问权限下手动授予访问权限并添加你的终端。没有这个，服务器会立即以 authorization denied 退出。

步骤 2：安装插件

在 Claude Code 中运行：

/plugin install imessage@claude-plugins-official

如果 Claude Code 报告在任何市场中找不到插件，你的市场要么缺失要么过时。运行 /plugin marketplace update claude-plugins-official 刷新它，或者如果你之前没有添加过，运行 /plugin marketplace add anthropics/claude-plugins-official。然后重试安装。

步骤 3：重启并启用频道

退出 Claude Code 并用频道标志重启：

claude --channels plugin:imessage@claude-plugins-official

步骤 4：给自己发消息

在登录你 Apple ID 的任何设备上打开 Messages 并给自己发消息。它会立即到达 Claude：自我聊天绕过访问控制，无需设置。

Claude 发送的第一条回复会触发 macOS 自动化提示，询问你的终端是否可以控制 Messages。点击 OK。

步骤 5：允许其他发送者

默认情况下，仅你自己的消息通过。要让其他联系人联系 Claude，添加他们的句柄：

/imessage:access allow +15551234567

句柄是 +country 格式的电话号码或如 user@example.com 的 Apple ID 邮箱。

你也可以为还没有插件的系统构建你自己的频道。

快速入门​

Fakechat 是官方支持的演示频道，在 localhost 上运行聊天 UI，无需认证也无需配置外部服务。

安装并启用 fakechat 后，你可以在浏览器中输入消息，消息会到达你的 Claude Code 会话。Claude 回复，回复会显示回浏览器。测试完 fakechat 界面后，尝试 Telegram、Discord 或 iMessage。

要尝试 fakechat 演示，你需要：

  • 带 claude.ai 账户的 Claude Code 已安装和认证
  • 已安装 Bun。预构建的频道插件是 Bun 脚本。用 bun --version 检查；如果失败，安装 Bun。
  • Team/Enterprise 用户：你的组织管理员必须启用频道在 managed 设置中

步骤 1：安装 fakechat 频道插件

启动 Claude Code 会话并运行安装命令：

/plugin install fakechat@claude-plugins-official

如果 Claude Code 报告在任何市场中找不到插件，你的市场要么缺失要么过时。运行 /plugin marketplace update claude-plugins-official 刷新它，或者如果你之前没有添加过，运行 /plugin marketplace add anthropics/claude-plugins-official。然后重试安装。

步骤 2：重启并启用频道

退出 Claude Code，然后用 --channels 重启并传递你安装的 fakechat 插件：

claude --channels plugin:fakechat@claude-plugins-official

fakechat 服务器自动启动。

提示：你可以将多个插件传递给 --channels，用空格分隔。

步骤 3：推送消息

在 http://localhost:8787 打开 fakechat UI 并输入消息：

hey, what's in my working directory?

消息作为 <channel source="fakechat"> 事件到达你的 Claude Code 会话。Claude 读取它，执行工作，并调用 fakechat 的 reply 工具。答案显示在聊天 UI 中。

如果你不在终端时 Claude 遇到权限提示，会话会暂停直到你响应。声明权限中继能力的频道服务器可以将这些提示转发给你，这样你可以远程批准或拒绝。对于无人值守使用，--dangerously-skip-permissions 完全绕过提示，但仅在你信任的环境中使用它。

安全​

每个批准的频道插件维护一个发送者白名单：只有你添加的 ID 可以推送消息，其他所有人都被静默丢弃。

Telegram 和 Discord 通过配对引导列表：

  1. 在 Telegram 或 Discord 中找到你的机器人并发送任何消息
  2. 机器人回复一个配对码
  3. 在你的 Claude Code 会话中，提示时批准码
  4. 你的发送者 ID 被添加到白名单

iMessage 工作方式不同：给自己发短信自动绕过网关，你用句柄通过 /imessage:access allow 添加其他联系人。

最重要的是，你用 --channels 控制每个会话启用哪些服务器，在 Team 和 Enterprise 计划上你的组织用 channelsEnabled 控制可用性。

在 .mcp.json 中不足以推送消息：服务器还必须用 --channels 命名。

如果频道声明了它，白名单还控制权限中继。任何可以通过频道回复的人都可以批准或拒绝你会话中的工具使用，因此仅将你信任的发送者加入白名单。

企业控制​

在 Team 和 Enterprise 计划上，频道默认关闭。管理员通过两个managed 设置控制可用性，用户无法覆盖：

设置用途未配置时
channelsEnabled主开关。必须为 true 任何频道才能传递消息。通过 claude.ai Admin 控制台切换或直接在 managed 设置中设置。关闭时阻止所有频道包括开发标志。频道被阻止
allowedChannelPlugins启用频道后哪些插件可以注册。设置时替换 Anthropic 维护的列表。仅在 channelsEnabled 为 true 时适用。应用 Anthropic 默认列表

没有组织的 Pro 和 Max 用户完全跳过这些检查：频道可用，用户用 --channels 按会话选择加入。

为你的组织启用频道​

管理员可以从 claude.ai → Admin settings → Claude Code → Channels 启用频道，或在 managed 设置中将 channelsEnabled 设置为 true。

启用后，你组织中的用户可以使用 --channels 选择将频道服务器加入单个会话。如果设置被禁用或未设置，MCP 服务器仍然连接且其工具工作，但频道消息不会到达。启动警告告诉用户让管理员启用该设置。

限制可以运行的频道插件​

默认情况下，Anthropic 维护的白名单上的任何插件都可以注册为频道。Team 和 Enterprise 计划的管理员可以通过在 managed 设置中设置 allowedChannelPlugins 用他们自己的替换该白名单。用它来限制允许哪些官方插件、批准来自你自己内部市场的频道，或两者兼有。每个条目命名一个插件和它来自的市场：

{
  "channelsEnabled": true,
  "allowedChannelPlugins": [
    { "marketplace": "claude-plugins-official", "plugin": "telegram" },
    { "marketplace": "claude-plugins-official", "plugin": "discord" },
    { "marketplace": "acme-corp-plugins", "plugin": "internal-alerts" }
  ]
}

设置 allowedChannelPlugins 时，它完全替换 Anthropic 白名单：只有列出的插件可以注册。不设置以回退到默认的 Anthropic 白名单。空数组阻止白名单中的所有频道插件，但 --dangerously-load-development-channels 仍然可以绕过它用于本地测试。要完全阻止频道包括开发标志，改为不设置 channelsEnabled。

此设置需要 channelsEnabled: true。如果用户将不在你列表上的插件传递给 --channels，Claude Code 正常启动但频道不注册，启动通知解释该插件不在组织的批准列表上。

研究预览​

频道是研究预览功能。可用性正在逐步推出，--channels 标志语法和协议合约可能会根据反馈更改。

在预览期间，--channels 仅接受来自 Anthropic 维护的白名单的插件，或者如果管理员设置了 allowedChannelPlugins 则接受你组织的白名单。claude-plugins-official 中的频道插件是默认批准的集合。如果你传递了不在有效白名单上的内容，Claude Code 正常启动但频道不注册，启动通知告诉你原因。

要测试你正在构建的频道，使用 --dangerously-load-development-channels。见在研究预览期间测试了解测试你构建的自定义频道的信息。

在 Claude Code GitHub 仓库 报告问题或反馈。

频道对比​

几个 Claude Code 功能连接到终端之外的系统，每种适合不同类型的工作：

功能它做什么适用于
Web 版 Claude Code在新的云沙箱中运行任务，从 GitHub 克隆委托以后检查的独立异步工作
Claude in Slack从频道或线程中的 @Claude 提及生成 web 会话直接从团队对话上下文启动任务
标准 MCP 服务器Claude 在任务期间查询它；没有任何内容推送到会话给 Claude 按需访问读取或查询系统
远程控制你从 claude.ai 或 Claude 移动应用驱动你的本地会话离开办公桌时操控进行中的会话

频道通过从非 Claude 源推送事件到你已经运行的本地会话来填补列表中的空白。

  • 聊天桥接：通过 Telegram、Discord 或 iMessage 从手机询问 Claude 问题，答案在同一个聊天中返回，同时工作在你的机器上针对真实文件运行。
  • Webhook 接收器：来自 CI、错误跟踪器、部署管道或其他外部服务的 webhook 到达 Claude 已经打开你的文件并记住你正在调试什么的地方。
