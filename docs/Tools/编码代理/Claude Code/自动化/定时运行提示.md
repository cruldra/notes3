# 定时运行提示

> 使用 /loop 和 cron 调度工具在 Claude Code 会话中重复运行提示、轮询状态或设置一次性提醒。

> **注意**：定时任务需要 Claude Code v2.1.72 或更高版本。用 `claude --version` 检查你的版本。

定时任务让你可以让 Claude 按间隔自动重新运行提示。用它们来轮询部署、看护 PR、检查长时间运行的构建，或提醒自己在会话稍后做某事。要在事件发生时响应而不是轮询，见[频道](/en/channels)：你的 CI 可以直接将失败推送到会话中。

任务是会话作用域的：它们存在于当前 Claude Code 进程中，退出时消失。对于能在重启后存活的持久化调度，使用 [Cloud](/en/web-scheduled-tasks) 或 [Desktop](/en/desktop-scheduled-tasks) 定时任务，或 [GitHub Actions](/en/github-actions)。

---

## 调度选项对比

Claude Code 提供三种方式来调度重复工作：

|                                | [Cloud](/en/web-scheduled-tasks) | [Desktop](/en/desktop-scheduled-tasks) | [`/loop`](/en/scheduled-tasks) |
| :----------------------------- | :------------------------------- | :------------------------------------- | :----------------------------- |
| 运行在                         | Anthropic 云                     | 你的机器                               | 你的机器                       |
| 需要机器开启                   | 否                               | 是                                     | 是                             |
| 需要打开会话                   | 否                               | 否                                     | 是                             |
| 重启后持久化                   | 是                               | 是                                     | 否（会话作用域）               |
| 访问本地文件                   | 否（新克隆）                     | 是                                     | 是                             |
| MCP 服务器                     | 每个任务配置的连接器             | [配置文件](/en/mcp) 和连接器           | 继承自会话                     |
| 权限提示                       | 否（自主运行）                   | 每个任务可配置                         | 继承自会话                     |
| 可自定义调度                   | 通过 CLI 中的 `/schedule`        | 是                                     | 是                             |
| 最小间隔                       | 1 小时                           | 1 分钟                                 | 1 分钟                         |

> **提示**：用 **Cloud 任务** 处理应在没有你的机器时可靠运行的工作。用 **Desktop 任务** 当你需要访问本地文件和工具时。用 **`/loop`** 在会话期间快速轮询。

## 用 /loop 重复运行提示

`/loop` [捆绑技能](/en/commands) 是在会话保持打开时重复运行提示的最快方式。间隔和提示都是可选的，你提供的内容决定了循环的行为方式。

| 你提供的内容             | 示例                        | 发生了什么                                                                                              |
| :----------------------- | :-------------------------- | :------------------------------------------------------------------------------------------------------ |
| 间隔和提示               | `/loop 5m check the deploy` | 你的提示按[固定调度](#固定间隔运行)运行                                                                 |
| 仅提示                   | `/loop check the deploy`    | 你的提示在每次迭代时以 [Claude 选择的间隔](#让-claude-选择间隔)运行                                     |
| 仅间隔，或不提供任何内容 | `/loop`                     | 运行[内置维护提示](#运行内置维护提示)，或者如果你有的话运行 `loop.md`                                   |

你也可以将另一个命令作为提示传递，例如 `/loop 20m /review-pr 1234`，在每次迭代时重新运行打包的工作流。

### 固定间隔运行

当你提供间隔时，Claude 将其转换为 cron 表达式，调度任务并确认节奏和任务 ID。

```text
/loop 5m check if the deployment finished and tell me what happened
```

间隔可以作为裸标记如 `30m` 引导提示，或作为从句如 `every 2 hours` 跟在后面。支持的单位是 `s`（秒）、`m`（分钟）、`h`（小时）和 `d`（天）。

秒向上舍入到最近的分钟，因为 cron 只有一分钟粒度。不映射到干净 cron 步骤的间隔（如 `7m` 或 `90m`）会舍入到最近的间隔，Claude 会告诉你它选择了什么。

### 让 Claude 选择间隔

当你省略间隔时，Claude 会动态选择一个，而不是在固定的 cron 调度上运行。每次迭代后，它根据观察到的内容选择 1 分钟到 1 小时之间的延迟：构建完成或 PR 活跃时等待较短，没有待处理事项时等待较长。选择的延迟和原因会在每次迭代结束时打印。

下面的示例检查 CI 和审查评论，一旦 PR 安静下来 Claude 会在迭代之间等待更长时间：

```text
/loop check whether CI passed and address any review comments
```

当你请求动态 `/loop` 调度时，Claude 可能直接使用 [Monitor 工具](/en/tools-reference#monitor-tool)。Monitor 运行后台脚本并将每个输出行流式返回，这完全避免了轮询，通常比按间隔重新运行提示更节省 token 且响应更快。

动态调度的循环像任何其他任务一样出现在你的[定时任务列表](#管理定时任务)中，因此你可以用相同方式列出或取消它。[抖动规则](#抖动)不适用于它，但[七天过期](#七天过期)适用：循环在你启动七天后自动结束。

> **注意**：在 Bedrock、Vertex AI 和 Microsoft Foundry 上，没有间隔的提示按固定的 10 分钟调度运行。

### 运行内置维护提示

当你省略提示时，Claude 使用内置维护提示而不是你提供的。每次迭代它按顺序处理以下内容：

- 继续对话中未完成的工作
- 处理当前分支的拉取请求：审查评论、失败的 CI 运行、合并冲突
- 当没有其他待处理事项时运行清理传递，如 bug 猎捕或简化

Claude 不会在该范围之外启动新的计划，不可逆的操作如推送或删除仅在它们继续转录已授权的内容时才会进行。

```text
/loop
```

单独的 `/loop` 以[动态选择的间隔](#让-claude-选择间隔)运行此提示。添加间隔，例如 `/loop 15m`，改为在固定调度上运行它。要用你自己的默认值替换内置提示，见[用 loop.md 自定义默认提示](#用-loop-md-自定义默认提示)。

> **注意**：在 Bedrock、Vertex AI 和 Microsoft Foundry 上，不带提示的 `/loop` 会打印使用消息而不是启动维护循环。

### 用 loop.md 自定义默认提示

`loop.md` 文件用你自己的指令替换内置维护提示。它为裸 `/loop` 定义单个默认提示，而不是单独的定时任务列表，并且在你从命令行提供提示时被忽略。要在其旁边调度额外的提示，使用 `/loop <prompt>` 或[直接询问 Claude](#管理定时任务)。

Claude 在两个位置查找文件并使用它找到的第一个。

| 路径                | 作用域                                                     |
| :------------------ | :--------------------------------------------------------- |
| `.claude/loop.md`   | 项目级别。当两个文件都存在时优先。                         |
| `~/.claude/loop.md` | 用户级别。适用于没有定义自己的任何项目。                   |

该文件是纯 Markdown，没有必需的结构。像直接输入 `/loop` 提示一样编写它。下面的示例保持发布分支健康：

```markdown
Check the `release/next` PR. If CI is red, pull the failing job log,
diagnose, and push a minimal fix. If new review comments have arrived,
address each one and resolve the thread. If everything is green and
quiet, say so in one line.
```

对 `loop.md` 的编辑在下次迭代时生效，因此你可以在循环运行时优化指令。当两个位置都不存在 `loop.md` 时，循环回退到内置维护提示。保持文件简洁：超过 25,000 字节的内容会被截断。

## 设置一次性提醒

对于一次性提醒，用自然语言描述你想要的而不是使用 `/loop`。Claude 调度一个单次触发任务，运行后自行删除。

```text
remind me at 3pm to push the release branch
```

```text
in 45 minutes, check whether the integration tests passed
```

Claude 使用 cron 表达式将触发时间固定到特定分钟和小时，并确认它何时触发。

## 管理定时任务

用自然语言询问 Claude 列出或取消任务，或直接引用底层工具。

```text
what scheduled tasks do I have?
```

```text
cancel the deploy check job
```

在底层，Claude 使用这些工具：

| 工具         | 用途                                                                                         |
| :----------- | :------------------------------------------------------------------------------------------- |
| `CronCreate` | 调度新任务。接受 5 字段 cron 表达式、要运行的提示，以及它是循环还是单次触发。                 |
| `CronList`   | 列出所有定时任务及其 ID、调度和提示。                                                        |
| `CronDelete` | 按 ID 取消任务。                                                                             |

每个定时任务有一个 8 字符 ID，你可以传递给 `CronDelete`。一个会话最多可以同时持有 50 个定时任务。

## 定时任务如何运行

调度器每秒检查一次到期任务并以低优先级将它们入队。定时提示在你的轮次之间触发，而不是在 Claude 响应中间。如果 Claude 在任务到期时正忙，提示会等待当前轮次结束。

所有时间都以你的本地时区解释。像 `0 9 * * *` 这样的 cron 表达式意味着你运行 Claude Code 所在位置的上午 9 点，而不是 UTC。

### 抖动

为避免所有会话在同一挂钟时刻同时命中 API，调度器给触发时间添加一个小的确定性偏移：

- 循环任务最多延迟其周期的 10% 触发，上限为 15 分钟。每小时作业可能在 `:00` 到 `:06` 之间的任何时间触发。
- 调度在整点或半点的一次性任务最多提前 90 秒触发。

偏移源自任务 ID，因此同一任务始终获得相同的偏移。如果精确时间很重要，选择不是 `:00` 或 `:30` 的分钟，例如 `3 9 * * *` 而不是 `0 9 * * *`，这样一次性抖动就不适用。

### 七天过期

循环任务在创建 7 天后自动过期。任务最后一次触发，然后删除自己。这限制了被遗忘的循环可以运行多长时间。如果你需要循环任务持续更长时间，在过期前取消并重新创建它，或使用 [Cloud 定时任务](/en/web-scheduled-tasks) 或 [Desktop 定时任务](/en/desktop-scheduled-tasks) 进行持久化调度。

## Cron 表达式参考

`CronCreate` 接受标准 5 字段 cron 表达式：`minute hour day-of-month month day-of-week`。所有字段支持通配符（`*`）、单个值（`5`）、步骤（`*/15`）、范围（`1-5`）和逗号分隔列表（`1,15,30`）。

| 示例           | 含义                     |
| :------------- | :----------------------- |
| `*/5 * * * *`  | 每 5 分钟                |
| `0 * * * *`    | 每小时整点               |
| `7 * * * *`    | 每小时第 7 分钟          |
| `0 9 * * *`    | 每天本地时间上午 9 点    |
| `0 9 * * 1-5`  | 工作日本地时间上午 9 点  |
| `30 14 15 3 *` | 3 月 15 日下午 2:30 本地 |

星期几使用 `0` 或 `7` 表示周日到 `6` 表示周六。不支持扩展语法如 `L`、`W`、`?` 和名称别名如 `MON` 或 `JAN`。

当月中日期和星期几都被约束时，如果任一字段匹配则日期匹配。这遵循标准 vixie-cron 语义。

## 禁用定时任务

在环境中设置 `CLAUDE_CODE_DISABLE_CRON=1` 完全禁用调度器。cron 工具和 `/loop` 变得不可用，任何已调度的任务停止触发。见[环境变量](/en/env-vars)获取禁用标志的完整列表。

## 限制

会话作用域调度有固有限制：

- 任务仅在 Claude Code 运行且空闲时触发。关闭终端或让会话退出会取消所有内容。
- 没有错过的触发补偿。如果任务的调度时间在 Claude 处理长时间运行的请求时过去，它会在 Claude 变为空闲时触发一次，而不是每个错过的间隔触发一次。
- 重启后不持久化。重启 Claude Code 会清除所有会话作用域任务。

对于需要无人值守运行的 cron 驱动自动化：

- [Cloud 定时任务](/en/web-scheduled-tasks)：在 Anthropic 管理的基础设施上运行
- [GitHub Actions](/en/github-actions)：在 CI 中使用 `schedule` 触发器
- [Desktop 定时任务](/en/desktop-scheduled-tasks)：在你的机器上本地运行
