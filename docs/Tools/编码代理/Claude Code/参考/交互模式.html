交互模式

Claude Code 会话中键盘快捷键、输入模式和交互功能的完整参考。

键盘快捷键​

注意：键盘快捷键可能因平台和终端而异。按 ? 查看你环境中可用的快捷键。

macOS 用户：Option/Alt 键快捷键（Alt+B、Alt+F、Alt+Y、Alt+M、Alt+P、Alt+T）需要在终端中将 Option 配置为 Meta：

  • iTerm2：设置 → Profiles → Keys → 将左/右 Option 键设置为"Esc+"
  • Terminal.app：设置 → Profiles → Keyboard → 勾选"Use Option as Meta Key"
  • VS Code：在 VS Code 设置中设置 "terminal.integrated.macOptionIsMeta": true

详情见终端配置。

命令​

在 Claude Code 中输入 / 查看所有可用命令，或输入 / 后跟任意字母进行过滤。/ 菜单显示你可以调用的所有内容：内置命令、捆绑和用户编写的技能，以及插件和 MCP 服务器贡献的命令。并非所有内置命令对每个用户都可见，因为某些命令取决于你的平台或计划。

见命令参考获取 Claude Code 中包含的命令完整列表。

命令历史​

Claude Code 维护当前会话的命令历史：

  • 输入历史按工作目录存储
  • 输入历史在运行 /clear 开始新会话时重置。之前会话的对话被保留且可以恢复。
  • 使用上/下箭头导航（见上面的键盘快捷键）
  • 注意：历史扩展（!）默认禁用

用 Ctrl+R 反向搜索​

按 Ctrl+R 交互式搜索你的命令历史：

  1. 开始搜索：按 Ctrl+R 激活反向历史搜索
  2. 输入查询：输入要在之前命令中搜索的文本。搜索词在匹配结果中高亮显示
  3. 导航匹配项：再次按 Ctrl+R 循环遍历更旧的匹配项
  4. 接受匹配项：
    • 按 Tab 或 Esc 接受当前匹配项并继续编辑
    • 按 Enter 接受并立即执行命令
  5. 取消搜索：
    • 按 Ctrl+C 取消并恢复原始输入
    • 在空搜索上按 Backspace 取消

搜索显示匹配的命令，搜索词高亮显示，因此你可以找到并复用之前的输入。

后台 bash 命令​

Claude Code 支持在后台运行 bash 命令，允许你在长时间运行的进程执行时继续工作。

后台工作原理​

当 Claude Code 在后台运行命令时，它异步运行命令并立即返回后台任务 ID。Claude Code 可以在命令继续在后台执行时响应新提示。

要在后台运行命令，你可以：

  • 提示 Claude Code 在后台运行命令
  • 按 Ctrl+B 将常规 Bash 工具调用移到后台。（由于 tmux 的前缀键，tmux 用户必须按两次 Ctrl+B。）

关键功能：

  • 输出写入文件，Claude 可以使用 Read 工具检索它
  • 后台任务有唯一 ID 用于跟踪和输出检索
  • 后台任务在 Claude Code 退出时自动清理
  • 当输出超过 5GB 时，后台任务自动终止，stderr 中有注释解释原因

要禁用所有后台任务功能，将 CLAUDE_CODE_DISABLE_BACKGROUND_TASKS 环境变量设置为 1。详情见环境变量。

常见的后台命令：

  • 构建工具（webpack、vite、make）
  • 包管理器（npm、yarn、pnpm）
  • 测试运行器（jest、pytest）
  • 开发服务器
  • 长时间运行的进程（docker、terraform）

带 ! 前缀的 Bash 模式​

通过在输入前加 ! 直接运行 bash 命令而无需经过 Claude：

! npm test
! git status
! ls -la

Bash 模式：

  • 将命令及其输出添加到对话上下文中
  • 显示实时进度和输出
  • 支持长时间运行命令的相同 Ctrl+B 后台化
  • 不需要 Claude 解释或批准命令
  • 支持基于历史的自动补全：输入部分命令并按 Tab 从当前项目中之前的 ! 命令完成
  • 在空提示上用 Escape、Backspace 或 Ctrl+U 退出
  • 将空格开头的文本粘贴到空提示中时自动进入 bash 模式，与键入 ! 行为匹配

这对于快速 shell 操作同时保持对话上下文很有用。

提示建议​

当你首次打开会话时，提示输入中会出现灰色示例命令来帮助你开始。Claude Code 从项目的 git 历史中选择，因此它反映你最近正在工作的文件。

Claude 响应后，建议会继续基于你的对话历史出现，如多部分请求的后续步骤或工作流的自然延续。

  • 按 Tab 或 右箭头 接受建议，或按 Enter 接受并提交
  • 开始打字以关闭它

建议作为后台请求运行，复用父对话的提示缓存，因此额外成本最小。当缓存冷时，Claude Code 跳过建议生成以避免不必要的成本。

建议在对话的第一轮之后、非交互模式和计划模式中自动跳过。

要完全禁用提示建议，设置环境变量或在 /config 中切换设置：

export CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION=false

用 /btw 边问​

使用 /btw 询问关于你当前工作的快速问题而不添加到对话历史中。这在你想要快速答案但不想弄乱主上下文或使 Claude 偏离长时间运行的任务时很有用。

/btw what was the name of that config file again?

边问对当前对话完全可见，因此你可以询问 Claude 已经读取的代码、它之前做出的决策或会话中的任何其他内容。问题和答案是短暂的：它们出现在可关闭的覆盖层中，从不进入对话历史。

  • 在 Claude 工作时可用：即使在 Claude 处理响应时你也可以运行 /btw。边问独立运行，不会中断主轮次。
  • 无工具访问：边问仅从已在上下文中的内容回答。Claude 在回答边问时无法读取文件、运行命令或搜索。
  • 单次响应：没有后续轮次。如果你需要来回交互，改用正常提示。
  • 低成本：边问复用父对话的提示缓存，因此额外成本最小。

按 Space、Enter 或 Escape 关闭答案并返回提示。

/btw 是子代理的反面：它看到你的完整对话但没有工具，而子代理有完整工具但从空上下文开始。用 /btw 询问 Claude 已经从此会话中知道的内容；用子代理去查找新内容。

任务列表​

在处理复杂的多步骤工作时，Claude 创建任务列表来跟踪进度。任务出现在终端的状态区域，带有指示器显示什么待处理、进行中或完成。

  • 按 Ctrl+T 切换任务列表视图。显示最多 10 个任务
  • 要查看所有任务或清除它们，直接询问 Claude："show me all tasks"或"clear all tasks"
  • 任务跨上下文压缩持久化，帮助 Claude 在较大项目上保持组织
  • 要跨会话共享任务列表，设置 CLAUDE_CODE_TASK_LIST_ID 使用 ~/.claude/tasks/ 中的命名目录：CLAUDE_CODE_TASK_LIST_ID=my-project claude

PR 审查状态​

当在带有开放拉取请求的分支上工作时，Claude Code 在底部显示可点击的 PR 链接（例如"PR #446"）。链接有带颜色的下划线指示审查状态：

  • 绿色：已批准
  • 黄色：待审查
  • 红色：请求更改
  • 灰色：草稿
  • 紫色：已合并

Cmd+点击（Mac）或 Ctrl+点击（Windows/Linux）链接在浏览器中打开拉取请求。状态每 60 秒自动更新。

注意：PR 状态需要安装并认证 gh CLI（gh auth login）。
