命令

Claude Code 中可用命令的完整参考，包括内置命令和捆绑技能。

命令在会话内控制 Claude Code。它们提供了一种快速方式来切换模型、管理权限、清除上下文、运行工作流等。

输入 / 查看所有可用命令，或输入 / 后跟字母进行过滤。

下表列出了 Claude Code 包含的所有命令。标记为 Skill 的条目是捆绑技能。它们使用与你自己编写的技能相同的机制：交给 Claude 的提示，Claude 也可以在相关时自动调用。其他都是内置命令，其行为编码在 CLI 中。要添加你自己的命令，见技能。

并非每个命令都对每个用户可见。可用性取决于你的平台、计划和环境。例如，/desktop 仅在 macOS 和 Windows 上显示，/upgrade 仅在 Pro 和 Max 计划上显示。

在下表中，<arg> 表示必需参数，[arg] 表示可选参数。

命令用途
/add-dir <path>为当前会话添加文件访问的工作目录。大多数 .claude/ 配置不会从添加的目录中发现
/agents管理代理配置
/autofix-pr [prompt]生成一个 Web 版 Claude Code 会话，监视当前分支的 PR 并在 CI 失败或审查者留下评论时推送修复。通过 gh pr view 从你检出的分支检测打开的 PR；要监视不同的 PR，先检出它的分支。默认情况下远程会话被告知修复每个 CI 失败和审查评论；传递提示给它不同的指令，例如 /autofix-pr only fix lint and type errors。需要 gh CLI 和 Web 版 Claude Code 访问权限
/batch <instruction>Skill。 并行编排代码库的大规模更改。研究代码库，将工作分解为 5 到 30 个独立单元，并呈现计划。批准后，在隔离的 git worktree 中为每个单元生成一个后台代理。每个代理实现其单元、运行测试并打开拉取请求。需要 git 仓库。示例：/batch migrate src/ from Solid to React
/btw <question>问一个快速边问而不添加到对话中
/chrome配置 Chrome 中的 Claude 设置
/claude-apiSkill。 加载你项目语言（Python、TypeScript、Java、Go、Ruby、C#、PHP 或 cURL）的 Claude API 参考材料和 Managed Agents 参考。涵盖工具使用、流式传输、批处理、结构化输出和常见陷阱。当你的代码导入 anthropic 或 @anthropic-ai/sdk 时也会自动激活
/clear清除对话历史并释放上下文。别名：/reset、/new
/color [color|default]为当前会话设置提示栏颜色。可用颜色：red、blue、green、yellow、purple、orange、pink、cyan。使用 default 重置
/compact [instructions]压缩对话，带可选的聚焦指令
/config打开设置界面以调整主题、模型、输出样式和其他偏好。别名：/settings
/context将当前上下文用量可视化为彩色网格。显示上下文密集型工具、内存膨胀和容量警告的优化建议
/copy [N]将最近的助手响应复制到剪贴板。传递数字 N 复制第 N 个最新响应：/copy 2 复制倒数第二个。当存在代码块时，显示交互式选择器以选择单个代码块或完整响应。在选择器中按 w 将选择写入文件而不是剪贴板，这在 SSH 上很有用
/cost显示 token 用量统计。见费用跟踪指南获取订阅特定的详情
/debug [description]Skill。 为当前会话启用调试日志记录并通过读取会话调试日志排查问题。除非你用 claude --debug 启动，否则调试日志默认关闭，因此在会话中途运行 /debug 从该点开始捕获日志。可选地描述问题以聚焦分析
/desktop在 Claude Code 桌面应用中继续当前会话。仅限 macOS 和 Windows。别名：/app
/diff打开交互式 diff 查看器显示未提交的更改和逐轮 diff。使用左右箭头在当前 git diff 和单个 Claude 轮次之间切换，使用上下箭头浏览文件
/doctor诊断并验证你的 Claude Code 安装和设置
/effort [low|medium|high|max|auto]设置模型努力级别。low、medium 和 high 跨会话持久化。max 仅适用于当前会话且需要 Opus 4.6。auto 重置为模型默认。不带参数时显示当前级别。立即生效，无需等待当前响应完成
/exit退出 CLI。别名：/quit
/export [filename]将当前对话导出为纯文本。带文件名时直接写入该文件。不带时打开对话框复制到剪贴板或保存到文件
/extra-usage配置额外用量以在达到速率限制时继续工作
/fast [on|off]切换快速模式开启或关闭
/feedback [report]提交关于 Claude Code 的反馈。别名：/bug
/branch [name]在此点创建当前对话的分支。别名：/fork
/help显示帮助和可用命令
/hooks查看工具事件的 hook 配置
/ide管理 IDE 集成并显示状态
/init用 CLAUDE.md 指南初始化项目。设置 CLAUDE_CODE_NEW_INIT=1 进行交互流程，还会浏览技能、hooks 和个人记忆文件
/insights生成分析报告，分析你的 Claude Code 会话，包括项目领域、交互模式和摩擦点
/install-github-app为仓库设置 Claude GitHub Actions 应用。引导你完成选择仓库和配置集成的过程
/install-slack-app安装 Claude Slack 应用。打开浏览器完成 OAuth 流程
/keybindings打开或创建你的键绑定配置文件
/login登录你的 Anthropic 账户
/logout退出 Anthropic 账户
/loop [interval] [prompt]Skill。 在会话保持打开时重复运行提示。省略间隔时 Claude 在迭代之间自定节奏。省略提示时 Claude 运行自主维护检查，或如果存在则运行 .claude/loop.md 中的提示。示例：/loop 5m check if the deploy finished。见定时运行提示
/mcp管理 MCP 服务器连接和 OAuth 认证
/memory编辑 CLAUDE.md 记忆文件，启用或禁用自动记忆，并查看自动记忆条目
/mobile显示 QR 码以下载 Claude 移动应用。别名：/ios、/android
/model [model]选择或更改 AI 模型。对于支持的模型，使用左右箭头调整努力级别。更改立即生效，无需等待当前响应完成
/passes与朋友分享一周免费的 Claude Code。仅在你的账户符合条件时可见
/permissions管理工具权限的允许、询问和拒绝规则。打开交互式对话框，你可以按作用域查看规则、添加或移除规则、管理工作目录和查看最近的自动模式拒绝。别名：/allowed-tools
/plan [description]直接从提示进入计划模式。传递可选描述以进入计划模式并立即开始该任务，例如 /plan fix the auth bug
/plugin管理 Claude Code 插件
/powerup通过带交互动画的快速课程发现 Claude Code 功能
/pr-comments [PR]在 v2.1.91 中移除。改为直接要求 Claude 查看拉取请求评论。在早期版本中，获取并显示 GitHub 拉取请求的评论；自动检测当前分支的 PR，或传递 PR URL 或编号。需要 gh CLI
/privacy-settings查看和更新你的隐私设置。仅适用于 Pro 和 Max 计划订阅者
/release-notes在交互式版本选择器中查看变更日志。选择特定版本查看其发行说明，或选择显示所有版本
/reload-plugins重新加载所有活动插件以应用待处理的更改而无需重启。报告每个重新加载组件的数量并标记任何加载错误
/remote-control使此会话可从 claude.ai 进行远程控制。别名：/rc
/remote-env配置用 --remote 启动的 web 会话的默认远程环境
/rename [name]重命名当前会话并在提示栏显示名称。不带名称时从对话历史自动生成一个
/resume [session]按 ID 或名称恢复对话，或打开会话选择器。别名：/continue
/review已弃用。改为安装 code-review 插件：claude plugin install code-review@claude-plugins-official
/rewind将对话和/或代码回滚到之前的点，或从选定的消息开始总结。见检查点。别名：/checkpoint
/sandbox切换沙箱模式。仅在支持的平台上可用
/schedule [description]创建、更新、列出或运行 Cloud 定时任务。Claude 通过对话引导你完成设置
/security-review分析当前分支上的待提交更改的安全漏洞。审查 git diff 并识别注入、认证问题和数据暴露等风险
/setup-bedrock通过交互式向导配置 Amazon Bedrock 认证、区域和模型固定。仅在设置 CLAUDE_CODE_USE_BEDROCK=1 时可见。首次使用 Bedrock 的用户也可以从登录屏幕访问此向导
/setup-vertex通过交互式向导配置 Google Vertex AI 认证、项目、区域和模型固定。仅在设置 CLAUDE_CODE_USE_VERTEX=1 时可见。首次使用 Vertex AI 的用户也可以从登录屏幕访问此向导
/simplify [focus]Skill。 审查你最近更改的文件的代码复用、质量和效率问题，然后修复它们。并行生成三个审查代理，汇总他们的发现并应用修复。传递文本以聚焦特定问题：/simplify focus on memory efficiency
/skills列出可用技能
/stats可视化每日用量、会话历史、连续天和模型偏好
/status打开设置界面（状态选项卡）显示版本、模型、账户和连接状态。在 Claude 响应时工作，无需等待当前响应完成
/statusline配置 Claude Code 的状态行。描述你想要的，或不带参数运行以从你的 shell 提示自动配置
/stickers订购 Claude Code 贴纸
/tasks列出和管理后台任务。也可用作 /bashes
/teleport将 Web 版 Claude Code 会话拉入此终端：打开选择器，然后获取分支和对话。也可用作 /tp。需要 claude.ai 订阅
/terminal-setup配置终端键绑定以实现 Shift+Enter 和其他快捷方式。仅在需要它的终端中可见，如 VS Code、Alacritty 或 Warp
/theme更改颜色主题。包括亮色和暗色变体、色盲友好（daltonized）主题和使用终端调色板的 ANSI 主题
/ultraplan <prompt>在 ultraplan 会话中起草计划，在浏览器中审查，然后远程执行或发送回终端
/upgrade打开升级页面以切换到更高级别的计划
/usage显示计划用量限制和速率限制状态
/vim在 v2.1.92 中移除。要在 Vim 和普通编辑模式之间切换，使用 /config → Editor mode
/voice切换按键说话语音听写。需要 Claude.ai 账户
/web-setup使用本地 gh CLI 凭据将你的 GitHub 账户连接到 Web 版 Claude Code。如果 GitHub 未连接，/schedule 会自动提示此操作

MCP 提示​

MCP 服务器可以暴露显示为命令的提示。这些使用 /mcp__<server>__<prompt> 格式，并从连接的服务器动态发现。详情见 MCP 提示。
