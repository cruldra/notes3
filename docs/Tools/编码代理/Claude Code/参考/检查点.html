检查点

跟踪、回滚和总结 Claude 的编辑和对话以管理会话状态。

Claude Code 在你工作时自动跟踪 Claude 的文件编辑，允许你快速撤销更改并回滚到之前的状态，以防出现问题。

检查点如何工作​

当你与 Claude 一起工作时，检查点会在每次编辑之前自动捕获你的代码状态。这个安全网让你可以进行雄心勃勃的大规模任务，知道你始终可以返回到之前的代码状态。

自动跟踪​

Claude Code 跟踪其文件编辑工具所做的所有更改：

  • 每个用户提示都会创建一个新的检查点
  • 检查点跨会话持久化，因此你可以在恢复的对话中访问它们
  • 在 30 天后与会话一起自动清理（可配置）

回滚和总结​

按两次 Esc（Esc + Esc）或使用 /rewind 命令打开回滚菜单。可滚动列表显示会话中的每个提示。选择你想要操作的点，然后选择一个操作：

  • 恢复代码和对话：将代码和对话都回滚到该点
  • 恢复对话：回滚到该消息同时保持当前代码
  • 恢复代码：撤销文件更改同时保持对话
  • 从此处总结：将从该点向前的对话压缩为摘要，释放上下文窗口空间
  • 算了：返回消息列表而不进行更改

在恢复对话或总结后，所选消息的原始提示会恢复到输入字段中，这样你可以重新发送或编辑它。

恢复 vs 总结​

三个恢复选项回滚状态：它们撤销代码更改、对话历史或两者。"从此处总结"的工作方式不同：

  • 所选消息之前的消息保持完整
  • 所选消息和所有后续消息被替换为紧凑的 AI 生成摘要
  • 磁盘上的文件不会更改
  • 原始消息在会话转录中被保留，因此 Claude 可以在需要时引用细节

这类似于 /compact，但是有针对性的：不是总结整个对话，而是保持早期上下文的完整细节，仅压缩占用空间的部分。你可以输入可选指令来指导摘要聚焦的内容。

注意：总结让你保持在同一会话中并压缩上下文。如果你想要分支并尝试不同的方法同时保持原始会话完整，改用 fork（claude --continue --fork-session）。

常见用例​

检查点在以下情况下特别有用：

  • 探索替代方案：尝试不同的实现方法而不丢失起点
  • 从错误中恢复：快速撤销引入 bug 或破坏功能的更改
  • 迭代功能：在实验变体时知道你可以回滚到工作状态
  • 释放上下文空间：从中间点向前总结冗长的调试会话，保持初始指令完整

限制​

不跟踪 bash 命令更改​

检查点不跟踪被 bash 命令修改的文件。例如，如果 Claude Code 运行：

rm file.txt
mv old.txt new.txt
cp source.txt dest.txt

这些文件修改无法通过回滚撤消。仅跟踪通过 Claude 的文件编辑工具进行的直接文件编辑。

不跟踪外部更改​

检查点仅跟踪在当前会话中已编辑的文件。你在 Claude Code 之外对文件的手动更改和其他并发会话的编辑通常不会被捕获，除非它们恰好修改了与当前会话相同的文件。

不是版本控制的替代​

检查点专为快速、会话级别的恢复而设计。对于永久版本历史和协作：

  • 继续使用版本控制（如 Git）进行提交、分支和长期历史
  • 检查点补充但不替代正确的版本控制
  • 将检查点视为"本地撤消"，将 Git 视为"永久历史"
