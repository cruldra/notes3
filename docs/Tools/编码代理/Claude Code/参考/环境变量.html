环境变量

控制 Claude Code 行为的环境变量完整参考。

Claude Code 支持以下环境变量来控制其行为。在启动 claude 之前在 shell 中设置它们，或在 settings.json 的 env 键下配置它们以应用于每个会话或推广到你的团队。

变量用途
ANTHROPIC_API_KEY作为 X-Api-Key 头部发送的 API 密钥。设置时，即使你已登录，此密钥也会代替你的 Claude Pro、Max、Team 或 Enterprise 订阅使用。在非交互模式（-p）中，密钥在存在时始终使用。在交互模式中，你会被提示批准一次密钥才能覆盖你的订阅。要改用订阅，运行 unset ANTHROPIC_API_KEY
ANTHROPIC_AUTH_TOKENAuthorization 头部的自定义值（你在这里设置的值将以 Bearer 为前缀）
ANTHROPIC_BASE_URL覆盖 API 端点以通过代理或网关路由请求。当设置为非第一方主机时，MCP 工具搜索默认禁用。如果你的代理转发 tool_reference 块，设置 ENABLE_TOOL_SEARCH=true
ANTHROPIC_BEDROCK_BASE_URL覆盖 Bedrock 端点 URL。用于自定义 Bedrock 端点或通过 LLM 网关路由时。见 Amazon Bedrock
ANTHROPIC_BEDROCK_MANTLE_BASE_URL覆盖 Bedrock Mantle 端点 URL。见 Mantle 端点
ANTHROPIC_BETAS要包含在 API 请求中的额外 anthropic-beta 头部值的逗号分隔列表。Claude Code 已经发送它需要的 beta 头部；用这个在 Claude Code 添加原生支持之前选择加入 Anthropic API beta。与需要 API 密钥认证的 --betas 标志不同，这个变量适用于所有认证方法包括 Claude.ai 订阅
ANTHROPIC_CUSTOM_HEADERS添加到请求的自定义头部（Name: Value 格式，多个头部用换行分隔）
ANTHROPIC_CUSTOM_MODEL_OPTION作为自定义条目添加到 /model 选择器的模型 ID。用这个使非标准或网关特定的模型可选而不替换内置别名。见模型配置
ANTHROPIC_CUSTOM_MODEL_OPTION_DESCRIPTION/model 选择器中自定义模型条目的显示描述。未设置时默认为 Custom model (<model-id>)
ANTHROPIC_CUSTOM_MODEL_OPTION_NAME/model 选择器中自定义模型条目的显示名称。未设置时默认为模型 ID
ANTHROPIC_DEFAULT_HAIKU_MODEL见模型配置
ANTHROPIC_DEFAULT_HAIKU_MODEL_DESCRIPTION见模型配置
ANTHROPIC_DEFAULT_HAIKU_MODEL_NAME见模型配置
ANTHROPIC_DEFAULT_HAIKU_MODEL_SUPPORTED_CAPABILITIES见模型配置
ANTHROPIC_DEFAULT_OPUS_MODEL见模型配置
ANTHROPIC_DEFAULT_OPUS_MODEL_DESCRIPTION见模型配置
ANTHROPIC_DEFAULT_OPUS_MODEL_NAME见模型配置
ANTHROPIC_DEFAULT_OPUS_MODEL_SUPPORTED_CAPABILITIES见模型配置
ANTHROPIC_DEFAULT_SONNET_MODEL见模型配置
ANTHROPIC_DEFAULT_SONNET_MODEL_DESCRIPTION见模型配置
ANTHROPIC_DEFAULT_SONNET_MODEL_NAME见模型配置
ANTHROPIC_DEFAULT_SONNET_MODEL_SUPPORTED_CAPABILITIES见模型配置
ANTHROPIC_FOUNDRY_API_KEY用于 Microsoft Foundry 认证的 API 密钥（见 Microsoft Foundry）
ANTHROPIC_FOUNDRY_BASE_URLFoundry 资源的完整基础 URL（例如 https://my-resource.services.ai.azure.com/anthropic）。ANTHROPIC_FOUNDRY_RESOURCE 的替代方案（见 Microsoft Foundry）
ANTHROPIC_FOUNDRY_RESOURCEFoundry 资源名称（例如 my-resource）。如果未设置 ANTHROPIC_FOUNDRY_BASE_URL 则需要（见 Microsoft Foundry）
ANTHROPIC_MODEL要使用的模型设置名称（见模型配置）
ANTHROPIC_SMALL_FAST_MODEL[已弃用] 用于后台任务的 Haiku 类模型名称
ANTHROPIC_SMALL_FAST_MODEL_AWS_REGION使用 Bedrock 或 Bedrock Mantle 时覆盖 Haiku 类模型的 AWS 区域
ANTHROPIC_VERTEX_BASE_URL覆盖 Vertex AI 端点 URL。用于自定义 Vertex 端点或通过 LLM 网关路由时。见 Google Vertex AI
ANTHROPIC_VERTEX_PROJECT_IDVertex AI 的 GCP 项目 ID。使用 Google Vertex AI时需要
API_TIMEOUT_MSAPI 请求的超时时间（毫秒）（默认：600000，即 10 分钟；最大：2147483647）。在网络慢或通过代理路由时增加此值。超过最大值的值会溢出底层定时器并导致请求立即失败
AWS_BEARER_TOKEN_BEDROCKBedrock API 密钥用于认证（见 Bedrock API 密钥）
BASH_DEFAULT_TIMEOUT_MS长时间运行的 bash 命令的默认超时时间（默认：120000，即 2 分钟）
BASH_MAX_OUTPUT_LENGTHbash 输出中的最大字符数，超过时从中间截断
BASH_MAX_TIMEOUT_MS模型可以为长时间运行的 bash 命令设置的最大超时时间（默认：600000，即 10 分钟）
CCR_FORCE_BUNDLE设置为 1 强制 claude --remote 打包并上传你的本地仓库，即使 GitHub 访问可用
CLAUDECODE在 Claude Code 生成的 shell 环境（Bash 工具、tmux 会话）中设置为 1。不在 hooks 或状态行命令中设置。用于检测脚本是否在 Claude Code 生成的 shell 中运行
CLAUDE_AGENT_SDK_DISABLE_BUILTIN_AGENTS设置为 1 禁用所有内置子代理类型如 Explore 和 Plan。仅适用于非交互模式（-p 标志）。对想要空白画布的 SDK 用户有用
CLAUDE_AGENT_SDK_MCP_NO_PREFIX设置为 1 在 SDK 创建的 MCP 服务器的工具名称上跳过 mcp__<server>__ 前缀。工具使用它们的原始名称。仅限 SDK 使用
CLAUDE_AUTOCOMPACT_PCT_OVERRIDE设置自动压缩触发的上下文容量百分比（1-100）。默认情况下，自动压缩在约 95% 容量时触发。使用较低的值如 50 以更早压缩。高于默认阈值的值无效。适用于主对话和子代理。此百分比与状态行中可用的 context_window.used_percentage 字段对齐
CLAUDE_AUTO_BACKGROUND_TASKS设置为 1 强制启用长时间运行的代理任务的自动后台化。启用时，子代理在运行约两分钟后被移到后台
CLAUDE_BASH_MAINTAIN_PROJECT_WORKING_DIR在主会话中每个 Bash 或 PowerShell 命令后返回到原始工作目录
CLAUDE_CODE_ACCESSIBILITY设置为 1 保持原生终端光标可见并禁用反文本光标指示器。允许屏幕放大器如 macOS Zoom 跟踪光标位置
CLAUDE_CODE_ADDITIONAL_DIRECTORIES_CLAUDE_MD设置为 1 从 --add-dir 指定的目录加载 CLAUDE.md 文件。默认情况下，附加目录不加载记忆文件
CLAUDE_CODE_API_KEY_HELPER_TTL_MS凭证应刷新的间隔毫秒数（使用 apiKeyHelper时）
CLAUDE_CODE_AUTO_COMPACT_WINDOW设置用于自动压缩计算的 token 上下文容量。默认为模型的上下文窗口：标准模型为 200K 或扩展上下文模型为 1M。在 1M 模型上使用较低的值如 500000 以将窗口视为 500K 用于压缩目的。该值以模型的实际上下文窗口为上限。CLAUDE_AUTOCOMPACT_PCT_OVERRIDE 作为此值的百分比应用。设置此变量将压缩阈值与状态行的 used_percentage 解耦，后者始终使用模型的完整上下文窗口
CLAUDE_CODE_AUTO_CONNECT_IDE覆盖自动 IDE 连接。默认情况下，Claude Code 在受支持的 IDE 集成终端内启动时自动连接。设置为 false 以防止此行为。设置为 true 在自动检测失败时强制连接尝试，如 tmux 遮蔽父终端时
CLAUDE_CODE_CERT_STORETLS 连接的 CA 证书源的逗号分隔列表。bundled 是随 Claude Code 提供的 Mozilla CA 集。system 是操作系统信任库。默认是 bundled,system。需要原生二进制发行版才能与系统集成。在 Node.js 运行时上，无论此值如何，仅使用捆绑集
CLAUDE_CODE_CLIENT_CERT用于 mTLS 认证的客户端证书文件路径
CLAUDE_CODE_CLIENT_KEY用于 mTLS 认证的客户端私钥文件路径
CLAUDE_CODE_CLIENT_KEY_PASSPHRASE加密的 CLAUDE_CODE_CLIENT_KEY 的密码短语（可选）
CLAUDE_CODE_DEBUG_LOGS_DIR覆盖调试日志文件路径。尽管名称如此，这是文件路径而不是目录。需要通过 --debug 或 /debug 单独启用调试模式：仅设置此变量不会启用日志记录。--debug-file标志同时完成这两项。默认为 ~/.claude/debug/<session-id>.txt
CLAUDE_CODE_DEBUG_LOG_LEVEL写入调试日志文件的最低日志级别。值：verbose、debug（默认）、info、warn、error。设置为 verbose 以包含高容量诊断如完整状态行命令输出，或提高到 error 以减少噪音
CLAUDE_CODE_DISABLE_1M_CONTEXT设置为 1 禁用 100 万上下文窗口支持。设置时，100 万模型变体在模型选择器中不可用。适用于有合规要求的企业环境
CLAUDE_CODE_DISABLE_ADAPTIVE_THINKING设置为 1 为 Opus 4.6 和 Sonnet 4.6 禁用自适应推理。禁用时，这些模型回退到由 MAX_THINKING_TOKENS 控制的固定思考预算
CLAUDE_CODE_DISABLE_ATTACHMENTS设置为 1 禁用附件处理。带 @ 语法的文件提及作为纯文本发送而不是扩展为文件内容
CLAUDE_CODE_DISABLE_AUTO_MEMORY设置为 1 禁用自动记忆。设置为 0 在逐渐推出期间强制开启自动记忆。禁用时，Claude 不创建或加载自动记忆文件
CLAUDE_CODE_DISABLE_BACKGROUND_TASKS设置为 1 禁用所有后台任务功能，包括 Bash 和子代理工具上的 run_in_background 参数、自动后台化和 Ctrl+B 快捷键
CLAUDE_CODE_DISABLE_CLAUDE_MDS设置为 1 防止将任何 CLAUDE.md 记忆文件加载到上下文中，包括用户、项目和自动记忆文件
CLAUDE_CODE_DISABLE_CRON设置为 1 禁用定时任务。/loop 技能和 cron 工具变得不可用，任何已调度的任务停止触发，包括会话中途运行的任务
CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS设置为 1 从 API 请求中剥离 Anthropic 特定的 anthropic-beta 请求头部和 beta 工具 schema 字段（如 defer_loading 和 eager_input_streaming）。当代理网关拒绝带有"Unexpected value(s) for the anthropic-beta header"或"Extra inputs are not permitted"等错误的请求时使用。标准字段（name、description、input_schema、cache_control）被保留
CLAUDE_CODE_DISABLE_FAST_MODE设置为 1 禁用快速模式
CLAUDE_CODE_DISABLE_FEEDBACK_SURVEY设置为 1 禁用"Claude 做得怎么样？"会话质量调查。当设置 DISABLE_TELEMETRY 或 CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC 时也会禁用调查。见会话质量调查
CLAUDE_CODE_DISABLE_FILE_CHECKPOINTING设置为 1 禁用文件检查点。/rewind 命令将无法恢复代码更改
CLAUDE_CODE_DISABLE_GIT_INSTRUCTIONS设置为 1 从 Claude 的系统提示中移除内置的 commit 和 PR 工作流指令以及 git 状态快照。使用你自己的 git 工作流技能时有用。设置时优先于 includeGitInstructions设置
CLAUDE_CODE_DISABLE_LEGACY_MODEL_REMAP设置为 1 防止在 Anthropic API 上将 Opus 4.0 和 4.1 自动重新映射到当前 Opus 版本。当你有意想要固定到旧模型时使用。重新映射不在 Bedrock、Vertex 或 Foundry 上运行
CLAUDE_CODE_DISABLE_MOUSE设置为 1 禁用全屏渲染中的鼠标跟踪。使用 PgUp 和 PgDn 的键盘滚动仍然有效。用这个保持终端的原生复制选择行为
CLAUDE_CODE_DISABLE_NONESSENTIAL_TRAFFIC等同于同时设置 DISABLE_AUTOUPDATER、DISABLE_FEEDBACK_COMMAND、DISABLE_ERROR_REPORTING 和 DISABLE_TELEMETRY
CLAUDE_CODE_DISABLE_NONSTREAMING_FALLBACK设置为 1 在流式请求中途失败时禁用非流式回退。流式错误传播到重试层。当代理或网关导致回退产生重复工具执行时有用
CLAUDE_CODE_DISABLE_OFFICIAL_MARKETPLACE_AUTOINSTALL设置为 1 跳过首次运行时自动添加官方插件市场
CLAUDE_CODE_DISABLE_TERMINAL_TITLE设置为 1 禁用基于对话上下文的自动终端标题更新
CLAUDE_CODE_DISABLE_THINKING设置为 1 强制禁用扩展思考，无论模型支持或其他设置如何。比 MAX_THINKING_TOKENS=0 更直接
CLAUDE_CODE_EFFORT_LEVEL设置支持的模型的努力级别。值：low、medium、high、max（仅 Opus 4.6）或 auto 使用模型默认。优先于 /effort 和 effortLevel 设置。见调整努力级别
CLAUDE_CODE_ENABLE_FINE_GRAINED_TOOL_STREAMING设置为 1 强制启用细粒度工具输入流式传输。没有这个，API 会在发送 delta 事件之前完全缓冲工具输入参数，这可能会延迟大工具输入的显示。仅限 Anthropic API：对 Bedrock、Vertex 或 Foundry 无效
CLAUDE_CODE_ENABLE_PROMPT_SUGGESTION设置为 false 禁用提示建议（/config 中的"Prompt suggestions"切换）。这些是在 Claude 响应后出现在提示输入中的灰色预测。见提示建议
CLAUDE_CODE_ENABLE_TASKS设置为 1 在非交互模式（-p 标志）中启用任务跟踪系统。任务在交互模式中默认开启。见任务列表
CLAUDE_CODE_ENABLE_TELEMETRY设置为 1 启用 OpenTelemetry 数据收集用于指标和日志记录。配置 OTel 导出器之前需要。见监控
CLAUDE_CODE_EXIT_AFTER_STOP_DELAY查询循环变为空闲后自动退出前等待的毫秒数。适用于使用 SDK 模式的自动化工作流和脚本
CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS设置为 1 启用代理团队。代理团队是实验性的且默认禁用
CLAUDE_CODE_FILE_READ_MAX_OUTPUT_TOKENS覆盖文件读取的默认 token 限制。当你需要完整读取更大文件时有用
CLAUDE_CODE_GIT_BASH_PATH仅限 Windows：Git Bash 可执行文件（bash.exe）的路径。当 Git Bash 已安装但不在 PATH 中时使用。见 Windows 设置
CLAUDE_CODE_GLOB_HIDDEN设置为 false 在 Claude 调用 Glob 工具时从结果中排除点文件。默认包含。不影响 @ 文件自动补全、ls、Grep 或 Read
CLAUDE_CODE_GLOB_NO_IGNORE设置为 false 使 Glob 工具尊重 .gitignore 模式。默认情况下，Glob 返回所有匹配文件包括 gitignore 的文件。不影响 @ 文件自动补全，它有自己的 respectGitignore 设置
CLAUDE_CODE_GLOB_TIMEOUT_SECONDSGlob 工具文件发现的超时秒数。大多数平台默认 20 秒，WSL 上默认 60 秒
CLAUDE_CODE_IDE_HOST_OVERRIDE覆盖连接到 IDE 扩展使用的主机地址。默认情况下 Claude Code 自动检测正确的地址，包括 WSL 到 Windows 路由
CLAUDE_CODE_IDE_SKIP_AUTO_INSTALL跳过 IDE 扩展的自动安装。等同于将 autoInstallIdeExtension 设置为 false
CLAUDE_CODE_IDE_SKIP_VALID_CHECK设置为 1 在连接期间跳过 IDE lockfile 条目的验证。当自动连接失败但 IDE 正在运行时使用
CLAUDE_CODE_MAX_OUTPUT_TOKENS设置大多数请求的最大输出 token 数。默认值和上限因模型而异；见max output tokens。增加此值会减少自动压缩触发前可用的有效上下文窗口
CLAUDE_CODE_MAX_RETRIES覆盖重试失败 API 请求的次数（默认：10）
CLAUDE_CODE_MAX_TOOL_USE_CONCURRENCY可以并行执行的只读工具和子代理的最大数量（默认：10）。更高的值增加并行性但消耗更多资源
CLAUDE_CODE_NEW_INIT设置为 1 使 /init 运行交互式设置流程。流程询问要生成哪些文件，包括 CLAUDE.md、技能和 hooks，然后在探索代码库并写入它们。没有这个变量，/init 自动生成 CLAUDE.md 而不提示
CLAUDE_CODE_NO_FLICKER设置为 1 启用全屏渲染，一项减少闪烁并在长对话中保持内存平稳的研究预览
CLAUDE_CODE_OAUTH_REFRESH_TOKENClaude.ai 认证的 OAuth 刷新令牌。设置时，claude auth login 直接交换此令牌而不是打开浏览器。需要 CLAUDE_CODE_OAUTH_SCOPES。适用于在自动化环境中配置认证
CLAUDE_CODE_OAUTH_SCOPES刷新令牌被授予时的空格分隔 OAuth 作用域，如 "user:profile user:inference user:sessions:claude_code"。设置 CLAUDE_CODE_OAUTH_REFRESH_TOKEN 时需要
CLAUDE_CODE_OAUTH_TOKENClaude.ai 认证的 OAuth 访问令牌。SDK 和自动化环境中 /login 的替代方案。优先于钥匙串存储的凭证。用 claude setup-token 生成一个
CLAUDE_CODE_OTEL_FLUSH_TIMEOUT_MS刷新待处理 OpenTelemetry span 的超时毫秒数（默认：5000）。见监控
CLAUDE_CODE_OTEL_HEADERS_HELPER_DEBOUNCE_MS刷新动态 OpenTelemetry 头部的间隔毫秒数（默认：1740000 / 29 分钟）。见动态头部
CLAUDE_CODE_OTEL_SHUTDOWN_TIMEOUT_MSOpenTelemetry 导出器在关闭时完成的超时毫秒数（默认：2000）。如果退出时指标丢失则增加。见监控
CLAUDE_CODE_PERFORCE_MODE设置为 1 启用 Perforce 感知写保护。设置时，如果目标文件缺少 owner-write 位（Perforce 在同步文件上清除直到 p4 edit 打开它们），Edit、Write 和 NotebookEdit 会失败并提示 p4 edit <file>。这防止 Claude Code 绕过 Perforce 变更跟踪
CLAUDE_CODE_PLUGIN_CACHE_DIR覆盖插件根目录。尽管名称如此，这设置的是父目录而不是缓存本身：市场和插件缓存存在于该路径的子目录中。默认为 ~/.claude/plugins
CLAUDE_CODE_PLUGIN_GIT_TIMEOUT_MS安装或更新插件时 git 操作的超时毫秒数（默认：120000）。对于大仓库或慢网络连接增加此值。见git 操作超时
CLAUDE_CODE_PLUGIN_KEEP_MARKETPLACE_ON_FAILURE设置为 1 在 git pull 失败时保留现有市场缓存而不是擦除并重新克隆。适用于离线或气隙环境，其中重新克隆会以相同方式失败。见离线环境中市场更新失败
CLAUDE_CODE_PLUGIN_SEED_DIR一个或多个只读插件种子目录的路径，Unix 上用 : 分隔或 Windows 上用 ; 分隔。用这个将预填充的插件目录捆绑到容器镜像中。Claude Code 在启动时从这些目录注册市场并使用预缓存的插件而不重新克隆。见为容器预填充插件
CLAUDE_CODE_PROXY_RESOLVES_HOSTS设置为 1 允许代理执行 DNS 解析而不是调用方。适用于代理应处理主机名解析的环境
CLAUDE_CODE_RESUME_INTERRUPTED_TURN设置为 1 如果上一个会话在轮次中途结束则自动恢复。用于 SDK 模式，这样模型可以继续而无需 SDK 重新发送提示
CLAUDE_CODE_SCRIPT_CAPS当设置 CLAUDE_CODE_SUBPROCESS_ENV_SCRUB 时，JSON 对象限制每个会话中特定脚本可调用的次数。键是与命令文本匹配的子串；值是整数调用限制。例如 {"deploy.sh": 2} 允许 deploy.sh 最多调用两次。匹配是基于子串的，所以 ./scripts/deploy.sh $(evil) 等 shell 扩展技巧仍然计入上限。通过 xargs 或 find -exec 的运行时扇出不会被检测；这是纵深防御控制
CLAUDE_CODE_SCROLL_SPEED在全屏渲染中设置鼠标滚轮滚动倍增器。接受 1 到 20 的值。如果你的终端每个缺口发送一个滚轮事件而没有放大，设置为 3 以匹配 vim
CLAUDE_CODE_SESSIONEND_HOOKS_TIMEOUT_MSSessionEnd hooks 完成的最大毫秒数（默认：1500）。适用于会话退出、/clear 和通过交互式 /resume 切换会话。每个 hook 的 timeout 值也受此预算限制
CLAUDE_CODE_SHELL覆盖自动 shell 检测。当你的登录 shell 与首选工作 shell 不同时有用（例如 bash vs zsh）
CLAUDE_CODE_SHELL_PREFIX包装所有 bash 命令的命令前缀（例如用于日志记录或审计）。示例：/path/to/logger.sh 将执行 /path/to/logger.sh <command>
CLAUDE_CODE_SIMPLE设置为 1 以最小系统提示和仅 Bash、文件读取和文件编辑工具运行。来自 --mcp-config 的 MCP 工具仍然可用。禁用 hooks、技能、插件、MCP 服务器、自动记忆和 CLAUDE.md 的自动发现。--bare CLI 标志设置此项

CLAUDE_CODE_SKIP_FOUNDRY_AUTH跳过 Microsoft Foundry 的 Azure 认证（例如使用 LLM 网关时）
CLAUDE_CODE_SKIP_MANTLE_AUTH跳过 Bedrock Mantle 的 AWS 认证（例如使用 LLM 网关时）
CLAUDE_CODE_SKIP_VERTEX_AUTH跳过 Vertex 的 Google 认证（例如使用 LLM 网关时）
CLAUDE_CODE_SUBAGENT_MODEL见模型配置
CLAUDE_CODE_SUBPROCESS_ENV_SCRUB设置为 1 从子进程环境（Bash 工具、hooks、MCP stdio 服务器）中剥离 Anthropic 和云提供商凭证。父 Claude 进程保留这些凭证用于 API 调用，但子进程无法读取它们，减少通过 shell 扩展尝试泄露秘密的提示注入攻击的暴露。在 Linux 上，这还将 Bash 子进程运行在隔离的 PID 命名空间中，这样它们无法通过 /proc 读取宿主进程环境；副作用是 ps、pgrep 和 kill 无法看到或发信号给宿主进程。配置 allowed_non_write_users 时 claude-code-action 自动设置此项
CLAUDE_CODE_SYNC_PLUGIN_INSTALL在非交互模式（-p 标志）中设置为 1 在第一次查询前等待插件安装完成。没有这个，插件在后台安装且可能在第一轮不可用。与 CLAUDE_CODE_SYNC_PLUGIN_INSTALL_TIMEOUT_MS 组合以限制等待
CLAUDE_CODE_SYNC_PLUGIN_INSTALL_TIMEOUT_MS同步插件安装的超时毫秒数。超出时，Claude Code 在没有插件的情况下继续并记录错误。无默认值：没有此变量时，同步安装等待直到完成
CLAUDE_CODE_SYNTAX_HIGHLIGHT设置为 false 禁用 diff 输出中的语法高亮。当颜色干扰你的终端设置时有用
CLAUDE_CODE_TASK_LIST_ID跨会话共享任务列表。在多个 Claude Code 实例中设置相同的 ID 以协调共享任务列表。见任务列表
CLAUDE_CODE_TEAM_NAME此队友所属的代理团队名称。在代理团队成员上自动设置
CLAUDE_CODE_TMPDIR覆盖用于内部临时文件的临时目录。Claude Code 在此路径后附加 /claude-{uid}/（Unix）或 /claude/（Windows）。默认：macOS 上为 /tmp，Linux/Windows 上为 os.tmpdir()

CLAUDE_CODE_USE_VERTEX使用 Vertex
CLAUDE_CONFIG_DIR覆盖配置目录（默认：~/.claude）。所有设置、凭证、会话历史和插件都存储在此路径下。适用于并排运行多个账户：例如 alias claude-work='CLAUDE_CONFIG_DIR=~/.claude-work claude'
CLAUDE_ENABLE_STREAM_WATCHDOG设置为 1 中止 90 秒内没有数据的停滞 API 响应流。适用于无人值守环境中挂起的会话不会被注意到的情况，或在静默断开连接的代理后面。没有这个，停滞的流会无限期挂起会话，因为请求超时只覆盖初始连接。用 CLAUDE_STREAM_IDLE_TIMEOUT_MS 配置超时
CLAUDE_ENV_FILEClaude Code 在每个 Bash 命令之前 source 的 shell 脚本路径。用于跨命令保持 virtualenv 或 conda 激活。也由 SessionStart、CwdChanged 和 FileChanged hooks 动态填充
CLAUDE_REMOTE_CONTROL_SESSION_NAME_PREFIX未提供显式名称时自动生成的远程控制会话名称前缀。默认为你的机器主机名，产生如 myhost-graceful-unicorn 的名称。--remote-control-session-name-prefix CLI 标志为单次调用设置相同的值
CLAUDE_STREAM_IDLE_TIMEOUT_MS流式空闲看门狗关闭停滞连接之前的超时毫秒数。默认：90000（90 秒）。需要 CLAUDE_ENABLE_STREAM_WATCHDOG=1。如果长时间运行的工具或慢速网络导致过早超时错误，增加此值
