优化你的终端设置

当终端正确配置时，Claude Code 的效果最佳。遵循这些指南来优化你的体验。

主题和外观​

Claude 无法控制终端的主题。这由你的终端应用程序处理。你可以通过 /config 命令随时将 Claude Code 的主题与终端匹配。

要对 Claude Code 界面本身进行额外定制，你可以配置自定义状态行在终端底部显示当前模型、工作目录或 git 分支等上下文信息。

换行​

你有几种选项可以在 Claude Code 中输入换行：

  • 快速转义：输入 \ 后跟 Enter 创建新行
  • Ctrl+J：发送换行符，在任何终端中无需配置即可作为新行工作
  • Shift+Enter：在 iTerm2、WezTerm、Ghostty 和 Kitty 中开箱即用
  • 键盘快捷键：在其他终端中设置键绑定以插入新行

为其他终端设置 Shift+Enter

在 Claude Code 中运行 /terminal-setup 自动为 VS Code、Alacritty、Zed 和 Warp 配置 Shift+Enter。

注意：/terminal-setup 命令仅在需要手动配置的终端中可见。如果你使用 iTerm2、WezTerm、Ghostty 或 Kitty，你不会看到这个命令，因为 Shift+Enter 已经原生工作。

设置 Option+Enter（VS Code、iTerm2 或 macOS Terminal.app）

对于 Mac Terminal.app：

  1. 打开设置 → 配置文件 → 键盘
  2. 勾选"Use Option as Meta Key"

对于 iTerm2：

  1. 打开设置 → 配置文件 → 键
  2. 在常规下，将左/右 Option 键设置为"Esc+"

对于 VS Code 终端：

在 VS Code 设置中设置 "terminal.integrated.macOptionIsMeta": true。

通知设置​

当 Claude 完成工作并等待你的输入时，它会触发通知事件。你可以通过终端将此事件显示为桌面通知，或使用通知 hooks运行自定义逻辑。

终端通知​

Kitty 和 Ghostty 支持无需额外配置的桌面通知。iTerm 2 需要设置：

  1. 打开 iTerm 2 设置 → 配置文件 → 终端
  2. 启用"Notification Center Alerts"
  3. 点击"Filter Alerts"并勾选"Send escape sequence-generated alerts"

如果通知没有出现，验证你的终端应用在操作系统设置中有通知权限。

在 tmux 内运行 Claude Code 时，通知和终端进度条只有在你的 tmux 配置中启用直通时才能到达外部终端（如 iTerm2、Kitty 或 Ghostty）：

set -g allow-passthrough on

没有这个设置，tmux 会拦截转义序列，它们不会到达终端应用程序。

其他终端，包括默认的 macOS Terminal，不支持原生通知。改用通知 hooks。

通知 hooks​

要在通知触发时添加自定义行为，如播放声音或发送消息，配置通知 hook。hooks 与终端通知一起运行，而不是替代。

减少闪烁和内存使用​

如果你在长时间会话中看到闪烁，或者当 Claude 工作时你的终端滚动位置跳到顶部，尝试全屏渲染。它使用备用渲染路径，保持内存平稳并添加鼠标支持。用 CLAUDE_CODE_NO_FLICKER=1 启用它。

处理大输入​

当处理大量代码或长指令时：

  • 避免直接粘贴：Claude Code 可能难以处理非常长的粘贴内容
  • 使用基于文件的工作流：将内容写入文件并要求 Claude 读取它
  • 注意 VS Code 的限制：VS Code 终端特别容易截断长粘贴

Vim 模式​

Claude Code 支持 Vim 键绑定的子集，可以通过 /config → Editor mode 启用。要直接在配置文件中设置模式，在 ~/.claude.json 中将 editorMode 全局配置键设置为 "vim"。

支持的子集包括：

  • 模式切换：Esc（到 NORMAL）、i/I、a/A、o/O（到 INSERT）
  • 导航：h/j/k/l、w/e/b、0/$/^、gg/G、f/F/t/T 带 ;/, 重复
  • 编辑：x、dw/de/db/dd/D、cw/ce/cb/cc/C、.（重复）
  • 复制/粘贴：yy/Y、yw/ye/yb、p/P
  • 文本对象：iw/aw、iW/aW、i"/a"、i'/a'、i(/a(、i[/a[、i{/a{
  • 缩进：>>/<<
  • 行操作：J（连接行）

见交互模式获取完整参考。
