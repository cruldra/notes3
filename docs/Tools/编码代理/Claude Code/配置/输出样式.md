# 输出样式

> 让 Claude Code 适配软件工程之外的用途

输出样式改变 Claude 的**回应方式**，而不是它知道什么。它们通过修改系统提示来设定角色、语气和输出格式，同时保留核心能力如运行脚本、读写文件和跟踪 TODO。当你每轮都要反复提示相同的声音或格式时使用，或者当你想让 Claude 充当软件工程师以外的角色时使用。

关于项目的指令、约定或代码库信息，请改用 [CLAUDE.md](/en/memory)。

---

## 内置输出样式

Claude Code 的 **Default** 输出样式是现有的系统提示，旨在帮助你高效地完成软件工程任务。

还有两个额外的内置输出样式，专注于教你理解代码库和 Claude 的工作方式：

- **Explanatory（解释型）**：在帮助你完成软件工程任务的同时，提供教育性的"见解（Insights）"。帮助你理解实现选择和代码库模式。

- **Learning（学习型）**：协作式、边做边学的模式，Claude 不仅会在编码时分享"见解"，还会要求你贡献少量战略性代码。Claude Code 会在你的代码中添加 `TODO(human)` 标记让你来实现。

---

## 输出样式如何工作

输出样式直接修改 Claude Code 的系统提示。

- 自定义输出样式会排除编码相关的指令（如用测试验证代码），除非 `keep-coding-instructions` 为 true。
- 所有输出样式都有自己的自定义指令，添加到系统提示的末尾。
- 所有输出样式都会在对话中触发提醒，让 Claude 遵守输出样式的指令。

token 用量取决于样式。向系统提示添加指令会增加输入 token，但提示缓存会在会话中第一次请求之后降低这个成本。内置的 Explanatory 和 Learning 样式默认产生比 Default 更长的响应，这会增加输出 token。对于自定义样式，输出 token 用量取决于你的指令让 Claude 生成什么内容。

## 更改输出样式

运行 `/config` 并选择 **Output style** 从菜单中选择样式。你的选择会被保存到 `.claude/settings.local.json` 的[本地项目级别](/en/settings)。

要不用菜单直接设置样式，在设置文件中直接编辑 `outputStyle` 字段：

```json
{
  "outputStyle": "Explanatory"
}
```

因为输出样式在会话启动时通过系统提示设置，更改会在下次启动新会话时生效。这让系统提示在对话期间保持稳定，因此提示缓存可以降低延迟和成本。

## 创建自定义输出样式

自定义输出样式是带 frontmatter 的 Markdown 文件，以及将被添加到系统提示中的文本：

```markdown
---
name: My Custom Style
description:
  此样式的简要描述，将显示给用户
---

# 自定义样式指令

你是一个交互式 CLI 工具，帮助用户完成软件工程任务。[你的自定义指令在此...]

## 特定行为

[定义此样式中助手应如何行为...]
```

你可以将这些文件保存在 user 级别（`~/.claude/output-styles`）或 project 级别（`.claude/output-styles`）。[插件](/en/plugins-reference)也可以在 `output-styles/` 目录中附带输出样式。

### Frontmatter

输出样式文件支持 frontmatter 来指定元数据：

| Frontmatter                | 用途                                                                     | 默认值                |
| :------------------------- | :----------------------------------------------------------------------- | :-------------------- |
| `name`                     | 输出样式的名称，如果不是文件名                                           | 继承自文件名          |
| `description`              | 输出样式的描述，显示在 `/config` 选择器中                                | 无                    |
| `keep-coding-instructions` | 是否保留 Claude Code 系统提示中与编码相关的部分                           | false                 |

## 与相关功能的比较

### 输出样式 vs. CLAUDE.md vs. --append-system-prompt

输出样式完全"关闭" Claude Code 默认系统提示中特定于软件工程的部分。CLAUDE.md 和 `--append-system-prompt` 都不编辑 Claude Code 的默认系统提示。CLAUDE.md 将内容作为用户消息添加在 Claude Code 默认系统提示**之后**。`--append-system-prompt` 将内容**追加**到系统提示中。

### 输出样式 vs. [子代理](/en/sub-agents)

输出样式直接影响主代理循环，且仅影响系统提示。代理被调用来处理特定任务，可以包含额外的设置，如使用的模型、可用的工具以及何时使用代理的一些上下文。

### 输出样式 vs. [技能](/en/skills)

输出样式修改 Claude 的回应方式（格式、语气、结构），一旦选择就始终激活。技能是特定于任务的提示，你可以用 `/skill-name` 调用或在相关时由 Claude 自动加载。用输出样式保持一致的格式偏好；用技能实现可复用的工作流和任务。
