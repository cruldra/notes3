语音听写

使用按键说话语音听写在 Claude Code CLI 中口述你的提示，而不是打字。

按住一个键并说话来口述你的提示。你的语音会实时转录到提示输入中，因此你可以在同一条消息中混合语音和打字。用 /voice 启用听写。默认按键说话键是 Space；重新绑定到修饰键组合以在第一次按键时激活而不是短暂按住后激活。

注意：语音听写需要 Claude Code v2.1.69 或更高版本。用 claude --version 检查你的版本。

要求​

语音听写将你的录音流式传输到 Anthropic 的服务器进行转录。音频不会在本地处理。语音转文本服务仅在使用 Claude.ai 账户认证时可用，当 Claude Code 配置为直接使用 Anthropic API 密钥、Amazon Bedrock、Google Vertex AI 或 Microsoft Foundry 时不可用。关于 Anthropic 如何处理你的数据，见数据使用。

语音听写还需要本地麦克风访问，因此它不能在远程环境（如 Web 版 Claude Code 或 SSH 会话）中工作。在 WSL 中，语音听写需要 WSLg 来访问音频，这包含在 Windows 11 的 WSL2 中。在 Windows 10 或 WSL1 上，改为在原生 Windows 中运行 Claude Code。

音频录制在 macOS、Linux 和 Windows 上使用内置原生模块。在 Linux 上，如果原生模块无法加载，Claude Code 回退到 ALSA utils 的 arecord 或 SoX 的 rec。如果两者都不可用，/voice 会为你的包管理器打印安装命令。

启用语音听写​

运行 /voice 切换语音听写开启。首次启用时，Claude Code 运行麦克风检查。在 macOS 上，如果从未授予过终端系统麦克风权限提示，这会触发它。

/voice
Voice mode enabled. Hold Space to record. Dictation language: en (/config to change).

语音听写跨会话持久化。再次运行 /voice 关闭它，或者直接在用户设置文件中设置：

{
  "voiceEnabled": true
}

启用语音听写时，当提示为空时，输入底部会显示 hold Space to speak 提示。如果你配置了自定义状态行，提示不会出现。

录制提示​

按住 Space 开始录制。Claude Code 通过监视来自终端的快速按键重复事件来检测按住键，因此在录制开始之前有一个短暂的预热期。底部在预热期间显示 keep holding…，然后在录制激活后切换到实时波形。

在预热期间，前几个按键重复字符会输入到输入中，并在录制激活时自动移除。单击 Space 仍然输入空格，因为按住检测仅在快速重复时触发。

提示：要跳过预热，重新绑定到修饰键组合如 meta+k。修饰键组合在第一次按键时开始录制，无需预热。

你的语音在你说的时候出现在提示中，在转录最终确定之前变暗。松开 Space 停止录制并最终确定文本。转录插入在你的光标位置，光标停留在插入文本的末尾，因此你可以按任意顺序混合打字和听写。再次按住 Space 追加另一段录音，或先移动光标以在提示中的其他位置插入语音：

> refactor the auth middleware to ▮
  # 按住 Space，说"use the new token validation helper"
> refactor the auth middleware to use the new token validation helper▮

转录针对编码词汇进行了优化。常见的开发术语如 regex、OAuth、JSON 和 localhost 被正确识别，你当前的项目名称和 git 分支名称会自动添加为识别提示。

在 /config 或直接在设置中设置语言。你可以使用 BCP 47 语言代码或语言名称：

{
  "language": "japanese"
}

如果你的 language 设置不在支持列表中，/voice 在启用时会警告你并回退到英语进行听写。Claude 的文本响应不受此回退影响。

重新绑定按键说话键​

按键说话键绑定在 Chat 上下文中的 voice:pushToTalk，默认为 Space。在 ~/.claude/keybindings.json 中重新绑定：

{
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "meta+k": "voice:pushToTalk",
        "space": null
      }
    }
  ]
}

设置 "space": null 移除默认绑定。如果你想要两个键都激活，省略它。

因为按住检测依赖于按键重复，避免绑定裸字母键如 v，因为它在预热期间会输入到提示中。使用 Space，或使用修饰键组合如 meta+k 在第一次按键时开始录制而无需预热。有关完整键绑定语法，见自定义键盘快捷键。

故障排除​

语音听写不激活或录制时的常见问题：

  • Voice mode requires a Claude.ai account：你使用 API 密钥或第三方提供商认证。运行 /login 用 Claude.ai 账户登录。
  • Microphone access is denied：在系统设置中授予终端麦克风权限。在 macOS 上，前往系统设置 → 隐私与安全性 → 麦克风并启用你的终端应用，然后再次运行 /voice。在 Windows 上，前往设置 → 隐私与安全 → 麦克风并为桌面应用打开麦克风访问，然后再次运行 /voice。如果你的终端未列在 macOS 设置中，见终端未列在 macOS 麦克风设置中。
  • No audio recording tool found（Linux）：原生音频模块无法加载且没有安装回退。用错误消息中显示的命令安装 SoX，例如 sudo apt-get install sox。
  • 按住 Space 时什么都没发生：观察提示输入当你按住时。如果空格不断累积，语音听写关闭；运行 /voice 启用它。如果只出现一两个空格然后什么都没有，语音听写已开启但按住检测未触发。按住检测需要你的终端发送按键重复事件，因此如果在操作系统级别禁用了按键重复，它无法检测到按住键。
  • 转录混乱或语言错误：听写默认为英语。如果你用另一种语言口述，先在 /config 中设置。见更改听写语言。

终端未列在 macOS 麦克风设置中​

如果你的终端应用没有出现在系统设置 → 隐私与安全 → 麦克风下，没有你可以启用的切换。重置终端的权限状态，以便下次 /voice 运行时触发新的 macOS 权限提示。

步骤 1：重置终端的麦克风权限

运行 tccutil reset Microphone <bundle-id>，将 <bundle-id> 替换为你的终端的标识符：内置 Terminal 为 com.apple.Terminal，或 iTerm2 为 com.googlecode.iterm2。对于其他终端，用 osascript -e 'id of app "AppName"' 查找标识符。

警告：你可以在没有 bundle ID 的情况下运行 tccutil reset Microphone，但这会撤销 Mac 上每个应用的麦克风访问，包括 Zoom 或 Slack 等应用。每个应用都需要在下次使用时重新请求访问，因此不要在活跃通话期间运行它。

步骤 2：退出并重新启动你的终端

macOS 不会重新提示已在运行的进程。用 Cmd+Q 退出终端应用，而不仅是关闭窗口，然后再次打开它。

步骤 3：触发新的提示

启动 Claude Code 并运行 /voice。macOS 提示麦克风访问；允许它。
