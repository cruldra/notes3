自定义键盘快捷键

使用键绑定配置文件自定义 Claude Code 中的键盘快捷键。

注意：可自定义键盘快捷键需要 Claude Code v2.1.18 或更高版本。用 claude --version 检查你的版本。

Claude Code 支持自定义键盘快捷键。运行 /keybindings 创建或打开位于 ~/.claude/keybindings.json 的配置文件。

配置文件​

键绑定配置文件是一个带 bindings 数组的对象。每个块指定一个上下文和按键到操作的映射。

注意：对键绑定文件的更改会自动检测并应用，无需重启 Claude Code。

字段描述
$schema可选的 JSON Schema URL，用于编辑器自动补全
$docs可选的文档 URL
bindings按上下文分组的绑定块数组

这个示例将 Ctrl+E 绑定为在聊天上下文中打开外部编辑器，并解绑 Ctrl+U：

{
  "$schema": "https://www.schemastore.org/claude-code-keybindings.json",
  "$docs": "https://code.claude.com/docs/en/keybindings",
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "ctrl+e": "chat:externalEditor",
        "ctrl+u": null
      }
    }
  ]
}

上下文​

每个绑定块指定一个应用绑定的上下文：

上下文描述
Global应用于应用中的任何地方
Chat主聊天输入区域
Autocomplete自动补全菜单打开时
Settings设置菜单
Confirmation权限和确认对话框
Tabs选项卡导航组件
Help帮助菜单可见时
Transcript转录查看器
HistorySearch历史搜索模式（Ctrl+R）
Task后台任务正在运行时
ThemePicker主题选择器对话框
Attachments选择对话框中的图片附件导航
Footer底部指示器导航（任务、团队、diff）
MessageSelector回溯和总结对话框消息选择
DiffDialogDiff 查看器导航
ModelPicker模型选择器努力级别
Select通用选择/列表组件
Plugin插件对话框（浏览、发现、管理）
Scroll全屏模式下的对话滚动和文本选择

按键语法​

修饰键​

使用修饰键与 + 分隔符：

  • ctrl 或 control — Control 键
  • alt、opt 或 option — Alt/Option 键
  • shift — Shift 键
  • meta、cmd 或 command — Meta/Command 键

例如：

ctrl+k          带修饰键的单键
shift+tab       Shift + Tab
meta+p          Command/Meta + P
ctrl+shift+c    多个修饰键

大写字母​

独立的大写字母隐含 Shift。例如，K 等效于 shift+k。这对于 vim 风格的绑定很有用，大写和小写键有不同的含义。

带修饰键的大写字母（如 ctrl+K）被视为风格化的，不隐含 Shift：ctrl+K 与 ctrl+k 相同。

组合键​

组合键是用空格分隔的按键序列：

ctrl+k ctrl+s   按 Ctrl+K，松开，然后按 Ctrl+S

特殊键​

  • escape 或 esc — Escape 键
  • enter 或 return — Enter 键
  • tab — Tab 键
  • space — 空格键
  • up、down、left、right — 方向键
  • backspace、delete — 删除键

解绑默认快捷键​

将操作设置为 null 以解绑默认快捷键：

{
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "ctrl+s": null
      }
    }
  ]
}

这也适用于组合键绑定。解绑共享前缀的每个组合键可以释放该前缀用作单键绑定：

{
  "bindings": [
    {
      "context": "Chat",
      "bindings": {
        "ctrl+x ctrl+k": null,
        "ctrl+x ctrl+e": null,
        "ctrl+x": "chat:newline"
      }
    }
  ]
}

如果你解绑了前缀上的一些但不是全部组合键，按下前缀仍然会进入剩余绑定的组合键等待模式。

保留快捷键​

这些快捷键不能重新绑定：

快捷键原因
Ctrl+C硬编码的中断/取消
Ctrl+D硬编码的退出
Ctrl+M在终端中与 Enter 相同（两者都发送 CR）

终端冲突​

某些快捷键可能与终端复用器冲突：

快捷键冲突
Ctrl+Btmux 前缀（按两次发送）
Ctrl+AGNU screen 前缀
Ctrl+ZUnix 进程挂起（SIGTSTP）

Vim 模式交互​

当通过 /config → Editor mode 启用 vim 模式时，键绑定和 vim 模式独立运行：

  • Vim 模式在文本输入级别处理输入（光标移动、模式、动作）
  • 键绑定在组件级别处理操作（切换待办、提交等）
  • vim 模式中的 Escape 键将 INSERT 切换到 NORMAL 模式；它不会触发 chat:cancel
  • 大多数 Ctrl+键 快捷键通过 vim 模式传递到键绑定系统
  • 在 vim NORMAL 模式中，? 显示帮助菜单（vim 行为）

验证​

Claude Code 验证你的键绑定并为以下内容显示警告：

  • 解析错误（无效的 JSON 或结构）
  • 无效的上下文名称
  • 保留快捷键冲突
  • 终端复用器冲突
  • 同一上下文中的重复绑定

运行 /doctor 查看任何键绑定警告。
