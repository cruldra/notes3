镜像管理

构建镜像​

# 基础语法
docker buildx build [选项] 路径 | URL | -

基础选项：
-f, --file string        指定Dockerfile的路径和名称 (默认: "PATH/Dockerfile")
-t, --tag stringArray    指定镜像名称和标签 (格式: "name:tag")
--pull                   总是尝试拉取所有引用的基础镜像
--push                   构建完成后推送到镜像仓库
--load                   构建完成后加载到本地docker引擎
-q, --quiet             安静模式,只显示成功时的镜像ID

缓存相关：
--no-cache              构建时不使用缓存
--cache-from            指定外部缓存源
--cache-to              指定缓存导出目标

构建参数：
--build-arg             设置构建时变量
--target                指定构建目标阶段
--network               设置RUN指令的网络模式 (默认 "default")
--platform              设置目标平台

输出选项：
-o, --output           指定输出目标 (格式: "type=local,dest=path")
--progress             设置进度输出类型 ("auto", "plain", "tty", "rawjson")
--iidfile              将镜像ID写入文件

安全相关：
--secret               向构建过程暴露密钥
--ssh                  暴露SSH agent或密钥给构建过程

示例

# 基本构建
docker build -t myapp:1.0 .

# 指定Dockerfile
docker build -f Dockerfile.prod -t myapp:prod .

# 带构建参数
docker build --build-arg VERSION=1.0 -t myapp .

# 多平台构建
docker build --platform linux/amd64,linux/arm64 -t myapp .
