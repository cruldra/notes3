管理界面


kafka-ui:
    image: provectuslabs/kafka-ui:latest
    profiles: ["infra"]
    ports:
      - "8190:8080"  # 浏览器访问端口
    environment:
      - KAFKA_CLUSTERS_0_NAME=smart-sales-cluster  # 给你的集群起个帅气的名字
      - KAFKA_CLUSTERS_0_BOOTSTRAPSERVERS=kafka:9092 # 告诉 UI 去哪里连 Kafka
    depends_on:
      - kafka
    networks:
      - smart-sales-net
    restart: unless-stopped

仪表台​

登录成功后，默认进入 Dashboard（仪表台） 页面，这里是整个 Kafka 集群的概览视图：

界面概览​

仪表台主要分为以下几个区域：

1. 顶部统计卡片​

显示当前集群的连接状态统计：

  • Online：在线集群数量（绿色标签）
  • Offline：离线集群数量（灰色标签）

2. 集群列表​

表格展示所有已配置的 Kafka 集群信息，包含以下列：

列名说明
Cluster name集群名称（如 smart-sales-cluster）
VersionKafka 版本号
Brokers countBroker 节点数量
Partitions总分区数
TopicsTopic 总数
Production生产消息总大小
Consumption消费消息总大小

3. 筛选功能​

  • Only offline clusters 开关：可勾选只显示离线的集群

4. 左侧导航菜单​

菜单项功能
Dashboard集群概览（当前页面）
Brokers查看 Broker 节点详情
Topics管理 Topic（创建、查看、删除等）
Consumers查看消费者组和消费进度

操作提示​

  • 点击表格中的集群名称可进入该集群的详细视图
  • 各列支持点击表头进行排序
  • 当集群状态异常时，对应的行会有视觉提示

Brokers（Broker 节点）​

点击左侧菜单 Brokers 可查看集群中所有 Broker 节点的详细信息和运行状态：

界面概览​

该页面主要分为两个区域：

1. 顶部统计卡片​

分为 Uptime 和 Partitions 两组统计信息：

Uptime 组​

指标说明
Broker CountBroker 节点总数
Active Controller当前活跃的 Controller 节点数量（正常应为 1）
VersionKafka 版本号

Partitions 组​

指标说明
Online在线分区数 / 总分区数
URPUnder-Replicated Partitions（副本不足的分区数，正常应为 0）
In Sync Replicas同步副本数 / 总副本数
Out Of Sync Replicas未同步的副本数（正常应为 0）

2. Broker 列表​

表格展示每个 Broker 节点的详细信息：

列名说明
Broker IDBroker 的唯一标识（绿色圆点表示在线状态）
Disk usage磁盘使用量及 segment 文件数量
Partitions skew分区分布倾斜度（- 表示均匀）
Leaders该 Broker 作为 Leader 的分区数量
Leader skewLeader 分布倾斜度
Online partitions在线分区数量
PortBroker 监听端口
HostBroker 所在主机 IP 地址

关键指标说明​

  • URP > 0：表示存在副本同步延迟，需要关注数据可靠性
  • Out Of Sync Replicas > 0：部分副本未同步，可能影响高可用
  • Partitions skew / Leader skew：数值越大表示分布越不均匀，可能导致热点问题

操作提示​

  • 点击 Broker ID 可查看该节点的详细配置和指标
  • 当 URP 或 Out Of Sync Replicas 不为 0 时，需要排查集群健康状态

Broker 详情界面​

在 Broker 列表中点击具体的 Broker ID 后，可以进入该节点的详情页面：

界面概览​

1. 顶部状态概览​

  • Segment Size: 该节点上所有日志分段的总大小。
  • Segment Count: 该节点上的日志分段总数。
  • Port: 该 Broker 监听的端口。
  • Host: 该 Broker 所在的主机地址。

2. 详情页签 (Tabs)​

页面提供以下三个维度的详细数据：

  • Log directories: 展示日志目录的使用情况（默认展示项）。
  • Configs: 查看和搜索该 Broker 的所有配置项（如 log.retention.ms 等）。
  • Metrics: 查看该节点的实时性能指标（JMX 指标）。

3. 日志目录 (Log directories) 详情​

在 Log directories 页签下，可以查看具体存储路径的状态：

  • Name: 日志存储的绝对路径（如 /tmp/kafka-logs）。
  • Error: 该目录是否存在读写异常。
  • Topics: 该目录下存储的 Topic 数量。
  • Partitions: 该目录下存储的分区总数。

4. 配置 (Configs) 详情​

在 Configs 页签下，可以查看该 Broker 的完整配置列表，支持搜索和动态修改：

界面功能：​

  • Search: 顶部的搜索框支持按配置项名称（Key）或当前值（Value）进行过滤。
  • Edit 按钮: 允许直接在 UI 界面修改支持动态更新的配置项。点击后会弹出编辑框，修改后立即生效（无需重启 Broker，取决于配置项类型）。
  • Source: 显示配置的来源，常见的包括：
    • DEFAULT_CONFIG: Kafka 默认配置。
    • DYNAMIC_DEFAULT_BROKER_CONFIG: 动态配置的默认值。
    • STATIC_BROKER_CONFIG: 配置文件中硬编码的静态配置（通常不可通过 UI 动态修改）。

表格列说明：​

列名说明
Key配置项名称（如 log.retention.ms, min.insync.replicas 等）
Value当前生效的值
Source配置来源说明（点击旁边的 i 图标可查看更多解释）

操作建议：

  • 并非所有配置都带有 Edit 按钮。如果某些关键配置（如端口、日志目录等）需要修改且没有 Edit 按钮，通常意味着需要修改物理配置文件并重启服务。
  • 修改动态配置前，建议先确认该修改对生产环境的影响。

Topics（主题）​

点击左侧菜单 Topics 可以查看和管理集群中的所有主题：

界面概览​

1. 顶部操作栏配置说明​

  • Search: 按名称快速搜索 Topic。
  • Show Internal Topics: 开关。开启后会显示 Kafka 内部使用的主题（如 __consumer_offsets），这些主题通常带有 IN 标识。
  • Add a Topic: 点击创建新的主题。
  • 批量操作按钮: (需选中 Topic 后激活)
    • Delete selected topics: 批量删除选中的主题。
    • Copy selected topic: 复制选中主题的配置。
    • Purge messages of selected topics: 清空选中主题中的所有消息（保留主题结构）。

2. 主题列表字段说明​

表格展示了主题的核心指标：索引和排序功能支持点击表头。

列名说明
Topic Name主题名称（IN 标签表示内部主题）
Partitions分区数量
Out of sync replicas未同步副本数（正常应为 0）
Replication Factor副本因子（建议至少为 2，生产环境通常为 3）
Number of messages消息总数
Size磁盘占用大小

3. 主题操作项​

点击表格最右侧的 三个点 (⋮) 按钮，可以对单个主题执行以下操作：

  • Clear Messages: 清空该主题的消息（注意：仅在清理策略为 DELETE 时允许执行）。
  • Recreate Topic: 重新创建主题（先删除再按原配置创建）。
  • Remove Topic: 删除该主题。

主题详情界面​

点击主题列表中的特定 Topic Name（例如 smart-sales-events），即可进入该主题的详情页面：

界面概览​

该页面主要分为三个区域：顶部操作栏、状态概览卡片和详细内容页签。

1. 顶部操作栏​

  • Produce Message: 允许你在 UI 界面直接向该主题发送测试消息（可指定分区、Key、Value 及 Headers）。
  • ⋮ 菜单: 提供与主题列表相同的快捷操作（如 Clear Messages、Recreate Topic 等）。

2. 状态概览卡片​

展示当前主题的全局健康状态和容量指标：

指标说明
Partitions该主题的总分区数。
Replication Factor副本因子（每个分区的副本总数）。
URPUnder-Replicated Partitions（副本同步落后的分区数，正常应为 0，绿色点表示健康）。
In Sync Replicas已同步副本数 / 总副本数（正常情况下两值应相等）。
Type主题类型（如 External 表示外部业务主题，Internal 表示 Kafka 内部主题）。
Segment Size该主题所有日志段占用的总磁盘空间。
Segment Count日志段文件总数。
Clean Up Policy数据清理策略（通常为 DELETE 按时间/大小删除，或 COMPACT 按 Key 压实）。
Message Count该主题当前包含的总消息数（近似值）。

3. 详细内容页签 (Tabs)​

页面提供五个维度的详细管理视图：

Overview (概览 - 默认)​

展示该主题下所有 分区 (Partition) 的详细分布和状态：

  • Partition ID: 分区编号。
  • Replicas: 该分区副本所在的 Broker ID 列表。绿色的 ID 表示该副本当前是 Leader。
  • First Offset: 该分区中最早可用消息的位点（Offset）。
  • Next Offset: 下一条将要写入消息的位点（通常等于最新位点 + 1）。
  • Message Count: 该分区内的消息总数（Next Offset - First Offset）。
  • ⋮ 操作: 针对具体分区，可以直接 Clear Messages（清空该分区消息）。

Messages (消息浏览)​

在 Messages 页签下，可以实时检索和浏览主题中的消息内容：

1. 检索与筛选配置​
  • Seek Type: 设置查询的起始方式（如 Offset 按位点、Timestamp 按时间戳等）。
  • Partitions: 选择要查询的分区（支持全选或指定特定分区）。
  • Key/Value Serde: 设置消息键（Key）和值（Value）的序列化/反序列化方式（如 String, JSON, Avro 等）。界面会自动尝试按所选方式解析内容。
  • Search: 输入关键字进行全文检索。
  • + Add Filters: 允许添加更复杂的过滤条件（如基于字段值的过滤）。
  • 排序方式: 顶部右侧可选择 Oldest First（由旧到新）或 Newest First（由新到旧）。
  • Submit: 点击开始检索。查询完成后会显示耗时、数据大小及消费的消息数量。
2. 消息列表说明​
列名说明
+展开按钮，点击可查看消息的完整 JSON 内容、Headers 及 Metadata。
Offset消息在分区中的唯一位点。
Partition消息所属的分区编号。
Timestamp消息生产的时间戳。
Key消息的键（点击 Preview 可快速查看）。
Value消息的内容主体（点击 Preview 可快速查看解析后的数据）。

Consumers（消费者组）​

点击左侧菜单 Consumers 可以查看和管理集群中的所有消费者组及其消费状态：

界面概览​

1. 顶部操作栏​

  • Search: 按消费者组 ID (Group ID) 快速搜索特定的消费者组。

2. 消费者组列表字段说明理论说明：​

列名说明
Group ID消费者组的唯一标识名称。
Num Of Members该组内当前活跃的消费者成员数量。
Num Of Topics该组正在消费的主题（Topic）数量。
Consumer Lag消费延迟（落后于最新消息的数量）。N/A 表示暂时无法获取或该组不活跃。
Coordinator负责管理该组状态的 Broker ID。
State消费者组的当前状态。

3. 常见状态说明 (State)​

  • STABLE (绿色): 状态正常，消费者组已平衡且正在正常工作。
  • EMPTY (灰色): 组内没有任何活跃的成员（通常表示消费者应用已停止）。
  • REBALANCING: 组内成员发生变化，正在进行分区重平衡。
  • DEAD: 消费者组已失效或即将被清理。
  • PREPARING_REBALANCE: 准备开始重平衡。

操作提示​

  • 点击特定的 Group ID 可以进入详情页面，查看该组在各个分区上的详细消费位点 (Offset) 和延迟 (Lag) 情况。

消费者组详情界面​

点击消费者组列表中的 Group ID，即可进入该组的详细消费视图：

界面概览​

1. 顶部状态概览卡片​

展示该组的全局运行指标：

  • State: 当前组状态（如 EMPTY, STABLE 等）。
  • Members: 组内当前活跃的消费者成员数。
  • Assigned Topics: 该组分配到的主题总数。
  • Assigned Partitions: 该组分配到的分区总数。
  • Coordinator ID: 负责管理该组的 Broker 节点 ID。
  • Total lag: 该组在所有主题和分区上的总延迟（消息堆积量）。这是监控消费能力的最核心指标。

2. 主题与分区消费详情​

界面按主题分组展示具体的消费进度：

  • Search by Topic Name: 可在组内快速搜索特定的主题。
  • Topic 列表: 每个主题作为一个折叠项显示其总的 Consumer Lag。展开后可查看具体的 分区 (Partition) 详情：
列名说明
Partition分区编号。
Consumer ID消费该分区的具体成员标识（如果是 EMPTY 状态则为空）。
Host消费者成员所在的主机地址。
Consumer Lag该分区上的消费延迟。
Current Offset消费者组当前已提交的消费位点。
End offset该分区当前的最新消息位点（Log End Offset）。

关键指标监控建议​

  • Total lag / Consumer Lag: 如果延迟持续增加，说明消费者的处理速度慢于消息生产速度，需要增加消费者实例或优化处理逻辑。
  • State: 如果业务正在运行但状态为 EMPTY，通常意味着消费者客户端已掉线。
  • Current Offset vs End offset: 当两者相等时，说明消息已全部消费完成。
