**智能体运行时——执行智能体代码的基础设施层，内置解决硬核运维问题的方案**

Inference.sh 是一个**智能体运行时**（agent runtime）：负责执行你智能体代码的基础设施层，并对那些棘手的运维问题提供了开箱即用的解决方案。

## Runtime vs Framework

**Framework** 给你的是构建块。你引入一个库，定义智能体的逻辑，然后自己想办法部署到某处。

**Runtime** 是执行你智能体的基础设施，并对以下方面提供明确保障：

- **持久性**：状态在故障和重启后依然存在
- **可观测性**：每一次决策和动作都被自动记录
- **工具编排**：托管式集成，认证由平台处理
- **人工监督**：对关键操作的审批关卡

这个区分非常重要，因为智能体工作负载具有独特的特征，通用基础设施处理起来效果很差。

## 为什么智能体需要 Runtime

智能体与传统软件不同：

- 它们做出一长串决策，中间穿插着外部调用
- 它们需要在多次交互中维护对话上下文
- 它们调用的工具可能失败、超时，或返回意外结果
- 它们的运行时长不可预测——有时几秒，有时几小时
- 它们对敏感操作需要人工监督

标准 Web 框架假定的是以毫秒为单位的请求-响应周期。容器编排假定的是无状态工作负载。消息队列假定的是独立作业。智能体工作负载与这些模式都不匹配。

## Runtime 处理什么

当你在 Inference.sh 上运行智能体时，你获得：

| 能力 | 作用 |
|------|------|
| [持久化执行](/docs/runtime/durable-execution) | 每一步后保存状态检查点；故障后恢复 |
| [可观测性](/docs/runtime/observability) | 自动追踪每一次决策和工具调用 |
| [人机协同](/docs/runtime/human-in-the-loop) | 对关键操作的审批关卡 |
| [工具编排](/docs/runtime/tool-orchestration) | 250+ 集成，认证托管 |

你编写智能体逻辑，Runtime 负责运维。
