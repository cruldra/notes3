Appllication

UnityEngine.Application 是 Unity 引擎中一个非常重要的静态类（Static Class）。它不挂载在对象上，也不能被实例化。

它的主要作用是提供关于当前运行的应用程序的全局信息以及控制运行时环境的方法。简单来说，它是你访问游戏程序本身（而不是游戏内容）的接口。

以下是 Application 类的核心功能分类介绍：

1. 文件路径访问 (File Paths)​

这是 Application 类最常用的功能之一，用于确定在不同平台（Windows, Android, iOS等）上读写文件的正确位置。

  • Application.persistentDataPath (最重要)
    • 用途：用于存储在游戏运行期间产生的数据（如存档文件、设置文件、下载的缓存）。
    • 特点：这个路径下的目录是可读写的，并且在应用程序更新后数据通常会被保留。
    • 示例： Windows: C:\Users\Username\AppData\LocalLow\CompanyName\ProductName Android: /storage/emulated/0/Android/data/package_name/files
  • Application.dataPath
    • 用途：包含游戏数据文件的路径。
    • 特点：在编辑器中指向项目的 Assets 文件夹。在发布出的应用中，它指向只读的数据区域（特别是移动平台）。不要尝试在移动端向此路径写入数据。
  • Application.streamingAssetsPath
    • 用途：指向 StreamingAssets 文件夹的路径。
    • 特点：用于放置需要原样保留、不被 Unity 压缩或打包成二进制的原始文件（如视频、AssetBundles）。

2. 应用程序控制 (Control)​

  • Application.Quit()
    • 用途：退出应用程序。
    • 注意：在 Unity 编辑器模式下调用此方法无效（应用不会关闭）。通常需要配合预处理指令或编辑器代码来停止播放模式。
    • 返回值：void（在某些平台上通过 exit code 退出）。
  • Application.targetFrameRate
    • 用途：设置游戏尝试渲染的目标帧率。
    • 场景：在移动端，通常设置为 60 或 30 以平衡流畅度和耗电量。设置为 -1 表示使用平台默认帧率。
    • 公式：Target Interval = 1 / targetFrameRate。
  • Application.runInBackground
    • 用途：当应用程序失去焦点（比如用户切到了别的窗口）时，游戏是否应该继续运行。
    • 场景：多人联机游戏通常设为 true，防止玩家切屏导致断线。

3. 应用程序信息 (Info)​

  • Application.platform
    • 用途：由于是枚举类型 RuntimePlatform，用于判断当前运行在哪个平台上（WindowsEditor, Android, IPhonePlayer 等）。
    • 场景：编写跨平台逻辑。
    • 代码：if (Application.platform == RuntimePlatform.Android) { ... }
  • Application.version
    • 用途：获取在 "Project Settings > Player" 中设置的版本号（如 "1.0.2"）。
    • 场景：在 UI 上显示版本号。
  • Application.identifier
    • 用途：获取包名（Bundle Identifier），如 com.MyCompany.MyGame。
  • Application.internetReachability
    • 用途：检查网络连接状态（无网络、移动数据、WiFi）。
    • 注意：只能检测是否连上局域网，不能保证能连上外网。

4. 外部交互​

  • Application.OpenURL(string url)
    • 用途：调用操作系统默认浏览器打开指定的网页链接。
    • 代码：Application.OpenURL("https://unity.com");

5. 事件回调 (Events)​

Application 提供了一些静态事件，允许你监听全局状态变化：

  • Application.logMessageReceived
    • 当 Debug.Log 写入日志时触发。常用于制作游戏内的 Debug Console（控制台）或者收集报错信息发送给服务器。
  • Application.focusChanged
    • 当游戏失去或获得焦点时触发。
  • Application.wantsToQuit
    • 当游戏即将退出时触发。你可以在这里返回 false 来阻止退出（例如提示“是否保存”）。

代码示例​

以下是一个涵盖了路径打印、退出逻辑和帧率设置的通用管理器示例：

using UnityEngine;

public class GameManager : MonoBehaviour
{
    void Awake()
    {
        // 1. 设置目标帧率为 60
        Application.targetFrameRate = 60;

        // 2. 允许后台运行（适用于网络游戏）
        Application.runInBackground = true;

        // 3. 打印重要路径 (用于 Debug)
        Debug.Log($"Save Data Path: {Application.persistentDataPath}");
    }

    public void OnClickQuitButton()
    {
        // 4. 处理退出逻辑
#if UNITY_EDITOR
        // 如果在编辑器中运行，停止播放模式
        UnityEditor.EditorApplication.isPlaying = false;
#else
        // 如果是打包后的应用，执行退出
        Application.Quit();
#endif
    }

    public void OpenWiki()
    {
        // 5. 打开网页
        Application.OpenURL("https://docs.unity3d.com");
    }
}

历史遗留说明 (Deprecation)​

如果你看到旧教程中使用 Application.LoadLevel() 或 Application.loadedLevelName 来切换场景，请不要使用它。

  • 旧方法：Application.LoadLevel("Level1")
  • 新标准：这些功能已被移动到 UnityEngine.SceneManagement.SceneManager 类中。
    • 请使用：SceneManager.LoadScene("Level1")。

总结​

UnityEngine.Application 是连接你的游戏代码与操作系统环境的纽带。在这个类中，最重要的属性是 persistentDataPath（用于存档），最常用的方法是 Quit()（用于退出）。
