> 我有一个创建棋盘的脚本，然后我在场景里面修改背景色，我得把场景关掉再打开才变化


```gdscript
## 棋盘格大小 (像素)
@export var cell_size: float = 80.0

## 棋盘外边距 (像素)
@export var margin: float = 60.0

## 线条颜色
@export var line_color: Color = Color("#4a3728")

## 线条宽度
@export var line_width: float = 2.0

## 背景颜色 (模拟木纹深度)
@export var background_color: Color = Color("#dcb35c")
```


这是一个非常经典的问题！虽然你的代码里已经正确使用了 `@tool` ，这让脚本能在编辑器里运行，但变量修改后**不会自动通知** Godot "画面需要刷新"。

Godot 为了性能，不会因为变量值变了就自动重绘界面。你需要手动告诉引擎："嘿，颜色变了，请重新执行一次 `_draw()` 函数。"

### 解决方法：使用 `set` 函数 + `queue_redraw()`

你需要给 `background_color`（以及其他影响外观的变量）添加一个 **setter (设置器)**。当你在检查器里修改数值时，setter 会被触发，我们就在那里手动请求重绘。

请修改你的脚本中变量定义的部分：

```gdscript
## 背景颜色 (模拟木纹深度)
# 修改前：
# @export var background_color: Color = Color("#dcb35c")

# 修改后：
@export var background_color: Color = Color("#dcb35c"):
	set(value):
		background_color = value
		queue_redraw() # <--- 关键！这句代码会强制触发 _draw()

```

### 为什么这样做有效？

1. **`set(value)`**: 当你在编辑器面板修改 `background_color` 时，Godot 会自动调用这个函数。
2. 
**`queue_redraw()`**: 这个函数会向引擎发送信号，标记当前节点为 "脏（dirty）" 状态。Godot 会在下一帧自动重新调用 `_draw()` ，从而用新的颜色绘制矩形。



### 建议：把所有视觉变量都加上

为了让你调整 `cell_size` 、`margin` 或 `line_width`  时也能实时看到变化，建议把它们也都改成这种写法：

```gdscript
@export var cell_size: float = 80.0:
	set(v):
		cell_size = v
		queue_redraw()

@export var margin: float = 60.0:
	set(v):
		margin = v
		queue_redraw()

# ... 其他变量同理

```

这样修改后，你在编辑器里拖动颜色或滑块，棋盘就会实时刷新，完全不需要重启场景。
