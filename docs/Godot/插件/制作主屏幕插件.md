## 本教程涵盖的内容

主屏幕插件允许你在编辑器的中心部分创建新的 UI，这些 UI 会出现在“2D”、“3D”、“Script”、“Game”和“AssetLib”按钮旁边。此类编辑器插件被称为“主屏幕插件”。

本教程将引导你创建一个基本的主屏幕插件。为了简单起见，我们的主屏幕插件将包含一个向控制台打印文本的按钮。

## 初始化插件

首先从“插件”菜单创建一个新插件。对于本教程，我们将把它放在名为 `main_screen` 的文件夹中，但你可以使用任何你喜欢的名称。

插件脚本将带有 `_enter_tree()` 和 `_exit_tree()` 方法，但对于主屏幕插件，我们需要添加一些额外的方法。添加五个额外的方法，使脚本看起来像这样：

**GDScript**

```gdscript
@tool
extends EditorPlugin

func _enter_tree():
    pass

func _exit_tree():
    pass

func _has_main_screen():
    return true

func _make_visible(visible):
    pass

func _get_plugin_name():
    return "Main Screen Plugin"

func _get_plugin_icon():
    return EditorInterface.get_editor_theme().get_icon("Node", "EditorIcons")
```

**C#**

```csharp
#if TOOLS
using Godot;

[Tool]
public partial class MainScreenPlugin : EditorPlugin
{
    public override void _EnterTree()
    {

    }

    public override void _ExitTree()
    {

    }

    public override bool _HasMainScreen()
    {
        return true;
    }

    public override void _MakeVisible(bool visible)
    {

    }

    public override string _GetPluginName()
    {
        return "Main Screen Plugin";
    }

    public override Texture2D _GetPluginIcon()
    {
        return EditorInterface.Singleton.GetEditorTheme().GetIcon("Node", "EditorIcons");
    }
}
#endif
```

脚本中的重要部分是 `_has_main_screen()` 函数，它被重载以返回 `true`。此函数在插件激活时由编辑器自动调用，告诉编辑器此插件向编辑器添加了一个新的中心视图。现在，我们将保持此脚本原样，稍后再回来。

## 主屏幕场景

创建一个以 `Control` 派生节点为根节点的新场景（对于此示例插件，我们将根节点设为 `CenterContainer`）。选择此根节点，在视口中，点击 `Layout`（布局）菜单并选择 `Full Rect`（全矩形）。你还需要在检查器中启用 `Expand`（扩展）垂直尺寸标志。面板现在使用主视口中的所有可用空间。

接下来，让我们向示例主屏幕插件添加一个按钮。添加一个 `Button` 节点，并将文本设置为 "Print Hello" 或类似内容。像这样给按钮添加脚本：

**GDScript**

```gdscript
@tool
extends Button

func _on_print_hello_pressed():
    print("Hello from the main screen plugin!")
```

**C#**

```csharp
using Godot;

[Tool]
public partial class PrintHello : Button
{
    private void OnPrintHelloPressed()
    {
        GD.Print("Hello from the main screen plugin!");
    }
}
```

然后将 "pressed" 信号连接到它自己。如果你需要有关信号的帮助，请参阅 [使用信号](https://docs.godotengine.org/en/stable/getting_started/step_by_step/signals.html#doc-signals) 文章。

我们完成了主屏幕面板。将场景保存为 `main_panel.tscn`。

## 更新插件脚本

我们需要更新 `main_screen_plugin.gd` 脚本，以便插件实例化我们的主面板场景并将其放置在需要的位置。这是完整的插件脚本：

**GDScript**

```gdscript
@tool
extends EditorPlugin

const MainPanel = preload("res://addons/main_screen/main_panel.tscn")

var main_panel_instance

func _enter_tree():
    main_panel_instance = MainPanel.instantiate()
    # Add the main panel to the editor's main viewport.
    EditorInterface.get_editor_main_screen().add_child(main_panel_instance)
    # Hide the main panel. Very much required.
    _make_visible(false)

func _exit_tree():
    if main_panel_instance:
        main_panel_instance.queue_free()

func _has_main_screen():
    return true

func _make_visible(visible):
    if main_panel_instance:
        main_panel_instance.visible = visible

func _get_plugin_name():
    return "Main Screen Plugin"

func _get_plugin_icon():
    # Must return some kind of Texture for the icon.
    return EditorInterface.get_editor_theme().get_icon("Node", "EditorIcons")
```

**C#**

```csharp
#if TOOLS
using Godot;

[Tool]
public partial class MainScreenPlugin : EditorPlugin
{
    PackedScene MainPanel = ResourceLoader.Load<PackedScene>("res://addons/main_screen/main_panel.tscn");
    Control MainPanelInstance;

    public override void _EnterTree()
    {
        MainPanelInstance = (Control)MainPanel.Instantiate();
        // Add the main panel to the editor's main viewport.
        EditorInterface.Singleton.GetEditorMainScreen().AddChild(MainPanelInstance);
        // Hide the main panel. Very much required.
        _MakeVisible(false);
    }

    public override void _ExitTree()
    {
        if (MainPanelInstance != null)
        {
            MainPanelInstance.QueueFree();
        }
    }

    public override bool _HasMainScreen()
    {
        return true;
    }

    public override void _MakeVisible(bool visible)
    {
        if (MainPanelInstance != null)
        {
            MainPanelInstance.Visible = visible;
        }
    }

    public override string _GetPluginName()
    {
        return "Main Screen Plugin";
    }

    public override Texture2D _GetPluginIcon()
    {
        // Must return some kind of Texture for the icon.
        return EditorInterface.Singleton.GetEditorTheme().GetIcon("Node", "EditorIcons");
    }
}
#endif
```

添加了几行特定的代码。`MainPanel` 是一个保存场景引用的常量，我们将其实例化到 `main_panel_instance` 中。

`_enter_tree()` 函数在 `_ready()` 之前调用。这是我们实例化主面板场景并将它们作为子项添加到编辑器特定部分的地方。我们使用 `EditorInterface.get_editor_main_screen()` 获取主编辑器屏幕，并将我们的主面板实例作为子项添加到其中。我们调用 `_make_visible(false)` 函数来隐藏主面板，以便在首次激活插件时它不会争夺空间。

`_exit_tree()` 函数在插件停用时调用。如果主屏幕仍然存在，我们调用 `queue_free()` 释放实例并将其从内存中移除。

覆盖 `_make_visible()` 函数以根据需要隐藏或显示主面板。当用户点击编辑器顶部的主视口按钮时，编辑器会自动调用此函数。

`_get_plugin_name()` 和 `_get_plugin_icon()` 函数控制插件主视口按钮的显示名称和图标。

你可以添加的另一个函数是 `handles()` 函数，它允许你处理节点类型，当选择该类型时自动聚焦主屏幕。这类似于点击 3D 节点会自动切换到 3D 视口。

## 尝试插件

在项目设置中激活插件。你会观察到在主视口上方的 2D、3D、Script 旁边有一个新按钮。点击它将带你进入新的主屏幕插件，中间的按钮将打印文本。

如果你想尝试此插件的完整版本，请查看此处的插件演示：[https://github.com/godotengine/godot-demo-projects/tree/master/plugins](https://github.com/godotengine/godot-demo-projects/tree/master/plugins)

如果你想查看主屏幕插件功能的更完整示例，请查看此处的 2.5D 演示项目：[https://github.com/godotengine/godot-demo-projects/tree/master/misc/2.5d](https://github.com/godotengine/godot-demo-projects/tree/master/misc/2.5d)
