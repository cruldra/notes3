> **注意**
>
> 本教程假设你已经知道如何制作通用插件。如果有疑问，请参考 [制作插件](制作插件.md) 页面。这也假设你熟悉 Godot 的导入系统。

## 简介

导入插件是一种特殊类型的编辑器工具，允许 Godot 导入自定义资源并将其视为一等公民资源。编辑器本身捆绑了许多导入插件来处理常见资源，如 PNG 图像、Collada 和 glTF 模型、Ogg Vorbis 声音等。

本教程展示了如何创建一个导入插件，以将自定义文本文件作为材质资源加载。此文本文件将包含三个由逗号分隔的数值，代表颜色的三个通道，生成的颜色将用作导入材质的反照率（主颜色）。在此示例中，它包含纯蓝色（零红，零绿，全蓝）：

```text
0,0,255
```

## 配置

首先，我们需要一个通用插件来处理导入插件的初始化和销毁。让我们先添加 `plugin.cfg` 文件：

```ini
[plugin]

name="Silly Material Importer"
description="Imports a 3D Material from an external text file."
author="Yours Truly"
version="1.0"
script="material_import.gd"
```

然后我们需要 `material_import.gd` 文件以便在需要时添加和移除导入插件：

```gdscript
# material_import.gd
@tool
extends EditorPlugin

var import_plugin

func _enter_tree():
    import_plugin = preload("import_plugin.gd").new()
    add_import_plugin(import_plugin)

func _exit_tree():
    remove_import_plugin(import_plugin)
    import_plugin = null
```

当此插件激活时，它将创建一个导入插件的新实例（我们很快就会制作）并使用 `add_import_plugin()` 方法将其添加到编辑器。我们将它的引用存储在类成员 `import_plugin` 中，以便以后在移除它时可以引用它。`remove_import_plugin()` 方法在插件停用时调用，以清理内存并让编辑器知道导入插件不再可用。

注意，导入插件是一种引用类型，因此不需要使用 `free()` 函数显式释放内存。当它超出作用域时，引擎会自动释放它。

## EditorImportPlugin 类

本次展示的主角是 [EditorImportPlugin](https://docs.godotengine.org/en/stable/classes/class_editorimportplugin.html#class-editorimportplugin) 类。它负责实现 Godot 在需要知道如何处理文件时调用的方法。

让我们开始编写插件代码，一次一个方法：

```gdscript
# import_plugin.gd
@tool
extends EditorImportPlugin

func _get_importer_name():
    return "demos.sillymaterial"
```

第一个方法是 `_get_importer_name()`。这是插件的唯一名称，Godot 使用它来了解特定文件使用了哪个导入器。当文件需要重新导入时，编辑器将知道调用哪个插件。

```gdscript
func _get_visible_name():
    return "Silly Material"
```

`_get_visible_name()` 方法负责返回它导入类型的名称，它将显示在“导入”面板中给用户看。

你应该将此名称选作“Import as”（导入为）的延续，例如 *"Import as Silly Material"*。你可以随意命名，但我们建议为你的插件起一个描述性的名称。

```gdscript
func _get_recognized_extensions():
    return ["mtxt"]
```

Godot 的导入系统通过扩展名检测文件类型。在 `_get_recognized_extensions()` 方法中，你返回一个字符串数组来表示此插件可以理解的每个扩展名。如果一个扩展名被多个插件识别，用户可以在导入文件时选择使用哪一个。

> **提示**
>
> 像 `.json` 和 `.txt` 这样的常见扩展名可能会被许多插件使用。此外，项目中可能存在只是游戏数据而不应导入的文件。导入时必须小心验证数据。永远不要期望文件格式正确。

```gdscript
func _get_save_extension():
    return "material"
```

导入的文件保存在项目根目录下的 `.import` 文件夹中。它们的扩展名应与你正在导入的资源类型匹配，但由于 Godot 无法判断你将使用什么（因为同一资源可能有多个有效扩展名），你需要声明将在导入中使用的扩展名。

由于我们正在导入材质，我们将使用此类资源类型的特殊扩展名。如果你正在导入场景，可以使用 `scn`。通用资源可以使用 `res` 扩展名。但是，引擎并没有强制执行这一点。

```gdscript
func _get_resource_type():
    return "StandardMaterial3D"
```

导入的资源具有特定类型，因此编辑器可以知道它属于哪个属性槽。这允许从文件系统面板拖放到检查器中的属性。

在我们的例子中，它是 [StandardMaterial3D](https://docs.godotengine.org/en/stable/classes/class_standardmaterial3d.html#class-standardmaterial3d)，可以应用于 3D 对象。

> **注意**
>
> 如果你需要从同一扩展名导入不同类型，则必须创建多个导入插件。你可以将导入代码抽象到另一个文件中，以避免在这方面重复。

## 选项和预设

你的插件可以提供不同的选项，允许用户控制资源的导入方式。如果一组选定的选项很常见，你还可以创建不同的预设，使用户更轻松。下图显示了选项如何在编辑器中显示：

![导入插件选项](https://docs.godotengine.org/en/stable/_images/import_plugin_options.png)

由于可能有很多预设，并且它们是用数字标识的，因此使用枚举是一个好习惯，这样你就可以使用名称来引用它们。

```gdscript
@tool
extends EditorImportPlugin

enum Presets { DEFAULT }

...
```

现在定义了枚举，让我们继续查看导入插件的方法：

```gdscript
func _get_preset_count():
    return Presets.size()
```

`_get_preset_count()` 方法返回此插件定义的预设数量。我们现在只有一个预设，但我们可以通过返回 `Presets` 枚举的大小来使此方法面向未来。

```gdscript
func _get_preset_name(preset_index):
    match preset_index:
        Presets.DEFAULT:
            return "Default"
        _:
            return "Unknown"
```

这里我们有 `_get_preset_name()` 方法，它为预设提供名称，正如它们将呈现给用户的那样，所以请务必使用简短明了的名称。

我们可以使用 `match` 语句使代码更加结构化。这样以后添加新预设很容易。我们使用捕获所有模式来返回一些东西。虽然 Godot 不会要求超出你定义的预设计数的预设，但为了安全起见总是更好的。

如果你只有一个预设，你可以简单地直接返回它的名称，但是如果你这样做，当你添加更多预设时必须小心。

```gdscript
func _get_import_options(path, preset_index):
    match preset_index:
        Presets.DEFAULT:
            return [{
                       "name": "use_red_anyway",
                       "default_value": false
                    }]
        _:
            return []
```

这是定义可用选项的方法。`_get_import_options()` 返回一个字典数组，每个字典包含几个键，用于检查以自定义显示给用户的选项。下表显示了可能的键：

| 键 | 类型 | 描述 |
| --- | --- | --- |
| `name` | String | 选项的名称。显示时，下划线变为空格，首字母大写。 |
| `default_value` | Any | 此预设的选项默认值。 |
| `property_hint` | Enum value | 用作提示的 [PropertyHint](https://docs.godotengine.org/en/stable/classes/class_%40globalscope.html#enum-globalscope-propertyhint) 值之一。 |
| `hint_string` | String | 属性的提示文本。与在 GDScript 中 `export` 语句中添加的相同。 |
| `usage` | Enum value | 定义用法的 [PropertyUsageFlags](https://docs.godotengine.org/en/stable/classes/class_%40globalscope.html#enum-globalscope-propertyusageflags) 值之一。 |

`name` 和 `default_value` 键是 **强制性** 的，其余是可选的。

注意 `_get_import_options` 方法接收预设编号，因此你可以为每个不同的预设配置选项（尤其是默认值）。在这个例子中我们使用 `match` 语句，但是如果你有很多选项并且预设只改变值，你可能想先创建选项数组，然后根据预设更改它。

> **警告**
>
> 即使你不定义预设（通过使 `_get_preset_count` 返回零），也会调用 `_get_import_options` 方法。你必须返回一个数组，即使它是空的，否则你会得到错误。

```gdscript
func _get_option_visibility(path, option_name, options):
    return true
```

对于 `_get_option_visibility()` 方法，我们只返回 `true`，因为我们的所有选项（即我们定义的唯一一个）始终可见。

如果你需要让某个选项仅在另一个选项设置为特定值时才可见，你可以在此方法中添加逻辑。

## `import` 方法

该过程的繁重部分，即负责将文件转换为资源的部分，由 `_import()` 方法涵盖。我们的示例代码有点长，所以让我们将其拆分为几部分：

```gdscript
func _import(source_file, save_path, options, r_platform_variants, r_gen_files):
    var file = FileAccess.open(source_file, FileAccess.READ)
    if file == null:
        return FileAccess.get_open_error()

    var line = file.get_line()
```

我们要导入方法的第一部分打开并读取源文件。我们使用 [FileAccess](https://docs.godotengine.org/en/stable/classes/class_fileaccess.html#class-fileaccess) 类来执行此操作，传递编辑器提供的 `source_file` 参数。

如果打开文件时出错，我们会返回它以让编辑器知道导入不成功。

```gdscript
    var channels = line.split(",")
    if channels.size() != 3:
        return ERR_PARSE_ERROR

    var color
    if options.use_red_anyway:
        color = Color.from_rgba8(255, 0, 0)
    else:
        color = Color.from_rgba8(int(channels[0]), int(channels[1]), int(channels[2]))
```

这段代码获取它之前读取的文件行，并将其拆分为由逗号分隔的部分。如果值多于或少于三个，则认为文件无效并报告错误。

然后它创建一个新的 [Color](https://docs.godotengine.org/en/stable/classes/class_color.html#class-color) 变量，并根据输入文件设置其值。如果启用了 `use_red_anyway` 选项，则将其设置为纯红色。

```gdscript
    var material = StandardMaterial3D.new()
    material.albedo_color = color
```

这部分制作了一个新的 [StandardMaterial3D](https://docs.godotengine.org/en/stable/classes/class_standardmaterial3d.html#class-standardmaterial3d)，即导入的资源。我们创建一个它的新实例，然后将其反照率颜色设置为我们之前获得的值。

```gdscript
    return ResourceSaver.save(material, "%s.%s" % [save_path, _get_save_extension()])
```

这是最后一部分，也是非常重要的一部分，因为这里我们将制作的资源保存到磁盘。保存文件的路径由编辑器生成并通过 `save_path` 参数通知。注意这 **不带** 扩展名，所以我们使用 [字符串格式化](https://docs.godotengine.org/en/stable/tutorials/scripting/gdscript/gdscript_format_string.html#doc-gdscript-printf) 添加它。为此我们调用之前定义的 `_get_save_extension` 方法，这样我们可以确保它们不会不同步。

我们还返回 `ResourceSaver.save()` 方法的结果，所以如果此步骤出错，编辑器就会知道。

## 平台变体和生成的文件

你可能已经注意到我们的插件忽略了 `import` 方法的两个参数。这些是 *返回参数*（因此名称开头有 `r`），这意味着编辑器将在调用你的导入方法后从中读取。它们都是可以填充信息的数组。

如果需要根据目标平台不同地导入资源，则使用 `r_platform_variants` 参数。虽然它被称为 *平台* 变体，但它是基于 [特性标签](https://docs.godotengine.org/en/stable/tutorials/export/feature_tags.html#doc-feature-tags) 的存在的，因此即使是同一平台也可以根据设置有多个变体。

要导入平台变体，你需要在扩展名之前使用特性标签保存它，然后将标签推送到 `r_platform_variants` 数组，以便编辑器知道你这样做了。

例如，假设我们为移动平台保存不同的材质。我们需要做类似以下的事情：

```gdscript
    r_platform_variants.push_back("mobile")
    return ResourceSaver.save(mobile_material, "%s.%s.%s" % [save_path, "mobile", _get_save_extension()])
```

`r_gen_files` 参数用于在导入过程中生成并需要保留的额外文件。编辑器将查看它以了解依赖关系并确保不会无意中删除额外文件。

这也是一个数组，应填充你保存的文件的完整路径。例如，让我们为下一个 pass 创建另一个材质并将其保存在不同的文件中：

```gdscript
    var next_pass = StandardMaterial3D.new()
    next_pass.albedo_color = color.inverted()
    var next_pass_path = "%s.next_pass.%s" % [save_path, _get_save_extension()]

    err = ResourceSaver.save(next_pass, next_pass_path)
    if err != OK:
        return err
    r_gen_files.push_back(next_pass_path)
```

## 尝试插件

这是理论上的，但现在导入插件已完成，让我们测试一下。确保创建了示例文件（包含介绍部分描述的内容）并将其保存为 `test.mtxt`。然后在项目设置中激活插件。

如果一切顺利，导入插件将被添加到编辑器中，并扫描文件系统，使自定义资源出现在文件系统面板上。如果你选择它并聚焦“导入”面板，你可以看到那里唯一的选择选项。

在场景中创建一个 MeshInstance3D 节点，并为其 Mesh 属性设置一个新的 SphereMesh。在检查器中展开 Material 部分，然后将文件从文件系统面板拖到材质属性。对象将在视口中更新为导入材质的蓝色。

![尝试导入插件](https://docs.godotengine.org/en/stable/_images/import_plugin_trying.png)

转到“导入”面板，启用 "Use Red Anyway"（无论如何使用红色）选项，然后点击 "Reimport"（重新导入）。这将更新导入的材质，并且应该自动更新视图，显示红色。

就是这样！你的第一个导入插件完成了！现在发挥创意，为你自己钟爱的格式制作插件吧。这对于以自定义格式编写数据然后像原生资源一样在 Godot 中使用它非常有用。这展示了导入系统的强大和可扩展性。
