## 关于插件

插件是扩展编辑器实用工具的好方法。它们完全可以用 GDScript 和标准场景制作，甚至不需要重新加载编辑器。与模块不同，你不需要创建 C++ 代码或重新编译引擎。虽然这使得插件的功能稍微受限，但你可以用它们做很多事情。请注意，插件类似于你已经可以制作的任何场景，只是它是使用脚本创建的，以添加编辑器功能。

本教程将指导你创建两个插件，以便你了解它们的工作原理并能够开发自己的插件。第一个是可以添加到项目中任何场景的自定义节点，另一个是添加到编辑器的自定义停靠面板 (Dock)。

## 创建插件

开始之前，请在任何你想要的位置创建一个新的空项目。这将作为开发和测试插件的基础。

要让编辑器识别新插件，你需要创建两个文件：用于配置的 `plugin.cfg` 和具有功能的工具脚本。插件在项目文件夹中有像 `addons/plugin_name` 这样的标准路径。Godot 提供了一个对话框来生成这些文件并将它们放置在正确的位置。

在主工具栏中，点击 **项目 (Project)** 下拉菜单。然后点击 **项目设置 (Project Settings...)**。转到 **插件 (Plugins)** 选项卡，然后点击右上角的 **创建新插件 (Create New Plugin)** 按钮。

你会看到如下对话框出现：

![创建插件对话框](https://docs.godotengine.org/en/stable/_images/making_plugins-create_plugin_dialog.webp)

每个字段中的占位符文本描述了它如何影响插件文件的创建和配置文件的值。

要继续此示例，请使用以下值：

**GDScript**

*   **插件名称 (Plugin Name)**: My Custom Node
*   **子文件夹 (Subfolder)**: my_custom_node
*   **描述 (Description)**: A custom node made to extend the Godot Engine.
*   **作者 (Author)**: Your Name Here
*   **版本 (Version)**: 1.0.0
*   **语言 (Language)**: GDScript
*   **脚本名称 (Script Name)**: custom_node.gd

**C#**

*   **插件名称 (Plugin Name)**: My Custom Node
*   **子文件夹 (Subfolder)**: MyCustomNode
*   **描述 (Description)**: A custom node made to extend the Godot Engine.
*   **作者 (Author)**: Your Name Here
*   **版本 (Version)**: 1.0.0
*   **语言 (Language)**: C#
*   **脚本名称 (Script Name)**: CustomNode.cs

> **警告**
>
> 在 C# 中，EditorPlugin 脚本需要编译，这需要构建项目。构建项目后，可以在“项目设置”的“插件”选项卡中启用插件。

你最终应该得到如下所示的目录结构：

![插件文件夹结构](https://docs.godotengine.org/en/stable/_images/making_plugins-my_custom_mode_folder.webp)

`plugin.cfg` 是一个包含插件元数据的 INI 文件。名称和描述有助于人们了解它的作用。你的名字有助于你获得应有的归属。版本号有助于其他人知道他们是否拥有过时的版本；如果你不确定如何确定版本号，请查看 [语义化版本控制 (Semantic Versioning)](https://semver.org/)。主脚本文件将指示 Godot 在插件激活后在编辑器中做什么。

### 脚本文件

创建插件后，对话框将自动为你打开 EditorPlugin 脚本。脚本有两个你不能更改的要求：它必须是 `@tool` 脚本，否则它将无法在编辑器中正确加载，并且它必须继承自 [EditorPlugin](https://docs.godotengine.org/en/stable/classes/class_editorplugin.html#class-editorplugin)。

> **警告**
>
> 除了 EditorPlugin 脚本之外，你的插件使用的任何其他 GDScript *也*必须是工具脚本。编辑器使用的任何没有 `@tool` 的 GDScript 都将表现得像空文件一样！

处理资源的初始化和清理很重要。一个好的做法是使用虚函数 `_enter_tree()` 来初始化插件，使用 `_exit_tree()` 来清理它。幸运的是，对话框为你生成了这些回调。你的脚本应该如下所示：

**GDScript**

```gdscript
@tool
extends EditorPlugin

func _enter_tree():
    # Initialization of the plugin goes here.
    pass

func _exit_tree():
    # Clean-up of the plugin goes here.
    pass
```

**C#**

```csharp
#if TOOLS
using Godot;

[Tool]
public partial class CustomNode : EditorPlugin
{
    public override void _EnterTree()
    {
        // Initialization of the plugin goes here.
    }

    public override void _ExitTree()
    {
        // Clean-up of the plugin goes here.
    }
}
#endif
```

这是创建新插件时使用的一个很好的模板。

## 自定义节点

有时你想在许多节点中拥有某种行为，例如可重用的自定义场景或控件。实例化在很多情况下很有用，但在许多项目中使用它可能会很繁琐。对此的一个好解决方案是制作一个插件，添加一个具有自定义行为的节点。

在本教程中，我们将创建一个在点击时打印消息的按钮。为此，我们需要一个继承自 [Button](https://docs.godotengine.org/en/stable/classes/class_button.html#class-button) 的脚本。如果你愿意，它也可以继承 [BaseButton](https://docs.godotengine.org/en/stable/classes/class_basebutton.html#class-basebutton)：

**GDScript**

```gdscript
# Optional, add to execute in the editor.
@tool

# Icons are optional.
# Alternatively, you may use the UID of the icon or the absolute path.
@icon("icon.png")

# Automatically register the node in the Create New Node dialog
# and make it available for use with other scripts.
class_name MyButton
extends Button

func _enter_tree():
    pressed.connect(clicked)

func clicked():
    print("You clicked me!")
```

**C#**

```csharp
using Godot;

// Optional, add to execute in the editor.
[Tool]

// Icons are optional.
// Alternatively, you may use the UID of the icon or the absolute path.
[Icon("icon.png")]

// Automatically register the node in the Create New Node dialog
// and make it available for use with other scripts.
[GlobalClass]
public partial class MyButton : Button
{
    public override void _EnterTree()
    {
        Pressed += Clicked;
    }

    public void Clicked()
    {
        GD.Print("You clicked me!");
    }
}
```

基本的按钮就这样了。你可以将其保存为 `my_button.gd` 在插件文件夹中。你可能有一个 16×16 的图标显示在场景树中。如果你没有，你可以从引擎中获取默认图标并将其保存到你的 `addons/my_custom_node` 文件夹中作为 icon.png，或者使用默认的 Godot Logo (`preload("res://icon.svg")`)。

> **提示**
>
> 用作自定义节点图标的 SVG 图像应启用 **Editor > Scale With Editor Scale** 和 **Editor > Convert Colors With Editor Theme** [导入选项](https://docs.godotengine.org/en/stable/tutorials/assets_pipeline/importing_images.html#doc-importing-images-editor-import-options)。这允许图标遵循编辑器的缩放和主题设置，如果图标的设计颜色调色板与 Godot 自己的图标相同。

![自定义节点图标](https://docs.godotengine.org/en/stable/_images/making_plugins-custom_node_icon.png)

完成后，插件应该已经在 **项目设置** 的插件列表中可用，按照 [检查结果](#检查结果) 中的说明激活它。

然后通过添加你的新节点来尝试它：

![创建自定义节点](https://docs.godotengine.org/en/stable/_images/making_plugins-custom_node_create.webp)

当你添加节点时，你会看到它已经附加了你创建的脚本。给按钮设置一些文本，保存并运行场景。当你点击按钮时，你可以在控制台看到一些文本：

![自定义节点控制台输出](https://docs.godotengine.org/en/stable/_images/making_plugins-custom_node_console.webp)

## 自定义停靠面板 (Dock)

有时，你需要扩展编辑器并添加始终可用的工具。一种简单的方法是用插件添加一个新的停靠面板。停靠面板只是基于 Control 的场景，所以它们的创建方式类似于通常的 GUI 场景。

创建自定义停靠面板就像自定义节点一样。在 `addons/my_custom_dock` 文件夹中创建一个新的 `plugin.cfg` 文件，然后向其中添加以下内容：

**GDScript**

```ini
[plugin]

name="My Custom Dock"
description="A custom dock made so I can learn how to make plugins."
author="Your Name Here"
version="1.0"
script="custom_dock.gd"
```

**C#**

```ini
[plugin]

name="My Custom Dock"
description="A custom dock made so I can learn how to make plugins."
author="Your Name Here"
version="1.0"
script="CustomDock.cs"
```

然后在同一文件夹中创建脚本 `custom_dock.gd`。用我们 [之前看到的模板](#脚本文件) 填充它以获得良好的开端。

由于我们试图添加一个新的自定义停靠面板，我们需要创建停靠面板的内容。这不过是一个标准的 Godot 场景：只需在编辑器中创建一个新场景然后编辑它。

对于编辑器停靠面板，根节点 **必须** 是 [Control](https://docs.godotengine.org/en/stable/classes/class_control.html#class-control) 或其子类之一。对于本教程，你可以创建一个单独的按钮。别忘了给你的按钮添加一些文本。

![自定义停靠面板场景](https://docs.godotengine.org/en/stable/_images/making_plugins-my_custom_dock_scene.webp)

将此场景保存为 `my_dock.tscn`。现在，我们需要抓取我们创建的场景，然后将其作为停靠面板添加到编辑器中。为此，你可以依赖 [EditorPlugin](https://docs.godotengine.org/en/stable/classes/class_editorplugin.html#class-editorplugin) 类中的 `add_dock()` 函数。

你需要选择一个停靠面板位置并定义要添加的控件（即你刚刚创建的场景）。别忘了在插件停用时 **移除停靠面板**。脚本可能如下所示：

**GDScript**

```gdscript
@tool
extends EditorPlugin

# A class member to hold the dock during the plugin life cycle.
var dock

func _enter_tree():
    # Initialization of the plugin goes here.
    # Load the dock scene and instantiate it.
    var dock_scene = preload("res://addons/my_custom_dock/my_dock.tscn").instantiate()

    # Create the dock and add the loaded scene to it.
    dock = EditorDock.new()
    dock.add_child(dock_scene)

    dock.title = "My Dock"

    # Note that LEFT_UL means the left of the editor, upper-left dock.
    dock.default_slot = DOCK_SLOT_LEFT_UL

    # Allow the dock to be on the left or right of the editor, and to be made floating.
    dock.available_layouts = EditorDock.DOCK_LAYOUT_VERTICAL | EditorDock.DOCK_LAYOUT_FLOATING

    add_dock(dock)

func _exit_tree():
    # Clean-up of the plugin goes here.
    # Remove the dock.
    remove_dock(dock)
    # Erase the control from the memory.
    dock.queue_free()
```

**C#**

```csharp
#if TOOLS
using Godot;

[Tool]
public partial class CustomDock : EditorPlugin
{
    private EditorDock _dock;

    public override void _EnterTree()
    {
        var _dock_scene = GD.Load<PackedScene>("res://addons/MyCustomDock/MyDock.tscn").Instantiate<Control>();
        AddControlToDock(DockSlot.LeftUl, _dock);

        // Create the dock and add the loaded scene to it.
        _dock = new EditorDock();
        _dock.AddChild(dock_scene);

        _dock.Title = "My Dock";

        // Note that LeftUl means the left of the editor, upper-left dock.
        _dock.DefaultSlot = DockSlot.LeftUl;

        // Allow the dock to be on the left or right of the editor, and to be made floating.
        _dock.AvailableLayouts = DockLayout.Horizontal | DockLayout.Floating;

        AddDock(_dock);
    }

    public override void _ExitTree()
    {
        // Clean-up of the plugin goes here.
        // Remove the dock.
        RemoveDock(_dock);
        // Erase the control from the memory.
        _dock.QueueFree();
    }
}
#endif
```

请注意，虽然停靠面板最初将出现在其指定位置，但用户可以自由更改其位置并保存结果布局。

### 检查结果

现在是检查工作结果的时候了。打开 **项目设置** 并点击 **插件** 选项卡。你的插件应该是列表中唯一的插件。

![项目设置插件列表](https://docs.godotengine.org/en/stable/_images/making_plugins-project_settings.webp)

你可以看到插件未启用。点击 **启用 (Enable)** 复选框以激活插件。在关闭设置窗口之前，停靠面板应该就会变得可见。你现在应该拥有了一个自定义停靠面板：

![自定义停靠面板](https://docs.godotengine.org/en/stable/_images/making_plugins-custom_dock.webp)

## 在插件中注册自动加载 (Autoloads)/单例

编辑器插件可以在启用插件时自动注册 [自动加载 (Autoloads)](https://docs.godotengine.org/en/stable/tutorials/scripting/singletons_autoload.html#doc-singletons-autoload)。这也包括在禁用插件时注销自动加载。

这使得用户设置插件更快，因为如果你的编辑器插件需要使用自动加载，他们不再需要手动将自动加载添加到他们的项目设置中。

使用以下代码从编辑器插件注册单例：

**GDScript**

```gdscript
@tool
extends EditorPlugin

# Replace this value with a PascalCase autoload name, as per the GDScript style guide.
const AUTOLOAD_NAME = "SomeAutoload"

func _enable_plugin():
    # The autoload can be a scene or script file.
    add_autoload_singleton(AUTOLOAD_NAME, "res://addons/my_addon/some_autoload.tscn")

func _disable_plugin():
    remove_autoload_singleton(AUTOLOAD_NAME)
```

**C#**

```csharp
#if TOOLS
using Godot;

[Tool]
public partial class MyEditorPlugin : EditorPlugin
{
    // Replace this value with a PascalCase autoload name.
    private const string AutoloadName = "SomeAutoload";

    public override void _EnablePlugin()
    {
        // The autoload can be a scene or script file.
        AddAutoloadSingleton(AutoloadName, "res://addons/MyAddon/SomeAutoload.tscn");
    }

    public override void _DisablePlugin()
    {
        RemoveAutoloadSingleton(AutoloadName);
    }
}
#endif
```

## 使用子插件

通常一个插件会添加多个东西，例如自定义节点和面板。在这些情况下，为每个功能拥有单独的插件脚本可能会更容易。子插件可用于此目的。

首先像普通插件一样创建所有插件和子插件：

![子插件创建](https://docs.godotengine.org/en/stable/_images/sub_plugin_creation.webp)

然后将子插件移动到主插件文件夹中：

![子插件移动](https://docs.godotengine.org/en/stable/_images/sub_plugin_moved.webp)

Godot 将从插件列表中隐藏子插件，以便用户无法启用或禁用它们。相反，主插件脚本应该像这样启用和禁用子插件：

**GDScript**

```gdscript
@tool
extends EditorPlugin

# The main plugin is located at res://addons/my_plugin/
const PLUGIN_NAME = "my_plugin"

func _enable_plugin():
    EditorInterface.set_plugin_enabled(PLUGIN_NAME + "/node", true)
    EditorInterface.set_plugin_enabled(PLUGIN_NAME + "/panel", true)

func _disable_plugin():
    EditorInterface.set_plugin_enabled(PLUGIN_NAME + "/node", false)
    EditorInterface.set_plugin_enabled(PLUGIN_NAME + "/panel", false)
```

## 进阶

既然你已经学会了如何制作基本插件，你可以通过多种方式扩展编辑器。可以使用 GDScript 向编辑器添加大量功能；这是一种创建专用编辑器而无需深入研究 C++ 模块的强大方法。

你可以制作自己的插件来帮助自己，并在 [资产库 (Asset Library)](https://godotengine.org/asset-library/) 中分享它们，以便人们可以从你的工作中受益。
