## 什么是 `@tool`？

`@tool` 是一行强大的代码，当添加到脚本顶部时，它会使脚本在编辑器中执行。你还可以决定脚本的哪些部分在编辑器中执行、哪些在游戏中执行、哪些在两者中都执行。

你可以用它做很多事情，但它在关卡设计中最为有用，用于直观地展示那些我们自己难以预测的事物。以下是一些用例：

*   如果你有一个发射受物理（重力）影响的炮弹的大炮，你可以在编辑器中绘制炮弹的轨迹，使关卡设计变得更加容易。
*   如果你有跳跃高度不同的跳板，你可以绘制玩家跳到上面能达到的最高高度，这同样能让关卡设计变得更容易。
*   如果你的玩家不使用精灵图（sprite），而是使用代码绘制自己，你可以让该绘制代码在编辑器中执行以看到你的玩家。

> **危险**
>
> `@tool` 脚本在编辑器内运行，并允许你访问当前编辑场景的场景树。这是一个强大的功能，但也伴随着风险，因为编辑器不包含针对潜在滥用 `@tool` 脚本的保护措施。在操作场景树时要**极其**小心，特别是通过 `Node.queue_free`，因为如果你在编辑器运行涉及该节点的逻辑时释放了它，可能会导致崩溃。

## 如何使用 `@tool`

要将脚本转换为工具，请在代码顶部添加 `@tool` 注解。

要检查当前是否处于编辑器中，请使用：`Engine.is_editor_hint()`。

例如，如果你只想在编辑器中执行某些代码，请使用：

**GDScript**

```gdscript
if Engine.is_editor_hint():
    # 在编辑器中执行的代码。
```

**C#**

```csharp
if (Engine.IsEditorHint())
{
    // 在编辑器中执行的代码。
}
```

另一方面，如果你只想在游戏中执行代码，只需对上述语句取反：

**GDScript**

```gdscript
if not Engine.is_editor_hint():
    # 在游戏中执行的代码。
```

**C#**

```csharp
if (!Engine.IsEditorHint())
{
    // 在游戏中执行的代码。
}
```

不包含上述两种条件的代码片段将在编辑器和游戏中同时运行。

以下是 `_process()` 函数可能的写法：

**GDScript**

```gdscript
func _process(delta):
    if Engine.is_editor_hint():
        # 在编辑器中执行的代码。

    if not Engine.is_editor_hint():
        # 在游戏中执行的代码。

    # 在编辑器和游戏中都执行的代码。
```

**C#**

```csharp
public override void _Process(double delta)
{
    if (Engine.IsEditorHint())
    {
        // 在编辑器中执行的代码。
    }

    if (!Engine.IsEditorHint())
    {
        // 在游戏中执行的代码。
    }

    // 在编辑器和游戏中都执行的代码。
}
```

## 重要信息

通用规则是：**你的工具脚本使用的任何其他 GDScript 也必须是工具（tool）**。编辑器无法从没有 `@tool` 的 GDScript 文件构建实例，这意味着你无法调用它们的方法或引用其中的成员变量。然而，由于静态方法、常量和枚举可以在不创建实例的情况下使用，因此可以从 `@tool` 脚本调用或引用其他非工具脚本中的这些内容。一个例外是[静态变量](https://docs.godotengine.org/en/stable/tutorials/scripting/gdscript/gdscript_basics.html#doc-gdscript-basics-static-variables)。如果你尝试在没有 `@tool` 的脚本中读取静态变量的值，它将始终返回 `null`，但执行此操作时不会打印警告或错误。此限制不适用于静态方法，无论目标脚本是否处于工具模式，都可以调用静态方法。

继承 `@tool` 脚本并不会自动使子脚本也成为 `@tool`。如果子脚本中省略了 `@tool`，超类的工具行为将被禁用。因此，子脚本也应该指定 `@tool` 注解。

在编辑器中的修改是永久性的，且无法撤销/重做。例如，在下一节中当我们移除脚本时，节点将保持其旋转。请小心避免进行不必要的修改。考虑设置[版本控制](https://docs.godotengine.org/en/stable/tutorials/best_practices/version_control_systems.html#doc-version-control-systems)以防出错时丢失工作。

## 调试

虽然调试器和断点无法直接用于工具脚本，但可以启动一个新的编辑器实例并从中进行调试。为此，导航到 **Debug > Customize Run Instances...**（调试 > 自定义运行实例...），并在 **Main Run Args**（主运行参数）中指定 `--editor`。

有关更多信息，请参阅[调试工具概览](https://docs.godotengine.org/en/stable/tutorials/scripting/debug/overview_of_debugging_tools.html#doc-overview-of-debugging-tools)。

此外，你也可以使用 `print` 语句来显示变量的内容。

## 尝试 `@tool`

在场景中添加一个 `Sprite2D` 节点，并将纹理设置为 Godot 图标。附加并打开一个脚本，并将其更改为：

**GDScript**

```gdscript
@tool
extends Sprite2D

func _process(delta):
    rotation += PI * delta
```

**C#**

```csharp
using Godot;

[Tool]
public partial class MySprite : Sprite2D
{
    public override void _Process(double delta)
    {
        Rotation += Mathf.Pi * (float)delta;
    }
}
```

保存脚本并返回编辑器。你现在应该能看到对象在旋转。如果你运行游戏，它也会旋转。

> **警告**
>
> 你可能需要重启编辑器。这是所有 Godot 4 版本中存在的一个已知 Bug：[GH-66381](https://github.com/godotengine/godot/issues/66381)。

![在编辑器中旋转](https://docs.godotengine.org/en/stable/_images/rotating_in_editor.gif)

> **注意**
>
> 如果你看不到变化，请重新加载场景（关闭它并重新打开）。

现在让我们选择哪些代码在何时运行。修改你的 `_process()` 函数如下：

**GDScript**

```gdscript
func _process(delta):
    if Engine.is_editor_hint():
        rotation += PI * delta
    else:
        rotation -= PI * delta
```

**C#**

```csharp
public override void _Process(double delta)
{
    if (Engine.IsEditorHint())
    {
        Rotation += Mathf.Pi * (float)delta;
    }
    else
    {
        Rotation -= Mathf.Pi * (float)delta;
    }
}
```

保存脚本。现在对象在编辑器中将顺时针旋转，但如果你运行游戏，它将逆时针旋转。

## 编辑变量

向脚本添加并导出（export）一个变量 `speed`。为了更新速度并重置旋转角度，添加一个 setter `set(new_speed)`，它会随着检查器（inspector）中的输入而执行。修改 `_process()` 以包含旋转速度。

**GDScript**

```gdscript
@tool
extends Sprite2D

@export var speed = 1:
    # 更新速度并重置旋转。
    set(new_speed):
        speed = new_speed
        rotation = 0

func _process(delta):
    rotation += PI * delta * speed
```

**C#**

```csharp
using Godot;

[Tool]
public partial class MySprite : Sprite2D
{
    private float _speed = 1;

    [Export]
    public float Speed
    {
        get => _speed;
        set
        {
            // 更新速度并重置旋转。
            _speed = value;
            Rotation = 0;
        }
    }

    public override void _Process(double delta)
    {
        Rotation += Mathf.Pi * (float)delta * _speed;
    }
}
```

> **注意**
>
> 来自其他节点的代码不会在编辑器中运行。你对其他节点的访问是受限的。你可以访问树和节点及其默认属性，但无法访问用户变量。如果你想这样做，其他节点也必须在编辑器中运行。

## 当资源改变时获得通知

有时你希望你的工具使用一个资源（resource）。然而，当你在编辑器中更改该资源的属性时，工具的 `set()` 方法将不会被调用。

**GDScript**

```gdscript
@tool
class_name MyTool
extends Node

@export var resource: MyResource:
    set(new_resource):
        resource = new_resource
        _on_resource_set()

# 这仅在创建、删除或粘贴资源时被调用。
# 调整其属性时你不会收到更新。
func _on_resource_set():
    print("My resource was set!")
```

**C#**

```csharp
using Godot;

[Tool]
public partial class MyTool : Node
{
    private MyResource _resource;

    [Export]
    public MyResource Resource
    {
        get => _resource;
        set
        {
            _resource = value;
            OnResourceSet();
        }
    }

    // 这仅在创建、删除或粘贴资源时被调用。
    // 调整其属性时你不会收到更新。
    private void OnResourceSet()
    {
        GD.Print("My resource was set!");
    }
}
```

要解决这个问题，你首先必须让你的资源也成为工具，并让它在设置属性时发出 `changed` 信号：

**GDScript**

```gdscript
# 让你的资源成为工具。
@tool
class_name MyResource
extends Resource

@export var property = 1:
    set(new_setting):
        property = new_setting
        # 当属性改变时发出信号。
        changed.emit()
```

**C#**

```csharp
using Godot;

[Tool]
public partial class MyResource : Resource
{
    private float _property = 1;

    [Export]
    public float Property
    {
        get => _property;
        set
        {
            _property = value;
            // 当属性改变时发出信号。
            EmitChanged();
        }
    }
}
```

然后，你想在设置新资源时连接信号：

**GDScript**

```gdscript
@tool
class_name MyTool
extends Node

@export var resource: MyResource:
    set(new_resource):
        resource = new_resource
        # 一旦添加新资源就连接 changed 信号。
        if resource != null:
            resource.changed.connect(_on_resource_changed)

func _on_resource_changed():
    print("My resource just changed!")
```

**C#**

```csharp
using Godot;

[Tool]
public partial class MyTool : Node
{
    private MyResource _resource;

    [Export]
    public MyResource Resource
    {
        get => _resource;
        set
        {
            _resource = value;
            // 一旦添加新资源就连接 Changed 信号。
            if (_resource != null)
            {
                _resource.Changed += OnResourceChanged;
            }
        }
    }

    private void OnResourceChanged()
    {
        GD.Print("My resource just changed!");
    }
}
```

最后，记得断开信号，因为旧资源在别处被使用和改变会导致不必要的更新。

**GDScript**

```gdscript
@export var resource: MyResource:
    set(new_resource):
        # 如果之前的资源不为空，断开信号。
        if resource != null:
            resource.changed.disconnect(_on_resource_changed)
        resource = new_resource
        if resource != null:
            resource.changed.connect(_on_resource_changed)
```

**C#**

```csharp
[Export]
public MyResource Resource
{
    get => _resource;
    set
    {
        // 如果之前的资源不为空，断开信号。
        if (_resource != null)
        {
            _resource.Changed -= OnResourceChanged;
        }
        _resource = value;
        if (_resource != null)
        {
            _resource.Changed += OnResourceChanged;
        }
    }
}
```

## 报告节点配置警告

Godot 使用“节点配置警告”系统来警告用户节点配置不正确。当节点配置不正确时，场景停靠面板中该节点名称旁边会出现一个黄色警告标志。当你悬停或点击该图标时，会弹出警告消息。你可以在脚本中使用此功能来帮助你和你的团队在设置场景时避免错误。

使用节点配置警告时，当任何应影响或移除警告的值发生变化时，你需要调用 `update_configuration_warnings`。默认情况下，警告仅在关闭并重新打开场景时更新。

**GDScript**

```gdscript
# 使用 setter 自动更新配置警告。
@export var title = "":
    set(p_title):
        if p_title != title:
            title = p_title
            update_configuration_warnings()

@export var description = "":
    set(p_description):
        if p_description != description:
            description = p_description
            update_configuration_warnings()

func _get_configuration_warnings():
    var warnings = []

    if title == "":
        warnings.append("请将 `title` 设置为非空值。")

    if description.length() >= 100:
        warnings.append("`description` 的长度应少于 100 个字符。")

    # 返回空数组意味着“无警告”。
    return warnings
```

## 使用 EditorScript 运行一次性脚本

有时，你需要运行一次代码来自动化编辑器中没有开箱即用提供的某些任务。一些示例可能是：

*   作为 GDScript 或 C# 脚本的游乐场，无需运行项目即可测试。`print()` 输出显示在编辑器的“输出”面板中。
*   缩放当前编辑场景中的所有灯光节点，因为你注意到在放置好灯光后，关卡看起来太暗或太亮。
*   将通过复制粘贴得到的节点替换为场景实例，以便以后更容易修改。

这在 Godot 中可以通过在脚本中扩展 `EditorScript` 来实现。这提供了一种在编辑器中运行单个脚本的方法，而无需创建编辑器插件。

要创建 EditorScript，请在文件系统面板中右键点击文件夹或空白处，然后选择 **New > Script...**（新建 > 脚本...）。在脚本创建对话框中，点击树形图标以选择要继承的对象（或直接在左侧字段中输入 `EditorScript`，注意这区分大小写）：

![在脚本编辑器创建对话框中创建编辑器脚本](https://docs.godotengine.org/en/stable/_images/running_code_in_the_editor_creating_editor_script.webp)

这将自动选择适合 EditorScript 的脚本模板，其中已经插入了 `_run()` 方法：

```gdscript
@tool
extends EditorScript

# 当脚本执行时调用（使用 脚本编辑器中的 文件 -> 运行）。
func _run():
    pass
```

当你使用 **File > Run**（文件 > 运行）或键盘快捷键 `Ctrl + Shift + X` 时，此 `_run()` 方法会执行（前提是 EditorScript 是当前脚本编辑器中打开的脚本）。此键盘快捷键仅在当前聚焦于脚本编辑器时有效。

扩展 EditorScript 的脚本必须是 `@tool` 脚本才能运行。

> **注意**
>
> EditorScript 只能从 Godot 脚本编辑器中运行。如果你使用的是外部编辑器，请在 Godot 脚本编辑器内打开脚本以运行它。

> **危险**
>
> EditorScript 没有撤销/重做功能，所以如果脚本旨在修改任何数据，**请确保在运行之前保存你的场景**。

要访问当前编辑场景中的节点，请使用 `EditorScript.get_scene` 方法，它会返回当前编辑场景的根节点。以下是一个递归获取当前编辑场景中所有节点并将所有 `OmniLight3D` 节点范围翻倍的示例：

```gdscript
@tool
extends EditorScript

func _run():
    for node in get_all_children(get_scene()):
        if node is OmniLight3D:
            # 不要操作实例化的子场景节点，因为重新加载场景时更改会丢失。
            # 参见下文“实例化场景”部分对 `owner` 的描述。
            var is_instanced_subscene_child = node != get_scene() and node.owner != get_scene()
            if not is_instanced_subscene_child:
                node.omni_range *= 2.0

# 此函数是递归的：它根据需要调用自身以获取更深层级的子节点。
# `children_acc` 是允许此函数工作的累加器参数。
# 当你直接调用此函数时，应将其保留为默认值。
func get_all_children(in_node, children_acc = []):
    children_acc.push_back(in_node)
    for child in in_node.get_children():
        children_acc = get_all_children(child, children_acc)

    return children_acc
```

> **提示**
>
> 即使在脚本视图打开时，你也可以更改编辑器顶部当前编辑的场景。这将影响 `EditorScript.get_scene` 的返回值，因此在运行脚本之前，请确保你已经选择了打算迭代的场景。

## 实例化场景

你可以像往常一样实例化打包场景（packed scenes）并将它们添加到编辑器中当前打开的场景。默认情况下，使用 `Node.add_child(node)` 添加的节点或场景在场景树停靠面板中是**不**可见的，也**不会**保存到磁盘。如果你希望节点或场景在场景树停靠面板中可见并在保存场景时持久化到磁盘，你需要将子节点的 `owner` 属性设置为当前编辑场景的根节点。

如果你正在使用 `@tool`：

**GDScript**

```gdscript
func _ready():
    var node = Node3D.new()
    add_child(node) # 父节点可以是场景中的任何节点

    # 下面这行代码是必需的，以便让节点在场景树面板中可见，
    # 并将工具脚本所做的更改持久化到保存的场景文件中。
    node.owner = get_tree().edited_scene_root
```

**C#**

```csharp
public override void _Ready()
{
    var node = new Node3D();
    AddChild(node); // 父节点可以是场景中的任何节点

    // 下面这行代码是必需的，以便让节点在场景树面板中可见，
    // 并将工具脚本所做的更改持久化到保存的场景文件中。
    node.Owner = GetTree().EditedSceneRoot;
}
```

如果你正在使用 `EditorScript`：

**GDScript**

```gdscript
func _run():
    # `parent` 可以是场景中的任何节点。
    var parent = get_scene().get_node("Parent")
    var node = Node3D.new()
    parent.add_child(node)

    # 下面这行代码是必需的，以便让节点在场景树面板中可见，
    # 并将工具脚本所做的更改持久化到保存的场景文件中。
    node.owner = get_scene()
```

**C#**

```csharp
public override void _Run()
{
    // `parent` 可以是场景中的任何节点。
    var parent = GetScene().GetNode("Parent");
    var node = new Node3D();
    parent.AddChild(node);

    // 下面这行代码是必需的，以便让节点在场景树面板中可见，
    // 并将工具脚本所做的更改持久化到保存的场景文件中。
    node.Owner = GetScene();
}
```

> **警告**
>
> 不当地使用 `@tool` 可能会导致许多错误。建议先按照你想要的方式编写代码，然后再在顶部添加 `@tool` 注解。此外，确保将编辑器内运行的代码与游戏中运行的代码分开。这样，你可以更容易地发现错误。
