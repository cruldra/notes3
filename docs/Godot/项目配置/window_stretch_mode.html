window_stretch_mode

project.godot文件中的window/stretch/mode="canvas_items"这行代码是什么意思

这行代码配置了 Godot 游戏在窗口大小发生变化（或者在不同分辨率的屏幕上运行）时的画面缩放策略。

window/stretch/mode="canvas_items" 的意思是：“保持逻辑分辨率不变，但在渲染时使用屏幕的物理像素，以保证画面清晰锐利。”

这是现代高清 2D 游戏（非像素风）和应用软件最常用的模式。

下面是详细的拆解和对比：

1. 核心作用：清晰度 (Sharpness)​

当你把游戏窗口拉大，或者在 4K 屏幕上全屏运行时，Godot 需要决定如何处理画面。

  • 如果选 viewport (视口模式): 引擎会先把画面渲染在一个小的画布上（比如你设定的 1280x720），然后像把一张小图片拉大一样，粗暴地把画面放大。

  • 结果： 画面变模糊，或者是马赛克变大（适合像素游戏）。

  • 如果选 canvas_items (当前模式): 引擎会计算出物体放大后的位置，直接利用屏幕的原生分辨率（比如 1920x1080 或 4K）来绘制图像和文字。

  • 结果： 文字边缘极其清晰，圆形非常圆润，没有任何锯齿或模糊。

2. 坐标系不变​

虽然画面变清晰了，但你写代码时的坐标系不会变。

  • 假设你的“项目设置”里基础分辨率是 1280x720。
  • 屏幕中心点是 (640, 360)。
  • 当你把窗口拉大到 2560x1440 (2倍) 时：
  • 代码逻辑上： 中心点依然是 (640, 360)，你不需要修改任何代码。
  • 视觉上： 那个物体依然在屏幕正中间，但是它占用的物理像素变多了，看起来更精细了。

3. 三种模式的对比 (一图胜千言)​

模式 (Mode)渲染逻辑视觉效果适用场景
disabled不缩放窗口变大时，UI 不变大，你只是看到了更宽广的视野（如果是 UI 则会留黑边）。极少使用，或者特殊的地图编辑器工具。
canvas_items重绘 (Redraw)清晰、锐利。文字和矢量图形在高分屏上完美显示。高清 2D 游戏 (如《空洞骑士》)、软件应用、文字游戏。
viewport放大 (Upscale)模糊 (如果没开过滤) 或 大颗粒像素 (如果开了最近邻过滤)。复古像素风游戏 (Pixel Art)。确保像素点是正方形且清晰。

4. 编辑器中的位置​

你不需要手动改 project.godot 文件。这行代码对应编辑器设置：

  1. 点击顶部菜单 项目 (Project) -> 项目设置 (Project Settings)。
  2. 进入 显示 (Display) -> 窗口 (Window)。
  3. 找到 拉伸 (Stretch) -> 模式 (Mode)。
  4. 下拉菜单中选择了 canvas_items。

总结​

  • 如果你在做 像素艺术 (Pixel Art) 游戏：这个设置可能是错的。你应该改用 viewport，否则像素旋转时会出现“歪歪扭扭”的子像素瑕疵。
  • 如果你在做 高清手绘、矢量风格或 UI 为主 的游戏：这个设置是完美的。

你是正在制作像素风游戏，还是高清/手绘风格的游戏？这决定了你要不要修改这个设置。
