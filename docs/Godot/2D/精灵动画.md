## 简介 (Introduction)

在本教程中，你将学习如何使用 `AnimatedSprite2D` 类和 `AnimationPlayer` 来创建 2D 动画角色。通常，当你创建或下载一个动画角色时，它会有两种提供方式：作为一组独立的图像文件，或者是作为包含所有动画帧的单张精灵表（Sprite Sheet）。在 Godot 中，这两种方式都可以使用 `AnimatedSprite2D` 类进行动画处理。

首先，我们将使用 [AnimatedSprite2D](https://docs.godotengine.org/en/stable/classes/class_animatedsprite2d.html) 来制作一组独立图像的动画。然后，我们将使用该类制作精灵表的动画。最后，我们将学习另一种通过 [AnimationPlayer](https://docs.godotengine.org/en/stable/classes/class_animationplayer.html) 和 [Sprite2D](https://docs.godotengine.org/en/stable/classes/class_sprite2d.html) 的 *Animation* 属性制作精灵表动画的方法。

> **注意**
>
> 以下示例中使用的美术资产由 [ansimuz](https://opengameart.org/users/ansimuz) 和 tgfcoder 提供。

## 使用 AnimatedSprite2D 处理独立图像 (Individual images with AnimatedSprite2D)

在这种情况下，你有一组图像，每张图像包含角色动画的一个帧。对于这个例子，我们将使用以下运行动画：

![运行动画预览](https://docs.godotengine.org/en/stable/_images/2d_animation_run_preview.gif)

你可以从这里下载图像资源：[2d_sprite_animation_assets.zip](https://github.com/godotengine/godot-docs-project-starters/releases/download/latest-4.x/2d_sprite_animation_assets.zip)

解压图像并将其放入你的项目文件夹。使用以下节点设置你的场景树：

![场景树设置](https://docs.godotengine.org/en/stable/_images/2d_animation_tree1.webp)

> **注意**
>
> 根节点也可以是 [Area2D](https://docs.godotengine.org/en/stable/classes/class_area2d.html) 或 [RigidBody2D](https://docs.godotengine.org/en/stable/classes/class_rigidbody2d.html)。动画的制作方式是相同的。一旦动画完成，你可以为 `CollisionShape2D` 分配一个形状。详见 [物理系统简介](https://docs.godotengine.org/en/stable/tutorials/physics/physics_introduction.html)。

现在选择 `AnimatedSprite2D`，在其 **SpriteFrames** 属性中选择 “新建 SpriteFrames”。

![新建 SpriteFrames](https://docs.godotengine.org/en/stable/_images/2d_animation_new_spriteframes.webp)

点击新建的 SpriteFrames 资源，你会看到编辑器窗口底部出现一个新面板：

![SpriteFrames 编辑器面板](https://docs.godotengine.org/en/stable/_images/2d_animation_spriteframes.webp)

从左侧的 “文件系统” 面板中，将 8 张独立图像拖入 SpriteFrames 面板的中央区域。在左侧，将动画名称从 “default” 更改为 “run”。

![完成后的 SpriteFrames 设置](https://docs.godotengine.org/en/stable/_images/2d_animation_spriteframes_done.webp)

使用 “播放” 按钮预览动画。你现在应该能看到动画在视口中播放。但是，它看起来有点慢。为了修复这个问题，在 SpriteFrames 面板中将 **Speed (FPS)** 设置改为 10。

你可以通过点击 “添加动画” 按钮并添加更多图像来创建额外的动画。

### 通过代码控制动画 (Controlling the animation)

动画准备就绪后，你可以通过代码使用 `play()` 和 `stop()` 方法来控制它。下面是一个简单的示例：当按住右方向键时播放动画，释放时停止。

```gdscript
extends CharacterBody2D

@onready var _animated_sprite = $AnimatedSprite2D

func _process(_delta):
	if Input.is_action_pressed("ui_right"):
		_animated_sprite.play("run")
	else:
		_animated_sprite.stop()
```

```csharp
using Godot;

public partial class Character : CharacterBody2D
{
    private AnimatedSprite2D _animatedSprite;

    public override void _Ready()
    {
        _animatedSprite = GetNode<AnimatedSprite2D>("AnimatedSprite2D");
    }

    public override void _Process(double delta)
    {
        if (Input.IsActionPressed("ui_right"))
        {
            _animatedSprite.Play("run");
        }
        else
        {
            _animatedSprite.Stop();
        }
    }
}
```

## 使用 AnimatedSprite2D 处理精灵表 (Sprite sheet with AnimatedSprite2D)

你也可以使用 `AnimatedSprite2D` 类轻松地从精灵表制作动画。我们将使用这张公共领域的精灵表：

![青蛙精灵表](https://docs.godotengine.org/en/stable/_images/2d_animation_frog_spritesheet.png)

右键点击图片并选择 “图片另存为” 下载它，然后将其拷贝到项目文件夹中。

按照之前处理独立图像的方式设置场景树。选择 `AnimatedSprite2D` 并在其 **SpriteFrames** 属性中选择 “新建 SpriteFrames”。

点击新建的资源。这一次，在底部面板出现时，选择 **从精灵表添加帧 (Add frames from a Sprite Sheet)** 按钮。

![从精灵表添加帧](https://docs.godotengine.org/en/stable/_images/2d_animation_add_from_spritesheet.webp)

在弹出的文件对话框中选择你的精灵表。

一个新窗口会打开并显示你的精灵表。你首先需要做的是更改精灵表中垂直和水平图像的数量。在这张精灵表中，我们有 4 列（水平）和 2 行（垂直）。

![设置行和列](https://docs.godotengine.org/en/stable/_images/2d_animation_spritesheet_select_rows.webp)

接下来，选择你想要包含在动画中的帧。我们将选择顶部的四个，然后点击 “添加 4 帧” 来创建动画。

![选择动画帧](https://docs.godotengine.org/en/stable/_images/2d_animation_spritesheet_selectframes.webp)

你现在会在底部面板的动画列表中看到你的动画。双击 “default” 将动画名称更改为 “jump”。

![重命名动画](https://docs.godotengine.org/en/stable/_images/2d_animation_spritesheet_animation.webp)

最后，点击 SpriteFrames 编辑器上的播放按钮，看看你的青蛙跳跃吧！

![预览青蛙跳跃](https://docs.godotengine.org/en/stable/_images/2d_animation_play_spritesheet_animation.webp)

## 使用 AnimationPlayer 处理精灵表 (Sprite sheet with AnimationPlayer)

另一种在使用精灵表时进行动画处理的方法是：使用标准的 [Sprite2D](https://docs.godotengine.org/en/stable/classes/class_sprite2d.html) 节点显示纹理，然后使用 [AnimationPlayer](https://docs.godotengine.org/en/stable/classes/class_animationplayer.html) 动画化纹理的变化。

考虑这张包含 6 帧动画的精灵表：

![跑步精灵表](https://docs.godotengine.org/en/stable/_images/2d_animation_player-run.png)

下载该图片并存入项目文件夹。我们的目标是循环显示这些图像。首先设置场景树：

![场景树设置 2](https://docs.godotengine.org/en/stable/_images/2d_animation_tree2.webp)

> **注意**
>
> 根节点也可以是 [Area2D](https://docs.godotengine.org/en/stable/classes/class_area2d.html) 或 [RigidBody2D](https://docs.godotengine.org/en/stable/classes/class_rigidbody2d.html)。

将精灵表拖入 Sprite2D 的 **Texture** 属性，你会看到整张表显示在屏幕上。为了将其切分成单个帧，在检查器中展开 **Animation** 部分，并将 **Hframes** 设置为 `6`。*Hframes* 和 *Vframes* 分别是精灵表中的水平和垂直帧数。

![设置 Hframes](https://docs.godotengine.org/en/stable/_images/2d_animation_setframes.webp)

现在尝试更改 **Frame** 属性的值。你会发现它的范围是 `0` 到 `5`，并且 `Sprite2D` 显示的图像会相应改变。这正是我们要动画化的属性。

选择 `AnimationPlayer` 节点，点击 “动画” 按钮并选择 “新建”。将新动画命名为 “walk”。将动画长度设置为 `0.6` 秒，并点击 “循环” 按钮。

![新建动画](https://docs.godotengine.org/en/stable/_images/2d_animation_new_animation.webp)

现在选择 `Sprite2D` 节点，并在检查器中点击 **Frame** 属性旁的钥匙图标来添加一个新轨道。

![添加轨道](https://docs.godotengine.org/en/stable/_images/2d_animation_new_track.webp)

继续在时间轴的每个点（默认间隔 `0.1` 秒）添加帧，直到包含从 0 到 5 的所有帧。你会看到这些帧实际出现在了动画轨道中：

![完整的动画轨道](https://docs.godotengine.org/en/stable/_images/2d_animation_full_animation.webp)

点击播放按钮预览效果。

![运行动画效果](https://docs.godotengine.org/en/stable/_images/2d_animation_running.gif)

### 通过代码控制 AnimationPlayer 动画 (Controlling an AnimationPlayer animation)

与 `AnimatedSprite2D` 类似，你可以使用 `play()` 和 `stop()` 方法通过代码控制动画。

```gdscript
extends CharacterBody2D

@onready var _animation_player = $AnimationPlayer

func _process(_delta):
	if Input.is_action_pressed("ui_right"):
		_animation_player.play("walk")
	else:
		_animation_player.stop()
```

> **注意**
>
> 如果你同时更新动画和另一个独立属性（例如，一个平台游戏角色转向时同时开始“转身”动画并修改 `flip_h`），请务必记住 `play()` 并不是立即应用的。它会在下一次 `AnimationPlayer` 处理时（通常是下一帧）生效。如果这导致了“闪烁”或一帧的动作错误，你可以在调用 `play()` 之后调用 `advance(0)` 来强制动画立即更新。

## 总结 (Summary)

以上示例展示了 Godot 中可用于 2D 动画的两个类。`AnimationPlayer` 比 `AnimatedSprite2D` 略微复杂一些，但它提供了额外的功能，因为你还可以动画化位置、缩放等其他属性。此外，`AnimationPlayer` 也可以与 `AnimatedSprite2D` 配合使用。根据你的具体需求进行选择和实验。
