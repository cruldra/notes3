## 简介 (Introduction)

在 3D 中，网格（Meshes）用于显示世界。而在 2D 中，由于图像（Images）更为常用，网格的使用相对较少。Godot 的 2D 引擎是纯二维引擎，因此它无法直接显示 3D 网格（尽管可以通过 `Viewport` 和 `ViewportTexture` 间接实现）。

> **另请参阅**
>
> 如果你对在 2D 视口中显示 3D 网格感兴趣，请参阅 [使用 SubViewport 作为纹理](https://docs.godotengine.org/en/stable/tutorials/shaders/using_viewport_as_texture.html) 教程。

2D 网格是包含二维几何信息（Z 轴被省略或忽略）的网格。你可以尝试通过代码使用 `SurfaceTool` 创建它们，并在 `MeshInstance2D` 节点中显示。

目前，在编辑器中生成 2D 网格的唯一方法是：要么将 OBJ 文件作为网格导入，要么从 `Sprite2D` 转换而来。

## 优化像素绘制 (Optimizing pixels drawn)

这种工作流在某些情况下对于优化 2D 绘制非常有用。当绘制带有透明区域的大型图像时，Godot 会将整个矩形块（Quad）绘制到屏幕上，即使是大量的透明区域也会参与绘制。

这会影响性能，特别是在移动设备上，或者在绘制非常大的图像（通常是屏幕大小）以及叠放多张具有大面积透明区域的图像时（例如使用 `ParallaxBackground` 时）。

将图像转换为网格可以确保只绘制不透明的部分，而忽略其余部分。

## 将 Sprite2D 转换为 2D 网格 (Converting Sprite2Ds to 2D meshes)

你可以通过将 `Sprite2D` 转换为 `MeshInstance2D` 来利用这种优化。首先选择一个边缘包含大量透明区域的图像，比如这棵树：

![包含大量透明区域的树](https://docs.godotengine.org/en/stable/_images/mesh2d1.png)

将其放入 `Sprite2D` 中，然后从菜单中选择 **Convert to MeshInstance2D (转换为 MeshInstance2D)**：

![菜单中的转换选项](https://docs.godotengine.org/en/stable/_images/mesh2d2.webp)

随后会弹出一个对话框，展示 2D 网格创建的预览：

![转换预览对话框](https://docs.godotengine.org/en/stable/_images/mesh2d3.webp)

默认值在大多数情况下已经足够好，但你也可以根据需要调整 **Growth (膨胀)** 和 **Simplification (简化)** 参数：

![调整参数](https://docs.godotengine.org/en/stable/_images/mesh2d4.webp)

最后，点击 **Convert 2D Mesh** 按钮，你的 `Sprite2D` 就会被替换：

![转换后的结果](https://docs.godotengine.org/en/stable/_images/mesh2d5.webp)
