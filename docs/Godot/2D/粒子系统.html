粒子系统

简介 (Intro)​

粒子系统用于模拟复杂的物理效果，如火花、火焰、魔法粒子、烟雾、薄雾等。

其基本理念是：以固定的时间间隔发射具有固定寿命的“粒子”。在寿命期间，每个粒子都遵循相同的基本行为。使每个粒子看起来不同且更具生动感的是与每个参数相关联的“随机性”。本质上，创建粒子系统意味着设置基础物理参数，然后为其添加随机性。

粒子节点 (Particle nodes)​

Godot 为 2D 粒子提供了两种不同的节点：GPUParticles2D 和 CPUParticles2D。

  • GPUParticles2D：更高级，使用 GPU 处理粒子效果。
  • CPUParticles2D：由 CPU 驱动，功能与 GPUParticles2D 几乎一致，但在粒子数量较多时性能较低。不过，在低端系统或 GPU 瓶颈的情况下，其表现可能更好。

GPUParticles2D 是通过 ParticleProcessMaterial（以及可选的自定义着色器）进行配置的，而 CPUParticles2D 的对应选项则是直接作为节点属性提供的。

目前没有计划为 CPUParticles2D 添加新功能，因此除非有明确理由，否则我们建议优先使用 GPUParticles2D。

你可以在工具栏中通过 CPUParticles2D > Convert to GPUParticles2D 将 CPU 粒子转换为 GPU 粒子：

时间参数 (Time parameters)​

寿命 (Lifetime)​

每个粒子存活的时间（秒）。当寿命结束，系统会创建一个新粒子来替换它。

Lifetime: 0.5

Lifetime: 4.0

单次发射 (One Shot)​

启用后，节点将发射完所有粒子一次后停止，不再循环。

预处理 (Preprocess)​

粒子系统最初从零粒子开始发射。预处理允许系统在第一次实际绘制之前先运行若干秒，这对于场景加载时就需要“已经燃起来”的火把或弥漫的雾气非常有用。

速度缩放 (Speed Scale)​

调整整个粒子系统的运行快慢。增加此值会让粒子运行得更快。

爆炸性 (Explosiveness)​

  • 0：按常规间隔发射粒子（默认）。
  • 1：所有粒子同时喷发。 用于创建爆炸或瞬间喷溅效果：

随机性 (Randomness)​

所有的物理参数都可以随机化，范围为 0 到 1。

固定 FPS (Fixed FPS)​

让粒子系统以固定的帧率进行渲染。例如设为 2 会让粒子以每秒 2 帧的速率更新。注意这不会减慢物理模拟本身。

碎分数增量 (Fract Delta)​

启用此项可以获得更精确电计算和更平滑的粒子显示效果，特别是在具有高随机性或快速移动粒子的系统中。这有助于保持视觉一致性，但会有性能代价。

绘制参数 (Drawing parameters)​

可见性矩形 (Visibility Rect)​

控制粒子在屏幕上的可见区域。如果此矩形在视口外，引擎将停止渲染以优化性能。

你可以通过 2D 视图上方的工具栏让 Godot 自动生成：选择节点并点击 Particles > Generate Visibility Rect。

本地坐标 (Local Coords)​

  • 开启（默认）：粒子随发射节点一起移动。
  • 禁用：粒子发射到全局空间。如果节点移动，已经发射出去的粒子不会跟着动。

绘制顺序 (Draw Order)​

控制粒子的重叠顺序：Index（按发射顺序）或 Lifetime（按剩余寿命）。

关于 ParticleProcessMaterial 中各设置项的详细信息，请参阅 此页面。
