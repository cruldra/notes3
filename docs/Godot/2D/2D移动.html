2D移动

每个初学者都遇到过这样的问题：“我该如何让我的角色移动？”根据你正在制作的游戏类型，你可能会有特殊的要求，但一般来说，大多数 2D 游戏中的移动都是基于少数几种设计的。

在这些示例中，我们将使用 CharacterBody2D，但其原理也适用于其他节点类型（Area2D、RigidBody2D）。

设置​

下面的每个示例都使用相同的场景设置。首先创建一个带有两个子节点的 CharacterBody2D：Sprite2D 和 CollisionShape2D。你可以使用 Godot 图标 ("icon.png") 作为 Sprite2D 的纹理，或者使用你拥有的任何其他 2D 图像。

打开 项目 -> 项目设置 并选择 “输入映射” 选项卡。添加以下输入动作（详见 InputEvent）：

8 向移动​

在这种场景中，你希望用户按下四个方向键（上/左/下/右 或 W/A/S/D）并向所选方向移动。“8 向移动” 这个名字源于玩家可以通过同时按下两个键来进行斜向移动。

  • GDScript
  • C#
extends CharacterBody2D

@export var speed = 400

func get_input():
	# 使用 Input.get_vector() 检查四个按键事件并返回一个方向向量。
	var input_direction = Input.get_vector("left", "right", "up", "down")
	velocity = input_direction * speed

func _physics_process(delta):
	get_input()
	move_and_slide()
using Godot;

public partial class Movement : CharacterBody2D
{
    [Export]
    public int Speed { get; set; } = 400;

    public void GetInput()
    {
        // 使用 Input.GetVector() 检查四个按键事件并返回一个方向向量。
        Vector2 inputDirection = Input.GetVector("left", "right", "up", "down");
        Velocity = inputDirection * Speed;
    }

    public override void _PhysicsProcess(double delta)
    {
        GetInput();
        MoveAndSlide();
    }
}

在 get_input() 函数中，我们使用 Input 的 get_vector() 方法来检查四个按键事件，并返回一个相加后的方向向量。

然后，我们可以通过将这个长度为 1 的方向向量乘以我们期望的速度来设置速度（velocity）。

提示：如果你以前从未接触过向量数学，或者需要温习一下，可以在 向量数学 中查看 Godot 中向量使用的说明。

注意：如果按下按键时上面的代码没有任何反应，请仔细检查你是否按照本教程 设置 部分的说明正确设置了输入动作。

旋转 + 移动​

这种移动类型有时被称为 “Asteroids 风格”，因为它类似于那款经典街机游戏的运作方式。按下左/右键会旋转角色，而按上/下键则会根据角色所面对的方向向前或向后移动。

  • GDScript
  • C#
extends CharacterBody2D

@export var speed = 400
@export var rotation_speed = 1.5

var rotation_direction = 0

func get_input():
	rotation_direction = Input.get_axis("left", "right")
	# 使用 transform.x（指向角色“前方”的向量）乘以输入轴和速度来设置 velocity。
	velocity = transform.x * Input.get_axis("down", "up") * speed

func _physics_process(delta):
	get_input()
	rotation += rotation_direction * rotation_speed * delta
	move_and_slide()
using Godot;

public partial class Movement : CharacterBody2D
{
    [Export]
    public int Speed { get; set; } = 400;

    [Export]
    public float RotationSpeed { get; set; } = 1.5f;

    private float _rotationDirection;

    public void GetInput()
    {
        _rotationDirection = Input.GetAxis("left", "right");
        // 使用 Transform.X（指向角色“前方”的向量）乘以输入轴和速度来设置 Velocity。
        Velocity = Transform.X * Input.GetAxis("down", "up") * Speed;
    }

    public override void _PhysicsProcess(double delta)
    {
        GetInput();
        Rotation += _rotationDirection * RotationSpeed * (float)delta;
        MoveAndSlide();
    }
}

这里我们添加了两个变量来跟踪旋转方向和旋转速度。旋转直接应用到物体的 rotation 属性上。

为了设置速度（velocity），我们使用物体的 transform.x，这是一个指向物体 “前方” 方向的向量，然后将其乘以速度。

旋转 + 移动 (鼠标)​

这种移动风格是前一种的变体。这一次，方向是由鼠标位置而不是键盘设置的。角色将始终 “看向” 鼠标指针。不过，向前/向后的输入保持不变。

  • GDScript
  • C#
extends CharacterBody2D

@export var speed = 400

func get_input():
	# 使玩家看向鼠标位置。
	look_at(get_global_mouse_position())
	velocity = transform.x * Input.get_axis("down", "up") * speed

func _physics_process(delta):
	get_input()
	move_and_slide()
using Godot;

public partial class Movement : CharacterBody2D
{
    [Export]
    public int Speed { get; set; } = 400;

    public void GetInput()
    {
        // 使玩家看向鼠标位置。
        LookAt(GetGlobalMousePosition());
        Velocity = Transform.X * Input.GetAxis("down", "up") * Speed;
    }

    public override void _PhysicsProcess(double delta)
    {
        GetInput();
        MoveAndSlide();
    }
}

这里我们使用了 Node2D 的 look_at() 方法来使玩家指向鼠标的位置。如果没有这个函数，你也可以通过像这样设置角度来达到同样的效果：

  • GDScript
  • C#
rotation = get_global_mouse_position().angle_to_point(position)
var rotation = GetGlobalMousePosition().AngleToPoint(Position);

点击移动​

最后一个示例仅使用鼠标来控制角色。点击屏幕将导致玩家移动到目标位置。

  • GDScript
  • C#
extends CharacterBody2D

@export var speed = 400

var target = position

func _input(event):
	# 使用 is_action_pressed 仅接受单击作为输入，而不是鼠标拖动。
	if event.is_action_pressed(&"click"):
		target = get_global_mouse_position()

func _physics_process(delta):
	velocity = position.direction_to(target) * speed
	# look_at(target) # 如果需要，可以取消注释使角色转向目标方向
	if position.distance_to(target) > 10:
		move_and_slide()
using Godot;

public partial class Movement : CharacterBody2D
{
    [Export]
    public int Speed { get; set; } = 400;

    private Vector2 _target;

    public override void _Input(InputEvent @event)
    {
        // 使用 IsActionPressed 仅接受单击作为输入，而不是鼠标拖动。
        if (@event.IsActionPressed("click"))
        {
            _target = GetGlobalMousePosition();
        }
    }

    public override void _PhysicsProcess(double delta)
    {
        Velocity = Position.DirectionTo(_target) * Speed;
        // LookAt(_target); // 如果需要，可以取消注释使角色转向目标方向
        if (Position.DistanceTo(_target) > 10)
        {
            MoveAndSlide();
        }
    }
}

注意我们在移动之前进行的 distance_to() 检查。如果没有这个测试，物体在到达目标位置时会产生 “抖动”，因为它会稍微移过该位置并尝试移回，结果又移得太远，如此往复。

如果你愿意，取消注释 look_at() 行也会使物体转向其运动方向。

提示：这项技术也可以用作 “跟随” 角色的基础。target 位置可以是你想移动到的任何物体的坐标。

总结​

你可能会发现这些代码示例对于你自己的项目来说是很有用的起点。请随意使用并尝试它们，看看你能做出什么。
