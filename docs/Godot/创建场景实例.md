> **注意**
>
> 本教程讨论的是在编辑器中实例化场景。要学习如何通过代码实例化场景，请参阅 [节点与场景实例](https://docs.godotengine.org/en/stable/tutorials/scripting/nodes_and_scene_instances.html)。
>
> Godot 下面描述的 “实例化 (instancing)” 方法不应与用于快速渲染大量相似物体的硬件实例化混淆。如有此类需求，请参阅 [使用 MultiMeshes 进行优化](https://docs.godotengine.org/en/stable/tutorials/performance/using_multimesh.html)。

在上一部分中，我们了解到场景是组织在树状结构中的节点集合，并以单个节点作为根。你可以将项目拆分为任意数量的场景。此功能有助于你分解并组织游戏的不同组件。

你可以根据需要创建任意多个场景，并将其保存为扩展名为 `.tscn` 的文件（代表 “文本场景 (text scene)”）。上一课中的 `label.tscn` 文件就是一个例子。我们将这些文件称为 “打包场景 (Packed Scenes)”，因为它们打包了关于你场景内容的信息。

下面是一个球的示例。它由一个名为 “Ball” 的 [RigidBody2D](https://docs.godotengine.org/en/stable/classes/class_rigidbody2d.html) 节点作为根（允许球下落并在墙上反弹）、一个 [Sprite2D](https://docs.godotengine.org/en/stable/classes/class_sprite2d.html) 节点和一个 [CollisionShape2D](https://docs.godotengine.org/en/stable/classes/class_collisionshape2d.html) 组成。

![Ball Scene](https://docs.godotengine.org/en/stable/_images/instancing_ball_scene.webp)

一旦保存了场景，它就可以作为一个蓝图：你可以在其他场景中根据需要重复使用它任意多次。这种从模板复制对象的过程称为 **实例化 (instancing)**。

![Ball Instances Example](https://docs.godotengine.org/en/stable/_images/instancing_ball_instances_example.webp)

正如我们在上一部分中提到的，实例化的场景表现得像一个节点：编辑器默认隐藏它们的内容。当你实例化 “Ball” 时，你只能看到 Ball 节点。另外请注意，每个副本都有一个唯一的名称。

“Ball” 场景的每个实例都以与 `ball.tscn` 相同的结构和属性开始。但是，你可以独立修改每一个实例，例如改变它们的反弹方式、重量，或者源场景暴露的任何属性。

## 实践操作

让我们通过实践来了解实例化在 Godot 中是如何工作的。我们邀请你下载我们为你准备的球示例项目：[instancing_starter.zip](https://github.com/godotengine/godot-docs-project-starters/releases/download/latest-4.x/instancing_starter.zip)。

在你的电脑上解压压缩包。要导入它，你需要使用项目管理器。通过打开 Godot 进入项目管理器；如果你已经打开了 Godot，请点击 **项目 > 退出到项目列表** (Ctrl + Shift + Q)。

在项目管理器中，点击 **导入 (Import)** 按钮。

![Import Button](https://docs.godotengine.org/en/stable/_images/instancing_import_button.webp)

在弹出的窗口中，导航到你解压的文件夹。双击 `project.godot` 文件打开它。

![Import Project File](https://docs.godotengine.org/en/stable/_images/instancing_import_project_file.webp)

最后，点击 **导入并编辑 (Import & Edit)** 按钮。

![Import and Edit Button](https://docs.godotengine.org/en/stable/_images/instancing_import_and_edit_button.webp)

可能会出现一个窗口提醒你该项目上次是用旧版本 Godot 打开的，这不是问题。点击 “确定” 打开项目。

该项目包含两个打包场景：`main.tscn`（包含球与其碰撞的墙壁）和 `ball.tscn`。主场景应该会自动打开。如果你看到的是空的 3D 场景而不是主场景，请点击屏幕顶部的 **2D** 按钮。

![2D Scene Select](https://docs.godotengine.org/en/stable/_images/instancing_2d_scene_select.webp) ![Main Scene](https://docs.godotengine.org/en/stable/_images/instancing_main_scene.webp)

让我们添加一个球作为 Main 节点的子节点。在 “场景” 面板中，选择 Main 节点。然后，点击场景面板顶部的 **链条图标**。此按钮允许你将场景的实例添加为当前所选节点的子节点。

![Scene Link Button](https://docs.godotengine.org/en/stable/_images/instancing_scene_link_button.webp)

双击 `ball` 场景将其实例化。

![Instance Child Window](https://docs.godotengine.org/en/stable/_images/instancing_instance_child_window.webp)

球出现在视口的左上角。

![Ball Instanced](https://docs.godotengine.org/en/stable/_images/instancing_ball_instanced.webp)

点击并拖动它到视图中心。

![Ball Moved](https://docs.godotengine.org/en/stable/_images/instancing_ball_moved.webp)

按 **F5** (macOS 为 **Cmd + B**) 运行游戏。你应该能看到它下落。

现在，我们想创建更多 Ball 节点的实例。在球仍被选中的状态下，按 **Ctrl + D** (macOS 为 **Cmd + D**) 调用复制命令。点击并拖动以将新球移动到不同的位置。

![Ball Duplicated](https://docs.godotengine.org/en/stable/_images/instancing_ball_duplicated.webp)

你可以重复此过程，直到场景中有多个球。

![Main Scene with Balls](https://docs.godotengine.org/en/stable/_images/instancing_main_scene_with_balls.webp)

再次运行游戏。你现在应该看到每个球都独立下落。这就是实例的作用：每一个都是模板场景的独立复制。

## 编辑场景与实例

实例还有更多功能。通过此功能，你可以：

1.  使用检查器更改其中一个球的属性，而不会影响其他球。
2.  通过打开 `ball.tscn` 场景并在那里修改 Ball 节点，来更改每个球的默认属性。保存后，项目中 Ball 的所有实例都会更新其值。

> **注意**
>
> 更改实例上的属性始终会覆盖来自对应打包场景的值。

让我们尝试一下。在文件系统中双击 `ball.tscn` 打开它。

![Ball Scene Open](https://docs.godotengine.org/en/stable/_images/instancing_ball_scene_open.webp)

在左侧的 “场景” 面板中，选择 Ball 节点。然后，在右侧的 “检查器” 中，点击 `PhysicsMaterial` 属性以展开它。

![Physics Material Expand](https://docs.godotengine.org/en/stable/_images/instancing_physics_material_expand.webp)

通过点击数字字段，输入 `0.5` 并按回车，将其 `Bounce` (反弹) 属性设置为 `0.5`。

![Property Bounce Updated](https://docs.godotengine.org/en/stable/_images/instancing_property_bounce_updated.webp)

按 **F5** 运行游戏，观察现在所有的球反弹得更厉害了。由于 Ball 场景是所有实例的模板，修改并保存它会导致所有实例相应更新。

现在让我们调整单个实例。点击视口上方的对应选项卡回到 Main 场景。

![Scene Tabs](https://docs.godotengine.org/en/stable/_images/instancing_scene_tabs.webp)

选择其中一个实例化的 Ball 节点，并在检查器中将其 `Gravity Scale` (重力缩放) 值设置为 `10`。

![Property Gravity Scale](https://docs.godotengine.org/en/stable/_images/instancing_property_gravity_scale.webp)

调整后的属性旁边会出现一个灰色的 “重置” 按钮。

![Property Revert Icon](https://docs.godotengine.org/en/stable/_images/instancing_property_revert_icon.webp)

此图标表示你正在覆盖源打包场景中的值。即使你修改了原始场景中的该属性，实例中覆盖的值也会被保留。点击重置图标会将该属性还原为保存场景中的值。

重新运行游戏，观察这个球现在下落的速度比其他球快得多。

> **注意**
>
> 你可能会发现无法更改球的 `PhysicsMaterial` 的值。这是因为 `PhysicsMaterial` 是一个 **资源 (resource)**，在链接到其原始场景的场景中编辑它之前，需要将其设为唯一。要为一个实例使资源唯一，请在检查器的 `Physics Material` 属性上右键点击，然后在上下文菜单中点击 **设为唯一 (Make Unique)**。
>
> 资源是 Godot 游戏的另一个基本构建块，我们将在以后的课程中介绍。

## 场景实例化作为一种设计语言

Godot 中的实例和场景提供了一种出色的设计语言，使该引擎脱颖而出。我们从底层开始围绕这一概念设计了 Godot。

我们建议在使用 Godot 制作游戏时放弃诸如模型-视图-控制器 (MVC) 或实体-关系图之类的架构代码模式。相反，你可以从想象玩家在你的游戏中看到的元素开始，并围绕它们组织你的代码。

例如，你可以这样分解一款射击游戏：

![Shooter Diagram](https://docs.godotengine.org/en/stable/_images/instancing_diagram_shooter.png)

几乎任何类型的游戏都可以画出这样的图。每个矩形代表玩家视角中游戏中可见的一个实体。箭头指向每个场景的实例化者。

一旦有了图表，我们建议为其中列出的每个元素创建一个场景来开发你的游戏。你将使用实例化（通过代码或直接在编辑器中）来构建你的场景树。

程序员往往会花费大量时间设计抽象架构，并试图将组件塞入其中。基于场景的设计使开发更快、更直观，让你专注于游戏逻辑本身。由于大多数游戏组件直接映射到场景，使用基于场景实例化的设计意味着你几乎不需要其他的架构代码。

下面是一个拥有大量资产和嵌套元素的开放世界游戏的场景图示例：

![Open World Diagram](https://docs.godotengine.org/en/stable/_images/instancing_diagram_open_world.png)

想象一下我们从创建房间开始。我们可以制作几个不同的房间场景，里面有独特的家具布置。稍后，我们可以制作一个房屋场景，在内部使用多个房间实例。我们会用许多实例化的房屋创建一个城堡，以及一个放置城堡的大型地形。每一个都是实例化一个或多个子场景的场景。

稍后，我们可以创建代表守卫的场景并将它们添加到城堡中。它们会被间接地添加到整个游戏世界中。

使用 Godot，像这样迭代你的游戏很容易，因为你所需要做的就是创建并实例化更多的场景。我们将编辑器设计得让程序员、设计师和艺术家都能上手。一个典型的团队开发过程可能涉及 2D 或 3D 艺术家、关卡设计师、游戏设计师和动画师，他们都在使用 Godot 编辑器工作。

## 总结

实例化（从蓝图生成对象的过程）有许多方便的用途。配合场景，它为你提供了：

*   将游戏分解为可重用组件的能力。
*   一种构建和封装复杂系统的工具。
*   一种以自然的方式思考游戏项目结构的语言。
