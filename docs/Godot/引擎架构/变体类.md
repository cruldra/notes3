## 关于 (About)

`Variant` 是 Godot 中最重要的数据类型。在 64 位平台上，一个 `Variant` 仅占用 24 字节（在 32 位平台上为 20 字节），它可以存储引擎内几乎任何数据类型。`Variant` 很少用于长时间持有信息，它们主要用于通信、编辑、序列化以及通用的数据传输。

一个 `Variant` 可以：

*   存储几乎任何数据类型。
*   在许多变体之间执行操作（GDScript 使用 `Variant` 作为其原子/原生数据类型）。
*   进行哈希处理，以便可以快速与其他变体进行比较。
*   用于在数据类型之间进行安全转换。
*   用于抽象方法调用及其参数（Godot 通过变体导出其所有函数）。
*   用于延迟调用或在线程之间移动数据。
*   序列化为二进制并存储到磁盘，或通过网络传输。
*   序列化为文本，并用于打印值和可编辑设置。
*   作为导出属性工作，以便编辑器可以对其进行通用编辑。
*   用于字典、数组、解析器等。

基本上，得益于 `Variant` 类，编写 Godot 本身变得容易得多，因为它允许以很少的代价实现 C++ 中不常见的、高度动态的结构。今天就成为 `Variant` 的朋友吧。

> **注意**
>
> `Variant` 内部除了 `Nil` 和 `Object` 之外的所有类型都 **不能** 为 `null`，且必须始终存储有效值。因此，`Variant` 内部的这些类型被称为 *非空 (non-nullable)* 类型。
>
> `Variant` 类型之一是 *Nil*，它只能存储值 `null`。因此，尽管除 `Nil` 和 `Object` 之外的所有 `Variant` 类型都是非空的，但 `Variant` 仍然可能包含值 `null`。

## 容器：Array 和 Dictionary (Containers: Array and Dictionary)

[Array](https://docs.godotengine.org/en/stable/classes/class_array.html) 和 [Dictionary](https://docs.godotengine.org/en/stable/classes/class_dictionary.html) 都是使用变体实现的。一个 `Dictionary` 可以将任何作为键使用的类型匹配到任何其他数据类型。一个 `Array` 只是持有一个 `Variant` 数组。当然，一个 `Variant` 也可以在内部持有一个 `Dictionary` 或一个 `Array`，这使得它更加灵活。

对容器的修改将修改对它的所有引用。如果需要进行 [多线程访问](https://docs.godotengine.org/en/stable/tutorials/performance/using_multiple_threads.html)，应创建一个 [Mutex](https://docs.godotengine.org/en/stable/engine_details/architecture/core_types.html#doc-core-concurrency-types) 来对其进行锁定。
