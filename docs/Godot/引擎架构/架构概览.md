## 场景层 (Scene Layer)

场景层是 Godot 架构的最高层，提供了场景系统，这是构建和组织应用程序或游戏的主要方式。有关更多信息，请参阅 [SceneTree](https://docs.godotengine.org/en/stable/classes/class_scenetree.html) / [使用 SceneTree](https://docs.godotengine.org/en/stable/tutorials/scripting/scene_tree.html) 和 [Node](https://docs.godotengine.org/en/stable/classes/class_node.html)。

对应源代码：[/scene/*](https://github.com/godotengine/godot/tree/master/scene)

## 服务器层 (Server Layer)

服务器组件实现了 Godot 的大部分子系统（渲染、音频、物理等）。它们是在引擎启动时初始化的单例对象。

[为什么 Godot 使用服务器和 RID？](https://godotengine.org/article/why-does-godot-use-servers-and-rids)

对应源代码：[/servers/*](https://github.com/godotengine/godot/tree/master/servers)

## 驱动 / 平台接口 (Drivers / Platform Interface)

该层抽象了底层平台特定的细节，包含图形 API 驱动程序、音频后端和操作系统接口（所有平台特定的 [OS](https://docs.godotengine.org/en/stable/classes/class_os.html) 和 [DisplayServer](https://docs.godotengine.org/en/stable/classes/class_displayserver.html) 实现）。

对应源代码：[/drivers/*](https://github.com/godotengine/godot/tree/master/drivers) 和 [/platform/*](https://github.com/godotengine/godot/tree/master/platform)

## 核心 (Core)

引擎的核心包含了在整个引擎中使用的基本功能和数据结构，如 [Object](https://docs.godotengine.org/en/stable/classes/class_object.html) 和 [ClassDB](https://docs.godotengine.org/en/stable/classes/class_classdb.html)、[内存管理](https://docs.godotengine.org/en/stable/engine_details/architecture/core_types.html)、[容器](https://docs.godotengine.org/en/stable/engine_details/architecture/core_types.html)、文件 I/O、[Variant](https://docs.godotengine.org/en/stable/engine_details/architecture/variant_class.html) 以及 [其他工具](https://docs.godotengine.org/en/stable/engine_details/architecture/common_engine_methods_and_macros.html)。

对应源代码：[/core/*](https://github.com/godotengine/godot/tree/master/core)

## 主入口 (Main)

主组件负责初始化和管理引擎生命周期，包括启动、关闭和主循环。有关更多详细信息，请参阅 [MainLoop](https://docs.godotengine.org/en/stable/classes/class_mainloop.html)。

对应源代码：[/main/*](https://github.com/godotengine/godot/tree/master/main)
