架构概览

以下图表描述了 Godot 架构中最重要的方面。它并非旨在穷举所有细节，而是为了对主要组件及其相互关系提供一个高层概览。

致谢：Hendrik Brucker

场景层 (Scene Layer)​

场景层是 Godot 架构的最高层，提供了场景系统，这是构建和组织应用程序或游戏的主要方式。有关更多信息，请参阅 SceneTree / 使用 SceneTree 和 Node。

对应源代码：/scene/*

服务器层 (Server Layer)​

服务器组件实现了 Godot 的大部分子系统（渲染、音频、物理等）。它们是在引擎启动时初始化的单例对象。

为什么 Godot 使用服务器和 RID？

对应源代码：/servers/*

驱动 / 平台接口 (Drivers / Platform Interface)​

该层抽象了底层平台特定的细节，包含图形 API 驱动程序、音频后端和操作系统接口（所有平台特定的 OS 和 DisplayServer 实现）。

对应源代码：/drivers/* 和 /platform/*

核心 (Core)​

引擎的核心包含了在整个引擎中使用的基本功能和数据结构，如 Object 和 ClassDB、内存管理、容器、文件 I/O、Variant 以及 其他工具。

对应源代码：/core/*

主入口 (Main)​

主组件负责初始化和管理引擎生命周期，包括启动、关闭和主循环。有关更多详细信息，请参阅 MainLoop。

对应源代码：/main/*
