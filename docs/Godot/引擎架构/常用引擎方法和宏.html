常用引擎方法和宏

Godot 的 C++ 代码库使用了数十种自定义的方法和宏，它们几乎出现在每一个文件中。本页面面向初学者贡献者，但对于编写自定义 C++ 模块的人来说也非常有用。

打印文本 (Print text)​

// 向标准输出打印一条消息。
print_line("Message");

// 非字符串参数会自动转换为字符串进行打印。
// 如果传递多个参数，它们将被连接在一起，每个参数之间有一个空格。
print_line("There are", 123, "nodes");

// 向标准输出打印一条消息，但仅当引擎使用 `--verbose` 命令行参数启动时才会打印。
print_verbose("Message");

// 使用 BBCode 向标准输出打印富文本消息。
// 这支持 RichTextLabel 支持的 BBCode 标签子集，并且也会在编辑器的“输出”面板中显示格式。
// 在 Windows 上，这需要 Windows 10 或更高版本才能在终端中正常工作。
print_line_rich("[b]Bold[/b], [color=red]Red text[/color]")

// 打印带有追踪信息的格式化错误或警告消息。
ERR_PRINT("Message");
WARN_PRINT("Message");

// 每个会话仅打印一次错误或警告消息。
// 这可以用来避免控制台输出被刷屏。
ERR_PRINT_ONCE("Message");
WARN_PRINT_ONCE("Message");

如果你需要在消息中添加占位符，请使用如下所述的格式化字符串。

格式化字符串 (Format a string)​

vformat() 函数返回一个格式化后的 String。它的行为类似于 C 语言的 sprintf()：

vformat("My name is %s.", "Godette");
vformat("%d bugs on the wall!", 1234);
vformat("Pi is approximately %f.", 3.1416);

// 将生成的 String 转换为 `const char *`。
// 如果将结果作为参数传递给期望 `const char *` 而非 String 的方法时，你可能需要这样做。
vformat("My name is %s.", "Godette").utf8().get_data();

在大多数情况下，尽量使用 vformat() 而不是字符串连接，因为这样代码可读性更高。

将整数或浮点数转换为字符串 (Convert an integer or float to a string)​

使用 print_line() 打印数字时不需要这样做，但在某些其他用例中，你可能仍需要执行手动转换。

// 使用整数转字符串转换存储字符串 "42"。
String int_to_string = itos(42);

// 使用实数转字符串转换存储字符串 "123.45"。
String real_to_string = rtos(123.45);

国际化字符串 (Internationalize a string)​

Godot 的代码库中有两种类型的国际化：

  • TTR(): 编辑器（“工具”）翻译 将仅在编辑器中处理。如果用户在其项目中使用相同的文本，除非他们自己提供翻译，否则该文本不会被翻译。在为引擎贡献代码时，这通常是你应该用于可本地化字符串的宏。
  • RTR(): 运行时翻译 如果项目为给定字符串提供了翻译，则会在项目中自动进行本地化。这种翻译不应用于仅限编辑器的代码。
// 返回与用户语言区域设置匹配的翻译字符串。
// 翻译位于 `editor/translations`。
// 本地化模板是自动生成的；请勿修改它。
TTR("Exit the editor?");

要在可本地化字符串中插入占位符，请将本地化宏包装在 vformat() 调用中，如下所示：

String file_path = "example.txt";
vformat(TTR("Couldn't open \"%s\" for reading."), file_path);

注意

同时使用 vformat() 和翻译宏时，务必将翻译宏包装在 vformat() 内部，而不是相反。否则，字符串将永远无法匹配翻译，因为在传递给 TranslationServer 时，占位符已经被替换了。

限制值 (Clamp a value)​

Godot 提供了用于限制值的宏，包括下限 (MAX)、上限 (MIN) 或两者 (CLAMP)：

int a = 3;
int b = 5;

MAX(b, 6); // 6
MIN(2, a); // 2
CLAMP(a, 10, 30); // 10

这适用于任何可以与其他值进行比较的类型（如 int 和 float）。

微基准测试 (Microbenchmarking)​

如果你想对一段代码进行基准测试，但不知道如何使用性能分析器 (profiler)，请使用此代码片段：

uint64_t begin = Time::get_singleton()->get_ticks_usec();

// 你的代码放在这里...

uint64_t end = Time::get_singleton()->get_ticks_usec();
print_line(vformat("Snippet took %d microseconds", end - begin));

这将打印 begin 声明和 end 声明之间花费的时间。

注意

如果尚未包含，你可能需要 #include "core/os/time.h"。

在提交拉取请求 (PR) 时，请务必移除此代码片段以及之前添加的 include。

获取项目/编辑器设置 (Get project/editor settings)​

有四个宏可用于此目的：

// 返回指定的项目设置值，
// 如果不存在则默认为 `false`。
GLOBAL_DEF("section/subsection/value", false);

// 返回指定的编辑器设置值，
// 如果不存在则默认为 "Untitled"。
EDITOR_DEF("section/subsection/value", "Untitled");

如果已在其他地方指定了默认值，请不要再次指定以避免重复：

// 返回项目设置的值。
GLOBAL_GET("section/subsection/value");
// 返回编辑器设置的值。
EDITOR_GET("section/subsection/value");

建议对每个设置仅使用一次 GLOBAL_DEF/EDITOR_DEF，并在所有其他引用的地方使用 GLOBAL_GET/EDITOR_GET。

错误宏 (Error macros)​

Godot 拥有许多错误宏，使错误报告更加方便。

警告

错误宏中的条件工作方式与 GDScript 内置的 assert() 函数 相反。如果内部条件评估为 true 而非 false，则会触发错误。

注意

这里仅记录了带有自定义消息的变体，因为在新的贡献中应始终使用这些变体。请确保提供的自定义消息包含足够的信息供人们诊断问题，即使他们不懂 C++。如果向方法传递了无效参数，你可以打印相关的无效值以方便调试。

对于不需要显示人类可读消息的内部错误检查，请移除宏名称末尾的 _MSG 并且不要提供消息参数。

此外，始终尝试返回可处理的数据，以便引擎可以继续良好运行。

// 有条件地打印错误消息并从函数返回。
// 在不返回值的（void）方法中使用。
ERR_FAIL_COND_MSG(!mesh.is_valid(), vformat("Couldn't load mesh at: %s", path));

// 有条件地打印错误消息并从函数返回 `0`。
// 在必须返回值的方法中使用。
ERR_FAIL_COND_V_MSG(rect.x < 0 || rect.y < 0, 0,
		"Couldn't calculate the rectangle's area.");

// 如果 `index` < 0 或 >= `SomeEnum::QUALITY_MAX`，则打印错误消息，
// 然后从函数返回。
ERR_FAIL_INDEX_MSG(index, SomeEnum::QUALITY_MAX,
		vformat("Invalid quality: %d. See SomeEnum for allowed values.", index));

// 如果 `index` < 0 或 >= `some_array.size()`，则打印错误消息，
// 然后从函数返回 `-1`。
ERR_FAIL_INDEX_V_MSG(index, some_array.size(), -1,
		vformat("Item %d is out of bounds.", index));

// 无条件地打印错误消息并从函数返回。
// 仅当你需要执行复杂的错误检查时才使用。
if (!complex_error_checking_routine()) {
	ERR_FAIL_MSG("Couldn't reload the filesystem cache.");
}

// 无条件地打印错误消息并从函数返回 `false`。
// 仅当你需要执行复杂的错误检查时才使用。
if (!complex_error_checking_routine()) {
	ERR_FAIL_V_MSG(false, "Couldn't parse the input arguments.");
}

// 使引擎崩溃。除了测试崩溃处理代码外，通常绝不应使用。
// Godot 的理念是永不崩溃，无论是在编辑器中还是在导出的项目中。
CRASH_NOW_MSG("Can't predict the future! Aborting.");

另请参阅​

有关每个错误宏的更多信息，请参阅 Godot 代码库中的 core/error/error_macros.h。

某些函数返回错误代码（通过 Error 返回类型体现）。该值可以直接从错误宏返回。请参阅 core/error/error_list.h 中可用的错误代码列表。
