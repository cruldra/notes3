在本节课中，我们将添加玩家的移动、动画，并将其设置为检测碰撞。

为此，我们需要添加一些内置节点无法实现的功能，因此我们要添加一个脚本。点击 `Player` 节点并点击 “附加脚本” 按钮：

![Add Script Button](https://docs.godotengine.org/en/stable/_images/add_script_button.webp)

在脚本设置窗口中，你可以保持默认设置。只需点击 “创建”：

> **注意**
>
> 如果你创建的是 C# 脚本或其他语言，请在点击创建之前从语言下拉菜单中选择对应语言。

![Attach Node Window](https://docs.godotengine.org/en/stable/_images/attach_node_window.webp)

> **注意**
>
> 如果这是你第一次接触 GDScript，请在继续之前阅读 [脚本语言](../../step_by_step/脚本语言.md)。

首先声明该对象需要的成员变量：

```gdscript
extends Area2D

@export var speed = 400 # 玩家移动速度（像素/秒）。
var screen_size # 游戏窗口的大小。
```

```csharp
using Godot;

public partial class Player : Area2D
{
    [Export]
    public int Speed { get; set; } = 400; // 玩家移动速度（像素/秒）。

    public Vector2 ScreenSize; // 游戏窗口的大小。
}
```

在第一个变量 `speed` 上使用 `export` 关键字允许我们在检查器中设置它的值。对于那些你希望能够像调整节点内置属性一样进行调整的值，这非常方便。点击 `Player` 节点，你会看到该属性现在出现在检查器中一个以脚本名称命名的全新部分中。记住，如果你在这里更改了值，它将覆盖脚本中编写的值。

> **警告**
>
> 如果你使用 C#，每当你想要看到新的导出变量或信号时，都需要（重新）构建项目程序集。可以通过点击编辑器右上角的 **构建 (Build)** 按钮手动触发此构建。

![Build Dotnet](https://docs.godotengine.org/en/stable/_images/build_dotnet.webp)
![Export Variable](https://docs.godotengine.org/en/stable/_images/export_variable.webp)

你的 `player.gd` 脚本应该已经包含一个 `_ready()` 和一个 `_process()` 函数。如果你没有选择上面显示的默认模板，请在学习课程时创建这些函数。

当一个节点进入场景树时会调用 `_ready()` 函数，这是获取游戏窗口大小的好时机：

```gdscript
func _ready():
	screen_size = get_viewport_rect().size
```

```csharp
public override void _Ready()
{
    ScreenSize = GetViewportRect().Size;
}
```

现在我们可以使用 `_process()` 函数来定义玩家将要做什么。`_process()` 每帧都会被调用，因此我们将使用它来更新游戏中我们预期会经常变化的元素。对于玩家，我们需要执行以下操作：

*   检查输入。
*   沿给定方向移动。
*   播放合适的动画。

首先，我们需要检查输入——玩家是否按下了某个键？对于这款游戏，我们需要检查 4 个方向的输入。输入动作在项目设置的 “输入映射 (Input Map)” 下定义。在这里，你可以定义自定义事件并为其分配不同的按键、鼠标事件或其他输入。对于这款游戏，我们将把方向键映射到四个方向。

点击 **项目 -> 项目设置 (Project -> Project Settings)** 打开项目设置窗口，并点击顶部的 **输入映射 (Input Map)** 选项卡。在顶部栏输入 “move_right” 并点击 “添加” 按钮来添加 `move_right` 动作。

![Input Mapping Add Action](https://docs.godotengine.org/en/stable/_images/input-mapping-add-action.webp)

我们需要为这个动作分配一个按键。点击右侧的 “+” 图标，打开事件管理器窗口。

![Input Mapping Add Key](https://docs.godotengine.org/en/stable/_images/input-mapping-add-key.webp)

“正在监听输入...” 字段应该会自动被选中。按下键盘上的 “右” 方向键，菜单现在看起来应该是这样的。

![Input Mapping Event Configuration](https://docs.godotengine.org/en/stable/_images/input-mapping-event-configuration.webp)

选择 “确定” 按钮。“右” 方向键现在就与 `move_right` 动作关联起来了。

重复这些步骤以添加另外三个映射：

1.  `move_left` 映射到左方向键。
2.  `move_up` 映射到上方向键。
3.  `move_down` 映射到下方向键。

你的输入映射选项卡看起来应该是这样的：

![Input Mapping Completed](https://docs.godotengine.org/en/stable/_images/input-mapping-completed.webp)

点击 “关闭” 按钮关闭项目设置。

> **注意**
>
> 我们只为每个输入动作映射了一个按键，但你可以为同一个输入动作映射多个按键、手柄按钮或鼠标按钮。

你可以使用 `Input.is_action_pressed()` 来检测按键是否被按下，如果按下则返回 `true`，否则返回 `false`。

```gdscript
func _process(delta):
	var velocity = Vector2.ZERO # 玩家的移动向量。
	if Input.is_action_pressed("move_right"):
		velocity.x += 1
	if Input.is_action_pressed("move_left"):
		velocity.x -= 1
	if Input.is_action_pressed("move_down"):
		velocity.y += 1
	if Input.is_action_pressed("move_up"):
		velocity.y -= 1

	if velocity.length() > 0:
		velocity = velocity.normalized() * speed
		$AnimatedSprite2D.play()
	else:
		$AnimatedSprite2D.stop()
```

```csharp
public override void _Process(double delta)
{
    var velocity = Vector2.Zero; // 玩家的移动向量。

    if (Input.IsActionPressed("move_right"))
    {
        velocity.X += 1;
    }

    if (Input.IsActionPressed("move_left"))
    {
        velocity.X -= 1;
    }

    if (Input.IsActionPressed("move_down"))
    {
        velocity.Y += 1;
    }

    if (Input.IsActionPressed("move_up"))
    {
        velocity.Y -= 1;
    }

    var animatedSprite2D = GetNode<AnimatedSprite2D>("AnimatedSprite2D");

    if (velocity.Length() > 0)
    {
        velocity = velocity.Normalized() * Speed;
        animatedSprite2D.Play();
    }
    else
    {
        animatedSprite2D.Stop();
    }
}
```

我们首先将 `velocity` 设置为 `(0, 0)`——默认情况下，玩家不应该移动。然后我们检查每个输入并从 `velocity` 中增加/减去值以获得总方向。例如，如果你同时按住 `右` 和 `下`，生成的 `velocity` 向量将是 `(1, 1)`。在这种情况下，由于我们增加了水平和垂直移动，玩家斜向移动的速度会比单纯水平移动时 **更快**。

如果我们 **归一化 (normalize)** 速度，就可以防止这种情况，这意味着我们将向量的 **长度** 设置为 `1`，然后乘以预期的速度。这意味着不再有快速的斜向移动。

> **提示**
>
> 如果你以前从未接触过向量数学，或者需要复习一下，可以在 Godot 文档的 [向量数学](https://docs.godotengine.org/en/stable/tutorials/math/vector_math.html) 中看到关于向量用法的解释。了解一下是有好处的，但对于本教程的其余部分并不是必需的。

我们还会检查玩家是否正在移动，以便我们可以调用 `AnimatedSprite2D` 的 `play()` 或 `stop()`。

> **提示**
>
> `$` 是 `get_node()` 的简写。所以在上面的代码中，`$AnimatedSprite2D.play()` 等同于 `get_node("AnimatedSprite2D").play()`。
>
> 在 GDScript 中，`$` 返回相对于当前节点的路径处的节点，如果未找到该节点则返回 `null`。由于 `AnimatedSprite2D` 是当前节点的子节点，我们可以使用 `$AnimatedSprite2D`。

既然我们已经有了移动方向，我们就可以更新玩家的位置。我们还可以使用 `clamp()` 来防止它离开屏幕。**限制 (Clamping)** 一个值意味着将其约束在给定范围内。在 `_process` 函数的底部添加以下内容（确保它没有缩进在 `else` 下方）：

```gdscript
	position += velocity * delta
	position = position.clamp(Vector2.ZERO, screen_size)
```

```csharp
    Position += velocity * (float)delta;
    Position = new Vector2(
        x: Mathf.Clamp(Position.X, 0, ScreenSize.X),
        y: Mathf.Clamp(Position.Y, 0, ScreenSize.Y)
    );
```

> **提示**
>
> `_process()` 函数中的 `delta` 参数指的是 **帧长度 (frame length)**——即上一帧完成所花费的时间。使用此值可确保即使帧率发生变化，你的移动也能保持一致。

点击 “运行当前场景” (F6，macOS 为 Cmd + R) 并确认你可以向所有方向在屏幕上移动玩家。

> **警告**
>
> 如果你在 “调试器” 面板中得到一个错误，提示：
>
> `Attempt to call function 'play' in base 'null instance' on a null instance`
>
> 这很可能意味着你把 `AnimatedSprite2D` 节点的名字拼错了。节点名称是区分大小写的，`$NodeName` 必须与你在场景树中看到的名称完全匹配。

## 选择动画 (Choosing animations)

既然玩家可以移动了，我们需要根据玩家的方向更改 `AnimatedSprite2D` 正在播放的动画。我们有 “walk” 动画，它显示玩家向右走。对于向左移动，此动画应使用 `flip_h` 属性进行水平翻转。我们还有 “up” 动画，对于向下移动，它应使用 `flip_v` 进行垂直翻转。让我们将此代码放在 `_process()` 函数的末尾：

```gdscript
	if velocity.x != 0:
		$AnimatedSprite2D.animation = "walk"
		$AnimatedSprite2D.flip_v = false
		# 见下方关于以下布尔赋值的说明。
		$AnimatedSprite2D.flip_h = velocity.x < 0
	elif velocity.y != 0:
		$AnimatedSprite2D.animation = "up"
		$AnimatedSprite2D.flip_v = velocity.y > 0
```

```csharp
    if (velocity.X != 0)
    {
        animatedSprite2D.Animation = "walk";
        animatedSprite2D.FlipV = false;
        // 见下方关于以下布尔赋值的说明。
        animatedSprite2D.FlipH = velocity.X < 0;
    }
    else if (velocity.Y != 0)
    {
        animatedSprite2D.Animation = "up";
        animatedSprite2D.FlipV = velocity.Y > 0;
    }
```

> **注意**
>
> 上面代码中的布尔赋值是程序员常用的简写。既然我们正在进行比较测试（布尔值）并且还在 **分配** 一个布尔值，我们可以同时完成这两项工作。对比一下这段代码与上面的一行布尔赋值：
>
> ```gdscript
> if velocity.x < 0:
> 	$AnimatedSprite2D.flip_h = true
> else:
> 	$AnimatedSprite2D.flip_h = false
> ```

再次运行场景并检查每个方向的动画是否正确。

> **提示**
>
> 这里一个常见的错误是把动画的名字拼错了。`SpriteFrames` 面板中的动画名称必须与你在代码中输入的完全一致。如果你把动画命名为 `"Walk"`，那么在代码中你也必须使用大写的 “W”。

当你确定移动正常工作后，将这一行添加到 `_ready()` 中，这样玩家在游戏开始时就会被隐藏：

```gdscript
hide()
```

```csharp
Hide();
```

## 为碰撞做准备 (Preparing for collisions)

我们希望 `Player` 能够检测到何时被敌人击中，但我们还没有制作任何敌人！没关系，因为我们将使用 Godot 的 **信号 (signal)** 功能来实现它。

在脚本顶部添加以下内容。如果你使用 GDScript，将其添加在 `extends Area2D` 之后。如果你使用 C#，将其添加在 `public partial class Player : Area2D` 之后：

```gdscript
signal hit
```

```csharp
// 别忘了重新构建项目，以便编辑器知道新信号。

[Signal]
public delegate void HitEventHandler();
```

这定义了一个名为 “hit” 的自定义信号，我们将让玩家在与敌人碰撞时发出（发送出）该信号。我们将使用 `Area2D` 来检测碰撞。选中 `Player` 节点并点击检查器选项卡旁边的 **节点 (Node)** 选项卡，查看玩家可以发出的信号列表：

![Player Signals](https://docs.godotengine.org/en/stable/_images/player_signals.webp)

注意我们的自定义 “hit” 信号也在那里！由于我们的敌人将是 `RigidBody2D` 节点，我们需要 `body_entered(body: Node2D)` 信号。当一个物体接触到玩家时，将发出此信号。点击 “连接...”，出现 “连接信号” 窗口。

Godot 会直接在脚本中为你创建一个具有该确切名称的函数。你现在不需要更改默认设置。

> **警告**
>
> 如果你使用的是外部文本编辑器（例如 Visual Studio Code），目前的一个错误会阻止 Godot 自动创建函数。你会被引导到外部编辑器，但新函数不会出现在那里。
>
> 在这种情况下，你需要自己在 Player 的脚本文件中编写该函数。

![Player Signal Connection](https://docs.godotengine.org/en/stable/_images/player_signal_connection.webp)

注意绿色图标表示信号已连接到此函数；这并不意味着该函数已存在，只意味着信号将尝试连接到具有该名称的函数，所以请仔细检查函数的拼写是否完全匹配！

接着，向函数中添加这段代码：

```gdscript
func _on_body_entered(_body):
	hide() # 玩家被击中后消失。
	hit.emit()
	# 必须延迟执行，因为我们不能在物理回调中更改物理属性。
	$CollisionShape2D.set_deferred("disabled", true)
```

```csharp
// 我们还在编辑器的连接窗口中以 PascalCase 指定了此函数名称。
private void OnBodyEntered(Node2D body)
{
    Hide(); // 玩家被击中后消失。
    EmitSignal(SignalName.Hit);
    // 必须延迟执行，因为我们不能在物理回调中更改物理属性。
    GetNode<CollisionShape2D>("CollisionShape2D").SetDeferred(CollisionShape2D.PropertyName.Disabled, true);
}
```

每当敌人击中玩家时，都会发出该信号。我们需要禁用玩家的碰撞，以免多次触发 `hit` 信号。

> **注意**
>
> 如果禁用区域的碰撞形状发生在引擎碰撞处理的中间，可能会导致错误。使用 `set_deferred()` 告诉 Godot 等到安全时再禁用形状。

最后一部分是添加一个函数，我们可以在开始新游戏时调用该函数来重置玩家。

```gdscript
func start(pos):
	position = pos
	show()
	$CollisionShape2D.disabled = false
```

```csharp
public void Start(Vector2 position)
{
    Position = position;
    Show();
    GetNode<CollisionShape2D>("CollisionShape2D").Disabled = false;
}
```
