[AnimationPlayer](https://docs.godotengine.org/en/stable/classes/class_animationplayer.html#class-animationplayer) 节点允许你创建从简单到复杂的各种动画。

通过本指南，你将学习：

*   使用动画面板（Animation Panel）
*   为任何节点的任何属性制作动画
*   创建一个简单的动画

在 Godot 中，你可以为检查器（Inspector）中可用的任何内容制作动画，例如节点变换（Transform）、精灵（Sprite）、UI 元素、粒子、材质的可见性和颜色等等。你还可以修改脚本变量的值，甚至调用函数。

## 创建 AnimationPlayer 节点

要使用动画工具，我们首先必须创建一个 [AnimationPlayer](https://docs.godotengine.org/en/stable/classes/class_animationplayer.html#class-animationplayer) 节点。

AnimationPlayer 节点类型是动画的数据容器。一个 AnimationPlayer 节点可以容纳多个动画，这些动画可以自动相互切换。

![AnimationPlayer 节点](https://docs.godotengine.org/en/stable/_images/animation_create_animationplayer.webp)

创建 AnimationPlayer 节点后，点击它以打开位于视口底部的动画面板。

![动画面板位置](https://docs.godotengine.org/en/stable/_images/animation_animation_panel.webp)

动画面板由四个部分组成：

![动画面板概览](https://docs.godotengine.org/en/stable/_images/animation_animation_panel_overview.webp)

*   **动画控制**（例如添加、加载、保存和删除动画）
*   **轨道列表**（Tracks listing）
*   带有**关键帧**的**时间轴**
*   **时间轴和轨道控制**，例如你可以在此处缩放时间轴和编辑轨道。

## 计算机动画依赖于关键帧

关键帧（Keyframe）定义了属性在某个时间点的值。

时间轴中的菱形形状代表关键帧。两个关键帧之间的线条表示它们之间的值没有变化。

![Godot 中的关键帧](https://docs.godotengine.org/en/stable/_images/animation_keyframes.webp)

你设置节点属性的值并为它们创建动画关键帧。当动画运行时，引擎将插值（interpolate）关键帧之间的值，从而使它们随时间逐渐变化。

![两个关键帧即可获得平滑运动](https://docs.godotengine.org/en/stable/_images/animation_illustration.webp)

时间轴定义了动画所需的时间。你可以在各个点插入关键帧，并更改它们的时机。

![动画面板中的时间轴](https://docs.godotengine.org/en/stable/_images/animation_timeline.webp)

动画面板中的每一行都是一个动画轨道，引用节点的普通或变换属性。每个轨道存储一个指向节点的路径及其受影响的属性。例如，插图中的位置轨道指的是 Sprite2D 节点的 `position` 属性。

![普通动画轨道示例](https://docs.godotengine.org/en/stable/_images/animation_normal_track.webp)

:::tip 提示
如果你为错误的属性制作了动画，你可以随时通过双击轨道并输入新路径来编辑轨道的路径。使用“从头播放”按钮（![从头播放](https://docs.godotengine.org/en/stable/_images/animation_play_from_beginning.png) 或按键盘上的 Shift + D）播放动画以立即看到更改。
:::

## 教程：创建一个简单的动画

### 场景设置

在本教程中，我们将创建一个 Sprite 节点，并将其作为 AnimationPlayer 的父节点（或者将 AnimationPlayer 作为其子节点）。我们将制作精灵在屏幕上两个点之间移动的动画。

![我们的场景设置](https://docs.godotengine.org/en/stable/_images/animation_animation_player_tree.webp)

:::warning 警告
AnimationPlayer 继承自 Node 而不是 Node2D 或 Node3D，这意味着由于层级结构中存在一个基础 Node，子节点将不会继承父节点的变换（Transform）。

因此，不建议将具有 2D/3D 变换的节点添加为 AnimationPlayer 节点的子节点。
:::

精灵持有一个图像纹理。对于本教程，选择 Sprite2D 节点，在检查器中点击 Texture，然后点击加载（Load）。选择默认的 Godot 图标作为精灵的纹理。

### 添加动画

选择 AnimationPlayer 节点并点击动画编辑器中的“动画（Animation）”按钮。从列表中选择“新建（New）”（![添加动画](https://docs.godotengine.org/en/stable/_images/animation_add.png)）以添加新动画。在对话框中输入动画名称。

![添加新动画](https://docs.godotengine.org/en/stable/_images/animation_create_new_animation.webp)

### 管理动画库

为了重用性，动画被注册在动画库（Animation Library）资源的列表中。如果你向 AnimationPlayer 添加动画而不指定任何特定设置，动画将被注册在 AnimationPlayer 默认拥有的 [Global] 动画库中。

![管理动画](https://docs.godotengine.org/en/stable/_images/animation_library.webp)

如果存在多个动画库并且你尝试添加动画，则会出现一个带有选项的对话框。

![带库选项添加新动画](https://docs.godotengine.org/en/stable/_images/animation_library_dialog.webp)

### 添加轨道

要为我们的精灵添加新轨道，请选中它并查看工具栏：

![便捷按钮](https://docs.godotengine.org/en/stable/_images/animation_convenience_buttons.webp)

这些开关和按钮允许你为所选节点的位置（Location）、旋转（Rotation）和缩放（Scale）添加关键帧。由于我们只制作精灵位置的动画，请确保仅选中位置开关。选中的开关为蓝色。

点击钥匙按钮以创建第一个关键帧。由于我们还没有为 Position 属性设置轨道，Godot 会提议为我们创建它。点击 **创建（Create）**。

Godot 将创建一个新轨道并在时间轴的开头插入我们的第一个关键帧：

![精灵轨道](https://docs.godotengine.org/en/stable/_images/animation_track.webp)

### 第二个关键帧

我们需要设置精灵的结束位置以及到达那里所需的时间。

假设我们希望它在两点之间移动需要两秒钟。默认情况下，动画设置为仅持续一秒钟，因此在动画面板时间轴标题右侧的控件中将动画长度更改为 2。

![动画长度](https://docs.godotengine.org/en/stable/_images/animation_set_length.webp)

现在，将精灵向右移动到其最终位置。你可以使用工具栏中的“移动工具”或在“检查器”中设置“位置（Position）”的 X 值。

在动画面板中点击两秒标记附近的时间轴标题，然后点击工具栏中的钥匙按钮以创建第二个关键帧。

### 运行动画

点击“从头播放”（![从头播放](https://docs.godotengine.org/en/stable/_images/animation_play_from_beginning.png)）按钮。

太棒了！我们的动画运行了：

![动画演示](https://docs.godotengine.org/en/stable/_images/animation_simple.gif)

### 加载时自动播放

你可以设置动画在 AnimationPlayer 节点场景开始或加入另一个场景时自动播放。为此，请点击动画编辑器中的“加载时自动播放（Autoplay on load）”按钮，它就在编辑按钮旁边。

![自动播放按钮](https://docs.godotengine.org/en/stable/_images/autoplay_on_load.webp)

它的图标也会出现在动画名称的前面，这样你就可以轻松识别哪个是自动播放动画。

### 来回移动

Godot 有一个我们可以在动画中使用的有趣功能。当设置了“动画循环（Animation Looping）”但动画末尾没有指定关键帧时，第一个关键帧也是最后一个关键帧。

这意味着我们现在可以将动画长度延长到四秒，Godot 也会计算从最后一个关键帧到第一个关键帧的帧，使我们的精灵来回移动。

![动画循环](https://docs.godotengine.org/en/stable/_images/animation_loop.webp)

你可以通过更改轨道的循环模式来更改此行为。这将在下一章中介绍。

### 轨道设置

每个属性轨道末尾都有一个设置面板，你可以在其中设置其更新模式、轨道插值和循环模式。

![轨道设置](https://docs.godotengine.org/en/stable/_images/animation_track_settings.webp)

轨道的**更新模式**告诉 Godot 何时更新属性值。可以是：

*   **Continuous（连续）：** 每一帧更新属性
*   **Discrete（离散）：** 仅在关键帧上更新属性
*   **Capture（捕捉）：** 如果第一个关键帧的时间大于 `0.0`，则将记住属性的当前值并将其与第一个动画关键帧混合。例如，你可以使用捕捉模式将位于任何位置的节点移动到特定位置。

![轨道模式](https://docs.godotengine.org/en/stable/_images/animation_track_rate.webp)

你通常会使用“Continuous”模式。其他类型用于编写复杂的动画脚本。

**轨道插值**告诉 Godot 如何计算关键帧之间的帧值。支持以下插值模式：

*   Nearest（最近）：设置最近的关键帧值
*   Linear（线性）：根据两个关键帧之间的线性函数计算设置值
*   Cubic（立方）：根据两个关键帧之间的立方函数计算设置值
*   Linear Angle（线性角度，仅出现在旋转属性中）：具有最短路径旋转的线性模式
*   Cubic Angle（立方角度，仅出现在旋转属性中）：具有最短路径旋转的立方模式

![轨道插值](https://docs.godotengine.org/en/stable/_images/animation_track_interpolation.webp)

使用立方插值（Cubic），动画在关键帧处较慢，在它们之间较快，从而导致更自然的运动。立方插值通常用于角色动画。线性插值（Linear）以固定的步调制作动画，导致更机械化的效果。

Godot 支持两种循环模式，它们在动画设置为循环时影响动画：

![循环模式](https://docs.godotengine.org/en/stable/_images/animation_track_loop_modes.webp)

*   Clamp loop interpolation（夹紧循环插值）：选中此项时，动画在轨道的最后一个关键帧后停止。再次到达第一个关键帧时，动画将重置为其值。
*   Wrap loop interpolation（包裹循环插值）：选中此项时，Godot 会计算最后一个关键帧之后的动画，以再次达到第一个关键帧的值。

## 其他属性的关键帧

Godot 的动画系统不仅限于位置、旋转和缩放。你可以为任何属性制作动画。

如果在显示动画面板时选中精灵，Godot 将在“检查器”中为精灵的每个属性显示一个小的关键帧按钮。点击其中一个按钮可为当前动画添加轨道和关键帧。

![其他属性的关键帧](https://docs.godotengine.org/en/stable/_images/animation_properties_keyframe.webp)

## 编辑关键帧

你可以在动画时间轴中点击关键帧，以在“检查器”中显示和编辑其值。

![编辑关键帧](https://docs.godotengine.org/en/stable/_images/animation_keyframe_editor_key.webp)

你还可以在此处通过点击并拖动关键帧的缓动曲线（Easing curve）来编辑其缓动值。这告诉 Godot 在到达此关键帧时如何插值动画属性。

你可以通过这种方式微调你的动画，直到运动“看起来正确”。

## 使用 RESET 轨道

你可以设置一个特殊的 *RESET* 动画来包含“默认姿势”。这用于确保在保存场景并在编辑器中再次打开它时恢复默认姿势。

对于现有轨道，你可以添加名为“RESET”（区分大小写）的动画，然后为你想要重置的每个属性添加轨道。唯一的关键帧应该在时间 0，并为其提供每个轨道所需的默认值。

如果 AnimationPlayer 的 **Reset On Save** 属性设置为 `true`，则保存场景时将应用重置动画的效果（就像已寻道到时间 `0.0` 一样）。这仅影响保存的文件——编辑器中的属性轨道保持在原位。

如果你想重置编辑器中的轨道，选择 AnimationPlayer 节点，打开 **动画** 底部面板，然后在动画编辑器的 **编辑（Edit）** 下拉菜单中选择 **应用重置（Apply Reset）**。

在检查器中使用属性旁边的关键帧图标时，编辑器会询问你是否自动创建 RESET 轨道。

:::note 注意
RESET 轨道也用作混合的参考值。另请参阅[为了更好的混合](https://docs.godotengine.org/en/stable/tutorials/animation/animation_tree.html#for-better-blending)。
:::

## 洋葱皮 (Onion Skinning)

Godot 的动画编辑器允许你在创建动画时使用洋葱皮功能。要开启此功能，请点击动画编辑器右上角的洋葱图标。现在，动画中先前位置的内容将以透明的红色副本显示。

![洋葱皮](https://docs.godotengine.org/en/stable/_images/onion_skin.webp)

洋葱皮按钮旁边的三个点按钮会打开一个下拉菜单，允许你调整其工作方式，包括为未来的帧使用洋葱皮的功能。

## 动画标记 (Animation Markers)

动画标记可用于播放动画的特定部分而不是全部。这是一个用例示例：有一个动画文件包含一个执行两个不同动作的角色，而项目需要整个动画以及这两个单独的动作。与其制作两个额外的动画，不如在时间轴上放置标记，现在这两个动作都可以单独播放。

要向动画添加标记，请右键点击时间轴上方的空间并选择 **插入标记...（Insert Marker...）**。

![动画标记点击区域](https://docs.godotengine.org/en/stable/_images/animation_marker_click_area.webp)

所有标记在动画中都需要一个唯一的名称。你还可以设置标记的颜色以改善组织。

要播放两个标记之间的动画部分，请使用 [play_section_with_markers()](https://docs.godotengine.org/en/stable/classes/class_animationplayer.html#class-animationplayer-method-play-section-with-markers) 和 [play_section_with_markers_backwards()](https://docs.godotengine.org/en/stable/classes/class_animationplayer.html#class-animationplayer-method-play-section-with-markers-backwards) 方法。如果未指定开始标记，则使用动画的开头；如果未指定结束标记，则使用动画的结尾。

如果结束标记在动画结束之后，则 `AnimationPlayer` 将夹紧（clamp）该部分的末尾，使其不会超过动画的结束。

要预览两个标记之间的动画，请使用 Shift + 点击来选择标记。当选中两个标记时，它们之间的空间应以红色突出显示。

![选中的动画标记](https://docs.godotengine.org/en/stable/_images/animation_marker_selected.webp)

现在所有播放动画按钮都将表现得好像所选区域是整个动画一样。“从头播放动画”将把第一个标记视为动画的开始，“从结尾倒序播放动画”将把第二个标记视为结尾，依此类推。
