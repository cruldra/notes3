功能列表

本页旨在列出 Godot 目前支持的所有功能。

注意

本页列出的功能由当前 Godot 稳定版本支持。其中部分功能在 3.x 版本系列中不可用。

平台​

参见

有关硬件和软件版本要求，请参阅系统要求。

注意

有关主机支持的信息，请参阅 Godot 网站。

可以运行编辑器和导出项目：

  • Windows (x86 和 ARM，64 位和 32 位)。
  • macOS (x86 和 ARM，仅限 64 位)。
  • Linux (x86 和 ARM，64 位和 32 位)。
    • 二进制文件是静态链接的，如果在足够旧的基础发行版上编译，则可以在任何发行版上运行。
    • 官方二进制文件使用 Godot Engine buildroot 编译，确保二进制文件可以在常见的 Linux 发行版上运行。
  • Android (编辑器支持目前处于实验性阶段)。
  • Web 浏览器 (实验性功能)。在 4.0 中处于实验阶段，针对 HTML5 平台时建议使用 Godot 3.x。
注意

Linux 支持 rv64 (RISC-V)、ppc64 & ppc32 (PowerPC) 以及 loongarch64。但是你必须自己为该平台编译编辑器（以及导出模板），目前不提供官方下载。RISC-V 的编译指令可以在为 Linux, *BSD 编译页面找到。

运行导出的项目：

  • iOS。

Godot 旨在尽可能地实现平台无关性，并可以相对轻松地移植到新平台。

注意

使用 Godot 4 的 C# 项目目前无法导出到 Web 平台。要在该平台上使用 C#，请考虑使用 Godot 3。自 Godot 4.2 起支持 Android 和 iOS 平台，但处于实验性阶段且存在一些限制。

编辑器​

功能：

  • 场景树编辑器。
  • 内置脚本编辑器。
  • 支持外部脚本编辑器，如 Visual Studio Code 或 Vim。
  • GDScript 调试器。
    • 自 4.2 起支持在线程中进行调试。
  • 可视化分析器 (Profiler)，提供渲染流水线每一步的 CPU 和 GPU 时间指示。
  • 性能监控工具，包括自定义性能监控器。
  • 脚本热重载 (Live script reloading)。
  • 实时场景编辑 (Live scene editing)。
    • 更改将反映在编辑器中，并在关闭运行的项目后保留。
  • 远程检查器 (Remote inspector)。
    • 更改不会反映在编辑器中，也不会在关闭运行的项目后保留。
  • 实时摄像机复制。
    • 移动编辑器内的摄像机，并在运行的项目中实时查看结果。
  • 内置离线类参考文档。
  • 由社区贡献的数十种语言的编辑器界面。

插件：

  • 可以从资产库 (Asset Library)下载编辑器插件来扩展编辑器功能。
  • 使用 GDScript 创建自己的插件，以添加新功能或加快工作流程。
  • 在项目管理器中从资产库下载项目并直接导入。

渲染​

Godot 4 包含三个渲染器：

  • Forward+：最先进的渲染器，仅适用于桌面平台。在桌面平台上默认使用。该渲染器使用 Vulkan、Direct3D 12 或 Metal 作为渲染驱动，并使用 RenderingDevice 后端。
  • Mobile：功能较少，但渲染简单场景更快。适用于移动和桌面平台。在移动平台上默认使用。该渲染器使用 Vulkan、Direct3D 12 或 Metal 作为渲染驱动，并使用 RenderingDevice 后端。
  • Compatibility：有时称为 GL Compatibility。最基础的渲染器，适用于低端桌面和移动平台。在 Web 平台上默认使用。该渲染器使用 OpenGL 作为渲染驱动。

请参阅渲染器概述以获取渲染方法的详细比较。

2D 图形​

  • 精灵 (Sprite)、多边形和线段渲染。
    • 用于绘制线条和多边形的高级工具，如 Polygon2D 和 Line2D，支持纹理贴图。
  • AnimatedSprite2D 用于辅助创建动画精灵。
  • 视差层 (Parallax layers)。
    • 支持伪 3D，包括在编辑器中预览。
  • 支持法线贴图和高光贴图的 2D 灯光。
    • 点光源（全向/聚光）和方向 2D 灯光。
    • 硬阴影或软阴影（可按灯光单独调整）。
    • 自定义着色器可以访问基于 LightOccluder2D 节点的 2D 场景实时 SDF 表示，可用于改进 2D 照明效果，包括 2D 全局光照。
  • 使用位图、FreeType 栅格化或多通道有符号距离场 (MSDF) 的字体渲染。
    • 位图字体可以使用 BMFont 等工具导出，或从图像导入（仅限等宽字体）。
    • 动态字体支持单色字体以及彩色字体（例如表情符号）。支持的格式包括 TTF、OTF、WOFF1 和 WOFF2。
    • 动态字体支持可选的字体轮廓，宽度和颜色可调。
    • 动态字体支持可变字体和 OpenType 功能（包括连字）。
    • 当字体文件缺少粗体和斜体样式时，动态字体支持模拟粗体和斜体。
    • 动态字体支持过采样 (Oversampling)，以在高分辨率下保持字体清晰。
    • 动态字体支持子像素定位 (Subpixel positioning)，使字体在小尺寸下更清晰。
    • 动态字体支持 LCD 子像素优化，使字体在小尺寸下更加清晰。
    • 有符号距离场 (SDF) 字体可以在任何分辨率下缩放而无需重新栅格化。与单通道 SDF 字体相比，多通道使用使 SDF 字体在缩小到较小尺寸时表现更好。
  • 支持自定义粒子着色器的基于 GPU 的粒子。
  • 基于 CPU 的粒子。
  • 可选的 2D HDR 渲染，以获得更好的辉光 (Glow) 效果。

2D 工具​

  • 用于 2D 瓷砖式关卡设计的 TileMaps。
  • 内置平滑和拖动边距的 2D 摄像机。
  • Path2D 节点，用于表示 2D 空间中的路径。
    • 可以在编辑器中绘制或程序化生成。
    • PathFollow2D 节点，使节点跟随 Path2D。
  • 2D 几何辅助类 (Geometry2D)。

2D 物理​

物理主体：

  • 静态体 (Static bodies)。
  • 可动画体 (Animatable bodies)，用于仅通过脚本或动画移动的对象，如门和平台。
  • 刚体 (Rigid bodies)。
  • 角色体 (Character bodies)。
  • 关节 (Joints)。
  • 区域 (Areas)，用于检测主体进入或离开。

碰撞检测：

  • 内置形状：线、矩形、圆形、胶囊形、世界边界（无限平面）。
  • 碰撞多边形（可以在编辑器中手动绘制或从精灵生成）。

3D 图形​

  • 带 sRGB 的 HDR 渲染。
  • 透视、正交和视锥偏移摄像机。
  • 使用 Forward+ 渲染器时，会使用深度预处理 (Depth Prepass) 来通过减少过度绘制 (Overdraw) 的成本来提高复杂场景的性能。
  • 在 Forward+ 和 Mobile 中，在支持的 GPU 上支持可变速率着色 (Variable Rate Shading)。

基于物理的渲染 (内置材质功能)：

  • 遵循 Disney PBR 模型。
  • 支持 Burley、Lambert、Lambert Wrap (half-Lambert) 和 Toon 漫反射着色模式。
  • 支持 Schlick-GGX、Toon 和 Disabled 高光着色模式。
  • 使用粗糙度-金属感 (Roughness-Metallic) 工作流，支持 ORM 纹理。
  • 使用地平线高光遮蔽 (Horizon Specular Occlusion, Filament 模型) 来改进材质外观。
  • 法线贴图 (Normal mapping)。
  • 视差/浮雕贴图 (Parallax/Relief mapping)，支持基于距离的自动细节层级 (LOD)。
  • 反照率 (Albedo) 和法线贴图的细节贴图 (Detail mapping)。
  • 次表面散射 (Sub-surface scattering) 和透射 (Transmittance)。
  • 支持材质粗糙度的屏幕空间折射（产生模糊折射）。
  • 近距离淡入 (Proximity fade, 软粒子) 和距离淡入 (Distance fade)。
  • 距离淡入可以使用 Alpha 混合或抖动 (Dithering) 来避免进入透明流水线。
  • 抖动可以按像素或按对象确定。

实时照明：

  • 方向光 (Directional lights，太阳/月亮)。每个场景最多 4 个。
  • 全向光 (Omnidirectional lights)。
  • 具有可调锥角和衰减的聚光灯 (Spot lights)。
  • 高光、间接光和体积雾能量可以按灯光单独调整。
  • 可调节的灯光“尺寸”，用于伪面积光（也会使阴影更模糊）。
  • 可选的距离淡入系统，用于淡出远处的光源及其阴影，从而提高性能。
  • 使用 Forward+ 渲染器（桌面平台默认）时，灯光使用分簇前向 (Clustered Forward) 优化进行渲染，以降低其各自的成本。分簇渲染还消除了对网格上可使用灯光数量的任何限制。
  • 使用 Mobile 渲染器时，每个网格资源最多可显示 8 个全向光和 8 个聚光灯。如果需要，可以使用烘焙照明来克服此限制。

阴影贴图：

  • DirectionalLight：正交 (最快)、PSSM 2 分割和 4 分割。支持分割之间的混合。
  • OmniLight：双抛物面 (快) 或立方体贴图 (较慢但更准确)。支持全景图形式的彩色投影纹理。
  • SpotLight：单一纹理。支持彩色投影纹理。
  • 阴影法线偏移偏置 (Shadow normal offset bias) 和阴影扁平化 (Shadow pancaking)，以减少可见的阴影痤疮 (Shadow Acne) 和彼得潘现象 (Peter-panning)。
  • 类似于 PCSS 的阴影模糊，基于灯光尺寸和阴影投射表面之间的距离。
  • 可按灯光单独调整阴影模糊。

具有间接照明的全局光照：

  • 烘焙光照贴图 (Baked Lightmaps)（速度快，但无法在运行时更新）。
    • 支持仅烘焙间接光或同时烘焙直接光和间接光。烘焙模式可以按灯光单独调整，以允许混合光照烘焙设置。
    • 支持使用自动和手动放置的探针 (Probes) 为动态物体照明。
    • 可选支持基于球面谐波 (Spherical Harmonics) 的方向性照明和粗糙反射。
    • 光照贴图使用计算着色器在 GPU 上烘焙（比 CPU 烘焙快得多）。烘焙只能在编辑器中进行，不能在导出的项目中进行。
    • 支持使用 JNLM 进行基于 GPU 的去噪，或使用 OIDN 进行基于 CPU/GPU 的去噪。
  • 基于体素的 GI 探针 (Voxel-based GI probes)。支持动态灯光和动态遮挡物，同时支持反射。需要一个快速的烘焙步骤，可以在编辑器中或在运行时（包括从导出的项目）执行。
  • 专为大型开放世界设计的有符号距离场 GI (SDFGI)。支持动态灯光，但不支持动态遮挡物。支持反射。无需烘焙。
  • 半分辨率或全分辨率的屏幕空间间接照明 (SSIL)。完全实时并支持任何类型的自发光光源（包括贴花）。
  • VoxelGI 和 SDFGI 使用延迟处理 (Deferred pass) 来允许以半分辨率渲染 GI 以提高性能（同时仍具有功能性的 MSAA 支持）。

反射：

  • 基于体素的反射（使用 GI 探针时）和基于 SDF 的反射（使用有符号距离场 GI 时）。体素反射在透明表面上可见，而粗糙的基于 SDF 的反射在透明表面上可见。
  • 使用 ReflectionProbe 进行快速烘焙反射或慢速实时反射。可以选择启用视差框校正 (Parallax box correction)。
  • 支持材质粗糙度的屏幕空间反射 (SSR)。
  • 反射技术可以混合在一起以获得更高的准确性或可扩展性。
  • 使用 Forward+ 渲染器（桌面平台默认）时，反射探针使用分簇前向优化进行渲染，以降低其各自的成本。分簇渲染还消除了对网格上可使用反射探针数量的任何限制。
  • 使用 Mobile 渲染器时，每个网格资源最多可显示 8 个反射探针。使用 Compatibility 渲染器时，每个网格资源最多可显示 2 个反射探针。

贴花 (Decals)：

  • 支持反照率 (Albedo)、自发光、ORM 和法线贴图。
  • 纹理通道平滑地覆盖在底层材质之上，支持仅法线/仅 ORM 的贴花。
  • 支持法线淡出，根据入射角淡出贴花。
  • 不依赖于运行时网格生成。这意味着贴花可以用于复杂的蒙皮网格而没有性能损失，即使贴花每帧都在移动。
  • 支持最近邻、双线性、三线性或各向异性纹理过滤（全局配置）。
  • 可选的距离淡入系统，用于淡出远处的贴花，从而提高性能。
  • 使用 Forward+ 渲染器（桌面平台默认）时，贴花使用分簇前向优化进行渲染，以降低其各自的成本。分簇渲染还消除了对网格上可使用贴花数量的任何限制。
  • 使用 Mobile 渲染器时，每个网格资源最多可显示 8 个贴花。

天空：

  • 全景天空 (使用 HDRI)。
  • 响应场景中 DirectionalLights 的程序化天空和基于物理的天空。
  • 支持自定义天空着色器，可以进行动画处理。
  • 用于环境光和高光光的辐射图 (Radiance Map) 可以根据所选的质量设置实时更新。

雾 (Fog)：

  • 指数深度雾 (Exponential depth fog)。
  • 指数高度雾 (Exponential height fog)。
  • 支持根据天空颜色自动设置雾颜色（大气透视）。
  • 支持雾中的阳光散射。
  • 支持控制雾渲染对天空的影响程度，对传统雾和体积雾有单独的控制。
  • 支持使特定材质忽略雾。

体积雾 (Volumetric fog)：

  • 响应灯光和阴影的全局体积雾。
  • 使用 VoxelGI 或 SDFGI 时，体积雾可以将间接光考虑在内。
  • 可以放置雾体积节点，以便向特定区域添加雾（或从特定区域移除雾）。支持的形状包括方盒、椭圆、圆锥、圆柱和基于 3D 纹理的密度图。
  • 每个雾体积都可以拥有自己的自定义着色器。
  • 可以与传统雾一起使用。

粒子 (Particles)：

  • 基于 GPU 的粒子，支持子发射器 (2D + 3D)、拖尾 (2D + 3D)、吸引器 (仅限 3D) 和碰撞 (2D + 3D)。
    • 支持的 3D 粒子吸引器形状：方盒、球体和 3D 向量场。
    • 支持的 3D 粒子碰撞形状：方盒、球体、烘焙有符号距离场和实时高度图（适用于开放世界天气效果）。
    • 2D 粒子碰撞使用根据场景中的 LightOccluder2D 节点实时生成的有符号距离场进行处理。
    • 拖尾可以使用内置的带状拖尾和管状拖尾网格，或带有骨架的自定义网格。
    • 支持带有手动发射功能的自定义粒子着色器。
  • 基于 CPU 的粒子。

后期处理 (Post-processing)：

  • 色调映射 (Tonemapping: Linear, Reinhard, Filmic, ACES, AgX)。
  • 基于视口亮度的自动曝光调整（以及手动曝光覆盖）。
  • 具有可调散景模拟（方盒、六边形、圆形）的近距离和远距离景深。
  • 半分辨率或全分辨率的屏幕空间环境遮蔽 (SSAO)。
  • 具有可选双立方上采样的辉光/绽放 (Glow/Bloom)，提供多种混合模式：Screen, Soft Light, Add, Replace, Mix。
  • 辉光可以拥有彩色脏迹贴图 (Dirt Map) 纹理，充当镜头污渍效果。
  • 辉光可以用作屏幕空间模糊效果。
  • 使用一维渐变或 3D LUT 纹理进行颜色分级。
  • 粗糙度限制器 (Roughness limiter)，用于减少高光锯齿的影响。
  • 亮度、对比度和饱和度调整。

纹理过滤：

  • 最近邻、双线性、三线性或各向异性过滤。
  • 过滤选项是在使用时定义的，而不是在每个纹理基础上定义的。

纹理压缩：

  • Basis Universal (较慢，但生成的文件较小)。
  • BPTC 用于高质量压缩 (macOS 不支持)。
  • ETC2 (macOS 不支持)。
  • S3TC (移动/Web 平台不支持)。

抗锯齿 (Antialiasing)：

  • 时间抗锯齿 (TAA)。
  • AMD FidelityFX Super Resolution 2.2 抗锯齿 (FSR2)，可以在原生分辨率下作为一种高质量的时间抗锯齿形式使用。
  • 多重采样抗锯齿 (MSAA)，用于 2D 抗锯齿和 3D 抗锯齿。
  • 快速近似抗锯齿 (FXAA)。
  • 使用双线性 3D 缩放和 1.0 以上 3D 分辨率比例的超级采样抗锯齿 (SSAA)。
  • 按材质进行 Alpha 抗锯齿、MSAA Alpha 至覆盖 (Alpha to Coverage) 和 Alpha 哈希 (Alpha Hashing)。

分辨率缩放：

  • 支持以较低分辨率渲染 3D，同时保持 2D 渲染在原始比例。这可以用于提高低端系统的性能或提高高端系统的视觉效果。
  • 分辨率缩放使用双线性过滤、AMD FidelityFX Super Resolution 1.0 (FSR1) 或 AMD FidelityFX Super Resolution 2.2 (FSR2)。
  • 纹理 Mipmap LOD 偏置会自动调整，以提高低分辨率比例下的质量。也可以手动偏移进行修改。

上述大多数效果都可以为了更好的性能进行调整，或者进一步提高质量。这在使用 Godot 进行离线渲染时会很有帮助。

3D 工具​

  • 内置网格：立方体、圆柱体/圆锥体、(半)球体、棱柱、平面、四边形、圆环、带状、管状。
  • 用于 3D 瓷砖式关卡设计的 GridMaps。
  • 构造实体几何 (CSG)（旨在用于原型设计）。
  • 用于程序化几何生成的工具。
  • Path3D 节点，用于表示 3D 空间中的路径。
    • 可以在编辑器中绘制或程序化生成。
    • PathFollow3D 节点，使节点跟随 Path3D。
  • 3D 几何辅助类 (Geometry3D)。
  • 支持将当前场景导出为 glTF 2.0 文件，既可以在编辑器中导出，也可以在运行时从导出的项目中导出。

3D 物理​

物理主体：

  • 静态体 (Static bodies)。
  • 可动画体 (Animatable bodies)，用于仅通过脚本或动画移动的对象，如门和平台。
  • 刚体 (Rigid bodies)。
  • 角色体 (Character bodies)。
  • 载具体 (Vehicle bodies)（旨在用于娱乐物理而非真实模拟）。
  • 关节 (Joints)。
  • 软体 (Soft bodies)。
  • 布娃娃 (Ragdolls)。
  • 区域 (Areas)，用于检测主体进入或离开。

碰撞检测：

  • 内置形状：长方体、球体、胶囊形、圆柱体、世界边界（无限平面）。
  • 在编辑器中为任何网格生成三角形碰撞形状。
  • 在编辑器中为任何网格生成一个或多个凸碰撞形状。

着色器 (Shaders)​

  • 2D：自定义顶点、片段和光照着色器。
  • 3D：自定义顶点、片段、光照和天空着色器。
  • 基于文本的着色器，使用受 GLSL 启发的着色语言。
  • 可视化着色器编辑器。
    • 支持可视化着色器插件。

脚本 (Scripting)​

通用：

  • 具有扩展节点的脚本的面向对象设计模式。
  • 用于脚本间通信的信号 (Signals) 和组 (Groups)。
  • 支持跨语言脚本编程。
  • 许多 2D、3D 和 4D 线性代数数据类型，如向量和变换 (Transforms)。

GDScript:

  • 具有可选静态类型的高级解释型语言。
  • 语法深受 Python 启发。但是，GDScript 并非基于 Python。
  • GitHub 提供了语法高亮显示。
  • 使用线程来执行异步操作或利用多核处理器。

C#:

  • 打包在单独的二进制文件中，以减小文件大小和依赖项。
  • 支持 .NET 8 及更高版本。
    • 完全支持 C# 12.0 语法和功能。
  • 支持 Windows、Linux 和 macOS。自 Godot 4.2 起，还提供对 Android 和 iOS 的实验性支持。
    • 在 iOS 平台上仅支持部分架构：arm64。
    • 目前不支持 Web 平台。要在该平台上使用 C#，请考虑使用 Godot 3。
  • 建议使用外部编辑器以受益于 IDE 功能。

GDExtension (C, C++, Rust, D, ...):

  • 当需要时，链接到原生库以获得更高的性能和第三方集成。
    • 对于脚本编写游戏逻辑，如果 GDScript 或 C# 的性能合适，建议使用它们。
  • C 和 C++ 的官方 GDExtension 绑定。
    • 使用任何你希望的构建系统和语言功能。
  • 社区提供的、正在积极开发的 D、Swift 和 Rust 绑定。（其中一些绑定可能处于实验阶段，尚未达到生产级别）。

音频​

功能：

  • 单声道、立体声、5.1 和 7.1 输出。
  • 2D 和 3D 中的非定位和定位播放。
    • 2D 和 3D 中的可选多普勒效应。
  • 支持可重定向的音频总线 (Audio Buses)和效果，内置数十种效果。
  • 支持复音 (Polyphony)（从单个 AudioStreamPlayer 节点播放多个声音）。
  • 支持随机音量和音高。
  • 支持实时音高缩放。
  • 支持顺序/随机样本选择，包括在使用随机样本选择时防止重复。
  • Listener2D 和 Listener3D 节点，用于从不同于摄像机的位置进行监听。
  • 支持程序化音频生成。
  • 用于录制麦克风的音频输入。
  • MIDI 输入。
    • 尚不支持 MIDI 输出。

使用的 API：

  • Windows：WASAPI。
  • macOS：CoreAudio。
  • Linux：PulseAudio 或 ALSA。

导入​

  • 支持自定义导入插件。

格式：

  • 图像：参见导入图像。
  • 音频：
    • 具有可选 IMA-ADPCM 压缩的 WAV。
    • Ogg Vorbis。
    • MP3。
  • 3D 场景：参见导入 3D 场景。
    • glTF 2.0 （推荐）。
    • .blend (通过透明地调用 Blender 的 glTF 导出功能)。
    • FBX (通过透明地调用 FBX2glTF)。
    • Collada (.dae)。
    • Wavefront OBJ (仅限静态场景，可以直接加载为网格或导入为 3D 场景)。
  • 支持在运行时加载 glTF 2.0 场景，包括从导出的项目中加载。
  • 3D 网格在导入时使用 Mikktspace 生成切线，确保与其他 3D 应用程序（如 Blender）保持一致。

输入​

  • 使用硬编码输入事件或可重映射输入动作的输入映射系统。
    • 轴值可以映射到两个不同的动作，并具有可配置的死区 (Deadzone)。
    • 使用相同的代码支持键盘和游戏手柄。
  • 键盘输入。
    • 按键可以映射在“物理”模式下，以独立于键盘布局。
  • 鼠标输入。
    • 鼠标光标可以是可见的、隐藏的、捕获的或限制在窗口内的。
    • 捕获时，在 Windows 和 Linux 上将使用原始输入，以规避操作系统的鼠标加速设置。
  • 游戏手柄输入（最多支持 8 个控制器同时输入）。
  • 支持压力感应的笔/平板电脑输入。

导航​

  • 2D 和 3D 中的 A* 算法。
  • 2D 和 3D 中具有动态避障功能的导航网格。
  • 在编辑器中或在运行时（包括从导出的项目）生成导航网格。

网络​

  • 使用 StreamPeer 和 TCPServer 的低级 TCP 网络。
  • 使用 PacketPeer 和 UDPServer 的低级 UDP 网络。
  • 使用 HTTPClient 的低级 HTTP 请求。
  • 使用 HTTPRequest 的高级 HTTP 请求。
    • 使用捆绑的证书提供开箱即用的 HTTPS 支持。
  • 使用 UDP 和 ENet 的高级多人游戏 API。
    • 使用远程过程调用 (RPC) 的自动复制。
    • 支持不可靠、可靠和有序的传输。
  • WebSocket 客户端和服务器，可在所有平台上使用。
  • WebRTC 客户端和服务器，可在所有平台上使用。
  • 支持 UPnP，以在 NAT 后托管服务器时规避转发端口的要求。

国际化​

  • 全面支持 Unicode，包括表情符号。
  • 使用 CSV 或 gettext 存储本地化字符串。
    • 支持在编辑器中生成 gettext POT 和 PO 文件。
  • 在项目中的 GUI 元素中自动使用本地化字符串，或使用 tr() 函数。
  • 在使用 gettext 翻译时支持复数形式和翻译上下文。
  • 支持双向排版 (Bidirectional typesetting)、文本整形和 OpenType 本地化形式。
  • 针对从右到左语言环境的自动 UI 镜像。
  • 支持伪本地化 (Pseudolocalization)，以测试项目的国际化友好性。

窗口与操作系统集成​

  • 在单个进程中派生多个独立窗口。
  • 移动、调整大小、最小化和最大化由项目派生的窗口。
  • 更改窗口标题和图标。
  • 请求关注（在大多数平台上会导致标题栏闪烁）。
  • 全屏模式。
    • 在 Windows 上默认使用无边框全屏以实现快速 Alt-Tab 切换，但也可以选择使用独占全屏以减少输入延迟。
  • 无边框窗口（全屏或非全屏）。
  • 能够使窗口始终位于顶部。
  • macOS 上的全局菜单集成。
  • 以阻塞或非阻塞方式执行命令（包括运行同一项目的多个实例）。
  • 使用默认或自定义协议处理程序打开文件路径和 URL（如果在系统上已注册）。
  • 解析自定义命令行参数。
  • 通过使用 --headless 命令行参数启动，任何 Godot 二进制文件（编辑器或导出的项目）都可以用作无头服务器 (Headless Server)。这允许在没有 GPU 或显示服务器的情况下运行引擎。

移动平台​

  • Android 和 iOS 上的应用内购买。
  • 支持使用第三方模块进行广告投放。

XR 支持 (AR 和 VR)​

  • 开箱即用的 OpenXR 支持。
    • 包括对热门桌面头戴设备的驱动支持，如 Valve Index、WMR 头显以及 Quest (通过 Link)。
  • 通过插件支持使用 OpenXR 的 Android 头戴设备。
    • 包括对热门一体机头显的支持，如 Meta Quest 1/2/3 和 Pro, Pico 4, Magic Leap 2 和 Lynx R1。
  • 开箱即用的 visionOS Apple 头戴设备有限支持。
    • 目前仅支持导出在头显内平板上使用的应用程序。尚不支持沉浸式体验。
  • 通过 XR 插件结构支持其他设备。
  • 提供各种高级工具包，实现了 XR 应用程序所需的常用功能。

GUI 系统​

Godot 的 GUI 是使用与 Godot 制作游戏时相同的 Control 节点构建的。编辑器 UI 可以轻松使用加载项以多种方式进行扩展。

节点：

  • 按钮。
  • 复选框、检查按钮、单选按钮。
  • 使用 LineEdit（单行）和 TextEdit（多行）进行文本输入。TextEdit 还支持代码编辑功能，如显示行号和语法高亮。
  • 使用 PopupMenu 和 OptionButton 的下拉菜单。
  • 滚动条。
  • 标签 (Labels)。
  • RichTextLabel 用于使用 BBCode 格式化的文本，并支持动画自定义效果。
  • 树 (Trees，也可用于表示表格)。
  • 具有 RGB 和 HSV 模式的颜色选择器。
  • 控件可以进行旋转和缩放。

尺寸调整：

  • 锚点 (Anchors)，用于将 GUI 元素保持在特定的角落、边缘或中心。
  • 容器 (Containers)，用于按照某些规则自动放置 GUI 元素。
    • 堆叠布局。
    • 网格布局。
    • 流式布局（类似于文本自动换行）。
    • 边距、居中和纵横比布局。
    • 可拖动分隔栏布局。
  • 使用 canvas_items 或 viewport 拉伸模式缩放到多种分辨率。
  • 使用锚点和 expand 拉伸纵横比支持任何纵横比。

主题：

  • 内置主题编辑器。
    • 根据当前编辑器主题设置生成主题。
  • 使用 StyleBoxFlat 进行基于过程矢量的美化。
    • 支持圆角/倒角、投影、每条边宽度和抗锯齿。
  • 使用 StyleBoxTexture 进行基于纹理的美化。

Godot 极小的分发大小使其可以成为 Electron 或 Qt 等框架的合适替代方案。

动画​

  • 正向动力学 (FK) 和逆向动力学 (IK)。
  • 支持为任何属性制作动画，并具有可自定义的插值。
  • 支持在动画轨道中调用方法。
  • 支持在动画轨道中播放声音。
  • 支持动画中的贝塞尔曲线。

文件格式​

  • 场景和资源可以保存为基于文本或二进制格式。
    • 基于文本的格式是人类可读的，并且对版本控制更加友好。
    • 对于大型场景/资源，二进制格式保存/加载速度更快。
  • 使用 FileAccess 读取和写入文本或二进制文件。
    • 可以选择进行压缩或加密。
  • 读取和写入 JSON 文件。
  • 使用 ConfigFile 读取和写入 INI 风格的配置文件。
    • 可以序列化/反序列化任何 Godot 数据类型，包括 Vector2/3, Color 等。
  • 使用 XMLParser 读取 XML 文件。
  • 在导出的项目中加载和保存图像、音频/视频、字体和 ZIP 存档，而无需经过 Godot 的导入系统。
  • 将游戏数据打包到 PCK 文件（针对快速查找优化的自定义格式）、ZIP 存档中，或者直接打包到可执行文件中以进行单文件分发。
  • 导出额外的 PCK 文件，引擎可以读取这些文件以支持模组 (Mods) 和 DLC。

杂项​

  • 视频播放，内置对 Ogg Theora 的支持。
  • Movie Maker 模式，用于从运行的项目中录制视频，具有同步音频和完美的帧步进。
  • 对服务器的低级访问，允许在需要时绕过场景树的开销。
  • 用于自动化的命令行界面。
    • 使用持续集成平台导出和部署项目。
    • GitHub 上提供了针对 Bash, zsh 和 fish 的外壳补全脚本。
    • 使用 print_rich 在所有平台上向标准输出打印彩色文本。
  • 编辑器可以检测项目中使用的功能并创建编译配置文件，这可以用于创建禁用了不需要功能的较小导出模板二进制文件。
  • 支持静态链接到引擎二进制文件中的 C++ 模块。
    • 大多数内置模块都可以在编译时禁用，以减小自定义构建中的二进制文件大小。有关详细信息，请参阅优化构建大小。
  • 引擎和编辑器使用 C++17 编写。
    • 可以使用 GCC, Clang 和 MSVC 进行编译。也支持 MinGW。
    • 对打包者友好。在大多数情况下，可以使用系统库来代替 Godot 提供的库。构建系统不会下载任何内容。构建可以实现完全可重现。
  • 在宽松的 MIT 许可证下授权。
    • 开放的开发过程，欢迎贡献。
参见

Godot proposals 仓库列出了由社区请求的功能，并可能在未来的 Godot 版本中实现。
