## 简介 (Introduction)

由于 Godot 对项目结构或文件系统的使用没有限制，在学习引擎时组织文件可能会显得具有挑战性。本教程建议的工作流程应该是一个不错的起点。我们还将涵盖在 Godot 中使用版本控制的内容。

## 组织 (Organization)

Godot 本质上是基于场景的，并且直接使用文件系统，没有元数据或资产数据库。

与其他引擎不同，许多资源包含在场景内部，因此文件系统中的文件数量要少得多。

考虑到这一点，最常见的方法是 **尽可能将资产与场景分组存放**；当项目增长时，这会使其更具可维护性。

例如，通常可以将基本资产（如精灵图像、3D 模型网格、材质和音乐等）放入单个文件夹中。然后可以使用一个单独的文件夹来存储使用它们的已构建关卡。

```text
/project.godot
/docs/.gdignore  # 见下文“忽略特定文件夹”
/docs/learning.html
/models/town/house/house.dae
/models/town/house/window.png
/models/town/house/door.png
/characters/player/cubio.dae
/characters/player/cubio.png
/characters/enemies/goblin/goblin.dae
/characters/enemies/goblin/goblin.png
/characters/npcs/suzanne/suzanne.dae
/characters/npcs/suzanne/suzanne.png
/levels/riverdale/riverdale.scn
```

## 风格指南 (Style guide)

为了跨项目的一致性，我们建议遵循以下准则：

*   文件夹和文件名使用 **snake_case**（C# 脚本除外）。这可以规避在 Windows 上导出项目后可能出现的区分大小写问题。C# 脚本是此规则的一个例外，因为约定是根据类名命名，类名应使用 **PascalCase**。
*   节点名称使用 **PascalCase**，因为这与内置节点的命名一致。
*   通常情况下，将第三方资源保存在顶层的 `addons/` 文件夹中，即使它们不是编辑器插件。这使得跟踪哪些文件是第三方文件变得更容易。此规则也有一些例外；例如，如果你为某个角色使用第三方游戏资产，将它们包含在与该角色的场景和脚本相同的文件夹中会更有意义。

## 导入 (Importing)

3.0 之前的 Godot 版本是从项目外部的文件进行导入过程的。虽然这在大型项目中很有用，但对大多数开发人员来说造成了组织上的麻烦。

因此，资产现在透明地从项目文件夹内部进行导入。

### 忽略特定文件夹 (Ignoring specific folders)

为了防止 Godot 导入特定文件夹中包含的文件，在文件夹中创建一个名为 `.gdignore` 的空文件（开头的 `.` 是必需的）。这对于加快初始项目导入非常有用。

> **注意**
>
> 在 Windows 上创建以点开头的文件，请在文件名的开头和结尾都放一个点（".gdignore."）。Windows 会在确认名称后自动移除末尾的点。
>
> 或者，你可以使用 Notepad++ 等文本编辑器，或者在命令提示符中使用以下命令：`type nul > .gdignore`

一旦文件夹被忽略，该文件夹中的资源将无法再使用 `load()` 和 `preload()` 方法加载。忽略文件夹还会自动在文件系统（FileSystem）面板中隐藏它，这对于减少杂乱非常有用。

注意 `.gdignore` 文件的内容是被忽略的，这就是为什么文件应该是空的。它不支持像 `.gitignore` 文件那样的模式（patterns）。

## 区分大小写 (Case sensitivity)

Windows 和最近的 macOS 版本默认使用不区分大小写的文件系统，而 Linux 发行版默认使用区分大小写的文件系统。这可能会在导出项目后引起问题，因为 Godot 的 PCK 虚拟文件系统是区分大小写的。为了避免这种情况，建议对项目中的所有文件坚持使用 `snake_case` 命名（通常使用小写字母）。

> **注意**
>
> 当风格指南另有说明时（如 C# 风格指南），你可以打破这一规则。尽管如此，请保持一致以避免错误。

在 Windows 10 上，为了进一步避免与区分大小写相关的错误，你还可以将项目文件夹设为区分大小写。在启用“适用于 Linux 的 Windows 子系统（WSL）”功能后，在 PowerShell 窗口中运行以下命令：

```powershell
# 启用区分大小写：
fsutil file setcasesensitiveinfo <项目文件夹路径> enable

# 禁用区分大小写：
fsutil file setcasesensitiveinfo <项目文件夹路径> disable
```

如果你尚未启用 WSL，可以在 *以管理员身份运行* 的 PowerShell 窗口中输入以下行，然后在要求时重启：

```powershell
Enable-WindowsOptionalFeature -Online -FeatureName Microsoft-Windows-Subsystem-Linux
```
