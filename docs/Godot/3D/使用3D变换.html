使用3D变换

如果你以前从未制作过 3D 游戏，那么在三维空间中处理旋转起初可能会让人感到困惑。从 2D 转型过来，很自然的想法是 “哦，这就像在 2D 中旋转一样，只是现在旋转发生在 X、Y 和 Z 轴上”。

起初，这似乎很容易。对于简单的游戏，这种思维方式甚至可能已经足够。不幸的是，它通常是不正确的。

三维空间中的角度通常被称为“欧拉角 (Euler Angles)”。

欧拉角由数学家莱昂哈德·欧拉 (Leonhard Euler) 在 1700 年代初期引入。

这种表示 3D 旋转的方法在当时是开创性的，但在用于游戏开发时它有几个缺点（这对于一个戴着滑稽帽子的人来说是可以预见的）。本文档的目的是解释原因，并概述在编写 3D 游戏时处理变换的最佳实践。

欧拉角的问题​

虽然每个轴都有旋转似乎很直观，但事实是这并不实用。

轴顺序​

其主要原因在于没有唯一的方法来从角度构建朝向。没有一个标准的数学函数能同时处理所有角度并产生实际的 3D 旋转。从角度产生朝向的唯一方法是按一定的任意顺序逐个轴地旋转对象。

这可以通过先旋转 X，然后 Y，最后 Z 来完成。或者，你可以先旋转 Y，然后 Z，最后 X。任何顺序都可行，但根据顺序的不同，对象的最终朝向不一定相同。实际上，这意味着有多种方法可以从 3 个不同的角度构建朝向，具体取决于旋转的顺序。

想象你正在开发一个第一人称控制器（例如 FPS 游戏）。左右移动鼠标控制你平行于地面的视角，而上下移动鼠标则控制玩家视野的上下移动。

在这种情况下，为了达到预期的效果，必须先应用 Y 轴旋转（在这种情况下是“向上”，因为 Godot 使用“Y-Up”朝向），然后应用 X 轴旋转。

取决于所需的项目类型或效果，你希望应用轴旋转的顺序可能会有所不同。因此，仅应用 X、Y 和 Z 旋转是不够的：你还需要一个旋转顺序。

插值​

使用欧拉角的另一个问题是插值。想象你想在两个不同的相机或敌人位置（包括旋转）之间过渡。一种合乎逻辑的方法是在两个位置之间对角度进行插值。人们期望它看起来像这样：

但在使用角度时，这并不总是能达到预期的效果：

相机实际上旋转到了相反的方向！

发生这种情况有几个原因：

  • 旋转不能线性映射到朝向，因此对它们进行插值并不总是能得到最短路径（例如，从 270 度到 0 度与从 270 度到 360 度不同，尽管角度是等效的）。
  • 万向节锁 (Gimbal lock) 在起作用（第一个和最后一个旋转轴重合，从而丢失了一个自由度）。有关此问题的详细解释，请参阅维基百科关于万向节锁的页面。

对欧拉角说不​

所有这一切的结果是，你不应该在 Godot 游戏中使用 Node3D 节点的 rotation 属性。它主要是为了在编辑器中使用，以便与 2D 引擎保持一致，以及用于简单的旋转（通常只有一个轴，或者在有限情况下的两个轴）。尽管你可能会受此诱惑，但不要使用它。

相反，有更好的方法来解决你的旋转问题。

变换 (Transforms) 简介​

Godot 使用 Transform3D 数据类型来表示朝向。每个 Node3D 节点都包含一个 transform 属性，如果父节点是 Node3D 派生类型，则它是相对于父节点变换的。

也可以通过 global_transform 属性访问世界坐标变换。

变换具有 基 (Basis)（transform.basis 子属性），它由三个 Vector3 向量组成。这些可以通过 transform.basis 属性访问，也可以通过 transform.basis.x、transform.basis.y 和 transform.basis.z 直接访问。每个向量都指向其轴旋转的方向，因此它们有效地描述了节点的总旋转。缩放（只要是均匀的）也可以从轴的长度中推导出来。基 也可以被解释为一个 3x3 矩阵，并作为 transform.basis[x][y] 使用。

默认基（未经修改）类似于：

  • GDScript
  • C#
var basis = Basis()
# 包含以下默认值：
basis.x = Vector3(1, 0, 0) # 沿 X 轴指向的向量
basis.y = Vector3(0, 1, 0) # 沿 Y 轴指向的向量
basis.z = Vector3(0, 0, 1) # 沿 Z 轴指向的向量
// 由于 C# 中结构体的技术限制，默认构造函数将包含所有字段的零值。
var defaultBasis = new Basis();
GD.Print(defaultBasis); // 打印: ((0, 0, 0), (0, 0, 0), (0, 0, 0))

// 相反，我们可以使用 Identity 属性。
var identityBasis = Basis.Identity;
GD.Print(identityBasis.X); // 打印: (1, 0, 0)
GD.Print(identityBasis.Y); // 打印: (0, 1, 0)
GD.Print(identityBasis.Z); // 打印: (0, 0, 1)

// Identity 基等效于：
var basis = new Basis(Vector3.Right, Vector3.Up, Vector3.Back);
GD.Print(basis); // 打印: ((1, 0, 0), (0, 1, 0), (0, 0, 1))

这也类似于一个 3x3 单位矩阵。

按照 OpenGL 的惯例，X 是 右 (Right) 轴，Y 是 上 (Up) 轴，Z 是 前 (Forward) 轴（注：在 3D 中 Z 通常指向后方，-Z 为前方）。

除了基，变换还有一个 原点 (origin)。这是一个 Vector3，指定了此变换距离实际原点 (0, 0, 0) 有多远。将基与原点结合，变换有效地在空间中表示了唯一的平移、旋转和缩放。

可视化变换的一种方法是在“本地空间 (local space)”模式下查看对象的 3D 控制轴。

控制轴的箭头显示了基的 X、Y 和 Z 轴（分别用红、绿、蓝表示），而控制轴的中心位于对象的原点。

有关向量和变换数学的更多信息，请阅读向量数学教程。

操作变换​

当然，变换不像角度那样直观，并且也有其自身的问题。

可以旋转变换，方法是将它的基与另一个基相乘（这称为累积），或者使用旋转方法。

  • GDScript
  • C#
var axis = Vector3(1, 0, 0) # 或 Vector3.RIGHT
var rotation_amount = 0.1
# 绕 X 轴旋转变换 0.1 弧度。
transform.basis = Basis(axis, rotation_amount) * transform.basis
# 简写形式
transform.basis = transform.basis.rotated(axis, rotation_amount)
Transform3D transform = Transform;
Vector3 axis = new Vector3(1, 0, 0); // 或 Vector3.Right
float rotationAmount = 0.1f;

// 绕 X 轴旋转变换 0.1 弧度。
transform.Basis = new Basis(axis, rotationAmount) * transform.Basis;
// 简写形式
transform.Basis = transform.Basis.Rotated(axis, rotationAmount);

Transform = transform;

Node3D 中的一种方法简化了此操作：

  • GDScript
  • C#
# 绕 X 轴旋转变换 0.1 弧度。
rotate(Vector3(1, 0, 0), 0.1)
# 简写形式
rotate_x(0.1)
// 绕 X 轴旋转变换 0.1 弧度。
Rotate(new Vector3(1, 0, 0), 0.1f);
// 简写形式
RotateX(0.1f);

这会相对于父节点旋转该节点。

要相对于对象空间（节点自身的变换）进行旋转，请使用以下方法：

  • GDScript
  • C#
# 绕对象的本地 X 轴旋转 0.1 弧度。
rotate_object_local(Vector3(1, 0, 0), 0.1)
// 绕对象的本地 X 轴旋转 0.1 弧度。
RotateObjectLocal(new Vector3(1, 0, 0), 0.1f);

轴应在对象的本地坐标系中定义。例如，要绕对象的本地 X、Y 或 Z 轴旋转，请分别为 X 轴使用 Vector3.RIGHT，Y 轴使用 Vector3.UP，Z 轴使用 Vector3.FORWARD。

精度误差​

对变换进行连续操作会导致由于浮点误差而丢失精度。这意味着每个轴的缩放可能不再正好是 1.0，并且它们之间的角度可能不再正好是 90 度。

如果变换每帧都在旋转，它最终会随着时间的推移开始变形。这是不可避免的。

有两种不同的方法来处理这个问题。第一种是在一段时间后（如果你每帧都修改它，可能每帧一次）对变换进行 正交规范化 (orthonormalize)：

  • GDScript
  • C#
transform = transform.orthonormalized()
transform = transform.Orthonormalized();

这将使所有轴的长度再次变为 1.0 并且彼此成 90 度。但是，应用于变换的任何缩放都将丢失。

建议你不要缩放将要被操作的节点；而是缩放它们的子节点（如 MeshInstance3D）。如果你绝对必须缩放该节点，请在最后重新应用缩放：

  • GDScript
  • C#
transform = transform.orthonormalized()
transform = transform.scaled(scale)
transform = transform.Orthonormalized();
transform = transform.Scaled(scale);

获取信息​

此时你可能会想：“好吧，但我该如何从变换中获取角度呢？”。答案仍然是：你不获取。你必须尽最大努力停止思考角度。

想象你需要朝着玩家面对的方向射击子弹。只需使用前向轴即可。

  • GDScript
  • C#
# 在 RigidBody3D 上。

# 请记住，-Z 是前方。
bullet.transform = transform
bullet.linear_velocity = -transform.basis.z * BULLET_SPEED
// 在 RigidBody3D 上。

// 请记住，-Z 是前方。
bullet.Transform = Transform;
bullet.LinearVelocity = -Transform.Basis.Z * BulletSpeed;

敌人是否正在看着玩家？为此使用点积（有关点积的解释，请参阅向量数学教程）：

  • GDScript
  • C#
# 获取从玩家到敌人的方向向量
var direction = enemy.transform.origin - player.transform.origin
if direction.dot(enemy.transform.basis.z) > 0:
	enemy.im_watching_you(player)
// 获取从玩家到敌人的方向向量
Vector3 direction = enemy.Transform.Origin - player.Transform.Origin;
if (direction.Dot(enemy.Transform.Basis.Z) > 0)
{
    enemy.ImWatchingYou(player);
}

左侧平移 (Strafe left)：

  • GDScript
  • C#
# 在 CharacterBody3D 上。

# 请记住，-X 是左侧。
if Input.is_action_pressed("strafe_left"):
	velocity = -transform.basis.x * MOVE_SPEED

move_and_slide()
// 在 CharacterBody3D 上。

// 请记住，-X 是左侧。
if (Input.IsActionPressed("strafe_left"))
{
    Velocity = -Transform.Basis.X * MoveSpeed;
}

MoveAndSlide();

跳跃：

  • GDScript
  • C#
# 在 CharacterBody3D 上。

# 请记住，+Y 是上方。
if Input.is_action_just_pressed("jump"):
	velocity.y = JUMP_SPEED

move_and_slide()
// 在 CharacterBody3D 上。

// 请记住，+Y 是上方。
if (Input.IsActionJustPressed("jump"))
{
    Velocity = Vector3.Up * JumpSpeed;
}

MoveAndSlide();

所有常见的行为和逻辑都可以仅用向量来完成。

设置信息​

当然，也有一些情况你确实想向变换设置信息。想象一个第一人称控制器或轨道摄像机。这些肯定是用角度完成的，因为你确实想要变换以特定的顺序发生。

对于这种情况，请将角度和旋转保留在变换之外，并每帧设置它们。不要尝试获取并复用它们，因为变换不适合以这种方式使用。

环视示例（FPS 风格）：

  • GDScript
  • C#
# 累加器
var rot_x = 0
var rot_y = 0

func _input(event):
	if event is InputEventMouseMotion and event.button_mask & 1:
		# 修改累加的鼠标旋转
		rot_x -= event.screen_relative.x * LOOKAROUND_SPEED
		rot_y -= event.screen_relative.y * LOOKAROUND_SPEED
		transform.basis = Basis() # 重置旋转
		rotate_object_local(Vector3(0, 1, 0), rot_x) # 先绕 Y 旋转
		rotate_object_local(Vector3(1, 0, 0), rot_y) # 然后绕 X 旋转
// 累加器
private float _rotationX = 0f;
private float _rotationY = 0f;

public override void _Input(InputEvent @event)
{
    if (@event is InputEventMouseMotion mouseMotion)
    {
        // 修改累加的鼠标旋转
        _rotationX -= mouseMotion.ScreenRelative.X * LookAroundSpeed;
        _rotationY -= mouseMotion.ScreenRelative.Y * LookAroundSpeed;

        // 重置旋转
        Transform3D transform = Transform;
        transform.Basis = Basis.Identity;
        Transform = transform;

        RotateObjectLocal(Vector3.Up, _rotationX); // 先绕 Y 轴旋转
        RotateObjectLocal(Vector3.Right, _rotationY); // 然后绕 X 轴旋转
    }
}

如你所见，在这种情况下，将旋转保留在外部甚至更简单，然后将变换用作最终朝向。

使用四元数 (Quaternions) 进行插值​

在两个变换之间进行插值可以使用四元数高效地完成。关于四元数如何工作的更多信息可以在互联网的其他地方找到。对于实际应用，只要了解它们的主要用途是进行最近路径插值就足够了。也就是说，如果你有两个旋转，四元数将允许使用最近的轴在它们之间平滑地进行插值。

将旋转转换为四元数非常简单。

  • GDScript
  • C#
# 将基转换为四元数，请记住缩放会丢失
var a = Quaternion(transform.basis)
var b = Quaternion(transform2.basis)
# 使用球面线性插值 (SLERP) 进行插值。
var c = a.slerp(b, 0.5) # 找到 a 和 b 之间的中间点
# 应用回来
transform.basis = Basis(c)
// 将基转换为四元数，请记住缩放会丢失
var a = new Quaternion(transform.Basis);
var b = new Quaternion(transform2.Basis);
// 使用球面线性插值 (SLERP) 进行插值。
var c = a.Slerp(b, 0.5f); // 找到 a 和 b 之间的中间点
// 应用回来
transform.Basis = new Basis(c);

Quaternion 类型参考中有关于该数据类型的更多信息（它也可以执行变换累积、变换点等，尽管这些较少使用）。如果你多次插值或对四元数应用操作，请记住它们最终需要被归一化。否则，它们也会受到数值精度误差的影响。

四元数在进行相机/路径等插值时非常有用，因为结果始终是正确且平滑的。

变换是你的好朋友​

对于大多数初学者来说，习惯于使用变换可能需要一些时间。然而，一旦你习惯了它们，你会欣赏它们的简单和强大。

不要犹豫在 Godot 的任何在线社区中寻求有关此主题的帮助，一旦你变得足够自信，请帮助他人！
