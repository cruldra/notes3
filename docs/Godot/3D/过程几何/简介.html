简介

在 Godot 中有许多种方法可以程序化地生成几何体。在本系列教程中，我们将探讨其中的几种。每种技术都有其自身的优缺点，因此最好了解每种技术以及它在特定情况下的用途。

  • 使用 ArrayMesh
  • 使用 MeshDataTool
  • 使用 SurfaceTool
  • 使用 ImmediateMesh
注意

这里描述的所有过程几何生成方法都在 CPU 上运行。Godot 目前还不支持在 GPU 上生成几何体。

什么是几何体？​

几何体（Geometry）是“形状”的一种花哨说法。在计算机图形学中，几何体通常由称为“顶点”（Vertices）的位置数组表示。在 Godot 中，几何体由 Mesh（网格）表示。

什么是 Mesh？​

Godot 中许多事物的名称中都带有 mesh：Mesh、ArrayMesh、ImmediateMesh、MeshInstance3D、MultiMesh 以及 MultiMeshInstance3D。虽然它们都相关，但用途略有不同。

Mesh 和 ArrayMesh 是使用 MeshInstance3D 节点绘制的资源。像 Mesh 和 ArrayMesh 这样的资源不能直接添加到场景中。MeshInstance3D 代表场景中网格的一个实例。你可以在多个 MeshInstance3D 中重用同一个网格，从而使用不同的材质或变换（缩放、旋转、位置等）在场景的不同部分绘制它。

如果你要多次绘制同一个对象，使用 MultiMesh 结合 MultiMeshInstance3D 会很有帮助。MultiMeshInstance3D 利用硬件实例化（Hardware Instancing），可以非常廉价地绘制数千次网格。使用 MultiMeshInstance3D 的缺点是，网格的每个表面在所有实例中只能使用一种材质。它使用一个实例数组来存储每个实例的不同颜色 and 变换，但每个表面的所有实例都使用相同的材质。

Mesh 的组成​

一个 Mesh 由一个或多个 Surface（表面）组成。表面是一个由多个子数组组成的数组，这些子数组包含顶点、法线、UV 等。通常，构建表面和网格的过程在 RenderingServer 中对用户是隐藏的，但通过 ArrayMesh，用户可以通过传入包含表面信息的数组来手动构建 Mesh。

表面 (Surfaces)​

每个表面都有自己的材质。或者，当你使用 MeshInstance3D 时，可以通过 material_override 属性覆盖 Mesh 中所有表面的材质。

表面数组 (Surface array)​

表面数组是一个长度为 ArrayMesh.ARRAY_MAX 的数组。数组中的每个位置都填充了一个包含逐顶点信息的子数组。例如，位于 ArrayMesh.ARRAY_NORMAL 的数组是一个包含顶点法线的 PackedVector3Array。详见 Mesh.ArrayType。

表面数组可以是索引的或非索引的。创建非索引数组非常简单，只需不在 ArrayMesh.ARRAY_INDEX 索引处分配数组即可。非索引数组为每个三角形存储唯一的顶点信息，这意味着当两个三角形共享一个顶点时，该顶点在数组中是重复的。索引表面数组仅为每个唯一顶点存储顶点信息，然后还存储一个索引数组，该数组映射了如何从顶点数组构造三角形。通常，使用索引数组更快，但这意味着你必须在三角形之间共享顶点数据，这并不总是你想要的（例如，当你想要逐面法线时）。

工具 (Tools)​

Godot 提供了访问和处理几何体的不同方式。以下教程将提供有关每种方式的更多信息。

ArrayMesh​

ArrayMesh 资源扩展了 Mesh，添加了一些生活质量功能，最重要的是，能够通过脚本构建 Mesh 表面。

有关 ArrayMesh 的更多信息，请参阅 ArrayMesh 教程。

MeshDataTool​

MeshDataTool 是一个资源，它将 Mesh 数据转换为可以在运行时修改的顶点、面和边的数组。

有关 MeshDataTool 的更多信息，请参阅 MeshDataTool 教程。

SurfaceTool​

SurfaceTool 允许使用类似 OpenGL 1.x 即时模式风格的接口来创建 Mesh。

有关 SurfaceTool 的更多信息，请参阅 SurfaceTool 教程。

ImmediateMesh​

ImmediateMesh 是一种使用即时模式风格接口（如 SurfaceTool）来绘制对象的网格。ImmediateMesh 与 SurfaceTool 的区别在于，ImmediateMesh 是通过代码动态直接绘制的，而 SurfaceTool 用于生成一个你可以随意处理的 Mesh。

ImmediateMesh 因其简单的 API 而对原型设计非常有用，但由于每次更改都会重新构建几何体，因此速度较慢。它最适用于添加简单的几何体进行视觉调试（例如通过绘制线条来可视化物理射线检测等）。

有关 ImmediateMesh 的更多信息，请参阅 ImmediateMesh 教程。

我应该使用哪一个？​

你使用哪种方法取决于你想要实现的目标以及你习惯的处理方式。

SurfaceTool 和 ArrayMesh 都非常适合生成不随时间变化的静态几何体（网格）。

使用 ArrayMesh 比使用 SurfaceTool 稍快，但 API 稍微更具挑战性。此外，SurfaceTool 具有一些生活质量方法，如 generate_normals() 和 index()。

ImmediateMesh 比 ArrayMesh 和 SurfaceTool 受到更多限制。但是，如果你需要每帧都更改几何体，它提供了一个更简单的接口，这可能比每帧生成 ArrayMesh 稍快。

MeshDataTool 不快，但它允许你访问网格的各种属性，而这些属性是其他工具无法提供的（边、面等）。当你需要此类数据来变换网格时，它非常有用，但如果不需要这些额外信息，最好不要使用它。如果你要使用的算法需要访问面或边数组，则最好使用 MeshDataTool。
