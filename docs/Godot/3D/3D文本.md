在项目中，有时需要创建文本作为 3D 场景的一部分，而不只是在 HUD（平视显示器）中。Godot 提供了两种方法来实现这一点：**Label3D** 节点和用于 MeshInstance3D 节点的 **TextMesh** 资源。

此外，Godot 还可以根据 3D 点在摄像机上的位置来定位 Control 节点。在 Label3D 和 TextMesh 不够灵活的情况下，这可以作为“真实”3D 文本的替代方案。

:::info 另请参阅
你可以通过 [3D 标签和文本示例项目](https://github.com/godotengine/godot-demo-projects/tree/master/3d/labels_and_texts)查看 3D 文本的实际效果。

本页**不**涵盖如何在 3D 环境中显示 GUI 场景。有关如何实现该功能的信息，请参阅 [3D 中的 GUI](https://github.com/godotengine/godot-demo-projects/tree/master/viewport/gui_in_3d) 示例项目。
:::

## Label3D

![Label3D 示例](https://docs.godotengine.org/en/stable/_images/label_3d.png)

Label3D 的行为类似于 Label 节点，但处于 3D 空间中。与 Label 节点不同，Label3D 节点**不会**继承 GUI 主题（Theme）的属性。然而，它的外观仍然是可定制的，并使用与 Control 节点相同的字体子资源（包括对 MSDF 字体渲染的支持）。

### 优点

*   **生成速度快**：Label3D 的生成速度比 TextMesh 快。虽然两者都使用缓存机制仅渲染一次新字形，但 Label3D 的（重新）生成速度仍然更快，尤其是对于长文本。这可以避免低端 CPU 或移动设备在游戏过程中出现卡顿。
*   **字体支持广泛**：Label3D 可以使用位图字体（Bitmap fonts）和动态字体（Dynamic fonts，无论是否带有 MSDF 或 mipmaps）。在这方面它比 TextMesh 更灵活，特别是对于渲染具有自相交轮廓的字体或彩色字体（表情符号/Emoji）。

:::info 另请参阅
有关配置字体导入的指南，请参阅[使用字体](https://docs.godotengine.org/en/stable/tutorials/ui/gui_using_fonts.html#doc-gui-using-fonts)。
:::

### 局限性

默认情况下，Label3D 与 3D 环境的交互有限。如果启用了 **Shaded**（受光）标志，它可以被几何体遮挡并被光源照射。然而，即使 Label3D 的 GeometryInstance3D 属性中的 **Cast Shadow** 设置为 **On**，它也不会投射阴影。这是因为该节点在内部生成了一个四边形网格（Quad Mesh，每个字形一个四边形）并带有透明纹理，具有与 Sprite3D 相同的限制。当多个 Label3D 重叠时，透明度排序问题也会变得明显，特别是当它们具有轮廓时。

这可以通过将 Label3D 的透明度模式设置为 **Alpha Cut** 来缓解，代价是文本渲染不够平滑。**Opaque Pre-Pass** 透明度模式可以在允许 Label3D 投射阴影的同时保持文本平滑，但仍会存在一些透明度排序问题。

有关更多信息，请参阅 3D 渲染局限性页面中的[透明度排序](https://docs.godotengine.org/en/stable/tutorials/3d/3d_rendering_limitations.html#doc-3d-rendering-limitations-transparency-sorting)部分。

当从远处观察 Label3D 时，文本渲染质量也可能下降。为了提高文本渲染质量，请[在字体上启用 mipmaps](https://docs.godotengine.org/en/stable/tutorials/ui/gui_using_fonts.html#doc-using-fonts-mipmaps) 或[切换字体使用 MSDF 渲染](https://docs.godotengine.org/en/stable/tutorials/ui/gui_using_fonts.html#doc-using-fonts-msdf)。

## TextMesh

![TextMesh 示例](https://docs.godotengine.org/en/stable/_images/text_mesh.png)

TextMesh 资源与 Label3D 有相似之处。它们都在 3D 场景中显示文本，并使用相同的字体子资源。然而，TextMesh 不生成透明四边形，而是生成代表字形轮廓的 3D 几何体，并具有网格（Mesh）的属性。因此，TextMesh 默认受光，并自动向环境投射阴影。TextMesh 还可以应用材质（包括自定义着色器）。

![TextMesh 纹理参考](https://docs.godotengine.org/en/stable/_images/text_mesh_texture.png) ![TextMesh 应用纹理效果](https://docs.godotengine.org/en/stable/_images/text_mesh_textured.png)

### 优点

TextMesh 与 Label3D 相比有几个优点：

*   **逐面纹理**：TextMesh 可以使用纹理来修改文本不同侧面的颜色。
*   **3D 厚度**：TextMesh 几何体可以具有实际的深度，赋予字形 3D 外观。
*   **支持着色器**：与 Label3D 不同，TextMesh 可以使用自定义着色器（Shaders）。

### 局限性

TextMesh 存在一些限制：

*   **无内置轮廓**：与 Label3D 不同，TextMesh 不支持内置轮廓。不过这可以通过自定义着色器来模拟。
*   **仅支持动态字体**：仅支持动态字体（`.ttf`, `.otf`, `.woff`, `.woff2`）。**不**支持 `.fnt` 或 `.font` 格式的位图字体。
*   **自相交轮廓问题**：具有自相交轮廓的字体将无法正确渲染。如果你发现从 Google Fonts 等网站下载的字体存在渲染问题，请尝试从字体作者的官方网站下载。
*   **抗锯齿需求**：文本渲染的抗锯齿需要启用全场景抗锯齿方法，如 MSAA、FXAA 和时间抗锯齿（TAA）。如果没有启用任何抗锯齿方法，文本会显得粗糙，尤其是在远处。有关更多信息，请参阅 [3D 抗锯齿](https://docs.godotengine.org/en/stable/tutorials/3d/3d_antialiasing.html#doc-3d-antialiasing)。

## 投影 Label 节点 (或其他 Control)

最后一种解决方案设置起来更复杂，但提供了最大的灵活性：将 2D 节点投影到 3D 空间。这可以通过在脚本的 `_process()` 函数中使用 Camera3D 节点的 [unproject_position](https://docs.godotengine.org/en/stable/classes/class_camera3d.html#class-camera3d-method-unproject-position) 方法返回的值来实现。然后应使用此返回值设置 Control 节点的 `position` 属性。

有关此示例，请参阅 [3D 航点 (Waypoints)](https://github.com/godotengine/godot-demo-projects/tree/master/3d/waypoints) 演示。

### 优点

*   **功能强大**：可以使用任何 Control 节点，包括 Label、RichTextLabel 甚至 Button 节点。这允许强大的格式化和 GUI 交互。
*   **定位自由**：基于脚本的方法允许完全自由的定位。例如，这使得在 3D 标记移出屏幕时将其钉在屏幕边缘变得相当容易。
*   **遵循主题**：遵循 Control 的主题设置，允许更轻松地进行适用于整个项目的全局自定义。

### 局限性

*   **无法被遮挡**：投影的 Control 无法以任何方式被 3D 几何体遮挡。如果目标位置被碰撞体遮挡，你可以使用 RayCast 来完全隐藏该控件，但这不允许控件部分隐藏在墙后。
*   **手动缩放**：通过调节 Control 的 `scale` 属性来随距离更改文本大小是可能的，但需要手动完成。Label3D 和 TextMesh 自动处理这一点，但灵活性较差（无法以像素为单位设置最小/最大文本大小）。
*   **分辨率处理**：在脚本中必须考虑到分辨率和长宽比的变化，这可能具有挑战性。

## 我该使用 Label3D、TextMesh 还是投影 Control？

在大多数情况下，建议使用 **Label3D**，因为它更容易设置且提供更高的渲染质量（特别是在禁用 3D 抗锯齿的情况下）。

对于高级用例，**TextMesh** 更灵活，因为它允许使用自定义着色器为文本设置样式。自定义着色器允许修改最终几何体，例如使文本沿表面弯曲。由于文本是实际的 3D 几何体，它可以选择性地具有深度，并且还可以参与全局照明（Global Illumination）。

如果你需要 BBCode 或 Control 主题支持等功能，那么使用投影的 **RichTextLabel** 节点是唯一的选择。
