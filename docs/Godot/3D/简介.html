简介

创建 3D 游戏是一项挑战。额外的 Z 轴坐标使得许多在 2D 游戏中行之有效的简单技术不再适用。为了帮助这一过渡，值得一提的是，Godot 为 2D 和 3D 使用了类似的 API。大多数节点是相同的，并且在 2D 和 3D 版本中都存在。事实上，值得查看 3D 平台游戏教程或 3D 运动学角色教程，它们与其 2D 对应版本几乎完全相同。

Godot 第三人称射击 (TPS) 演示，可在 Github 仓库 或 资源库 中找到。

在 3D 中，数学比 2D 稍微复杂一些。有关为游戏开发者（而非数学家或工程师）编写的相关数学介绍，请查看向量数学和使用 3D 变换。

3D 工作区​

3D 场景的编辑在 3D 工作区中完成。该工作区可以手动选择，但当选中 Node3D 节点时，它会自动被选中。

与 2D 类似，工作区选择器下方的选项卡用于在当前打开的场景之间切换，或使用加号 (+) 按钮创建一个新场景。左侧和右侧的停靠面板你应该在编辑器介绍中已经熟悉了。

在场景选择器下方是主工具栏，主工具栏下方是 3D 视口。

主工具栏​

主工具栏中的一些按钮与 2D 工作区中的按钮相同。如果将鼠标光标悬停在按钮上一秒钟，会显示带有快捷键的简短说明。如果执行另一个按键操作，某些按钮可能会有额外的功能。下面从左到右回顾了每个按钮的主要功能及其默认快捷键：

  • 选择模式 (Select Mode) (Q)：允许在视口中选择节点。左键点击节点即可选中。释放后，左键点击并拖动矩形会选中该矩形边界内的所有节点。选择时按住 Shift 可添加更多节点到选区。按住 Shift 点击已选中的节点可取消选择该节点。在此模式下，你可以使用控制轴 (Gizmos) 进行移动或旋转。
  • 移动模式 (Move Mode) (W)：为选中的节点启用移动（或平移）模式。详见空间与操作控制轴。
  • 旋转模式 (Rotate Mode) (E)：为选中的节点启用旋转模式。详见空间与操作控制轴。
  • 缩放模式 (Scale Mode) (R)：为选中的节点启用缩放并显示不同轴的缩放控制轴。详见空间与操作控制轴。
  • 显示点击位置的可选节点列表：正如描述所示，如果点击区域内有多个节点，这会以右键菜单的形式提供点击位置的可选节点列表。
  • 锁定 (Lock) (Ctrl + L)：锁定选中的节点，防止在视口中进行选择和移动。再次点击该按钮（或使用 Ctrl + Shift + L）可解锁选中的节点。锁定节点只能在场景树中选择。在场景树中，锁定节点的名字旁边会有一个挂锁图标，点击此挂锁也可以解锁节点。
  • 编组选中的节点 (Group selected nodes) (Ctrl + G)：如果选中了任何子节点，这允许选择根节点。使用 Ctrl + G 可取消编组。此外，点击场景树中的取消编组按钮也可以执行相同操作。
  • 标尺模式 (Ruler Mode) (M)：启用后，你可以点击并拖动以测量场景中的距离（单位为米）。
  • 使用本地空间 (Use Local Space) (T)：如果启用，节点的控制轴将使用当前节点的旋转角度绘制，而不是使用全局视口轴。
  • 使用吸附 (Use Snap) (Y)：如果启用，移动和旋转将吸附到网格。执行操作时按住 Ctrl 也可以临时激活吸附。更改吸附选项的设置在下面说明。
  • 切换预览太阳光：如果场景中不存在 DirectionalLight3D，可以使用预览太阳光作为光源。详见预览环境与光照。
  • 切换预览环境：如果场景中不存在 WorldEnvironment，可以使用预览环境作为占位符。详见预览环境与光照。
  • 编辑太阳和环境设置 (三个点)：打开菜单以配置预览太阳光和环境设置。详见预览环境与光照。
  • 变换 (Transform) 菜单：它有三个选项：
    • 将对象吸附到地面 (Snap Object to Floor)：将对象吸附到实体地面。
    • 变换对话框 (Transform Dialog)：打开对话框手动调节变换参数（平移、旋转、缩放和变换）。
    • 吸附设置 (Snap Settings)：允许你更改平移、旋转吸附（角度）和缩放吸附（百分比）设置。
  • 视图 (View) 菜单：控制视图选项并启用额外的视口：

在该菜单中，你还可以显示/隐藏网格（默认设置为 1x1 米）和原点（蓝色、绿色和红色轴线相交处）。此外，可以在此菜单中切换特定类型的控制轴 (Gizmos)。

睁开的眼睛表示控制轴可见，闭上的眼睛表示隐藏。半开的眼睛表示它在不透明表面后也可见。

点击视图菜单中的 设置 (Settings) 会打开一个窗口，用于更改 垂直视野 (VFOV) 参数（度）、近裁面 (Z-Near) 和 远裁面 (Z-Far) 值。

在视图菜单旁边，可能会看到额外的按钮。在本章开头的工具栏图像中，出现了一个额外的 Mesh (网格) 按钮，因为选中了一个 MeshInstance3D。该菜单提供了一些快速操作或工具来处理特定的节点或选区。

视口视图菜单​

在 选择 工具下方，在 3D 视口中，点击三个点可打开视口的 视图菜单。隐藏编辑器 3D 视图中显示的所有控制轴也可以通过此菜单完成：

该菜单还显示当前的视图类型，并允许快速调节视口的查看角度。此外，它还提供了修改视口内节点外观的选项。

坐标系统​

Godot 在 3D 中所有的东西都使用公制 (metric)系统，1 个单位等于 1 米。物理和其他领域都针对此比例进行了调整。因此，尝试使用不同的比例通常是一个坏主意（除非你知道自己在做什么）。

在处理 3D 资源时，最好始终以正确的比例工作（在你的 3D 建模软件中将单位设置为公制）。Godot 允许在导入后进行缩放，虽然这在大多数情况下有效，但在极少数情况下，它可能会在渲染或物理等精细区域引入浮点精度问题（从而导致故障或瑕疵）。确保你的美术师始终以正确的比例工作！

Y 坐标用于“向上”。至于水平的 X/Z 轴，Godot 使用 右手 (right-handed) 坐标系。这意味着对于大多数需要对齐的对象（如灯光或相机），Z 轴被用作“指向”方向。这一惯例大致意味着：

  • X 是左右
  • Y 是上下
  • Z 是前后

空间与操作控制轴​

在 3D 视图中移动、旋转和缩放对象是通过操作器控制轴 (Gizmos) 完成的。每个轴都用一种颜色代表：红、绿、蓝分别代表 X、Y、Z。这一惯例也适用于网格和其他控制轴（以及着色器语言、Vector3、Color 等组件的排序）。

一些有用的快捷键：

  • 要吸附放置或旋转，在移动、缩放或旋转时按住 Ctrl。
  • 要将视图中心对准选中的对象，请按 F。

在视口中，可以点击并按住箭头在某个轴上移动对象。可以点击并按住圆弧来旋转对象。要锁定一个轴并在其他两个轴上自由移动对象，可以点击、按住并拖动彩色矩形。

如果变换模式从 选择模式 (Select Mode) 更改为 缩放模式 (Scale Mode)，箭头将被立方体取代，可以拖动立方体来缩放对象，就像移动对象一样。

导航 3D 环境​

在 3D 环境中，调节你观察场景的视点或角度通常很重要。在 Godot 中，在视口（或空间编辑器）中导航 3D 环境可以通过多种方式完成。

默认的 3D 场景导航控制类似于 Blender（旨在在自由软件管道中保持某种一致性），但在“编辑器设置”中包含了自定义鼠标按键和行为以类似于其他工具的选项。要将控制更改为 Maya 或 Modo 控制，可以导航至 编辑器设置 > 编辑器 > 3D。然后，在 导航 (Navigation) 下搜索 导航方案 (Navigation Scheme)。

使用默认设置，以下快捷键控制视口中的导航：

按下鼠标中键并拖动鼠标允许你围绕屏幕中心旋转。

也可以点击并按住视口右上角的操作器控制轴来围绕中心旋转：

点击其中一个彩色圆圈会将视图设置为选定的正交视图，视口的视图菜单也会相应更新。

如果视口启用了 透视 (Perspective) 视图（可以在视口的“视图菜单”看到，而不是主工具栏上的“视图菜单”），在视口中按住鼠标右键或按下 Shift + F 会切换到“漫游 (free-look)”模式。在这种模式下，你可以移动鼠标来环视四周，使用 W A S D 键在视图中飞行，E 向上，Q 向下。要禁用此模式，请释放鼠标右键或再次按下 Shift + F。

在漫游模式下，你可以使用 Shift 临时增加飞行速度，或使用 Alt 减小速度。要更改并保持速度修饰符，请使用鼠标滚轮向上或向下，分别增加或减小速度。

在正交模式下，按住鼠标右键会改为平移视图。使用数字键盘 5 可以在透视图和正交视图之间切换。

使用 Blender 风格的变换快捷键​

自 Godot 4.2 起，你可以为平移、旋转和缩放节点启用 Blender 风格的快捷键。在 Blender 中，这些快捷键是：

  • G 用于平移 (Translating)
  • R 用于旋转 (Rotating)
  • S 用于缩放 (Scaling)

在聚焦 3D 编辑器视口时按下快捷键后，移动鼠标或输入数字即可将选中的节点在 3D 单位中移动指定的量。你可以通过指定轴的字母来约束移动到特定轴，然后输入距离（如果是通过键盘输入值）。

例如，要将选区向上移动 2.5 个单位，请按顺序输入以下序列（Godot 中 Y+ 是向上）：

G - Y - 2 - . - 5 - Enter

要在 Godot 中使用 Blender 风格的变换快捷键，请转到编辑器设置的 快捷键 (Shortcuts) 选项卡，然后在 Spatial Editor 部分：

  • 将 Begin Translate Transformation 绑定到 G。
  • 将 Begin Rotate Transformation 绑定到 R。
  • 将 Begin Scale Transformation 绑定到 S。
  • 最后，解除 Scale Mode 的绑定，以免其快捷键与 Begin Rotate Transformation 冲突。
提示

更多快捷键可以在 3D / 空间编辑器页面找到。

Node3D 节点​

Node2D 是 2D 的基础节点。Control 是所有 GUI 的基础节点。按照这一推理，3D 引擎对所有 3D 的东西使用 Node3D 节点。

Node3D 具有本地变换，这是相对于父节点的（只要父节点也是 或继承自 Node3D 类型）。该变换可以作为 3×4 Transform3D 访问，或者作为 3 个代表位置、欧拉旋转（X、Y 和 Z 轴角度）和缩放的 Vector3 成员访问。

3D 内容​

不像 2D 那样加载图像内容并绘制非常简单，3D 要困难一些。内容需要使用专门的 3D 工具（也称为数字内容创作工具，或 DCC）创建，并导出为交换文件格式以导入到 Godot 中。这是必须的，因为 3D 格式不像图像那样标准化。

手动创作的模型（使用 3D 建模软件）​

可以导入外部工具创建的 3D 模型到 Godot 中。取决于格式，你可以导入整个场景（与其在 3D 建模软件中的样子完全一致），包括动画、骨骼绑定、混合形状 (Blend Shapes)，或者作为简单的资源。

生成的几何体​

可以通过直接使用 ArrayMesh 资源来创建自定义几何体。只需创建你的数组并使用 ArrayMesh.add_surface_from_arrays() 函数即可。还有一个辅助类 SurfaceTool，它提供了更直接的 API 和索引、生成法线、切线等的辅助工具。

在任何情况下，这种方法都适用于生成静态几何体（不会经常更新的模型），因为创建顶点数组并将其提交给 3D API 会产生显著的性能成本。

注意：要了解在 Godot 内部进行原型设计或使用外部工具，请参阅使用 CSG 进行关卡原型设计。

即时几何体 (Immediate geometry)​

相反，如果你需要生成经常更新的简单几何体，Godot 提供了一个特殊的 ImmediateMesh 资源，可以在 MeshInstance3D 节点中使用。这提供了一个 OpenGL 1.x 风格的即时模式 API 来创建点、线、三角形等。

3D 中的 2D​

虽然 Godot 包含一个强大的 2D 引擎，但许多类型的游戏在 3D 环境中使用 2D。通过使用不旋转的固定相机（正交或透视），可以使用 Sprite3D 和 AnimatedSprite3D 等节点来创建 2D 游戏，从而利用与 3D 背景混合、更真实的视差、光照/阴影效果等优势。

缺点当然是增加了复杂性，且与纯 2D 相比性能降低，同时也缺乏以像素为单位工作的参考。

环境​

除了编辑场景，通常还需要编辑环境。Godot 提供了一个 WorldEnvironment 节点，允许更改背景颜色、模式（例如放入天空盒），并应用几种类型的内置后期处理效果。环境也可以在相机 (Camera) 中被覆盖。

预览环境与光照​

默认情况下，任何没有 WorldEnvironment 节点或 DirectionalLight3D 的 3D 场景，都会开启缺失部分光照场景的预览。

预览灯光和环境仅在编辑器中的场景中可见。如果你运行场景或导出项目，它们将不会影响场景。

预览灯光和环境可以从顶部菜单通过点击它们各自的图标来开启或关闭。

图标旁边的三个点下拉菜单可用于在预览环境和灯光启用时调节其属性。

同一个项目中的每个场景都使用相同的预览太阳和环境。因此，只需进行适用于所有需要预览灯光和环境场景的调整即可。

相机 (Cameras)​

无论 3D 空间中放置了多少对象，除非场景中也添加了 Camera3D，否则什么都不会显示。相机可以在正交或透视投影下工作：

相机与父级或祖父级视口相关联（且仅显示在该视口）。由于场景树的根节点是一个视口，相机默认会显示在其上，但如果需要子视口（作为渲染目标或画中画），它们需要自己的子级相机来显示。

处理多个相机时，每个视口强制执行以下规则：

  • 如果场景树中没有相机，第一个进入场景树的将成为激活相机。后续进入场景的相机将被忽略（除非它们被设为 当前 (current)）。
  • 如果一个相机设置了 "当前 (current)" 属性，无论场景中是否有其他相机，它都会被使用。如果设置了该属性，它将激活并取代之前的相机。
  • 如果一个激活的相机离开了场景树，场景树顺序中的第一个相机将取代它的位置。

灯光 (Lights)​

背景环境会发出一些环境光，显示在表面上。尽管如此，如果没有在场景中放置任何光源，除非背景环境非常明亮，否则场景会显得相当黑暗。

大多数室外场景都有一个方向光（太阳或月亮），而室内场景通常有几个位置光源（灯、火炬……）。详见 3D 灯光与阴影以获取更多关于在 Godot 中设置灯光的信息。
