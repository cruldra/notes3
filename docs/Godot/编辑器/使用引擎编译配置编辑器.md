Godot 附带了大量内置特性。虽然这很方便，但也意味着其二进制文件体积比实际需要的要大，特别是对于那些仅使用其中一小部分特性的项目。

为了帮助减小二进制体积，可以编译禁用了特定特性的自定义导出模板。这在 [为体积优化构建](https://docs.godotengine.org/en/stable/engine_details/development/compiling/optimizing_for_size.html) 中有详细描述。然而，确定需要禁用哪些特性是一项繁琐的任务。引擎编译配置编辑器旨在解决这个问题，它提供了一个易于查看和管理这些特性的界面，同时还能够检测项目中当前正在使用的特性。

**项目 > 工具 > 引擎编译配置编辑器 (Project > Tools > Engine Compilation Configuration Editor)** 允许你为 Godot 项目创建和管理构建配置文件（Build Profiles）。

目前，你有两种选择：

*   查看列表并手动取消勾选不需要的特性。
*   使用 **从项目检测 (Detect from Project)** 按钮自动检测项目中当前使用的特性，并禁用未使用的特性。注意，这会覆盖现有的特性列表，如果你之前手动取消勾选了一些项目，它们的状态将根据项目是否实际使用了该特性而重置。

![打开引擎编译配置编辑器](https://docs.godotengine.org/en/stable/_images/engine_compilation_configuration_editor_detect.webp)

点击“从项目检测”后，将运行项目检测步骤。根据项目大小，这可能需要几秒钟到几分钟不等。检测完成后，你会看到更新后的特性列表，其中一些特性已被禁用：

![使用特性检测后的更新列表](https://docs.godotengine.org/en/stable/_images/engine_compilation_configuration_editor_detected.webp)

> **警告**
>
> 在此对话框中取消勾选特性并 **不会** 直接减小导出时的二进制体积。由于只有在编译时才能真正从二进制文件中移除特性，你仍然需要使用指定的构建配置文件来编译自定义导出模板，才能真正从引擎编译配置编辑器中获益。

现在你可以通过点击顶部的 **另存为 (Save As)** 来保存构建配置文件。配置文件可以保存在任何位置，但建议将其保存在项目文件夹中并添加到版本控制中，以便日后需要时查看。这还允许使用版本控制来跟踪配置文件的更改。

构建配置文件是一个 JSON 文件（扩展名为 `.gdbuild`），在上述示例检测后看起来像这样：

```json
{
	"disabled_build_options": {
		"disable_navigation_3d": true,
		"disable_xr": true,
		"module_godot_physics_3d_enabled": false,
		"module_msdfgen_enabled": false,
		"module_openxr_enabled": false
	},
	"disabled_classes": [
		"AESContext",
		...
		"ZIPReader"
	],
	"type": "build_profile"
}
```

在 [编译](https://docs.godotengine.org/en/stable/engine_details/development/compiling/index.html) 导出模板时，可以将此文件作为 SCons 选项传递：

```bash
scons target=template_release build_profile=/path/to/profile.gdbuild
```

构建系统将使用它来禁用未使用的类，从而减小二进制体积。

## 局限性 (Limitations)

“从项目检测”功能依赖于读取项目的场景和脚本。在以下场景中，它将无法检测到已使用的特性：

*   在运行时动态创建并运行的 GDScript 中使用的特性。
*   在 [表达式 (Expressions)](https://docs.godotengine.org/en/stable/tutorials/scripting/evaluating_expressions.html) 中使用的特性。
*   在 [GDExtensions](https://docs.godotengine.org/en/stable/tutorials/scripting/gdextension/index.html) 中使用的特性，除非语言绑定允许定义使用的类且扩展利用了该功能。
*   在运行时加载的 [外部 PCK](https://docs.godotengine.org/en/stable/tutorials/export/exporting_pcks.html) 中使用的特性。
*   可能存在某些边缘情况。如果不确定，请在 GitHub 上提交 Issue 并附带最小复现项目。

> **另见**
>
> 你可以通过传递其他减小二进制体积的选项来实现进一步的精简。有关更多信息，请参阅 [为体积优化构建](https://docs.godotengine.org/en/stable/engine_details/development/compiling/optimizing_for_size.html)。
