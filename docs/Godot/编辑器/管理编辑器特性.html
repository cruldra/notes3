管理编辑器特性

简介 (Introduction)​

在某些情况下，限制 Godot 编辑器中可用的特性是有好处的。例如，团队中的 UI 设计师可能不需要看到 3D 特性，或者教育者希望向学生逐步介绍各项功能。Godot 拥有一套名为“特性配置 (feature profiles)”的内置系统来实现这一点。

通过特性配置，可以从编辑器中隐藏主要的特性和节点。这仅仅是 隐藏界面部分，而并非真正移除对这些特性的支持，因此依赖这些特性的场景和脚本仍然可以正常运行。这也意味着特性配置并不是一种优化技术（因为它不减小体积或提高性能）。有关如何优化 Godot 的信息，请参阅 性能 (Performance)。

创建配置 (Creating a profile)​

要管理编辑器特性，请转到 编辑器 > 管理编辑器特性 (Editor > Manage Editor Features)。这会打开 管理编辑器特性配置 窗口。默认情况下没有配置，点击 创建配置 (Create Profile) 并为其命名。随后你会看到 Godot 编辑器中所有特性的列表。

第一部分允许移除主要的编辑器功能，如 3D 编辑器或脚本编辑器。在主要特性下方列出了 Godot 中的每一个类和节点，它们也可以被禁用。点击某个节点，其所有的属性和选项都会列在 额外项目 (Extra Items) 框中，这些都可以单独禁用。

分享配置 (Sharing a profile)​

要在不同的编辑器之间共享配置，点击 导出 (Export) 按钮。将自定义配置保存为 .profile 文件。要在另一个编辑器中使用它，打开该编辑器的 管理编辑器特性配置 窗口，点击 导入 (Import) 并选择该 .profile 文件。

然而，如果大量电脑需要自定义配置，这个过程可能会很繁琐。作为替代方案，你可以为 Godot 启用 自包含模式 (self-contained mode)，这允许将所有编辑器配置放在与编辑器二进制文件相同的文件夹中。详见 自包含模式。
