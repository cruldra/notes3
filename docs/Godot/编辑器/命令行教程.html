命令行教程

有些开发者喜欢广泛地使用命令行。Godot 的设计对他们非常友好，因此本页介绍了完全通过命令行工作的步骤。鉴于引擎几乎不依赖外部库，其初始化时间非常快，非常适合这种工作流。

注意

在 Windows 和 Linux 上，你可以在终端中通过指定相对或绝对路径来运行 Godot 二进制文件。

在 macOS 上，过程有所不同，因为 Godot 包含在 .app 包（这是一个 文件夹，而不是文件）中。要在 macOS 终端中运行 Godot，你必须 cd 到 Godot 应用程序包所在的文件夹，然后运行 Godot.app/Contents/MacOS/Godot，后跟任何命令行参数。如果你已将应用程序包重命名，请相应地修改此命令行。

注意，未知的命令行参数不会产生任何影响。引擎 不会 在你使用特定构建类型中不存在的命令行参数时发出警告。

通用选项 (General options)​

命令描述
-h, --help![release] 显示命令行选项列表。
--version![release] 显示版本字符串。
-v, --verbose![release] 使用详细的标准输出 (stdout) 模式。
-q, --quiet![release] 安静模式，静默 stdout 消息。错误仍会显示。
--no-header![release] 启动时不打印引擎版本和渲染方法标题。

运行选项 (Run options)​

命令描述
--, ++![release] 用户自定义参数的分隔符。后面的参数引擎不会使用，但可以通过 OS.get_cmdline_user_args() 读取。
-e, --editor![editor] 启动编辑器而不是运行场景。
-p, --project-manager![editor] 启动项目管理器，即使自动检测到项目。
--recovery-mode![editor] 以恢复模式启动编辑器，禁用通常会导致启动崩溃的功能。
--quit![release] 在第一次迭代后退出。
--path <目录>![release] 项目路径（<目录> 必须包含 project.godot 文件）。
--scene <路径>![release] 项目中应启动的场景路径或 UID。
-u, --upwards![release] 向上递归扫描文件夹以查找 project.godot 文件。
--headless![release] 启用无头模式（不显示界面和声音）。对服务器和配合 --script 使用非常有用。
--write-movie <文件>![release] 以录制电影的方式运行引擎（通常为 .avi 或 .png 扩展名）。

调试选项 (Debug options)​

命令描述
-d, --debug![release] 调试（本地标准输出调试器）。
-b, --breakpoints![release] 断点列表，格式为 source::line 的逗号分隔对。
--debug-collisions![debug] 运行场景时显示碰撞形状。
--debug-paths![debug] 运行场景时显示路径线。
--max-fps <fps>![release] 设置渲染的最大每秒帧数。
--print-fps![release] 将每秒帧数打印到标准输出。

独立工具 (Standalone tools)​

命令描述
-s, --script <脚本>![release] 运行脚本。路径应相对于项目或为绝对路径。
--import![editor] 启动编辑器，等待资源导入完成后退出。
--export-release <预设> <路径>![editor] 使用给定预设和路径以发布模式导出项目。
--export-debug <预设> <路径>![editor] 类似上条，但使用调试模板。
--doctool [<路径>]![editor] 将引擎 API 参考导出为 XML 格式。

路径设置 (Path)​

建议将你的 Godot 编辑器二进制文件放入 PATH 环境变量中，这样你就可以在任何地方输入 godot 来运行它。

在 Windows 上，你可以使用 Scoop 轻松实现：

scoop bucket add extras
# 标准编辑器：
scoop install godot
# 带 C# 支持的编辑器（PATH 中可用名为 godot-mono）：
scoop install godot-mono

设置项目路径 (Setting the project path)​

运行编辑器或命令时，可以使用 --path 参数指定项目路径：

godot --path path_to_your_project [其他参数]

例如，从命令行导出游戏：

godot --headless --path path_to_your_project --export-release "预设名称" game.exe

运行编辑器 (Running the editor)​

使用 -e 标志运行编辑器。必须在项目目录内执行，或者指定项目路径：

godot -e

运行游戏 (Running the game)​

在项目目录内直接执行 Godot 即可运行游戏：

godot

如果想测试特定场景，将其路径传给命令行：

godot scene.tscn

导出 (Exporting)​

从命令行导出项目对持续集成 (CI) 非常有用。在没有 GPU 访问的平台上（如 CI 服务器），必须 使用 --headless 参数。

godot --headless --export-release "Linux/X11" /var/builds/project

运行脚本 (Running a script)​

你可以从命令行运行 .gd 脚本。脚本必须继承自 SceneTree 或 MainLoop。

示例 sayhello.gd：

#!/usr/bin/env -S godot -s
extends SceneTree

func _init():
    print("Hello!")
    quit()

运行方式：

godot -s sayhello.gd
