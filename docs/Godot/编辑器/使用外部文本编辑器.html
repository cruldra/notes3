使用外部文本编辑器

本页介绍如何使用外部文本编辑器进行编码。

注意

要在外部编辑器中编写 C# 代码，请参阅 配置外部编辑器 (C# 指南)。

Godot 可以配合外部文本编辑器使用，例如 Sublime Text 或 Visual Studio Code。请浏览至相关的编辑器设置：编辑器 > 编辑器设置 > 文本编辑器 > 外部 (Editor > Editor Settings > Text Editor > External)。

这里有两个文本框：执行路径 (Exec Path) 和 执行标志 (Exec Flags)。标志允许你将编辑器与 Godot 集成，向其传递要打开的文件路径和其他相关参数。Godot 会将标志字符串中的以下占位符进行替换：

执行标志中的字段替换内容
{project}项目目录的绝对路径
{file}文件的绝对路径
{col}错误的列号
{line}错误的行号

各种编辑器的 执行标志 (Exec Flags) 示例：

编辑器执行标志
Geany/Kate{file} --line {line} --column {col}
Atom{file}:{line}
JetBrains Rider{project} --line {line} {file}
Visual Studio Code{project} --goto {file}:{line}:{col}
Vim (gVim)"+call cursor({line}, {col})" {file}
Emacsemacs +{line}:{col} {file}
Sublime Text/Zed{project} {file}:{line}:{col}
Visual Studio*/edit "{file}"

*: 参数无法自动检测，必须手动填写。

自 Godot 4.5 起，系统会自动检测上述所有编辑器的 执行标志（标星号的除外）。除非你的编辑器执行文件名未被自动识别，否则你不需要从本页粘贴这些标志。

注意

对于 Windows 上的 Visual Studio Code，你需要指向 code.cmd 文件。

对于 Emacs，如果你使用服务器模式，可以调用 emacsclient 而非 emacs。

对于 Visual Studio，你需要手动打开解决方案文件 .sln 才能使用 IDE 功能。此外，它不会自动跳转到特定行。

自动重新加载更改 (Automatically reloading your changes)​

要让 Godot 编辑器自动重新加载由外部编辑器修改的任何脚本，请开启 编辑器 > 编辑器设置 > 文本编辑器 > 行为 > 外部修改时自动重新加载脚本 (Editor > Editor Settings > Text Editor > Behavior > Auto Reload Scripts on External Change)。

在调试器中使用外部编辑器 (Using External Editor in Debugger)​

在调试器中使用外部编辑器是由设置中的另一个独立选项决定的。详见 脚本编辑器调试工具和选项。

官方编辑器插件 (Official editor plugins)​

我们为以下编辑器提供了官方插件：

  • Visual Studio Code
  • Emacs

LSP/DAP 支持 (LSP/DAP support)​

Godot 支持用于代码补全的 LSP (Language Server Protocol) 和用于调试的 DAP (Debug Adapter Protocol)。如果你的编辑器支持这些协议，你就可以在不安装自定义插件的情况下使用这些功能。

要使用这些协议，必须有一个 Godot 实例正在运行当前项目。然后配置你的编辑器与 Godot 中运行的适配器端口通信。默认情况下，LSP 端口为 6005，DAP 端口为 6006。你可以在 编辑器设置 的 Network > Language Server 和 Network > Debug Adapter 部分更改这些端口。

Visual Studio Code​

你需要安装官方的 Visual Studio Code 插件。

对于 LSP，连接状态可以在状态栏查看：

对于 DAP，在你的 launch.json 文件中指定 debugServer 属性：

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "GDScript Godot",
            "type": "godot",
            "request": "launch",
            "project": "${workspaceFolder}",
            "port": 6007,
            "debugServer": 6006
        }
    ]
}
