## 简介 (Introduction)

Godot 引擎的脚本编辑器是一个功能强大且完全集成的文本编辑器。它不仅简化了编写和调试 GDScript 代码的过程，还支持处理纯文本文件，为开发者编写游戏逻辑和行为提供了一个无缝的环境。它可以高亮显示代码、自动缩进、执行语法检查等。你还可以创建断点来调试项目，而无需切换到另一个窗口。此外，脚本编辑器还充当了离线类参考浏览器。

![脚本编辑器界面概览](https://docs.godotengine.org/en/stable/_images/text_editor.webp)

## 特性 (Features)

脚本编辑器的一些核心特性包括：

*   完全集成的 GDScript 代码编辑器。
*   支持 GDScript 和 JSON 文件的语法高亮。
*   支持 GDScript 和 JSON 文件的语法检查。
*   支持书签和断点。
*   自动缩进。
*   代码折叠。
*   可自定义的主题。
*   多光标编辑（按住 Alt + 左键点击启用）。
*   变量、函数、常量等的自动补全。
*   通过选中符号并按 Ctrl + D 进行行内重构。
*   在项目文件中进行大规模查找和替换。

## 使用方法 (Usage)

如果你的项目使用 GDScript，Godot 的内置脚本编辑器提供了你所需的一切。几乎所有可以在 UI 界面调整的参数，也都可以直接通过代码进行修改。

> **注意**
>
> 如果你想使用外部文本编辑器或在项目中使用 C#，请参阅 [使用外部文本编辑器](https://docs.godotengine.org/en/stable/tutorials/editor/external_editor.html) 和 [配置外部编辑器](https://docs.godotengine.org/en/stable/tutorials/scripting/c_sharp/c_sharp_basics.html)。

> **提示**
>
> 脚本编辑器也可以通过 **编辑器 > 编辑器设置 > 文本编辑器 (Editor > Editor Settings > Text Editor)** 进行深度自定义。

![打开脚本编辑器](https://docs.godotengine.org/en/stable/_images/editor_ui_script_editor_open.webp)

你可以通过界面顶部中央的工作区选择器中的 **Script** 按钮打开脚本编辑器。此外，也可以点击场景树面板中节点旁的 **打开脚本** 按钮，或者在文件系统面板中双击 `.gd` 文件或已识别的文本文件来直接打开。

![脚本编辑器菜单](https://docs.godotengine.org/en/stable/_images/editor_ui_script_editor_menu.webp)

打开后，你会看到顶部的菜单栏。菜单旁边有打开在线文档或搜索内置类参考的按钮。右侧的两个导航箭头允许你在查看历史记录中往返。最后，你可以使用“浮动 (float)”按钮将脚本编辑器从 Godot 窗口中分离，这在多显示器环境下非常有用。

左侧是 **脚本面板 (Script Panel)**，中间是 **编码区域 (Coding area)**。编码区域下方是 **状态栏 (status bar)**，显示代码中的错误和警告数量。点击图标会列出错误及行号，点击某一项可跳转。你也可以在列表中点击 `Ignore` 来忽略特定的警告。状态栏还允许你通过百分比值或 Ctrl + 滚轮更改缩放级别，并显示当前光标的行列位置以及缩进方式（Tab 或空格）。

### 脚本面板 (Script Panel)

![脚本面板详情](https://docs.godotengine.org/en/stable/_images/editor_ui_script_editor_script_panel.webp)

左侧面板列出了已打开的文件和文档页。不同文件会有不同的图标：
*   ![script](https://docs.godotengine.org/en/stable/_images/Script.webp) 表示 GDScript。
*   ![scriptcsharp](https://docs.godotengine.org/en/stable/_images/ScriptCSharp.webp) 表示 C# 脚本。
*   ![documentation](https://docs.godotengine.org/en/stable/_images/Documentation.webp) 表示内置类参考。
*   ![toolscript](https://docs.godotengine.org/en/stable/_images/ToolScript.webp) 表示正在运行的工具脚本（参见 [tool 注解](https://docs.godotengine.org/en/stable/tutorials/plugins/running_code_in_the_editor.html)）。

在状态栏点击向左箭头可隐藏此面板，向右箭头可再次显示。文件名上的星号 (*) 表示该文件有未保存的更改。

你可以拖动文件来排序。右键点击文件提供保存、关闭、复制相对路径等选项。

面板下方显示当前文件的名称，旁边的按钮可切换方法列表的排序方式（按字母或按出现顺序）。下方是文件的 **大纲 (Outline)**，点击可跳转到相应的方法或章节。

### 菜单 (Menus)

![脚本编辑器菜单概览](https://docs.godotengine.org/en/stable/_images/text_editor_menu.webp)

> **提示**
>
> 带有星号 (*) 的操作也可以通过在编辑器中右键点击弹出的上下文菜单找到。

**文件 (File) 菜单：**
*   **新建脚本...**：创建并添加脚本到项目。
*   **保存/全部保存**：保存当前或所有已打开的脚本。
*   **运行**：如果脚本扩展自 `EditorScript`，可以在不运行项目的情况下运行该脚本。
*   **在文件系统中显示**：在文件系统面板中高亮定位当前文件。

**编辑 (Edit) 菜单：**
*   **撤销/重做、剪切/复制/粘贴**。
*   **评估选中项**：如果选中项是数学表达式（如 `pow(2,3)`），则计算其结果。
*   **切换行注释**：对选中行进行注释或取消注释。
*   **折叠/展开**：折叠或展开代码块或代码区域（Region）。
*   **创建代码区域 (Code Region)**：将选中代码包裹在可折叠的区域内。

**搜索 (Search) 菜单：**
*   **查找/替换**：在当前文件中搜索。
*   **在文件中查找/替换...**：在整个项目文件夹的文件中进行全局搜索和替换。**注意：“在文件中替换”操作无法撤销！**

**转到 (Go To) 菜单：**
*   **转到函数... / 转到行...**：快速跳转。
*   **书签**：添加蓝色的书签符号，便于后续快速跳转。
*   **断点**：用于调试。你可以点击行号左侧的空白区域来添加或移除红色圆圈形状的断点。

### 编码区域 (Coding area)

![编码区域详情](https://docs.godotengine.org/en/stable/_images/editor_ui_script_editor_coding_area.webp)

这是你编写脚本的地方。左侧显示行号，右侧导航箭头下方有一个可点击的 **小地图 (minimap)**，提供整个脚本的鸟瞰图。

脚本中函数定义左侧可能会有额外图标：
*   ![override](https://docs.godotengine.org/en/stable/_images/override.webp) 表示该函数是现有函数的 **重写 (override)**。
*   ![receiver](https://docs.godotengine.org/en/stable/_images/receiver.webp) 表示这是一个 **信号接收方法**。点击可查看信号来源。
*   ![foldable](https://docs.godotengine.org/en/stable/_images/Foldable.webp) 表示可折叠的代码块。

![编码区域标识符说明](https://docs.godotengine.org/en/stable/_images/text_editor_coding_area_indicators.webp)
*(图中：52/56/58行为可折叠块；57行为代码区域；62行为已折叠块；53行为书签；55行为断点。)*
