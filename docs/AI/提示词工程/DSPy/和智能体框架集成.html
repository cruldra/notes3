如何将 DSPy 训练模型集成到 CrewAI/Agno 智能体框架

本文档详细介绍了在获得满意的 DSPy 训练结果（即 litreview_dspy_model.json 权重文件）后，如何在 CrewAI 或 Agno 等智能体框架中使用该模型。

我们提供了两种主要的集成策略，强烈推荐方案一。

方案一：封装为自定义工具 (Custom Tool) - 🔥 推荐​

这是最符合软件工程原则的方案。我们将 DSPy 模块封装为一个独立的工具（Tool），Agent 像调用普通函数一样调用它。这种方式实现了模块解耦，且能充分利用 DSPy 的推理能力。

1. 实现原理​

CrewAI 的 Agent 不直接包含 DSPy 的权重，而是拥有一个“工具”，该工具内部加载 JSON 权重文件并执行推理。

2. 代码实现​

在项目中创建 tools/litreview_tool.py：

from crewai.tools import BaseTool
from pydantic import Field
import dspy
import os

# --- DSPy 定义部分 (必须与训练脚本 train_litreview_dspy.py 保持完全一致) ---

class GenerateLitReview(dspy.Signature):
    """
    根据提供的【相关文献摘要列表】或【论文摘要】，撰写一份高质量的中文文献综述。
    
    要求：
    1. 如果输入包含多篇文献摘要，请综合归纳这些文献的观点、方法和结论。
    2. 结构应包含：核心概念定义、主要理论基础、研究现状（分类综述）、现有研究的不足与趋势。
    3. 语言学术规范，逻辑严密。
    """
    abstract = dspy.InputField(desc="文本内容，可以是单篇论文的摘要，也可以是多篇参考文献摘要的集合列表。")
    literature_review = dspy.OutputField(desc="生成的中文文献综述章节。")

class LitReviewModule(dspy.Module):
    def __init__(self):
        super().__init__()
        self.generate = dspy.ChainOfThought(GenerateLitReview)
    
    def forward(self, abstract):
        return self.generate(abstract=abstract)

# --- CrewAI 工具定义部分 ---

class LiteratureReviewTool(BaseTool):
    name: str = "Literature Review Generator"
    description: str = (
        "专门用于生成学术文献综述的工具。输入可以是单篇摘要，也可以是多篇摘要的列表文本。"
        "该工具内部加载了经过优化的 DSPy 模型，能生成结构严谨的中文综述。"
    )
    model_path: str = Field(..., description="DSPy JSON 模型文件的相对或绝对路径")

    def _run(self, abstract_content: str) -> str:
        try:
            # 1. 环境与配置检查
            # 建议在项目入口处统一配置，这里做防御性编程
            from dotenv import load_dotenv
            load_dotenv()
            
            if not dspy.settings.lm:
                # 如果没有全局配置，则初始化一个
                lm = dspy.LM(
                    model="deepseek/deepseek-chat", # 或从环境变量读取
                    api_key=os.getenv("DEEPSEEK_API_KEY"),
                    api_base=os.getenv("DEEPSEEK_BASE_URL"),
                    max_tokens=4000
                )
                dspy.configure(lm=lm)
            
            # 2. 实例化并加载权重
            module = LitReviewModule()
            
            if os.path.exists(self.model_path):
                # 关键步骤：加载训练好的权重
                module.load(self.model_path)
            else:
                return f"Error: DSPy model file not found at {self.model_path}"
            
            # 3. 执行推理
            pred = module(abstract=abstract_content)
            
            return pred.literature_review
            
        except Exception as e:
            return f"Error generating review using DSPy tool: {str(e)}"

3. 在 CrewAI 中调用​

在你的 main.py 或 crew.py 中：

from crewai import Agent, Task, Crew
from tools.litreview_tool import LiteratureReviewTool

# 1. 实例化工具
lit_tool = LiteratureReviewTool(
    model_path="scripts/litreview_dspy_model.json" 
)

# 2. 创建 Agent
researcher = Agent(
    role='Academic Researcher',
    goal='撰写高质量的文献综述',
    backstory='你是一名资深的学术研究员，擅长从大量文献中提炼观点。',
    tools=[lit_tool],  # 将工具赋予 Agent
    verbose=True
)

# 3. 创建 Task
task = Task(
    description='请阅读以下文献摘要列表：\n{abstracts}\n\n使用你的文献综述生成工具，写一篇完整的综述。',
    expected_output='一篇结构完整的中文文献综述',
    agent=researcher
)

# 4. 运行
crew = Crew(agents=[researcher], tasks=[task])
result = crew.kickoff(inputs={'abstracts': '...这里传入你的摘要文本...'

方案二：提取 Prompt (硬编码) - ⚡ 仅限轻量级使用​

如果你不想在生产环境中依赖 dspy 库，或者你需要将 Prompt 迁移到非 Python 环境，可以使用此方法。

原理：DSPy 的产物 litreview_dspy_model.json 本质上包含了一组经过筛选的 Few-Shot Examples（包含推理过程 CoT）。我们可以把这些 Prompt 提取出来，直接作为 Agent 的 backstory 或 system_prompt。

1. 提取 Prompt​

创建一个临时脚本 scripts/inspect_prompt.py：

import dspy
from scripts.train_litreview_dspy import setup_dspy, LitReviewModule

lm = setup_dspy()
module = LitReviewModule()
module.load("scripts/litreview_dspy_model.json")

# 必须运行一次虚拟推理才能构建完整的 Prompt
module(abstract="test input")

# 获取最后一次发送给 LLM 的完整 Prompt
last_prompt = lm.history[-1]
print("=== 把下面的内容复制出来 ===")
if 'messages' in last_prompt:
    for msg in last_prompt['messages']:
        print(f"[{msg['role']}]: {msg['content']}")
else:
    print(last_prompt['prompt'])

2. 硬编码到 Agent​

将打印出来的 Prompt（通常包含 --- Example 1 ---, Reasoning: ... 等内容）整理后，放入 Agent 定义中：

agent = Agent(
    role="Review Writer",
    backstory="""你是一个学术写作专家。
    在撰写文献综述时，请严格参考以下思维过程和范例：
    
    [这里粘贴你从 DSPy 提取出来的 Few-Shot Examples]
    
    现在，请为新的输入撰写综述。
    """
)

总结​

特性方案一 (自定义工具)方案二 (提取 Prompt)
集成难度低 (直接复用代码)中 (需要手动提取和拼接字符串)
维护性高 (重新训练只需替换 json 文件)低 (需手动更新 Prompt 字符串)
依赖性需要 dspy 库无需 dspy 库
推理一致性完全一致 (复用 DSPy 逻辑)可能偏差 (完全依赖 LLM 理解长文本)
适用场景生产环境、复杂流程临时测试、跨语言环境

建议： 在 thesis_crew 项目中，请优先使用 方案一。
