简介

一项标准化提案，通过在网站上使用 /llms.txt 文件来帮助大语言模型（LLM）在推理时获取网站信息。

背景​

大语言模型越来越依赖网站信息，但面临一个关键限制：上下文窗口太小，无法完整处理大多数网站。将包含导航、广告和 JavaScript 的复杂 HTML 页面转换为 LLM 友好的纯文本既困难又不精确。

虽然网站同时服务于人类读者和 LLM，但后者更受益于在单一可访问位置收集的更简洁、专家级的信息。这对于开发环境等用例尤其重要，因为 LLM 需要快速访问编程文档和 API。

提案​

我们提议在网站上添加 /llms.txt Markdown 文件，以提供 LLM 友好的内容。该文件提供简要的背景信息、指导以及指向详细 Markdown 文件的链接。

llms.txt Markdown 既可被人类阅读，也可被 LLM 阅读，同时采用精确的格式，允许使用固定的处理方法（即解析器和正则表达式等经典编程技术）进行处理。

我们还提议，网站上包含可能对 LLM 有用信息的页面，应在与原始页面相同的 URL 提供该页面的纯净 Markdown 版本，只需在 URL 后追加 .md 即可。（没有文件名的 URL 应追加 index.html.md。）

FastHTML 项目的文档遵循了这两项提案。例如，这是 FastHTML 文档的 llms.txt。这是一个普通 HTML 文档页面的示例，以及完全相同的 URL 但带有 .md 扩展名的版本。

本提案不包含关于如何处理 llms.txt 文件的任何特定建议，因为这取决于具体应用场景。例如，FastHTML 项目选择使用基于 XML 的结构（适用于 Claude 等 LLM）自动将 llms.txt 扩展为两个包含链接 URL 内容的 Markdown 文件。这两个文件分别是：llms-ctx.txt（不包含可选 URL）和 llms-ctx-full.txt（包含可选 URL）。它们是使用 llms_txt2ctx 命令行应用程序创建的，FastHTML 文档中包含了用户如何使用它们的信息。

llms.txt 文件的多功能性意味着它们可以服务于多种目的——从帮助开发者浏览软件文档，到让企业概述其结构，甚至为利益相关者分解复杂的法规。它们对于个人网站同样有用，可以帮助回答有关某人简历的问题；对于电商网站，可以解释产品和政策；对于学校和大学，可以快速访问课程信息和资源。

注意，所有 nbdev 项目现在默认创建所有页面的 .md 版本。所有使用 nbdev 的 Answer.AI 和 fast.ai 软件项目都已使用此功能重新生成了文档。例如，请参阅 fastcore 的 docments 模块的 Markdown 版本。

格式​

目前，语言模型最广泛且最容易理解的格式是 Markdown。简单地展示关键 Markdown 文件的位置是很好的第一步。提供一些基本结构可以帮助语言模型找到所需信息的来源。

llms.txt 文件的独特之处在于它使用 Markdown 来组织信息，而不是 XML 等经典结构化格式。原因是我们预期这些文件中的许多将被语言模型和代理读取。话虽如此，llms.txt 中的信息遵循特定格式，可以使用标准的基于程序的工具进行读取。

llms.txt 文件规范适用于位于网站根路径 /llms.txt（或可选的子路径）的文件。遵循该规范的文件按特定顺序包含以下 Markdown 部分：

  • H1 标题，包含项目或网站名称。这是唯一必需的部分
  • 引用块，包含项目的简短摘要，包含理解文件其余部分所需的关键信息
  • 零个或多个 Markdown 部分（如段落、列表等），除标题外的任何类型，包含有关项目的更详细信息以及如何解释提供的文件
  • 零个或多个由 H2 标题分隔的 Markdown 部分，包含可获取更多详细信息的 URL "文件列表"
    • 每个"文件列表"是一个 Markdown 列表，包含必需的 Markdown 超链接 [名称](url)，然后可选地加上 : 和关于文件的说明。

以下是一个模拟示例：

# 标题

> 可选描述放在这里

可选详情放在这里

## 部分名称

- [链接标题](https://link_url): 可选链接详情

## Optional

- [链接标题](https://link_url)

注意，"Optional"部分具有特殊含义——如果包含它，当需要更短的上下文时，可以跳过其中提供的 URL。将其用于通常可以跳过的次要信息。

现有标准​

llms.txt 旨在与当前的 Web 标准共存。虽然站点地图（sitemap）为搜索引擎列出所有页面，但 llms.txt 为 LLM 提供精选概览。它可以通过为允许的内容提供上下文来补充 robots.txt。该文件还可以引用网站上使用的结构化数据标记，帮助 LLM 在上下文中理解如何解释这些信息。

为文件标准化路径的方法遵循了 /robots.txt 和 /sitemap.xml 的做法。robots.txt 和 llms.txt 有不同的目的——robots.txt 通常用于让自动化工具知道对网站的什么访问被认为是可接受的，例如搜索索引机器人。另一方面，llms.txt 信息通常会在用户明确请求有关某个主题的信息时按需使用，例如在项目中包含编码库的文档时，或在向具有搜索功能的聊天机器人询问信息时。我们预期 llms.txt 主要用于推理（inference），即用户寻求帮助时，而不是用于训练（training）。然而，如果 llms.txt 的使用变得广泛，未来的训练运行也可能利用 llms.txt 文件中的信息。

sitemap.xml 是网站上所有可索引的人类可读信息的列表。这不能替代 llms.txt，因为它：

  • 通常不会列出 LLM 可读版本的页面
  • 不包含指向外部网站的 URL，即使这些链接可能有助于理解信息
  • 通常涵盖的文档总量太大，无法放入 LLM 上下文窗口，并且会包含许多理解网站所不必要的信息。

示例​

以下是 llms.txt 的示例，这是用于 FastHTML 项目的文件的精简版本（另请参阅完整版本）：

# FastHTML

> FastHTML 是一个 Python 库，它将 Starlette、Uvicorn、HTMX 和 fastcore 的 `FT`"FastTags"整合到一个用于创建服务器渲染超媒体应用程序的库中。

重要说明：

- 尽管其 API 的部分内容受 FastAPI 启发，但它与 FastAPI 语法**不兼容**，也不针对创建 API 服务
- FastHTML 与 JS 原生 Web 组件和任何原生 JS 库兼容，但不与 React、Vue 或 Svelte 兼容。

## Docs

- [FastHTML 快速入门](https://fastht.ml/docs/tutorials/quickstart_for_web_devs.html.md): 许多 FastHTML 功能的简要概述
- [HTMX 参考](https://github.com/bigskysoftware/htmx/blob/master/www/content/reference.md): 所有 HTMX 属性、CSS 类、头信息、事件、扩展、js 库方法和配置选项的简要描述

## Examples

- [待办事项列表应用](https://github.com/AnswerDotAI/fasthtml/blob/main/examples/adv_app.py): 详细演练 FastHTML 中完整的 CRUD 应用程序，展示 FastHTML 和 HTMX 模式的惯用用法。

## Optional

- [Starlette 完整文档](https://gist.githubusercontent.com/jph00/809e4a4808d4510be0e3dc9565e9cbd3/raw/9b717589ca44cedc8aaf00b2b8cacef922964c0f/starlette-sml.md): 对 FastHTML 开发有用的 Starlette 文档子集。

要创建有效的 llms.txt 文件，请考虑以下指南：

  • 使用简洁、清晰的语言。
  • 链接到资源时，包含简短、信息丰富的描述。
  • 避免模棱两可的术语或未解释的行话。
  • 运行一个将 llms.txt 文件扩展为 LLM 上下文文件的工具，并测试多个语言模型，看它们是否能回答有关你内容的问题。

集成​

各种工具和插件可帮助将 llms.txt 规范集成到您的工作流程中：

  • llms_txt2ctx - 用于解析 llms.txt 文件和生成 LLM 上下文的 CLI 和 Python 模块
  • JavaScript 实现 - 示例 JavaScript 实现
  • vitepress-plugin-llms - VitePress 插件，按照 llms.txt 规范自动生成 LLM 友好的网站文档
  • docusaurus-plugin-llms - Docusaurus 插件，按照 llmtxt.org 标准生成 LLM 友好的文档
  • Drupal LLM Support - Drupal Recipe，为任何 Drupal 10.3+ 网站提供对 llms.txt 提案的完整支持
  • llms-txt-php - 用于编写和读取 llms.txt Markdown 文件的库
  • VS Code PagePilot Extension - PagePilot 是一个 VS Code Chat 参与者，可自动加载外部上下文（文档、API、README 文件）以提供增强的响应。

后续步骤​

llms.txt 规范对社区输入开放。一个 GitHub 仓库托管了这份非正式概述，允许版本控制和公开讨论。一个社区 Discord 频道可用于分享实施经验和讨论最佳实践。
