提示词

我们使用description和instructions以及其他一些设置来提示代理。这些设置用于构建发送给语言模型的系统消息。

了解这些提示词是如何创建的将帮助您构建更好的代理。

两个关键参数是：

  1. 描述（Description）：指导代理整体行为的描述。
  2. 指令（Instructions）：关于如何实现其目标的精确、特定任务的指令列表。

描述和指令只提供格式化好处，我们不会改变或抽象任何信息，您始终可以设置system_message来提供自己的系统提示。

系统消息​

系统消息使用description、instructions和其他一些设置创建。description被添加到系统消息的开头，instructions作为列表添加在Instructions之后。例如：

from agno.agent import Agent

agent = Agent(
    description="你是一位受邀为杂志撰写的著名短篇小说作家",
    instructions=["你是一名从夏威夷飞往日本的飞机上的飞行员。"],
    markdown=True,
    debug_mode=True,
)
agent.print_response("给我讲一个两句话的恐怖故事。", stream=True)

将转换为（设置debug_mode=True查看日志）：

直接设置系统消息​

您可以使用system_message参数手动设置系统消息。

from agno.agent import Agent

agent = Agent(system_message="分享一个关于以下内容的两句话故事")
agent.print_response("12000年的爱情。")

一些模型通过某些模型提供商，如Groq上的llama-3.2-11b-vision-preview，需要没有系统消息的其他消息。要删除系统消息，请设置create_default_system_message=False和system_message=None。此外，如果设置了markdown=True，它将添加一个系统消息，所以要么删除它，要么明确禁用系统消息。

用户消息​

发送给Agent.run()或Agent.print_response()函数的输入message被用作用户消息。

默认系统消息​

代理创建一个可以使用以下参数自定义的默认系统消息：

参数类型默认值描述
descriptionstrNone代理的描述，添加到系统消息的开头。
goalstrNone描述代理应该实现的任务。
instructionsList[str]None在系统提示中添加到<instructions>标签中的指令列表。根据markdown、output_model等值也会创建默认指令。
additional_contextstrNone添加到系统消息末尾的额外上下文。
expected_outputstrNone提供代理的预期输出。这会添加到系统消息的末尾。
markdownboolFalse添加一条使用markdown格式化输出的指令。
add_datetime_to_instructionsboolFalse如果为True，将当前日期时间添加到提示中，使代理有时间感。这允许在提示中使用相对时间，如"明天"。
system_messagestrNone系统提示：提供系统提示作为字符串。
system_message_rolestrsystem系统消息的角色。
create_default_system_messageboolTrue如果为True，使用代理设置构建默认系统提示并使用它。

通过设置create_default_system_message=False禁用默认系统消息。

默认用户消息​

代理创建一个默认用户消息，它要么是输入消息，要么是带有context的消息（如果enable_rag=True）。默认用户消息可以使用以下参数自定义：

参数类型默认值描述
contextstrNone添加到用户消息末尾的额外上下文。
add_contextboolFalse如果为True，将上下文添加到用户提示中。
resolve_contextboolTrue如果为True，在将上下文添加到用户提示之前解析上下文（即调用上下文中的任何函数）。
add_referencesboolFalse通过将知识库中的引用添加到提示中来启用RAG。
retrieverCallableNone获取要添加到user_message的引用的函数。当add_references为True时，如果提供了此函数，则会调用它。
references_formatLiteral["json", "yaml"]"json"引用的格式。
add_history_to_messagesboolFalse如果为True，将聊天历史添加到发送给模型的消息中。
num_history_responsesint3添加到消息中的历史响应数量。
user_messageUnion[List, Dict, str]None提供用户提示作为字符串。注意：这将忽略发送给run函数的消息。
user_message_rolestruser用户消息的角色。
create_default_user_messageboolTrue如果为True，使用引用和聊天历史构建默认用户提示。

通过设置create_default_user_message=False禁用默认用户消息。
