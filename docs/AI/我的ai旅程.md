# 我的AI旅程

## 第一部分：回顾我与AI的历史

### 2022年：起点

**2022年底，OpenAI的ChatGPT横空出世。**

当时我在合肥居家办公，为北京一家公司开发投资风控系统。说实话，我已经不太记得第一次使用ChatGPT时的具体感受了，但可以确定的是，从那一刻起，AI正式进入了我的工作和生活。

最开始是在工作中使用。我越来越频繁地向ChatGPT询问编程相关的知识：
- 把它当谷歌用，搜索开发时遇到的技术问题
- 把它当成某个库的官方文档，询问想要的API及代码示例
- 让它解释复杂的代码逻辑

### GitHub Copilot：结对编程的开端

**其实在ChatGPT之前，GitHub Copilot已经在2021年6月29日发布了技术预览版，并在2022年6月29日正式面向所有开发者开放。**

所以时间线是这样的：
- **2021年6月**：Copilot技术预览
- **2022年6月**：Copilot正式发布  
- **2022年11月**：ChatGPT发布

Copilot以VS Code、JetBrains IDE等产品的插件形式，更深入地参与到我的编程工作中。它能在我写代码时提供基于AI的代码补全，这和之前IDE内置的补全体验完全不同：

| 传统IDE补全 | GitHub Copilot |
|------------|---------------|
| 基于已安装的库 | 基于AI理解代码上下文 |
| 白盒，API明确 | 黑盒，猜测意图 |
| 确定性建议 | 创造性建议 |

Copilot会根据我光标前后的代码，猜测我下一步想要实现的功能并提供相应补全。这在当时来看非常惊艳。

这个概念当时被称为**"结对编程"（Pair Programming）**。但总的来说，开发人员仍然需要：
- 设计系统架构
- 编写核心业务逻辑
- 编写测试代码

AI在此时提供的作用远不如现在大，甚至经常出现：
- ❌ 胡乱建议
- ❌ 中文乱码
- ❌ 生成无效代码

当时知乎上还有专门针对"结对编程"概念的梗图，吐槽AI助手有时候比新手还让人头疼。

### 2022年：Stable Diffusion的惊喜

**Stable Diffusion在2022年8月22日由Stability AI公开发布**，这在当时也是一个重磅事件。和DALL-E 2不同，SD是开源的，可以在本地运行。我也在本机安装了Stable Diffusion WebUI和1.5版本的模型来生成图片，第一次感受到AI创作的乐趣。

### 2023-2024年：AI编程工具进化

**2023年3月，由MIT四名学生创立的Anysphere公司发布了Cursor**——一个基于VS Code的AI编程IDE。虽然初期知名度不高，但这已经标志着AI编程工具开始从"代码补全"向"代码生成"转变。

到了**2024年**，AI编程工具真正开始百花齐放：
- **Cursor** 开始大规模流行，功能也越来越强大
- **Windsurf** 等新兴工具出现
- 相比于2022-2023年的GitHub Copilot，这些工具的功能丰富得多

**核心变化**：它们不再局限于给出建议，而是开始可以代替程序员编写独立的功能模块了。

### 2024年：Augment Code登场

**2024年4月**，Augment Code公司成立，由前微软软件工程师Igor Ostrovsky和前谷歌AI研究员Guy Gur-Ari联合创立。

**2024年10月24日**，Augment Code正式发布，定位为"面向团队的开发者AI"。

它最让我印象深刻的特点是**长上下文（Long Context）**能力：
- 能在多达10万行代码的大型代码仓库中保持高智商
- 理解整个项目的上下文，而不需要像Cursor那样使用`@`提及一大堆文件
- 你只需告诉它想实现的功能，它会理解整个项目并在合适的地方编写代码

这解决了Cursor的一个痛点：在Cursor中如果你不主动提及文件，它可能会"乱写"，甚至连你的`package.json`都不看一眼。

### 2025年：爆发与收紧

**2025年感觉像是AI应用增长最快的一年。**

#### 🔥 开年：DeepSeek平地一声雷

**2025年1月20日**，DeepSeek发布了R1推理模型，以极低的训练成本和开源策略震惊业界：
- 性能媲美OpenAI的o1模型
- 完全开源，MIT许可证
- 训练成本仅为557万美元（相比之下，GPT-4等模型的训练成本高达数亿美元）

一时间，抖音、B站等平台上到处都是DeepSeek相关的视频，"中国AI逆袭"的话题引发热议。

#### 🚀 大模型军备竞赛

紧接着，各家巨头争先恐后发布先进大模型：
- **GPT-4** 系列持续迭代
- **Claude 4**（Opus 4 / Sonnet 4）在2025年5月22日发布
- **Gemini** 系列不断更新

整个行业看起来一片繁荣，AI应用也越来越广泛。尤其是**生成式AI**，已经不再局限于文本和图片：
- 🎬 视频生成（Sora、可灵等）
- 🎵 音频生成（音乐、语音克隆）
- 🎮 3D模型生成
- 🧬 生物分子设计

#### 💻 Claude Code正式发布

**2025年2月**，Anthropic的Claude Code正式发布。随后在**5月1日**，通过Claude Max计划提供固定费率的无限使用，让更多开发者能够无顾虑地使用。

#### 📉 2025年底：行业开始收紧

感觉好像是从2025年底的那几个月开始，这个行业的节奏开始放缓。

首先是**AI编程IDE频繁调整定价策略**：

**Augment Code的价格过山车：**
- 最初：$30/月（社区版免费）
- 2025年初：涨到$50/月（涨幅67%）
- 2025年10月20日：再次调整为按使用量计费

有用户计算发现，新计费模式下成本可能比之前高出**10倍以上**，引发大量抗议。

**Cursor**也在2025年经历了多次定价调整。

#### 🔒 2026年初：各大巨头采取收紧策略

到了2026年初，更多迹象表明AI行业正在经历调整：

- **Claude Code**禁止第三方应用接入
- **Google Gemini**大幅降低免费配额
- **Antigravity**等工具也减少了免费额度

结合reddit上的讨论：
- OpenAI被传"入不敷出"
- 免费版ChatGPT开始添加广告

种种迹象表明，各大巨头都在采取**收紧策略**，AI行业可能正在从"烧钱抢市场"转向"追求盈利"的阶段。

## 第二部分：大模型背后的技术原理及垂直领域大模型训练的必要性

### 我的学习之路：从兴趣到实践

我对了解这种**"神奇魔法"背后的原理**非常感兴趣。但现实总是充满各种限制：

#### 🚧 客观条件的限制

**作为一家初创小微公司的合伙人**，在公司生存问题解决之前，我没办法全身心投入到大模型学习中：

| 限制类型 | 具体情况 |
|---------|---------|
| **时间** | 公司生存优先，只能利用空闲时间学习 |
| **经济条件** | 大模型对算力要求极高，个人难以承担 |
| **硬件成本** | RTX 5090需约2万元，专业算力更贵 |
| **个人经历** | 曾购买3000元的RTX 2080Ti 22G魔改版，用了不到半年就坏了 |

#### 💻 技术背景的局限

我本人从**2013年开始从事软件开发**，一直在做应用层开发：
- ✅ 接触过一些算法
- ❌ 但并不是专业的算法工程师

#### 📚 学习实践历程

尽管有种种限制，**2024-2025年**我还是利用空闲时间了解了大模型背后的技术原理：

**核心概念学习：**
- **专家系统、RNN(循环神经网络)、CNN(卷积神经网络)** —— 大模型不是一夜之间冒出来的，而是几十年AI发展的积累
- **Deep Learning（深度学习）、Machine Learning（机器学习）** —— 大模型的核心技术基础
- **Transformer架构** —— 现代大模型的基础结构
- **注意力机制（Attention）** —— 让模型学会"关注重点"
- **嵌入（Embedding）** —— 将文字转化为计算机能理解的向量
- **分词（Tokenization）** —— 文本处理的第一步
- **预训练（Pre-training）** —— 在大规模数据上学习通用知识
- **微调（Fine-tuning）** —— 针对特定任务的进一步优化

**实际动手实践：**
通过GitHub上的 **[MiniMind](https://github.com/jingyaogong/minimind)** 开源项目，我训练出了自己的第一个小型语言模型。这是一个从零开始训练的小型LLM项目，适合初学者理解大模型的完整训练流程。

> 🤦‍♂️ **真实 confession**：时至今日，用于训练出这个模型的源代码有**90%我都看不懂**（手动狗头）

### 垂直领域大模型的必要性

#### 🏔️ 通用大模型的门槛

正如第一部分所述，通用大模型虽然能力强、适用性广，但同样伴随着极高的门槛：

| 资源需求 | 具体要求 |
|---------|---------|
| **算力** | 需要数千张顶级GPU训练数月 |
| **电力** | 训练一次消耗相当于数百户家庭一年的用电量 |
| **基础设施** | 需要专业的数据中心、冷却系统 |
| **资金** | 训练成本动辄数千万甚至数亿美元 |

**个人和小微企业很难承担得起。**

#### 🎯 垂直领域大模型的优势

因此，我觉得在可预见的未来，**垂直领域大模型训练是非常有必要的**：

1. **专业化优化**
   - 针对特定行业（医疗、法律、金融等）或应用场景深度优化
   - 提供更专业、更精准的解决方案

2. **降低成本**
   - 模型规模可以更小，算力需求大幅降低
   - 训练数据更少，成本可控

3. **提升效率**
   - 在特定领域性能可能超越通用大模型
   - 响应速度更快，部署更灵活

4. **实际价值**
   - 让人工智能低成本地服务于更多细分场景
   - 实打实地带来生产力的提升

#### 🚀 小微企业的生存路径

基于此，我认为：

> **小微企业，特别是偏技术类的公司，在未来很长一段时间内，在各大巨头未能完全覆盖的细分领域，通过训练、微调垂直领域大模型来提供定制化的AI解决方案，是为数不多的能够在AI时代生存下去的路径之一。**

### 总结与展望

所以总结起来我的策略是：

```
大模型虽然难，但还得咬牙学
    ↓
先理解底层原理
    ↓
利用好开源生态
    ↓
最终目标：拥有能够针对特定业务场景训练专业大模型的能力
```

**关键行动点：**
- 📖 **持续学习**：理解Transformer、注意力机制等核心原理
- 🛠️ **动手实践**：通过MiniMind等开源项目积累经验
- 🌐 **关注开源**：利用Hugging Face、GitHub等生态资源
- 🎯 **找准定位**：聚焦垂直领域，避开与巨头的正面竞争
- 💼 **业务结合**：将AI能力与实际业务场景深度结合

**最终的愿景**：让每一个小微企业都能用得起、用得好AI，在细分领域创造真正的价值。

## 第三部分：AI对教育、医疗、游戏、IT行业的变革

基于我对AI领域的观察和亲身经历，以下是我认为AI可能带来颠覆性变革的四个行业分析。

### 一、IT行业：从"手工编码"到"氛围编程"

> **这是目前AI影响最深，也是我个人经历最直接的行业。**

#### 📊 现状：AI已深入软件开发全生命周期

AI编程工具的进化速度远超想象，已经从最初的**代码补全**发展到**代码生成**，再到今天的**全流程参与**：

**软件开发各环节AI渗透情况：**

| 开发阶段 | AI应用场景 | 代表工具 |
|---------|-----------|---------|
| **需求分析** | 自然语言转需求文档、用户故事生成 | ChatGPT、Claude |
| **系统设计** | 架构设计建议、数据库建模、API设计 | GitHub Copilot、Cursor |
| **代码编写** | 代码生成、重构、补全、解释 | Cursor、Windsurf、Augment Code |
| **代码审查** | Bug检测、安全漏洞扫描、性能优化建议 | GitHub Copilot Chat、CodeRabbit |
| **测试** | 自动化测试用例生成、测试脚本编写 | Codium、Testim |
| **部署运维** | CI/CD配置、Dockerfile生成、日志分析 | GitHub Actions、K8s GPT |

**从"结对编程"到"氛围编程"（Vibe Coding）：**
- **2021-2022年**：GitHub Copilot提供代码补全，开发者 still 主导
- **2023-2024年**：Cursor、Windsurf实现代码生成，AI开始编写完整功能模块
- **2025年**："氛围编程"兴起——开发者描述意图，AI完成绝大部分编码工作

> **今天的现实**：一个完全不懂编程的人，通过 Google Gemini Canvas 或 v0.dev，可以在3分钟内创建一个漂亮的网页。

#### 🏆 成功案例与工具演进

| 工具/平台 | 核心能力 | 标志性突破 |
|----------|---------|-----------|
| **GitHub Copilot** | 代码补全、生成 | 首个大规模商用的AI编程助手 |
| **Cursor** | 代码生成、重构、理解项目 | 基于VS Code的AI原生IDE |
| **Windsurf** | 协作编码、实时生成 | Cascade流式编辑器 |
| **Augment Code** | 长上下文理解（10万行+） | 大型代码库智能分析 |
| **Claude Code** | Agent模式、全流程开发 | 终端直接操作文件系统 |
| **v0.dev** | UI组件生成 | 自然语言生成React组件 |
| **Gemini Canvas** | 可视化编程 | 零基础快速创建应用 |
| **Replit Agent** | 全栈应用生成 | 从需求到部署一站式完成 |

**具体应用场景：**
- **前端开发**：AI生成React/Vue组件、CSS样式、响应式布局
- **后端开发**：自动生成API接口、数据库模型、业务逻辑
- **全栈开发**：从需求描述到完整应用（如Replit Agent）
- **遗留代码维护**：AI理解和重构10年前的老旧代码
- **跨语言迁移**：将Java代码自动转为Python，保持逻辑一致

#### 💔 残酷的现实：初学者的困境

以我个人切身经历来看，AI已经让**编程初学者在职业市场上变得毫无竞争力**。

**对比分析：**

| 维度 | 编程初学者 | AI编程工具 |
|------|-----------|-----------|
| **代码质量** | 不稳定，Bug多 | 稳定，遵循最佳实践 |
| **开发速度** | 慢，需要大量查询 | 快，秒级生成 |
| **成本** | 月薪5k-15k | $20-60/月 |
| **情绪管理** | 有情绪波动 | 无情绪，永远稳定 |
| **工作时长** | 8小时/天，会累 | 24/7不间断 |
| **学习能力** | 需要培训周期 | 即时掌握新框架 |
| **犯错空间** | 需要成长时间 | 无成长曲线，即用即产出 |

**资本市场的逻辑：**
> 在资本只看重成本和利润的本性下，企业更愿意选择：**成本低 + 效率高 + 无情绪 + 不休息**的AI，而非需要培养、会犯错、有情绪的初学者。

**这意味着：**
- ❌ 初级程序员岗位大幅减少
- ❌ 实习生、应届生的成长空间被压缩
- ❌ 传统的"从入门到熟练"路径被打破
- ✅ 高级架构师、AI提示工程师需求增加
- ✅ "懂业务 + 会用AI"的复合型人才更值钱

#### 🔮 未来展望：IT行业的重构

**1. 开发者角色转型**

```
传统开发者                    AI时代的开发者
    ↓                              ↓
编写每一行代码    →         设计系统架构
调试Bug           →         审查AI生成代码
查阅文档          →         编写精准提示词
手动测试          →         设计测试策略
    ↓                              ↓
"代码工人"        →         "AI指挥官"
```

**2. 开发范式革命**

- **从代码优先到意图优先**：开发者专注描述"要什么"，而非"怎么写"
- **从人工调试到AI自修复**：AI自动检测并修复Bug
- **从手动部署到一键发布**：AI自动完成CI/CD全流程
- **从单兵作战到AI协作**：一人+AI = 传统一个团队的生产力

**3. 新兴岗位崛起**

| 新岗位 | 核心能力 | 薪资趋势 |
|--------|---------|---------|

- **动态叙事系统**：根据玩家行为实时生成剧情，每位玩家体验独一无二

**创新案例《遥远行星：建造师》：**
- 玩家与数百名拥有自主意识、性格、记忆的AI NPC互动
- AI生成世界任务及故事剧情
- 存在"上帝AI"实时调度故事走向
- 每位玩家看到的游戏剧情可能都截然不同
- 策划不再"掌控一切"，游戏成为"活的"世界

**马斯克的新动向：**
特斯拉创始人埃隆·马斯克宣布开设AI游戏工作室，目标"让游戏再次伟大"，进一步证明"AI+游戏"已成为全球游戏厂商新的竞争高地。

**挑战与思考：**
- **创意 vs 算法**：当AI生成内容泛滥，"真实性"和"创意"成为核心竞争力
- **玩家接受度**：玩家是否愿意接受无法预测的游戏体验？
- **开发成本**：AI NPC虽然智能，但训练和运行成本高昂
- **伦理边界**：AI生成的内容如何监管？暴力、色情内容如何过滤？

## 第四部分：中美AI发展对比与未来趋势

### 一、创新模式的差异：0→1 vs 1→N

经常在网上看到一种说法：**"欧美人更擅长做0到1的创新，而中国人更擅长做1到N的迭代"**。AI时代似乎再次验证了这种观点。

| 维度 | 美国（0→1创新） | 中国（1→N迭代） |
|------|----------------|----------------|
| **优势** | 原创突破、技术定义权 | 快速优化、工程化落地 |
| **典型案例** | ChatGPT、Transformer架构 | DeepSeek优化、国产模型追赶 |
| **研发周期** | 长周期、高风险高回报 | 短周期、快速迭代 |
| **竞争策略** | 技术领先、生态垄断 | 成本优势、开源开放 |

**美国的0→1创新优势：**
- **OpenAI**：ChatGPT横空出世，引领全球AI热潮
- **Anthropic**：Claude系列（Sonnet、Opus）长期占据编程模型TOP1
- **Google**：Gemini系列在多模态和视觉方面独树一帜

**中国的1→N迭代能力：**
- 在已有技术路线上快速优化
- 注重工程化和成本控制
- 开源策略打破技术垄断

| **Google** | Gemini 1.5/2.0 | 多模态、长上下文 | 追赶OpenAI |
| **Meta** | Llama系列 | 开源生态 | 开源社区主导 |

#### 🇨🇳 中国：从追赶者到并跑者

**早期探索阶段（2023-2024）：**
- **百度文心一言**：国内最早发布的大模型，但后续发展乏力
- **智谱AI GLM**、**商汤日日新**、**阿里通义千问**：各自探索，未形成突破

**转折点（2025年初）：**
- **DeepSeek R1**（2025年1月20日）：以557万美元训练成本，性能媲美OpenAI o1，震惊业界
  - 完全开源（MIT许可证）
  - 训练成本仅为GPT-4的1/100
  - 被业界认为是"第一梯队吊车尾"，但已属重大突破

**快速追赶（2025年底-2026年初）：**
| 模型 | 发布方 | 核心突破 | 与美国的差距 |
|------|--------|---------|-------------|
| **Kimi 2.5** | 月之暗面 | 长上下文、推理能力 | 接近Sonnet 4水平 |
| **GLM-5** | 智谱AI | 多模态、代码生成 | 基准测试追平Opus 4.5 |
| **MiniMax** | MiniMax | 多模态、情感理解 | 实际体验仍有差距 |

**个人使用感受：**
> 就我实际使用体验来说，DeepSeek R1比同时期的Claude 3.7 Sonnet还是差一些，但在推理能力和数学解题上已经非常接近。到了2026年初，Kimi 2.5和GLM-5在日常使用场景中，与Claude 4的差距已经缩小到可以接受的范围。

### 三、开源 vs 闭源：不同的战略选择

**一个很有意思的现象：中国的大模型大部分是开源的。**

| 策略 | 代表模型 | 优势 | 劣势 |
|------|---------|------|------|
| **开源** | DeepSeek、Qwen、Llama | 快速传播、社区共建、打破垄断 | 商业化变现困难、核心技术暴露 |
| **闭源** | GPT-4、Claude、Gemini | 技术保密、商业护城河、服务收费 | 创新速度受限、生态封闭 |

**开源策略的影响：**
- **DeepSeek**的开源引发全球关注，Reddit上大量开发者期待中国模型的更新
- 甚至有传言称"3月3日元宵节会发布DeepSeek-4"（虽然官方从未确认）
- 开源让中国模型在全球开发者社区获得大量支持

### 四、全球格局：两极分化

**当前AI世界格局：**

```
全球AI格局

│   ├── OpenAI、Anthropic、Google、Meta
│   └── 技术领先、商业化成熟、生态完善
├── 🇨🇳 中国AI（第一梯队）

    ├── 欧洲（Mistral等）
    ├── 加拿大（Cohere等）
    └── 中东（阿联酋Falcon等）
    └── 落后中美1-2代
```

**Reddit上有意思的说法：**
> "现在世界上就分两种AI——**美国的黄种人搞的AI**和**中国的黄种人搞的AI**"

### 五、未来展望：中国能否反超？

#### 🎯 我的判断：5年内中美将达到55开

**前提条件：**
1. **抛开开源光环**：单独比较商业闭源模型的顶尖能力
2. **补齐短板**：芯片和软件生态必须突破
3. **发挥优势**：电力成本和工程化能力

#### 📊 中美AI竞争力对比

| 竞争维度 | 美国优势 | 中国优势 | 未来趋势 |
|---------|---------|---------|---------|
| **顶尖算法** | ✅ 领先 | ⚠️ 追赶中 | 差距缩小 |
| **高端芯片** | ✅ 垄断 | ❌ 被制裁 | 关键瓶颈 |
| **软件生态** | ✅ CUDA、PyTorch | ⚠️ 起步晚 | 需要时间 |
| **高端人才** | ✅ 全球吸引 | ⚠️ 部分流失 | 差距缩小 |
| **数据规模** | ⚠️ 质量高 | ✅ 总量大 | 各有优势 |
| **电力成本** | ❌ 贵 | ✅ 便宜且充足 | 中国优势扩大 |
| **工程化能力** | ⚠️ 强 | ✅ 极强 | 中国优势 |
| **产业链完整度** | ❌ 部分外包 | ✅ 全链条 | 中国优势 |

#### 🔋 中国的"杀手锏"：电力

美国搞AI最大的隐性瓶颈是什么？**电力！**

- 一个大型数据中心的耗电量相当于一个小城市
- 美国电力基础设施建设缓慢，电费昂贵
- 中国拥有全球最完整的电力产业链，且电费便宜

**AI = 算法 + 算力 + 电力**

当算法差距缩小，算力（芯片）被制裁，**电力**就是中国最大的筹码。

#### 🚀  scenarios：3种可能

**Scenario 1：乐观情况（概率30%）**
- 中国芯片突破（华为昇腾等替代方案成熟）
- 国产软件生态完善（MindSpore等框架普及）
- **结果**：中国在2-3年内实现AI能力反超

**Scenario 2：中性情况（概率50%）**
- 芯片和软件生态缓慢追赶
- 依靠开源策略和工程化能力保持并跑
- **结果**：中美长期保持55开，各有优势领域

**Scenario 3：悲观情况（概率20%）**
- 芯片制裁持续加码
- 高端人才持续流失
- **结果**：中国落后美国1-2代，但仍有自主能力

### 六、对从业者的启示

**在中美AI竞争中，个人如何定位？**

1. **不迷信美国**：中国模型的进步速度远超预期
2. **不轻视中国**：开源生态和工程化能力是真实优势
3. **关注垂直领域**：通用大模型趋于同质化，垂直应用才是机会
4. **拥抱开源**：参与开源社区，既是贡献也是学习
5. **保持技术敏感**：关注国产替代（芯片、框架、模型）的进展

**最后的判断：**

> AI时代，中美两国基本算是旗鼓相当。未来3-5年，在顶尖模型能力上中美能达到55开。规模效应在美国已经不太玩得转（边际收益递减），中国的优势在于：**完整的产业链、充足的电力、强大的工程化能力、开源社区的合力**。
>
> 只要把**芯片**和**软件生态**这两块补上，再拿出准备已久的杀手锏——**廉价且充足的电力**，中国AI在未来几年内与美国保持55开甚至小有领先，是完全有可能的。
>
> **未来已来，中美双雄并立的格局，正在形成。**
