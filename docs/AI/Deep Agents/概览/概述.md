> 构建能够规划、使用子代理并利用文件系统处理复杂任务的代理

这是开始使用 LLM 构建代理和应用程序的最简单方式——内置任务规划、文件系统上下文管理、子代理生成和长期记忆功能。

你可以将 `deepagents` 视为一个 ["代理框架"](/oss/python/concepts/products#agent-harnesses-like-the-deep-agents-sdk)。它与其他代理框架使用相同的核心工具调用循环，但内置了工具和功能。

[`deepagents`](https://pypi.org/project/deepagents/) 是一个建立在 [LangChain](/oss/python/langchain/) 核心代理构建模块之上的独立库。它使用 [LangGraph](/oss/python/langgraph/) 运行时来实现持久执行、流式传输、人在回路（human-in-the-loop）等功能。

`deepagents` 库包含：

* **Deep Agents SDK**：用于构建能够处理任何任务的代理的 SDK 包
* [**Deep Agents CLI**](/oss/python/deepagents/cli)：基于 Deep Agents SDK 构建的终端编码代理

[LangChain](/oss/python/langchain/) 是提供代理核心构建块的框架。
要了解 LangChain、LangGraph 和 Deep Agents 之间的区别，请参阅 [框架、运行时和框架](/oss/python/concepts/products)。

## 创建 Deep Agent

```python
# pip install -qU deepagents
from deepagents import create_deep_agent

def get_weather(city: str) -> str:
    """获取指定城市的天气。"""
    return f"{city} 永远是晴天！"

agent = create_deep_agent(
    tools=[get_weather],
    system_prompt="你是一个有帮助的助手",
)

# 运行代理
agent.invoke(
    {"messages": [{"role": "user", "content": "旧金山天气如何"}]}
)
```

请参阅 [快速入门](/oss/python/deepagents/quickstart/) 和 [自定义指南](/oss/python/deepagents/customization/) 开始构建你自己的代理和应用程序。

> 💡 **提示**
> 使用 [LangSmith](/langsmith/home) 来追踪请求、调试代理行为并评估输出。设置 `LANGSMITH_TRACING=true` 和你的 API 密钥即可开始使用。

## 何时使用 Deep Agents

当你想要构建能够执行以下操作的代理时，使用 **Deep Agents SDK**：

* **处理复杂的、多步骤任务**，需要规划和分解
* **通过文件系统工具管理大量上下文**
* **交换文件系统后端**以使用内存状态、本地磁盘、持久存储、[沙盒](/oss/python/deepagents/sandboxes) 或[你自己的自定义后端](/oss/python/deepagents/backends)
* **委派工作**给专门的子代理以实现上下文隔离
* **跨对话和线程持久化记忆**

对于构建更简单的代理，可以考虑使用 LangChain 的 [`create_agent`](/oss/python/langchain/agents) 或构建自定义的 [LangGraph](/oss/python/langgraph/overview) 工作流。

当你想要一个在命令行上的编码代理时，使用 **Deep Agents CLI**，它基于 Deep Agents SDK 构建：

* **交互式或非交互式运行** — 将 CLI 用作聊天式编码代理，或使用 `-n` 参数进行脚本化、无头执行。
* **自定义**代理的技能和记忆。
* **教导**代理你的偏好、常见模式和自定义项目知识。
* **执行代码**在你的机器上或沙盒中。

## 核心功能

### 规划和任务分解

Deep Agents 包含一个内置的 [`write_todos`](/oss/python/langchain/middleware/built-in#to-do-list) 工具，使代理能够将复杂任务分解为离散的步骤、跟踪进度并在新信息出现时调整计划。

### 上下文管理

文件系统工具（[`ls`](/oss/python/deepagents/harness#virtual-filesystem-access)、[`read_file`](/oss/python/deepagents/harness#virtual-filesystem-access)、[`write_file`](/oss/python/deepagents/harness#virtual-filesystem-access)、[`edit_file`](/oss/python/deepagents/harness#virtual-filesystem-access)）允许代理将大量上下文卸载到内存或文件系统存储中，防止上下文窗口溢出并支持处理可变长度的工具结果。

### 可插拔文件系统后端

虚拟文件系统由[可插拔后端](/oss/python/deepagents/backends)提供支持，你可以根据用例进行切换。选择内存状态、本地磁盘、用于跨线程持久化的 LangGraph 存储、[用于隔离代码执行的沙盒](/oss/python/deepagents/sandboxes)（Modal、Daytona、Deno），或者使用复合路由组合多个后端。你还可以实现自己的自定义后端。

### 子代理生成

一个内置的 `task` 工具使代理能够生成专门的子代理以实现上下文隔离。这可以保持主代理的上下文清晰，同时仍然深入处理特定的子任务。

### 长期记忆

使用 LangGraph 的 [Memory Store](/oss/python/langgraph/persistence#memory-store) 跨线程扩展代理的持久记忆。代理可以保存和检索以前对话中的信息。
