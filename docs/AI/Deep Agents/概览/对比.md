> 比较 LangChain Deep Agents 与 Claude Agent SDK 和 Codex SDK，为你的用例选择合适的工具

本页面帮助你了解 [LangChain Deep Agents](/oss/python/deepagents/overview) 如何与 [Claude Agent SDK](https://platform.anthropic.com/docs/en/agent-sdk/overview) 和 [Codex SDK](https://developers.openai.com/codex/sdk) 进行比较。

## 概述

| 特性                         | **LangChain Deep Agents**                                          | **Claude Agent SDK**                                                              | **Codex SDK**                                                      |
| ---------------------------- | ------------------------------------------------------------------ | -------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| **使用场景**                 | 自定义通用代理（包括编码）                                          | 自定义 AI 编码代理                                                               | 可执行编码任务的预构建编码代理                                     |
| **模型支持**                 | 灵活且模型无关（Anthropic、OpenAI 及 100+ 其他）                   | 与 Claude 模型紧密集成（Anthropic、Azure、Vertex AI、AWS Bedrock）               | 与 OpenAI 模型紧密集成（GPT-5.3-Codex 及变体）                    |
| **架构**                     | Python SDK、TypeScript SDK 和 CLI                                  | Python SDK、TypeScript SDK                                                        | TypeScript SDK、CLI、桌面应用、IDE 扩展、云端界面                 |
| **执行环境**                 | 本地、远程沙盒、虚拟文件系统                                        | 本地                                                                             | 本地、云端                                                         |
| **部署方式**                 | LangGraph Platform                                                 | 自托管                                                                           | 不适用                                                           |
| **前端支持**                 | [React 集成](/oss/python/langchain/streaming/frontend)             | 仅服务端                                                                         | 仅服务端                                                         |
| **可观测性**                 | LangSmith 追踪与评估                                                | 不适用                                                                           | OpenAI 追踪与命令行评估                                           |
| **安全可配置性**             | 可组合的、按工具的人在回路                                           | 权限系统，含模式、规则和钩子                                                     | 内置层级，使用审批模式和操作系统级沙盒                            |
| **许可证**                   | MIT                                                                | MIT（底层 Claude Code 为专有软件）                                               | Apache-2.0                                                       |

### 主要差异

**LangChain Deep Agents**：

* **模型灵活性**：随时切换模型提供商（Anthropic、OpenAI 或 100+ 其他），并运行评估。
* **长期记忆**：使用 [Memory Store](/oss/python/deepagents/long-term-memory) 跨会话和线程持久化上下文
* **沙盒即工具模式**：在代理外部运行时，从[不同提供商](/oss/python/deepagents/sandboxes)的隔离沙盒中运行单个操作，或在沙盒内运行完整代理
* **虚拟文件系统**：使用可插拔后端（内存、磁盘、持久存储、沙盒）进行上下文和检查点管理
* **生产部署**：通过 [LangSmith](/oss/python/langgraph/deploy) 部署或自托管使用 Agent Server
* **可观测性**：使用 [LangSmith](/langsmith/home) 进行原生追踪和调试

**Claude Agent SDK**：

* **标准化使用 Claude**：为 Claude 模型提供 Anthropic、Azure、Vertex AI 和 AWS Bedrock 的一流支持
* **自定义托管**：构建你自己的 HTTP/WebSocket 层，并在容器中运行 SDK
* **钩子**：轻松拦截和控制代理行为

**Codex SDK**：

* **标准化使用 OpenAI**：GPT-5.3-Codex 和 OpenAI 专用工具
* **操作系统级沙盒模式**：使用内置的 `read-only`、`workspace-write` 或 `danger-full-access` 模式
* **MCP 服务器模式**：使用 `codex mcp-server` 将你的代理作为 MCP 服务器暴露
* **可观测性**：使用 [OpenAI Traces](https://developers.openai.com/codex/guides/agents-sdk/#trace-the-workflow)
