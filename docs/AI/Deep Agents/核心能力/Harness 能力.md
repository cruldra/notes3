# Harness 能力

代理 harness 是几种不同能力的组合，使构建长期运行的代理更加容易：

* [规划能力](#规划能力)
* [虚拟文件系统](#虚拟文件系统访问)
* [任务委派（子代理）](#任务委派子代理)
* [上下文和令牌管理](#上下文管理)
* [代码执行](#代码执行)
* [人在回路](#人在回路)

除了这些能力外，Deep Agents 还使用[技能](#技能)和[记忆](#记忆)来获取额外的上下文和指令。

## 规划能力

harness 提供了一个 `write_todos` 工具，代理可以使用它来维护结构化的任务列表。

**特性：**

* 跟踪多个任务及其状态（`'pending'`、`'in_progress'`、`'completed'`）
* 持久化在代理状态中
* 帮助代理组织复杂的多步骤工作
* 对长期运行任务和规划很有用

## 虚拟文件系统访问

harness 提供了一个可配置的虚拟文件系统，可以由不同的可插拔后端支持。
后端支持以下文件系统操作：

| 工具         | 描述                                                         |
| ------------ | ------------------------------------------------------------ |
| `ls`         | 列出目录中的文件及元数据（大小、修改时间）                   |
| `read_file`  | 读取文件内容，带行号，支持大文件的偏移/限制。还支持读取图片（`.png`、`.jpg`、`.jpeg`、`.gif`、`.webp`），以多模态内容块形式返回。 |
| `write_file` | 创建新文件                                                   |
| `edit_file`  | 在文件中执行精确字符串替换（支持全局替换模式）               |
| `glob`       | 查找匹配模式的文件（例如 `**/*.py`）                         |
| `grep`       | 搜索文件内容，支持多种输出模式（仅文件、带上下文的内容或计数） |
| `execute`    | 在环境中运行 shell 命令（仅 [沙盒后端](/oss/python/deepagents/sandboxes) 可用） |

虚拟文件系统被其他几个 harness 能力使用，如技能、记忆、代码执行和上下文管理。
你也可以在为 Deep Agents 构建自定义工具和中间件时使用文件系统。

有关更多信息，请参阅[后端](/oss/python/deepagents/backends)。

## 任务委派（子代理）

harness 允许主代理创建临时的"子代理"来执行隔离的多步骤任务。

**为什么有用：**

* **上下文隔离** - 子代理的工作不会干扰主代理的上下文
* **并行执行** - 多个子代理可以同时运行
* **专业化** - 子代理可以拥有不同的工具/配置
* **令牌效率** - 大型子任务上下文被压缩为单个结果

**工作原理：**

* 主代理有一个 `task` 工具
* 调用时，它会创建一个具有自己上下文的新代理实例
* 子代理自主执行直到完成
* 向主代理返回单个最终报告
* 子代理是无状态的（不能发送多条消息回去）

**默认子代理：**

* 自动可用的"通用"子代理
* 默认具有文件系统工具
* 可以使用额外的工具/中间件进行自定义

**自定义子代理：**

* 定义具有特定工具的专门子代理
* 示例：代码审查员、网络研究员、测试运行器
* 通过 `subagents` 参数配置

## 上下文管理

为了处理长期运行任务，Deep Agents 使用有效的上下文管理。

代理可以访问几种类型的上下文。
一些源在启动时提供给代理；其他源在运行时变得可用，如用户输入。

本节概述了你的 Deep Agent 可以访问和管理的不同类型的上下文。

### 输入上下文

输入上下文由在启动时提供给 Deep Agent 并添加到提示词中的信息源组成。

#### 提示词

系统提示词定义代理的角色、行为、能力和知识库。
如果你提供自定义系统提示词，它会被添加到内置系统提示词之前，后者包括有关如何使用内置工具（如规划工具、文件系统工具和子代理）的详细指导。

添加工具的中间件（如文件系统中间件）会自动将工具特定的说明附加到系统提示词，创建工具提示词来解释如何有效使用这些工具。

最终的 Deep Agent 提示词由以下部分组成：

1. 自定义 system_prompt（如果提供）
2. [基础代理提示词](https://github.com/langchain-ai/deepagents/blob/595f6fe66bd974eaba9d429a76217607cb02f7a8/libs/deepagents/deepagents/graph.py#L36)
3. 待办事项列表提示词：如何使用待办事项列表进行规划的说明
4. 记忆提示词：AGENTS.md + 记忆使用指南（仅当提供 `memory` 时）
5. 技能提示词：技能位置 + 带有 frontmatter 信息的技能列表 + 使用方法（仅当提供技能时）
6. 虚拟文件系统提示词（文件系统 + 执行工具文档，如适用）
7. 子代理提示词：任务工具使用
8. 用户提供的中间件提示词（如果提供自定义中间件）
9. 人在回路提示词（当设置 `interrupt_on` 时）
10. 本地上下文提示词：当前目录、项目信息...（在本地使用 CLI 时）

### 运行时上下文

上下文压缩在保留与任务相关的细节的同时减少代理工作内存中信息的大小。
以下技术是确保传递给 LLM 的上下文保持在其上下文窗口限制内的内置功能：

* [卸载大型工具输入和结果](#卸载大型工具输入和结果)
* [摘要](#摘要)

你还可以配置 Deep Agents 使用[长期记忆](#长期记忆)来允许它们在不同的线程和对话中存储信息。

#### 卸载大型工具输入和结果

Deep Agents 使用[内置文件系统工具](#虚拟文件系统访问)来自动卸载内容，并在需要时搜索和检索该卸载内容。
内容卸载发生在两种情况下：

1. **工具调用输入超过 20,000 个令牌**（可通过 `tool_token_limit_before_evict` 配置）：文件写入和编辑操作在代理的对话历史中留下包含完整文件内容的工具调用。
   由于此内容已持久化到文件系统，通常是冗余的。
   当会话上下文超过模型可用窗口的 85% 时，Deep Agents 将截断较旧的工具调用，用指向磁盘上文件的指针替换它们，从而减少活动上下文的大小。

2. **工具调用结果超过 20,000 个令牌**（可通过 `tool_token_limit_before_evict` 配置）：当发生这种情况时，Deep Agent 将响应卸载到配置的后端，并用文件路径引用和前 10 行的预览替换它。然后代理可以根据需要重新读取或搜索内容。

#### 摘要

当上下文大小超过模型的上下文窗口限制（例如 `max_input_tokens` 的 85%），并且没有更多符合卸载条件的上下文时，Deep Agent 会摘要消息历史。

此过程有两个组成部分：

* **上下文内摘要**：LLM 生成对话的结构化摘要，包括会话意图、创建的工件和后续步骤，这取代了代理工作内存中的完整对话历史。
* **文件系统保留**：完整的原始对话消息作为规范记录写入文件系统。

这种双重方法确保代理通过摘要保持对其目标和进展的意识，同时在需要时保留恢复特定细节的能力（通过文件系统搜索）。

**配置：**

* 从 [模型配置文件](/oss/python/langchain/models#model-profiles) 的 `max_input_tokens` 的 85% 处触发
* 保留 10% 的令牌作为近期上下文
* 如果模型配置文件不可用，回退到 170,000 令牌触发 / 保留 6 条消息
* 如果任何模型调用引发 `ContextOverflowError`，Deep Agents 立即回退到摘要并使用摘要 + 保留的近期消息重试
* 较旧的消息由模型摘要

**为什么有用：**

* 使非常长的对话不会达到上下文限制
* 保留近期上下文同时压缩历史记录
* 对代理透明（显示为特殊系统消息）

##### 摘要工具

Deep Agents 包括一个可选的摘要[工具](/oss/python/langchain/tools)，使代理能够在适当时机（如在任务之间）触发摘要，而不是在固定的令牌间隔。

你可以通过将其附加到中间件列表来启用此工具：

```python
from deepagents import create_deep_agent
from deepagents.backends import StateBackend
from deepagents.middleware.summarization import (
    create_summarization_tool_middleware,
)

backend = StateBackend  # 如果使用默认后端

model = "openai:gpt-5.4"
agent = create_deep_agent(
    model=model,
    middleware=[
        create_summarization_tool_middleware(model, backend),
    ],
)
```

启用此功能不会禁用模型上下文限制 85% 处的默认摘要操作。

有关详细信息，请参阅 [API 参考](https://reference.langchain.com/python/deepagents/middleware/summarization/SummarizationToolMiddleware)。

#### 长期记忆

使用默认文件系统时，你的 Deep Agent 将其工作内存文件存储在代理状态中，这只在单个线程内持久化。
长期记忆使你的 Deep Agent 能够在不同的线程和对话中持久化信息。
要使用长期记忆，你必须使用一个 `CompositeBackend`，它将特定路径（通常是 `/memories/`）路由到 LangGraph Store，后者提供持久的跨线程持久化。
`CompositeBackend` 是一个混合存储系统，其中一些文件无限期持久化，而其他文件则限定在单个线程内。

代理存储在长期记忆路径中的文件（例如 `/memories/preferences.txt`）在代理重启后仍然保留，并且可以从任何对话线程访问。
Deep Agents 可以使用这些文件来存储用户偏好、积累的知识、研究进展或任何应该超出单个会话的信息。

有关更多信息，请参阅[长期记忆](/oss/python/deepagents/long-term-memory)。

## 代码执行

当你使用[沙盒后端](/oss/python/deepagents/sandboxes)时，harness 暴露一个 `execute` 工具，允许代理在隔离环境中运行 shell 命令。这使代理能够安装依赖项、运行脚本并执行其任务的一部分代码。

**工作原理：**

* 沙盒后端实现 `SandboxBackendProtocol` — 检测到时，harness 将 `execute` 工具添加到代理的可用工具中
* 没有沙盒后端时，代理只有文件系统工具（`read_file`、`write_file` 等）且不能运行命令
* `execute` 工具返回组合的 stdout/stderr、退出代码，并截断大型输出（保存到文件供代理增量读取）

**为什么有用：**

* **安全性** — 代码在隔离中运行，保护你的主机系统免受代理操作的影响
* **干净的环境** — 使用特定的依赖项或操作系统配置而无需本地设置
* **可重现性** — 跨团队的一致执行环境

有关设置、提供商和文件传输 API，请参阅[沙盒](/oss/python/deepagents/sandboxes)。

## 人在回路

harness 可以在指定的工具调用处暂停代理执行，以允许人工批准或修改。此功能通过 `interrupt_on` 参数选择加入。

**配置：**

* 将 `interrupt_on` 传递给 `create_deep_agent`，并带有工具名称到中断配置的映射
* 示例：`interrupt_on={"edit_file": True}` 在每次编辑前暂停
* 你可以在提示时提供批准消息或修改工具输入

**为什么有用：**

* 破坏性操作的安全门
* 昂贵 API 调用前的用户验证
* 交互式调试和指导

## 技能

harness 支持技能，为你的 Deep Agent 提供专业化的工作流程和领域知识。

**工作原理：**

* 技能遵循 [Agent Skills 标准](https://agentskills.io/)
* 每个技能是一个包含带有说明和元数据的 `SKILL.md` 文件的目录
* 技能可以包括额外的脚本、参考文档、模板和其他资源
* 技能使用渐进式披露 — 它们仅在代理确定对当前任务有用时才加载
* 代理在启动时从每个 `SKILL.md` 文件读取 frontmatter，然后在需要时查看完整的技能内容

**为什么有用：**

* 通过仅在需要时加载相关技能来减少令牌使用
* 将能力捆绑在一起成为具有额外上下文的更大操作
* 提供专业知识而不会使系统提示词混乱
* 实现模块化、可重用的代理能力

有关更多信息，请参阅[技能](/oss/python/deepagents/skills)。

## 记忆

harness 支持持久化记忆文件，为你的 Deep Agent 跨对话提供额外的上下文。
这些文件通常包含一般的编码风格、偏好、约定和指导方针，帮助代理理解如何与你的代码库一起工作并遵循你的偏好。

**工作原理：**

* 使用 [`AGENTS.md` 文件](https://agents.md/) 提供持久上下文
* 与使用渐进式披露的技能不同，记忆文件始终加载
* 在创建代理时将一个或多个文件路径传递给 `memory` 参数
* 文件存储在代理的后端（StateBackend、StoreBackend 或 FilesystemBackend）
* 代理可以根据你的交互、反馈和识别的模式更新记忆

**为什么有用：**

* 提供不需要每次对话重新指定的持久上下文
* 适用于存储用户偏好、项目指导方针或领域知识
* 始终对代理可用，确保一致的行为

有关配置详细信息和示例，请参阅[记忆](/oss/python/deepagents/customization#memory)。
