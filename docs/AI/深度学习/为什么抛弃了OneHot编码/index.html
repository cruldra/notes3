为什么抛弃了OneHot编码

一、One-Hot 编码的本质：给每个词分配唯一“身份证号”​

想象你所在的班级有 1 万个学生，学校要给每个学生发一个 唯一的学号。

  • 如果学号是 5 位数，比如：
    • 张三：00001
    • 李四：00002
    • ...
    • 第1万号同学：10000

这个学号系统就是 One-Hot 编码的简化版！

  • 每一位数字 → 对应向量中的一个位置。
  • 学号中的1 → 表示“这是你的唯一标识”。
  • 其他位是0 → 表示“这些位置不属于你”。

二、为什么不能有多个1？​

1. 唯一性被破坏​

如果允许一个学号中有多个1，比如：

  • 张三的学号写成 10001（第1位和第5位是1），
  • 李四的学号写成 01001（第2位和第5位是1）。

问题来了：

  • 学号 10001 和 01001 都包含第5位的1 → 系统会误以为他俩有某种关联（比如都是“第5位俱乐部”成员），但实际上他们毫无关系！
  • One-Hot 编码的目标是让每个词完全独立，不允许任何隐含的“关联”。

2. 类比：开关灯​

假设房间里有 1 万盏灯，每个词对应一盏灯：

  • 提到“猫”时 → 只打开第3盏灯（其他灯全灭） → [0,0,1,0,...,0]
  • 提到“狗”时 → 只打开第4盏灯 → [0,0,0,1,...,0]

如果允许开多盏灯：

  • 同时打开第3和第4盏灯 → [0,0,1,1,...,0]
  • 系统会困惑：这到底代表“猫狗混合体”，还是“猫和狗同时出现”？
  • One-Hot 编码要求每次只能亮一盏灯，明确表示一个词。

三、One-Hot 编码的局限性​

虽然 One-Hot 简单直接，但它有两大缺陷：

  1. 无法表达语义关系：

    • “猫”和“狗”的向量是 [0,0,1,0,...] 和 [0,0,0,1,...]，它们的距离和“猫”与“飞机”的距离一样远。
    • 但现实中，“猫”和“狗”显然更相似，而 One-Hot 完全无法表达这一点。

  2. 维度灾难：

    • 1万个词需要1万维向量 → 存储和计算成本极高。
    • 实际应用中，词表可能达到百万级，One-Hot 完全不可行。

四、为什么后来用低维向量（Embedding）取代了 One-Hot？​

1. Embedding 的智慧​

Embedding 不再用“唯一身份证号”，而是用 低维向量中的数值组合 表示语义。
例如：

  • 猫 → [0.3, -0.2, 1.5]
  • 狗 → [0.4, -0.1, 1.6]
  • 飞机 → [-0.8, 1.0, 0.2]

这些数值可以表达：

  • 相似性：猫和狗的向量接近，和飞机差异大。
  • 语义关系：通过向量加减法，可以推导出“国王 - 男 + 女 ≈ 女王”。

2. Embedding 允许“多个1”吗？​

  • 可以！但不是简单的0/1，而是通过浮点数的组合。
  • 例如，向量中的某个维度可能是0.8（表示“有80%的动物属性”），另一个维度是-0.3（表示“有30%的负面情感”）。
  • 这种灵活性是 One-Hot 永远做不到的。

五、总结​

  • One-Hot 编码：用“唯一身份证号”表示词，只能有一个1 → 保证独立性，但牺牲了语义信息。
  • Embedding：用低维数值组合表示词，允许多维度的语义表达 → 更聪明但更复杂。

类比：

  • One-Hot 像给每个人发唯一的数字ID（如学号），只能区分身份，无法描述性格、爱好。
  • Embedding 像给每个人写一段评语（如“外向80%，善良90%，聪明70%”），能详细描述特征。
