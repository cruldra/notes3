欢迎阅读 **Godot Engine** 的官方文档！Godot 是一款免费、开源、社区驱动的 2D 和 3D 游戏引擎。在这些看似深奥的名字背后，你会发现一个强大且用户友好的工具，你可以用它开发任何类型的游戏，支持任何平台，且没有任何使用限制。

本页对引擎和文档进行了总体概述，无论你是初学者还是正在寻找特定功能信息的开发者，都能通过本页了解从何处开始。

## 在你开始之前

[教程与资源](https://docs.godotengine.org/en/stable/community/tutorials.html#doc-community-tutorials) 页面列出了由社区贡献的视频教程。如果你更喜欢视频而非文本，可以去看看。否则，[入门指南](https://docs.godotengine.org/en/stable/getting_started/introduction/index.html#doc-getting-started-intro) 是一个很好的起点。

如果你在教程或项目中遇到困难，可以在各种 [社区频道](https://godotengine.org/community/) 中寻求帮助，特别是 Godot [Discord](https://discord.gg/godotengine) 社区和 [论坛](https://forum.godotengine.org/)。

## 关于 Godot 引擎

游戏引擎是一个复杂的工具，很难用几句话来概括。以下是一个简短的简介，如果你需要快速介绍 Godot 引擎，可以自由复用：

> Godot Engine 是一款功能丰富、跨平台的游戏引擎，可以通过统一的界面创建 2D 和 3D 游戏。它提供了一套全面的常用工具，让用户可以专注于制作游戏，而不必重复造轮子。游戏可以一键导出到多个平台，包括主要的桌面平台（Linux、macOS、Windows）、移动平台（Android、iOS），以及 Web 平台和游戏主机。
>
> Godot 在 [宽松的 MIT 许可证](https://docs.godotengine.org/en/stable/about/complying_with_licenses.html#doc-complying-with-licenses) 下完全免费且开源。没有任何附加条件，没有版税，什么都没有。用户的游戏完全归用户所有，甚至包括引擎代码的最后一行。Godot 的开发完全独立且由社区驱动，使用户能够根据自己的期望协助塑造引擎。它由非营利组织 [Godot Foundation](https://godot.foundation/) 支持。

## 文档结构

本文档分为几个部分：

*   **关于 (About)**：包含此介绍以及有关引擎的信息、历史、许可、作者等。它还包含 [常见问题解答 (FAQ)](https://docs.godotengine.org/en/stable/about/faq.html#doc-faq)。
*   **入门指南 (Getting Started)**：包含使用引擎制作游戏所需的所有必要信息。它以 [简介](https://docs.godotengine.org/en/stable/getting_started/introduction/index.html#doc-getting-started-intro) 部分开始，这应该是所有新用户的入口。**如果你是新用户，这里是最好的开始地点！**
*   **手册 (Manual)**：可以根据需要以任何顺序阅读或参考。它包含特定功能的教程和文档。
*   **引擎详情 (Engine details)**：包含面向高级用户和贡献者的部分，包括有关编译引擎、开发编辑器或开发 C++ 模块的信息。
*   **社区 (Community)**：致力于 Godot 社区的生活，包含本文档之外的推荐第三方教程和材料列表。它还提供了有关资产库 (Asset Library) 的详细信息。
*   **类参考 (Class reference)**：记录了完整的 Godot API，也可以直接在引擎的脚本编辑器中查看。你可以在这里找到所有类、函数、信号等信息。

除了本说明文档，你可能还想查看各种 [Godot 示例项目 (Godot demo projects)](https://github.com/godotengine/godot-demo-projects)。
