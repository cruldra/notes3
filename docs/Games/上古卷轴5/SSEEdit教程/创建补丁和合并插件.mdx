随着你安装的MOD数量增加，创建自定义补丁和合并插件成为管理游戏稳定性和兼容性的重要技能。本教程将深入探讨如何使用SSEEdit创建各种类型的补丁和合并插件，帮助你解决复杂的MOD冲突并优化游戏性能。

## 创建自定义补丁

自定义补丁是解决特定MOD之间冲突的最灵活方法。以下是创建自定义补丁的详细步骤：

### 基本补丁创建流程

1. 启动SSEEdit并加载所有相关插件
2. 右键点击右侧面板，选择"Add"
3. 输入新补丁的名称（例如"MyCustomPatch.esp"）
4. 找到需要修复的冲突记录
5. 右键点击你想要保留的版本，选择"Copy as override into..."
6. 选择你的自定义补丁
7. 根据需要编辑复制的记录
8. 保存更改

### 创建特定类型的补丁

#### 装备和物品补丁

当多个MOD修改同一装备或物品时：

1. 展开"Armor"、"Weapon"或"Miscellaneous Item"类别
2. 找到有冲突的物品记录
3. 确定你想保留的属性（如伤害值、护甲值、重量等）
4. 将这些属性复制到你的补丁中
5. 保存更改

#### NPC外观补丁

当多个MOD修改同一NPC时：

1. 展开"Non-Player Character (Actor)"类别
2. 找到有冲突的NPC记录
3. 决定保留哪些特性（如外观、装备、AI包等）
4. 将这些特性复制到你的补丁中
5. 保存更改

#### 地点和单元格补丁

当多个MOD修改同一地点时：

1. 展开"Worldspace"或"Cell"类别
2. 找到有冲突的地点记录
3. 决定保留哪些更改（如物体放置、光照等）
4. 将这些更改复制到你的补丁中
5. 保存更改

## 使用脚本创建补丁

SSEEdit提供了多种脚本，可以自动化补丁创建过程：

### 合并补丁脚本

1. 右键点击任何插件，选择"Other"->"Create merged patch"
2. 输入补丁名称（默认为"Merged Patch.esp"）
3. 等待补丁创建完成
4. 检查新创建的补丁，确保它正确合并了所有冲突
5. 手动调整任何不正确的合并结果

### Bash Tag生成脚本

1. 右键点击任何插件，选择"Apply Script"
2. 选择"Generate Bash Tags"脚本
3. 等待脚本完成
4. 检查生成的标签，这些标签可以帮助Wrye Bash创建更好的Bashed Patch

### 其他实用脚本

SSEEdit包含许多其他有用的脚本，如：

- **Find ESP plugins which could be turned into ESL**：查找可以转换为ESL的ESP插件
- **Find records with missing or invalid FormIDs**：查找具有缺失或无效FormID的记录
- **Find records with missing or invalid EDID**：查找具有缺失或无效EDID的记录
- **Find records with missing or invalid NAME**：查找具有缺失或无效NAME的记录

## 合并插件技术

当你的插件数量接近255上限时，合并插件是一个有效的解决方案。

### 使用SSEEdit合并插件

1. 确定要合并的插件（最好选择相关性强、相互依赖或修改相似内容的插件）
2. 在SSEEdit中加载这些插件
3. 右键点击右侧面板，选择"Add"
4. 输入新合并插件的名称
5. 对于每个要合并的插件：
   - 右键点击插件名称，选择"Copy as override into..."
   - 选择你的新合并插件
6. 解决任何冲突
7. 保存更改
8. 在游戏中禁用原始插件，启用新的合并插件

### 使用专用合并工具

除了SSEEdit外，还有专门的工具用于合并插件：

1. **zMerge**（Mator Merge的继任者）：
   - 下载并安装zMerge
   - 启动zMerge并配置游戏路径
   - 创建新的合并项目
   - 选择要合并的插件
   - 设置合并选项
   - 构建合并
   - 在游戏中禁用原始插件，启用新的合并插件

2. **Merge Plugins Standalone**：
   - 下载并安装Merge Plugins
   - 启动Merge Plugins并配置游戏路径
   - 创建新的合并项目
   - 选择要合并的插件
   - 设置合并选项
   - 构建合并
   - 在游戏中禁用原始插件，启用新的合并插件

### 合并插件的最佳实践

1. **选择合适的插件**：
   - 合并相关性强的插件（如同一作者的系列MOD）
   - 避免合并经常更新的插件
   - 避免合并包含脚本的复杂插件
   - 避免合并主文件（ESM）

2. **测试合并结果**：
   - 在新的游戏存档中测试合并插件
   - 确保所有功能正常工作
   - 检查是否有任何错误或崩溃

3. **保留原始插件**：
   - 保留原始插件的备份
   - 记录合并的插件列表，以便将来参考

## 创建特殊类型的补丁

### 创建Smashed Patch

Smashed Patch是一种更高级的合并补丁，可以更智能地合并记录：

1. 下载并安装Mator Smash
2. 启动Mator Smash
3. 配置游戏路径和设置
4. 选择要包含在Smashed Patch中的插件
5. 点击"Smash"按钮创建补丁
6. 等待补丁创建完成
7. 在游戏中启用Smashed Patch

### 创建Bashed Patch

Bashed Patch主要用于合并等级列表和其他特定类型的记录：

1. 下载并安装Wrye Bash
2. 启动Wrye Bash
3. 右键点击"Bashed Patch, 0.esp"，选择"Rebuild Patch"
4. 选择要包含在Bashed Patch中的组件
5. 点击"Build Patch"按钮
6. 等待补丁创建完成
7. 在游戏中启用Bashed Patch

## 高级编辑技术

### 批量编辑记录

当你需要对多个记录进行相同的更改时：

1. 在SSEEdit中找到要编辑的记录类型
2. 使用过滤器缩小范围（右键点击，选择"Filter"）
3. 选择多个记录（按住Ctrl键点击）
4. 右键点击，选择"Copy as override into..."
5. 选择你的补丁
6. 编辑复制的记录
7. 保存更改

### 使用正则表达式查找和替换

对于复杂的文本替换：

1. 按Ctrl+F打开查找对话框
2. 勾选"Use regular expressions"选项
3. 在"Text to find"字段中输入正则表达式
4. 在"Replace with"字段中输入替换文本
5. 点击"Find All"或"Replace All"按钮

### 创建新记录

当你需要从头创建新记录时：

1. 找到类似的现有记录
2. 右键点击，选择"Copy"
3. 右键点击你的补丁，选择"Paste"
4. 编辑新记录的属性（特别是FormID和EditorID）
5. 保存更改

## 故障排除和常见问题

### 补丁创建后游戏崩溃

可能的原因和解决方案：

1. **记录冲突未正确解决**：
   - 重新检查补丁中的记录
   - 确保所有必要的字段都被正确复制

2. **缺少主文件依赖**：
   - 检查补丁的主文件依赖列表
   - 确保所有必要的主文件都被列为依赖

3. **FormID冲突**：
   - 检查是否有重复的FormID
   - 使用"Find duplicate FormIDs"脚本

### 合并插件后丢失功能

可能的原因和解决方案：

1. **脚本未正确合并**：
   - 检查原始插件中的脚本是否被正确复制
   - 考虑使用专门的合并工具

2. **资源文件丢失**：
   - 确保所有必要的资源文件（如纹理、网格等）都被包含
   - 检查BSA文件是否被正确处理

3. **加载顺序问题**：
   - 确保合并插件在加载顺序中的位置正确
   - 检查是否有其他插件依赖于原始插件
