清理插件和解决冲突

在掌握了 SSEEdit 的基础使用方法后，我们可以进一步学习如何利用这个强大的工具来清理插件中的"脏数据"和解决 MOD 之间的冲突。这些技能对于维护一个稳定的游戏环境至关重要，尤其是当你安装了大量 MOD 时。

什么是"脏数据"？​

"脏数据"是指插件中不必要的、冗余的或有潜在问题的记录，主要包括以下几种类型：

  1. ITM 记录（Identical To Master）：与主文件完全相同的记录，没有任何修改但被不必要地包含在插件中
  2. UDR 记录（Undeleted and Disabled References）：被标记为已删除但实际上仍然存在于插件中的记录
  3. 删除的导航网格数据：可能导致游戏中 NPC 寻路问题的已删除导航网格

这些"脏数据"可能导致游戏不稳定、性能下降或各种奇怪的 bug。

清理插件中的脏数据​

使用 QuickAutoClean（推荐方法）​

SSEEdit 提供了一个名为 SSEEditQuickAutoClean.exe 的工具，可以自动清理插件中的脏数据。

  1. 在 MO2 中设置 SSEEditQuickAutoClean：

    • 点击 MO2 右上角的"可执行程序"按钮
    • 点击"添加"按钮，选择"添加从文件..."
    • 浏览并选择 SSEEditQuickAutoClean.exe 文件
    • 在"标题"字段中输入"SSEEditQuickAutoClean"
    • 在"参数"字段中输入-cp:utf-8
    • 点击"确定"保存设置

  2. 使用 QuickAutoClean 清理插件：

    • 在 MO2 中，从右上角的下拉菜单中选择"SSEEditQuickAutoClean"
    • 点击"运行"按钮
    • 在命令行窗口中，输入要清理的插件名称（包括.esp 或.esm 扩展名）
    • 按 Enter 键开始清理过程
    • 等待清理完成，工具会自动关闭

手动清理（适用于高级用户）​

如果你想手动清理插件，可以按照以下步骤操作：

  1. 启动 SSEEdit 并只加载你想要清理的插件
  2. 右键点击插件名称，选择"Apply Filter for Cleaning"
  3. 等待过滤器应用完成
  4. 右键点击插件名称，选择"Remove ITM Records"
  5. 确认操作并等待完成
  6. 右键点击插件名称，选择"Undelete and Disable References"
  7. 确认操作并等待完成
  8. 保存更改

检测和解决 MOD 冲突​

当多个 MOD 修改同一个游戏元素时，就会产生冲突。SSEEdit 可以帮助你识别和解决这些冲突。

识别冲突​

  1. 启动 SSEEdit 并加载所有你想检查冲突的插件
  2. 等待加载完成后，浏览左侧的插件树
  3. 查找带有颜色标记的记录：
    • 红色背景：表示记录被删除
    • 绿色背景：表示新添加的记录
    • 黄色背景：表示修改的记录
  4. 当多个插件修改同一记录时，右侧面板会显示所有版本的记录，便于比较差异

解决冲突的方法​

1. 创建合并补丁​

合并补丁是解决多个 MOD 冲突的最常用方法：

  1. 右键点击任何插件，选择"Other"->"Create merged patch"
  2. 输入补丁名称（默认为"Merged Patch.esp"）
  3. 等待补丁创建完成
  4. 检查新创建的补丁，确保它正确合并了所有冲突

2. 手动创建自定义补丁​

对于更复杂的冲突，你可能需要手动创建自定义补丁：

  1. 右键点击右侧面板，选择"Add"
  2. 输入新补丁的名称
  3. 找到有冲突的记录
  4. 右键点击你想要保留的版本，选择"Copy as override into..."
  5. 选择你的自定义补丁
  6. 根据需要编辑复制的记录
  7. 保存更改

3. 调整加载顺序​

有时，简单调整插件的加载顺序就可以解决冲突：

  1. 确定哪个 MOD 的更改你想要保留
  2. 确保该 MOD 的插件在加载顺序中排在后面（在 MO2 的右侧面板中）
  3. 记住，加载顺序中后面的插件会覆盖前面插件的更改

解决特定类型的冲突​

Leveled Lists（等级列表）冲突​

等级列表控制游戏中物品、NPC 和生物的分布和出现概率。多个 MOD 修改同一等级列表会导致冲突。

  1. 在 SSEEdit 中加载所有相关插件
  2. 展开"Leveled Item"、"Leveled NPC"或"Leveled Spell"类别
  3. 查找有冲突的等级列表
  4. 创建一个新的补丁
  5. 右键点击最后一个修改的版本，选择"Copy as override into..."
  6. 选择你的补丁
  7. 手动添加其他插件中的条目到你的补丁版本中
  8. 保存更改

配方和制作冲突​

当多个 MOD 添加或修改制作配方时，可能会出现冲突：

  1. 在 SSEEdit 中加载所有相关插件
  2. 展开"Constructible Object"类别
  3. 查找有冲突的配方
  4. 创建一个新的补丁
  5. 复制并合并冲突的配方到你的补丁中
  6. 保存更改

高级清理技巧​

清理主文件​

游戏的主文件（如 Update.esm）通常包含脏数据，清理它们可以提高游戏稳定性：

  1. 使用 SSEEditQuickAutoClean 清理以下文件（一次只清理一个）：

    • Update.esm
    • Dawnguard.esm
    • HearthFires.esm
    • Dragonborn.esm

  2. 注意：不要清理 Skyrim.esm 主文件！

使用脚本进行批量操作​

SSEEdit 支持使用脚本进行批量操作：

  1. 右键点击任何插件，选择"Apply Script"
  2. 从列表中选择适当的脚本
  3. 按照脚本提示进行操作

注意事项和最佳实践​

  1. 始终备份：在进行任何清理或编辑操作前，确保备份你的插件和存档
  2. 逐步操作：一次只清理或编辑一个插件，避免同时进行多个更改
  3. 测试更改：每次更改后，测试游戏以确保一切正常运行
  4. 保持简单：不要过度编辑或创建过于复杂的补丁
  5. 学习模组描述：阅读 MOD 作者的说明，了解是否需要特殊的补丁或兼容性处理

结语​

掌握如何清理插件和解决 MOD 冲突是成为上古卷轴 5 高级玩家的重要一步。通过正确使用 SSEEdit，你可以显著提高游戏的稳定性和性能，同时允许更多 MOD 和谐共存。
