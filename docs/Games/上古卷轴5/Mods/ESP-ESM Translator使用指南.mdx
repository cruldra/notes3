# ESP-ESM Translator使用指南

ESP-ESM Translator是上古卷轴5（天际）和辐射系列游戏中常用的MOD汉化工具，它允许用户查看和编辑ESP/ESM文件中的文本内容。本文将详细介绍该工具的界面各列含义和使用方法。

## 软件简介

ESP-ESM Translator是由Epervier 666开发的一款专门用于Bethesda游戏（如上古卷轴系列、辐射系列）的MOD翻译工具。它能够读取ESP/ESM文件中的文本内容，并提供便捷的翻译界面，支持批量翻译、导入导出文本、使用翻译数据库等功能。

## 界面各列含义

从您提供的截图来看，ESP-ESM Translator的主界面包含以下几个主要列：

### GRUP（组别）

- **含义**：表示当前记录所属的组或类别
- **作用**：Bethesda游戏的ESP/ESM文件将不同类型的数据分组存储，GRUP列显示当前记录属于哪个组
- **常见值**：
  - `TES4`：文件头信息，包含插件的基本信息、作者、描述和依赖关系等
  - `MGEF`：魔法效果（Magic Effect）
  - `SPEL`：法术（Spell）
  - `ARMO`：盔甲（Armor）
  - `WEAP`：武器（Weapon）
  - `NPC_`：NPC角色
  - `CELL`：单元格/场景
  - `QUST`：任务
  - `BOOK`：书籍
  - `ALCH`：炼金物品/药水
  - `MISC`：杂项物品

### ID

- **含义**：记录的唯一标识符，通常是一个十六进制数值
- **作用**：每个记录在ESP/ESM文件中都有一个唯一的ID，用于内部引用
- **格式**：通常显示为8位十六进制数，如`0000B8..`、`000F3..`等

### EDID（Editor ID）

- **含义**：编辑器ID，是记录的可读名称
- **作用**：在Creation Kit等编辑器中使用的标识符，通常是有意义的英文名称
- **示例**：
  - `DisDamageMagickaRegen`：降低魔法恢复
  - `FoodRestoreHealth`：恢复生命的食物
  - `DiseaseAtaxia`：共济失调（疾病名称）

### 扩展显示识别数据

- **含义**：额外的识别信息，帮助区分相似的记录
- **作用**：当多个记录有相似的EDID或名称时，此列提供更多上下文信息

### 字段

- **含义**：指定当前显示的文本来自哪个字段
- **常见值**：
  - `FULL`：完整名称字段，通常是物品、法术等的显示名称
  - `DESC`：描述字段，包含详细说明
  - `DNAM`：数据名称字段，通常包含效果描述
  - `CNAM`：注释字段
  - `SNAM`：声音名称字段

### 原文

- **含义**：ESP/ESM文件中的原始文本（通常是英文）
- **作用**：显示需要翻译的原始文本内容

### 译文

- **含义**：翻译后的文本（如中文）
- **作用**：用户在此列输入或编辑翻译内容

## 颜色标记含义

在ESP-ESM Translator界面中，不同的颜色表示不同的翻译状态：

- **红色**：未翻译的条目
- **绿色**：已翻译的条目
- **灰色**：已翻译但可能需要审核的条目
- **蓝色**：从数据库自动翻译的条目

## 使用方法

### 基本操作流程

1. **加载ESP/ESM文件**：
   - 点击左上角的"文件" → "打开ESP/ESM"
   - 选择要翻译的ESP或ESM文件

2. **浏览和筛选记录**：
   - 使用顶部的筛选选项可以按GRUP、字段等筛选记录
   - 使用搜索框可以搜索特定文本

3. **翻译文本**：
   - 双击需要翻译的条目
   - 在弹出的编辑窗口中输入译文
   - 点击"确定"保存翻译

4. **批量翻译**：
   - 选择多个条目（使用Ctrl或Shift多选）
   - 右键点击，选择"批量翻译"
   - 可以使用API翻译或应用数据库中的翻译

5. **保存翻译结果**：
   - 点击"文件" → "保存"或"另存为"
   - 选择保存位置，替换原文件或创建新文件

### 高级功能

1. **使用翻译数据库**：
   - 点击"数据库" → "加载数据库"
   - 选择包含已有翻译的数据库文件（.bdd格式）
   - 加载后可以自动应用数据库中的翻译

2. **导出/导入文本**：
   - 可以将文本导出为TXT或CSV格式进行外部翻译
   - 完成翻译后再导入回软件

3. **验证翻译**：
   - 使用"验证"功能检查翻译是否完整
   - 可以快速定位未翻译或有问题的条目

4. **自定义筛选**：
   - 可以创建自定义筛选规则，只显示特定类型的记录
   - 对于大型MOD特别有用

## 特定字段的翻译注意事项

1. **FULL字段**：
   - 通常是物品、法术等的名称
   - 翻译应简洁明了，符合游戏风格

2. **DESC字段**：
   - 包含详细描述
   - 可以适当调整格式，但要保留原始信息

3. **DNAM字段**：
   - 通常包含效果描述
   - 注意保留`<mag>`等标签，这些是游戏中的变量占位符

4. **带标签的文本**：
   - 如`<mag>`、`<dur>`等是游戏中的变量标签
   - 翻译时必须保留这些标签，否则游戏中可能显示错误

## 常见问题解决

1. **无法保存翻译**：
   - 确保游戏未运行
   - 检查文件是否被其他程序占用
   - 尝试以管理员身份运行软件

2. **翻译后游戏显示乱码**：
   - 确保使用UTF-8编码
   - 检查是否正确保存了文件
   - 验证游戏是否支持中文显示

3. **部分文本无法翻译**：
   - 某些特殊字段可能不支持翻译
   - 尝试使用TES5Edit等工具查看完整记录结构

4. **批量翻译不准确**：
   - API翻译质量有限，建议手动审核
   - 对于专有名词，最好创建自定义词典

## 高效翻译技巧

1. **先翻译常见文本**：
   - 优先翻译FULL字段（物品名称）
   - 然后翻译DESC字段（描述）

2. **利用数据库**：
   - 使用官方汉化或社区汉化数据库
   - 为常用术语创建自己的翻译数据库

3. **分组翻译**：
   - 按GRUP分组翻译，如先翻译所有武器，再翻译所有盔甲
   - 这样可以保持翻译风格一致

4. **保存备份**：
   - 定期保存翻译进度
   - 创建不同版本的备份

5. **测试翻译**：
   - 在游戏中测试翻译效果
   - 注意检查特殊格式和标签是否正确显示

## 总结

ESP-ESM Translator是一款功能强大的MOD翻译工具，通过了解其界面各列的含义和基本操作流程，您可以高效地完成MOD的汉化工作。对于上古卷轴5的玩家和MOD制作者来说，掌握这一工具的使用方法将大大提升游戏体验和MOD制作效率。

记住，好的翻译不仅仅是准确转换文字，还需要理解游戏背景和术语，保持翻译风格的一致性，这样才能为玩家提供沉浸式的游戏体验。
